## 第一部分：提示词工程（怎么跟 AI 说话）

### 概念 1：什么是提示词？

**说白了**：提示词就是你跟 AI 说的话。

**举个例子**：

❌ **糟糕的提示词**：
```
帮我写个东西
```

AI 内心 OS：写啥？写给谁看？多长？啥风格？

✅ **好的提示词**：
```
你是一个专业的健身教练。
帮我写一份"上班族一周健身计划"。
要求：
- 每天不超过 30 分钟
- 不需要器械
- 包含热身和拉伸
- 用表格形式呈现

目标人群：25-35 岁，久坐办公室，没时间去健身房
```

AI：明白了！马上给你安排！

---

### 概念 2：提示词的四个核心要素

记不住专业术语？没事，记住这个**"身份 - 指令 - 例子 - 背景"**框架就行。

#### 要素 1：身份（Identity）

**概念**：告诉 AI 它是谁。

**为什么重要**：不同的身份，回答方式不一样。

**例子对比**：

```
❌ 不说身份：
"解释一下量子力学"
→ AI 可能给你一篇学术论文

✅ 说清身份：
"你是一个幼儿园老师，用 5 岁小孩能听懂的话解释量子力学"
→ AI 会说："想象一下，一个小球可以同时出现在两个地方..."
```

**常见身份设定**：
- 你是专业的健身教练
- 你是一个幽默的脱口秀演员
- 你是一个耐心的小学老师
- 你是一个经验丰富的程序员
- 你是一个犀利的投资顾问

---

#### 要素 2：指令（Instructions）

**概念**：告诉 AI 具体要做什么，有什么规则。

**为什么重要**：没有指令，AI 会自由发挥，可能跑偏。

**例子对比**：

```
❌ 没有指令：
"帮我写个产品介绍"
→ AI 可能写 1000 字，也可能写 100 字

✅ 有指令：
"帮我写个产品介绍：
- 不超过 200 字
- 突出 3 个核心功能
- 用年轻人喜欢的语气
- 最后加一句行动号召（比如'立即购买'）"
→ AI 知道该怎么写
```

**常见指令类型**：
- 长度限制（不超过 XX 字）
- 格式要求（用表格/列表/段落）
- 风格要求（正式/幽默/温暖）
- 内容要求（必须包含 XX 信息）
- 禁止事项（不要使用专业术语）

---

#### 要素 3：例子（Examples）

**概念**：给 AI 看几个样板，让它照着学。

**为什么重要**：例子比解释更直观。

**专业术语**：这叫 Few-shot Learning（少样本学习）

**例子**：

```
任务：给产品评论打标签（正面/中性/负面）

先给 AI 看几个例子：

例 1：
评论："这个耳机音质超棒，佩戴也舒服！"
标签：正面

例 2：
评论："电池还行吧，但耳垫有点夹耳朵。"
标签：中性

例 3：
评论："客服态度太差，再也不买了！"
标签：负面

现在请给这个评论打标签：
"物流很快，包装也很好，满意！"

AI 会回答：正面
```

**为什么有效**：AI 从例子中学到了模式，知道该怎么分类。

---

#### 要素 4：背景（Context）

**概念**：给 AI 提供额外的信息，让它回答更准确。

**为什么重要**：没有背景信息，AI 只能瞎猜。

**例子对比**：

```
❌ 没有背景：
"这个产品怎么样？"
→ AI：哪个产品？？？

✅ 有背景：
"我们公司是做智能手表的，目标用户是 25-35 岁的都市白领。
产品特点是：续航长、功能全、价格中等。
主要竞争对手是 Apple Watch 和华为手表。

请问：我们的产品在市场上的定位是什么？"
→ AI 可以给出有针对性的分析
```

**常见背景信息**：
- 公司信息（行业、规模、定位）
- 用户信息（年龄、职业、需求）
- 产品信息（功能、价格、竞品）
- 场景信息（使用场景、时间、地点）

---

### 概念 3：消息角色（谁在说话）

**概念**：AI 对话中有三种角色，优先级不一样。

**三种角色**：
1. **developer**（开发者）- 优先级最高
2. **user**（用户）- 优先级中等
3. **assistant**（助手）- AI 自己的回复

**比喻理解**：
- `developer` = 老板定规则
- `user` = 客户提需求
- `assistant` = 员工干活

**例子**：

```
developer: "你只能回答 Python 相关问题，其他问题一律说'我只懂 Python'"
user: "今天天气怎么样？"
assistant: "我只懂 Python"

user: "Python 怎么读取文件？"
assistant: "用 open() 函数，比如：with open('file.txt') as f: ..."
```

**为什么要知道这个**：
- 如果你想设定规则，用 developer 角色
- 如果你只是提问，用 user 角色
- 你不需要管 assistant 角色，那是 AI 的事

---

### 概念 4：格式化技巧（让 AI 听懂你的结构）

**概念**：用 Markdown 和 XML 标签，让提示词结构清晰。

**为什么重要**：结构清晰，AI 不容易搞混。

**例子**：

```markdown
# 身份
你是一个专业的健身教练

# 指令
- 写一份一周健身计划
- 每天不超过 30 分钟
- 不需要器械

# 例子
<示例 1>
用户：我想减脂
教练：推荐有氧运动，比如跑步、游泳...

<示例 2>
用户：我想增肌
教练：推荐力量训练，比如举重、俯卧撑...

# 背景
<用户信息>
年龄：30 岁
职业：程序员
运动经验：零基础
可用时间：工作日晚上 8 点后
</用户信息>
```

**好处**：AI 一眼就能看出哪部分是身份、哪部分是指令、哪部分是例子。

## 第二部分：Skills（AI 的超能力）

### 概念 1：什么是 Skills？

**大白话**：Skills 就是 AI 的"插件"或者"APP"。

**比喻理解**：
- AI 本身 = 一部智能手机
- Skills = 手机上的 APP（微信、地图、相机...）
- 没有 Skills 的 AI = 只能聊天
- 有了 Skills 的 AI = 能干各种事

**例子对比**：

```
❌ 没有 Skills：
用户："今天北京天气怎么样？"
AI："我不知道，我没有实时数据"

✅ 有 Web Search Skill：
用户："今天北京天气怎么样？"
AI：[调用搜索技能] → 查天气网站 → 返回结果
"今天北京晴，最高温度 25 度，适合出门～"
```

---

### 概念 2：常见的 Skills 类型

#### 类型 1：搜索类 🔍
**功能**：上网查资料
**例子**：
- Web Search：搜索任何信息
- Google Search：用谷歌搜索
- 学术搜索：查论文

**使用场景**：
- "帮我查一下最新的 AI 新闻"
- "某某公司的股票代码是多少"
- "最近有什么热门电影"

---

#### 类型 2：代码类 💻
**功能**：写代码、运行代码
**例子**：
- Code Interpreter：执行 Python 代码
- Debug Assistant：帮忙找 bug
- Code Review：代码审查

**使用场景**：
- "帮我写个 Python 脚本，批量重命名文件"
- "这段代码为什么报错？[粘贴代码]"
- "帮我计算 1 到 1000 的素数之和"

---

#### 类型 3：文件类 📁
**功能**：读写文件、搜索文档
**例子**：
- File Reader：读取文件内容
- Document Search：在文档中搜索
- PDF Parser：解析 PDF

**使用场景**：
- "帮我读取这个文件的内容"
- "在我的笔记里搜索'健身'相关的内容"
- "把这个 PDF 转成文字"

---

#### 类型 4：连接类 🔗
**功能**：调用外部 API、连接其他服务
**例子**：
- API Client：调用任何 API
- MCP Connector：连接 MCP 服务
- Database Connector：连接数据库

**使用场景**：
- "帮我调用天气 API，获取北京天气"
- "从数据库里查询上个月的销售额"
- "给我的 Slack 频道发个消息"

---

#### 类型 5：创作类 🎨
**功能**：生成图片、视频、音频
**例子**：
- Image Generation：AI 画图
- Video Creator：生成视频
- Music Maker：创作音乐

**使用场景**：
- "帮我画一只可爱的猫咪"
- "生成一个产品宣传视频"
- "创作一首轻松的背景音乐"

---

#### 类型 6：分析类 📊
**功能**：数据分析、图表生成
**例子**：
- Data Analyzer：分析数据
- Chart Maker：生成图表
- Statistics：统计分析

**使用场景**：
- "帮我分析这个销售数据，找出趋势"

- "计算一下平均值、标准差"

### 概念 3：Skills 怎么用？

**四步流程**：

```
第 1 步：安装
    命令：clawhub install <skill-name>
    例子：clawhub install web-search

第 2 步：配置
    有些 Skills 需要 API Key 或其他配置
    例子：配置搜索 API 的密钥

第 3 步：调用
    在对话中直接使用
    例子："帮我搜索一下今天的新闻"

第 4 步：迭代
    根据结果调整用法
    例子：搜索结果不理想 → 换关键词再搜
```

**实战例子**：

```
用户：安装一个搜索技能
AI：好的，执行命令：clawhub install web-search
    → 安装完成！

用户：帮我查一下 2026 年 AI 发展趋势
AI：[调用 Web Search Skill]
    → 搜索"2026 AI trends"
    → 获取搜索结果
    → 整理汇总

AI：根据搜索结果，2026 年 AI 发展主要有以下趋势：
1. AI Agent 普及化 - 越来越多的企业使用 AI 助手
2. 多模态模型成熟 - 能同时处理文字、图片、视频
3. 边缘 AI 部署增加 - AI 在本地设备运行，不依赖云端
...
```

## 第三部分：多 Agent 协作（AI 团队合作）

### 概念 1：什么是多 Agent？

**大白话**：就是让多个 AI 一起干活，每个 AI 负责不同的任务。

**比喻理解**：
- 单一 AI = 一个全能员工，什么都干，但什么都不精
- 多 Agent = 一个团队，有专员、有分析师、有作家，各司其职

**为什么需要多 Agent**：

```
单一 AI 的问题：
- 能力有限，复杂任务搞不定
- 记不住太多东西（上下文限制）
- 不够专业（什么都懂一点，但都不深）

多 Agent 的优势：
- 分工明确，每个 AI 做自己擅长的
- 可以处理更复杂的任务
- 质量更高（专业的人做专业的事）
```

---

### 概念 2：三种协作模式

#### 模式 1：流水线式 🏭

**概念**：像工厂流水线，一个接一个处理。

**流程**：
```
用户输入 → Agent A(分析) → Agent B(执行) → Agent C(审核) → 输出
```

**例子：写市场分析报告**

```
第 1 步：研究员 Agent
任务：收集市场数据
输出：10 篇相关行业报告

第 2 步：分析师 Agent
任务：分析数据，提取关键信息
输出：5 个核心洞察

第 3 步：作家 Agent
任务：撰写报告
输出：完整的市场分析报告

第 4 步：审核 Agent
任务：检查报告质量
输出：最终版本
```

**适用场景**：需要多步骤处理的任务

---

#### 模式 2：主从式 🎯

**概念**：一个主 AI 协调，多个从 AI 并行干活。

**流程**：
```
           主 Agent (协调者)
          /     |     \
         ↓      ↓      ↓
    Agent A  Agent B  Agent C
    (搜索)   (分析)   (写作)
         \      |      /
          ↓     ↓     ↓
            汇总输出
```

**例子：回答复杂问题**

```
用户："帮我分析一下特斯拉的投资价值"

主 Agent：分解任务
    ↓
同时调用：
- Agent A：搜索特斯拉最新财报
- Agent B：分析电动车行业趋势
- Agent C：对比竞争对手数据
    ↓
主 Agent 汇总所有信息，给出投资建议
```

**适用场景**：需要并行处理的任务

---

#### 模式 3：讨论式 💬

**概念**：多个 AI 讨论，最后达成共识。

**流程**：
```
Agent A (观点 1) ─┐
                 ├→ 讨论 → 共识 → 输出
Agent B (观点 2) ─┘
```

**例子：决策建议**

```
问题："应不应该投资某家公司？"

Agent A (乐观派)：应该投资，理由是...
Agent B (保守派)：谨慎投资，风险是...
Agent C (中立派)：需要更多数据，建议...

经过讨论：
- 认同的点：公司基本面不错
- 分歧的点：短期风险 vs 长期价值
- 最终建议：可以小额投资，观察 3 个月后再决定
```

**适用场景**：需要多角度分析的任务

---

### 概念 3：实战案例

#### 案例 1：自动化客服系统

```
用户问题 → 分类 Agent → 判断问题类型
                    ↓
        ┌───────────┼───────────┐
        ↓           ↓           ↓
   技术支援     账单查询     产品咨询
   Agent        Agent        Agent
        ↓           ↓           ↓
        └───────────┼───────────┘
                    ↓
              回复用户
```

**好处**：
- 响应快（不用人工介入）
- 准确（专业 Agent 处理专业问题）
- 24/7 在线（AI 不休息）

---

#### 案例 2：内容创作团队

```
任务：写一篇产品评测文章

选题 Agent：确定评测什么产品
    ↓
研究 Agent：收集产品信息、用户评价
    ↓
写作 Agent：撰写初稿
    ↓
编辑 Agent：修改润色
    ↓
SEO Agent：优化关键词
    ↓
发布 Agent：发布到各个平台
```

**好处**：
- 效率高（全自动）
- 质量好（每个环节都有专业 Agent）
- 可复制（可以批量生产内容）

## 第五部分：常见问题解答

### Q1: AI 会不会取代我的工作？

**答**：短期内不会，但会用 AI 的人会取代不会用 AI 的人。

**比喻**：
- 计算器没有取代数学家，但让数学家算得更快
- AI 不会取代你，但会让你效率更高

**建议**：学会用 AI，让它帮你干重复的活，你专注于创造性的工作。

---

### Q2: 我没有任何技术背景，能学会吗？

**答**：能！这个教程就是为零基础设计的。

**学习建议**：
1. 从最简单的开始（跟 AI 聊天）
2. 循序渐进（先学提示词，再学 Skills）
3. 多实践（光学不练没用）

### Q3: 学这个需要多长时间？

**答**：

- 精通：1 个月（边学边用）

**关键**：不是学多长时间，而是用多长时间。

2. **多实践**：每天都要用，用多了自然就熟了
3. **建库**：建立自己的提示词库、Skills 库、案例库
