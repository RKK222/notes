# 测试数据生成器 - 方案 C：AI 智能驱动

> 设计时间：2026-03-03 02:30  
> 核心理念：用自然语言描述需求，AI 自动分析和生成

---

## 🎯 方案概述

**用户输入**：自然语言描述业务需求  
**AI 处理**：理解需求 → 推断场景 → 选择模板 → 生成数据  
**输出**：完整的测试数据

**适用场景**：
- ✅ 完全不知道有哪些表
- ✅ 没有接口文档
- ✅ 没有合适模板
- ✅ 复杂业务场景

## 📋 使用流程

### 方式 1：对话式生成

```bash
# 启动 AI 助手
testdata-gen --ai

# 对话示例：
用户：我要测试电商下单流程，需要 10 个订单
AI:  好的，我会生成：
     - 50 个用户数据
     - 20 个商品数据  
     - 10 个订单数据（每个订单 2-5 个商品）
     是否开始生成？

用户：好的
AI:  正在生成... 完成！
     文件已保存到：./test-data/
```

### 方式 2：单行命令

```bash
# 简单描述
testdata-gen --prompt "生成 10 个电商订单测试数据"

# 详细描述
testdata-gen --prompt "我需要一个电商网站的测试数据，
包括用户、商品和订单，用户大约 50 个，商品 20 个，
订单 10 个，每个订单有 2-5 个商品"

# 指定约束
testdata-gen --prompt "生成支付交易数据，
要求：100 个账户，500 笔交易，
交易金额 1-10000 元，包含转账/支付/退款三种类型"
```

### 方式 3：AI 辅助配置

```bash
# 先让 AI 分析需求
testdata-gen --analyze "我要测试社交应用的点赞功能"

# AI 输出：
{
  "scenario": "social",
  "tables": ["users", "posts", "likes"],
  "counts": {
    "users": 100,
    "posts": 200,
    "likes": 1000
  },
  "relations": {
    "likes.user_id": "users.id",
    "likes.post_id": "posts.id"
  }
}

# 确认后生成
testdata-gen --confirm
```

```python
    """AI 需求分析器"""
    
    def analyze(self, user_input):
        """分析用户需求"""
        # 使用 LLM 理解：
        # 1. 业务场景（电商/支付/社交...）
        # 2. 需要的数据类型
        # 3. 数据量要求
        # 4. 特殊约束条件
        
    def extract_entities(self, text):
        """提取关键实体"""
        # 识别：场景、表名、字段、数量、约束
        
    def infer_scenario(self, keywords):
        """推断业务场景"""
        # 关键词匹配 + 语义理解
```

**示例**：
```
用户输入："我要测试电商下单，需要 10 个订单"

AI 分析结果：
{
  "scenario": "ecommerce",
  "intent": "order_testing",
  "entities": {
    "order_count": 10,
    "implied_tables": ["orders", "users", "products", "order_items"]
  },
  "inferred_counts": {
    "users": 50,      # 自动推算
    "products": 20,   # 自动推算
    "orders": 10,     # 用户指定
    "order_items": "2-5 per order"  # 默认规则
  }
}
```
