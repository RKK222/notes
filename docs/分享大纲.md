### 第 3 章：Vibe Coding 时代：不会编码的人需要什么技能？（10 分钟）

**3.1 什么是 Vibe Coding**
- 定义：自然语言描述需求，AI 生成代码
- 核心流程：描述→生成→审核→迭代
- 与传统编程对比表

**3.2 不会编码的人更需要的 5 个技能**
1. **需求抽象能力**
   - 差的描述 vs 好的描述
   - 如何提升（5W1H、示例、分步骤）
2. **审核判断能力**
   - 审核清单（功能/边界/异常/安全/性能）
   - 如何提升
3. **系统思维能力**
   - 差的思维 vs 好的思维
   - 如何提升
4. **沟通协作能力**
   - 与 AI 协作的核心方法
   - 如何提升
5. **持续学习能力**
   - 为什么重要
   - 如何提升

**3.3 测试人员编程处理问题的可能性**
- 场景 1：自动化测试脚本（3 天→30 分钟）
- 场景 2：数据验证工具（2 周→10 分钟）
- 场景 3：性能测试（1 个月→15 分钟）
- 场景 4：Mock 服务（2 周→10 分钟）

**3.4 测试人员的独特优势**
- 边界思维→发现 AI 忽略的边界
- 质量意识→审核 AI 生成代码
- 场景设计→设计全面测试
- 用户视角→评估功能
- 风险敏感→识别潜在风险

**3.5 学习路径建议**
- 阶段 1：入门（1-2 周）
- 阶段 2：进阶（1-2 月）
- 阶段 3：精通（3-6 月）

### 第 5 章：测试人员的工作与成长变化（5 分钟）

**5.1 工作内容变化**
- 时间分配对比（2023 vs 2026）
- 核心变化：从"执行"到"设计"

**5.2 能力要求变化**
- 重要性上升的能力表（需求分析/系统设计/代码审核/AI 协作/自动化架构）
- 重要性下降的能力表（手动执行/用例编写/脚本语法记忆/重复劳动）

**5.3 职业发展路径变化**
- 传统路径：初级→中级→高级→专家→经理
- 新路径：测试执行者→质量工程师→AI 协作专家→质量架构师→质量总监
- 新兴岗位：AI 测试工程师、质量自动化架构师等

**5.4 具体变化示例**
- 变化 1：测试用例设计（2-3 天→2-3 小时）
- 变化 2：自动化测试（3-4 周→1 小时）
- 变化 3：回归测试（2-3 天→2-3 小时）

**5.5 给测试人员的 5 条建议**
1. 拥抱 AI，不要抗拒
2. 提升抽象能力
3. 加强代码审核能力
4. 培养系统思维
5. 建立个人知识库

**5.6 最终结论**
- 三个核心观点：
  1. AI 不是替代，是增强
  2. 从"测试执行者"到"质量工程师"
  3. 现在就开始行动
- 最后一句话：最好的开始时间是昨天，其次是现在

### 表 1：传统 AI vs Agent
| 维度 | 传统 AI | Agent |
|------|--------|-------|
| 交互方式 | 你问它答 | 你吩咐它执行 |
| 执行能力 | 只能输出文本 | 能操作电脑、调用工具 |
| 记忆能力 | 说完就忘 | 记住偏好、历史、上下文 |
| 自主性 | 需要步步指导 | 能独立完成任务 |

### 表 2：常见 Skills
| Skill | 功能 | 使用场景 |
|-------|------|---------|
| UltimateSearchSkill | 双引擎网络搜索 | 查资料、找新闻 |
| yt-dlp-downloader | 视频下载 | 下载 B 站/YouTube 教程 |
| caldav-calendar | 日历同步 | 日程管理、提醒 |
| proactive-agent | 主动代理 | 定时任务、主动推送 |

### 表 3：传统编程 vs Vibe Coding
| 维度 | 传统编程 | Vibe Coding |
|------|---------|------------|
| 入门门槛 | 学习语法（数月） | 会说话就行 |
| 编码速度 | 手动写代码 | AI 生成 |
| 调试方式 | 人工找 bug | AI 修复 |
| 核心能力 | 语法记忆 | 需求表达 |
| 产出质量 | 依赖个人水平 | AI 保证下限 |
