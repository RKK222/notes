## 第二部分：大模型如何工作？

### Slide 5: 核心原理 (简化版)

## 大模型是怎么"思考"的？

### 三步走：

1. **输入**: 把你的话变成数字
   ```
   "你好" → [1234, 5678]
   ```

2. **处理**: 用神经网络预测下一个字
   ```
   "你" + "好" → 下一个字可能是"！"
   ```

3. **输出**: 把数字变回文字
   ```
   [9012] → "！"
   ```

**本质**: 基于概率的"文字接龙"

## 第四部分：大模型的局限与风险

### Slide 13: 能力边界

## 大模型不擅长什么？

### ⚠️ 当前局限

| 领域 | 问题 | 原因 |
|------|------|------|
| 🧮 复杂数学 | 计算错误率高 | 本质是语言模型，不是计算器 |
| 📅 实时信息 | 知识有截止日期 | 训练数据不是实时的 |
| 🎯 精确事实 | 可能"幻觉" | 基于概率生成，不保证准确 |
| 🔒 隐私数据 | 不应输入敏感信息 | 数据可能被用于训练 |
| 🎭 情感理解 | 缺乏真实情感 | 模拟而非真正理解 |
| 🏛️ 专业判断 | 不能替代专家 | 缺乏实际经验 |

### Slide 16.5: 提示词进阶框架（新增）

## 完整提示词结构

### 身份 - 指令 - 例子 - 背景框架

```markdown
# Identity（身份）
- 你是什么角色（AI 助手/专家/顾问）
- 沟通风格（正式/轻松/专业/幽默）
- 高层目标

# Instructions（指令）
- 遵循什么规则（格式要求/长度限制）
- 能做什么/不能做什么
- 如何调用工具/Skills

# Examples（示例）
- 输入输出示例（展示期望格式）
- Few-shot learning（少样本学习）

# Context（上下文）
- 专有数据（用户提供的背景信息）
- 参考材料（相关文档/链接）
```

### 实战对比

❌ **糟糕的提示词**：
```
帮我写个东西
```

✅ **好的提示词**：
```
你是一个专业的健身教练。
帮我写一份"上班族一周健身计划"。
要求：
- 每天不超过 30 分钟
- 不需要器械
- 包含热身和拉伸
- 用表格形式呈现

目标人群：25-35 岁，久坐办公室，没时间去健身房
```

### Slide 19.5: Skills - AI 的超能力（新增）

## 什么是 Skills？

**大白话**：Skills 就是 AI 的"插件"或者"APP"

**比喻理解**：
- AI 本身 = 一部智能手机
- Skills = 手机上的 APP（微信、地图、相机...）
- 没有 Skills 的 AI = 只能聊天
- 有了 Skills 的 AI = 能干各种事

### 6 大类 Skills

| 类型 | 功能 | 示例场景 |
|------|------|----------|
| 🔍 **搜索类** | 上网查资料 | "帮我查一下今天的新闻" |
| 💻 **代码类** | 写代码、运行代码 | "帮我计算 1 到 100 的素数之和" |
| 📁 **文件类** | 读写文件、搜索文档 | "在我的笔记里搜索'健身'" |
| 🔗 **连接类** | 调用 API、连接服务 | "从数据库查询销售额" |
| 🎨 **创作类** | 生成图片、视频 | "帮我画一只可爱的猫咪" |
| 📊 **分析类** | 数据分析、图表 | "分析销售数据，找出趋势" |

### Slide 19.6: 多 Agent 协作（新增）

## 什么是多 Agent？

**大白话**：让多个 AI 一起干活，每个 AI 负责不同的任务

**比喻理解**：
- 单一 AI = 一个全能员工，什么都干，但什么都不精
- 多 Agent = 一个团队，有专员、有分析师、有作家，各司其职

### 三种协作模式

#### 模式 1：流水线式
```
用户输入 → Agent A(分析) → Agent B(执行) → Agent C(审核) → 输出
```
**适用场景**：需要多步骤处理的任务

#### 模式 2：主从式
```
        主 Agent (协调者)
       /    |    \
      ↓     ↓     ↓
  Agent A  Agent B  Agent C
```
**适用场景**：需要并行处理的任务

#### 模式 3：讨论式
```
Agent A (观点 1) ─┐
                 ├→ 讨论 → 共识 → 输出
Agent B (观点 2) ─┘
```
**适用场景**：需要多角度分析的任务

### 实战案例：自动化客服系统

```
用户问题 → 分类 Agent → 路由到专业 Agent
                    ↓
        ┌───────────┼───────────┐
        ↓           ↓           ↓
   技术支援     账单查询     产品咨询
   Agent        Agent        Agent
```
