#### 2. LongCLI-Bench: 长程智能体编程基准 (2026-02-26)
- **arXiv**: 2602.14337
- **标题**: LongCLI-Bench: A Preliminary Benchmark and Study for Long-horizon Agentic Programming in Command-Line Interfaces
- **作者**: Yukang Feng 等 (19 位作者)
- **领域**: 软件工程 (cs.SE); 多智能体系统 (cs.MA)
- **核心贡献**:
  - 从 1000+ 计算机科学作业和真实工作流中精选 20 个高质量长程任务
  - 涵盖四类工程任务：从零开始、功能添加、Bug 修复、重构
  - 提出双集测试协议：需求满足 (fail-to-pass) 和回归避免 (pass-to-pass)
  - 引入步骤级评分以精确定位执行失败
- **关键发现**:
  - 即使是最先进的智能体，通过率也低于 20%
  - 大多数任务在不到 30% 完成度时就停滞
  - 关键失败通常发生在早期阶段
  - 自我修正仅提供边际增益
  - 人机协作 (计划注入和交互指导) 产生显著提高
- **应用价值**: 强调需要发展协同的人机工作流，以及智能体的规划和执行能力

#### 3. Automated Test Suite Enhancement Using LLMs (2026-02-12)
- **arXiv**: 2602.12256
- **标题**: Automated Test Suite Enhancement Using Large Language Models with Few-shot Prompting
- **作者**: Alex Chudic, Gül Çalıklı (US Booking Services Ltd., University of Glasgow)
- **领域**: 软件工程 (cs.SE)
- **会议**: Accepted to ICPC 2026 (34th International Conference on Program Comprehension)
- **核心贡献**:
  - 实证研究 few-shot prompting 与不同测试工件源 (人类、SBST、LLM) 如何影响 LLM 生成的单元测试质量
  - 评估维度：正确性、覆盖率、可读性、认知复杂度、可维护性
  - 在 HumanEval 和 ClassEval 数据集上使用 GPT-4o 进行实验
  - 评估基于检索的示例选择方法
- **关键发现**:
  - LLM 可以通过 few-shot prompting 生成高质量测试
  - 人类编写的示例产生最佳覆盖率和正确性
  - 基于问题描述和代码的组合相似度选择示例最有效
- **应用价值**: 为混合人机代码库中的测试套件增强提供指导

#### 4. SAFuzz: LLM 生成代码的语义引导自适应 Fuzzing (2026-02-11)
- **arXiv**: 2602.11209
- **标题**: SAFuzz: Semantic-Guided Adaptive Fuzzing for LLM-Generated Code
- **作者**: Ziyi Yang 等
- **领域**: 软件工程 (cs.SE); 密码学与安全 (cs.CR)
- **核心贡献**:
  - 混合测试框架，利用 LLM 引导的自适应 fuzzing 检测算法漏洞
  - 集成基于提示的行为多样化、特定问题的 oracle 的 harness 生成
  - LLM 基预测器实现自适应资源分配和动态早期停止
- **关键结果**:
  - 漏洞区分精度从 77.9% 提升到 85.7%
  - 与 SOTA GreenFuzz 相比，时间成本降低 1.71 倍
  - 与现有单元测试生成方法结合，Bug 检测召回率从 67.3% 提升到 79.5%
- **应用价值**: 高效检测 AI 生成代码中的算法漏洞
