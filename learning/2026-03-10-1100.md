#### 1. Agent Safehouse – macOS 原生 Agent 沙箱
- **地址：** https://agent-safehouse.dev/
- **来源：** Hacker News / GitHub
- **领域分类：** Testing
- **技术介绍：** Agent Safehouse 是一个 macOS 原生的本地 Agent 沙箱系统，专为本地运行的 AI 代理提供安全隔离环境。随着本地 AI 代理的普及，如何安全地运行这些具有系统访问能力的代理成为关键问题。该系统利用 macOS 的沙箱机制（Sandbox.kext）、系统扩展（System Extensions）和权限控制，为 AI 代理创建一个受限的执行环境。核心功能包括：文件系统访问限制、网络访问控制、系统调用过滤、剪贴板隔离、用户数据保护。采用声明式安全策略配置，支持细粒度的权限管理。采用"拒绝优先"访问模型，代理继承用户全部权限的翻转版本——除非明确授权，否则一切不可访问。
- **应用场景：** 本地 AI 代理安全运行、敏感数据处理、企业 AI 部署安全
- **为什么值得关注：** 首个专为 macOS 本地 Agent 设计的沙箱系统，解决本地 AI 安全痛点，单脚本部署无需构建步骤

#### 2. ESAA-Security: AI 生成代码安全审计架构
- **地址：** https://arxiv.org/abs/2603.06365
- **来源：** arXiv
- **领域分类：** Testing
- **技术介绍：** 《ESAA-Security: An Event-Sourced, Verifiable Architecture for Agent-Assisted Security Audits of AI-Generated Code》研究提出了一种事件溯源、可验证的架构，用于代理辅助的 AI 生成代码安全审计。随着 AI 生成代码的广泛使用，如何确保这些代码的安全性成为关键挑战。该架构采用事件溯源模式记录所有审计操作，确保审计过程的可追溯性和可验证性。系统包含多个专业代理：代码分析代理、漏洞检测代理、修复建议代理和合规检查代理。每个代理的操作都被记录为不可变事件，支持审计回放和验证。框架将审计结构化为治理执行管道，包含四个阶段：侦察、域审计执行、风险分类和最终报告，操作化为 26 个任务、16 个安全域和 95 个可执行检查。
- **应用场景：** AI 生成代码安全审计、合规检查、安全研究
- **为什么值得关注：** 首个针对 AI 生成代码的事件溯源安全审计架构，开源实现可用

#### 3. SAHOO - 递归自改进中的对齐保护框架
- **地址：** https://arxiv.org/abs/2603.06333
- **来源：** arXiv (ICLR 2026 Workshop)
- **领域分类：** Testing
- **技术介绍：** 《Safeguarded Alignment for High-Order Optimization Objectives in Recursive Self-Improvement》研究提出了 SAHOO 框架，用于监控和控制递归自改进系统中的对齐漂移。现代系统可以批评、修订和评估自己的输出，但迭代自修改存在微妙的对齐漂移风险。SAHOO 通过三个保护机制实现：(1) 目标漂移指数（GDI），一个学习的多信号检测器，结合语义、词汇、结构和分布测量；(2) 约束保持检查，强制执行安全关键不变量，如语法正确性和非幻觉；(3) 回归风险量化，标记撤销先前增益的改进周期。在代码生成、数学推理和真实性方面的 189 个任务中，SAHOO 产生了显著的质量增益，包括代码任务 18.3% 的改进和推理任务 16.8% 的改进。
- **应用场景：** 递归自改进系统、AI 对齐监控、自优化系统安全
- **为什么值得关注：** ICLR 2026 接受，首个系统性递归自改进对齐保护框架，使对齐保护可测量、可部署

#### 5. AegisUI - AI 代理 UI 协议行为异常检测
- **地址：** https://arxiv.org/abs/2603.05031
- **来源：** arXiv
- **领域分类：** Testing
- **技术介绍：** 针对 AI 代理系统即时构建用户界面的结构化协议载荷的行为异常检测框架。当前防御措施仅停留在语法检查层面，无法检测行为不匹配问题（如按钮显示"查看发票"但实际操作是删除账户）。AegisUI 框架生成结构化 UI 载荷，注入真实攻击，提取数值特征，端到端基准测试异常检测器。研究产生了 4000 个标注载荷（3000 良性，1000 恶意），涵盖五个应用领域和五类攻击：钓鱼界面、数据泄露、布局滥用、操纵性 UI 和工作流异常。提取 18 个特征，比较三种检测器：Isolation Forest、自编码器和 Random Forest。Random Forest 表现最佳（准确率 0.931，精确率 0.980，F1 0.843）。
- **应用场景：** AI 代理安全、UI 协议验证、安全分析、恶意界面检测
- **为什么值得关注：** 首个针对 AI 代理 UI 协议的安全分析框架，填补 AI 安全测试空白

#### 1. Agent Safehouse - macOS 原生 Agent 沙箱

**GitHub 地址：** https://github.com/eugene1g/agent-safehouse

**详细技术分析：**

Agent Safehouse 代表了本地 AI 代理安全领域的重要突破。随着 Claude Code、Codex、Amp 等本地 AI 代理的普及，这些代理具有系统访问能力，如何安全运行成为关键问题。

**技术架构：**
- **内核级沙箱：** 利用 macOS Sandbox.kext 和 sandbox-exec 配置文件，在内核级别强制执行访问控制
- **拒绝优先模型：** 与传统"继承用户全部权限"不同，Safehouse 采用"默认拒绝"模型，除非明确授权，否则一切不可访问
- **声明式策略：** 使用简单的配置文件定义每个目录的访问级别（读/写/拒绝）
- **单脚本部署：** 无需构建步骤，仅依赖 Bash 和 macOS 原生功能

**核心创新：**
1. **工作目录自动授权：** 自动授予选定工作目录（默认 git 根目录）的读/写访问权限
2. **工具链只读访问：** 自动授予已安装工具链的只读访问
3. **敏感区域默认拒绝：** SSH 密钥、AWS 凭证、其他仓库、个人文件默认拒绝
4. **Shell 函数集成：** 可通过 shell 函数使所有代理默认在沙箱中运行

**实现机制：**
```bash
# 示例配置文件
~/my-project: read-write
~/shared-lib: read-only
~/.ssh: denied
~/.aws: denied
~/other-repos: denied
```

**与现有技术对比：**
- 传统沙箱：需要复杂配置，学习曲线陡峭
- 容器方案：重量级，不适合本地代理
- Agent Safehouse：轻量级、声明式、零配置启动

**优势：**
- 内核级安全，进程在访问被拒绝数据前就被阻止
- 零依赖，仅需 Bash 和 macOS
- 与所有主流代理兼容（Claude、Codex、Amp、Gemini）
- 可证明的安全性（可尝试读取 SSH 密钥验证）

**局限：**
- 仅支持 macOS
- 需要 sandbox-exec 配置文件知识
- 对跨目录工作流需要手动配置

**未来发展潜力：**
- 扩展到 Linux（使用 AppArmor/SELinux）
- 自动配置文件生成（使用 LLM 分析用户环境）
- 细粒度网络访问控制
- 与代理原生集成

**个人技术理解：**
Agent Safehouse 解决了本地 AI 代理的核心安全痛点。其"拒绝优先"模型是安全设计的黄金标准，而声明式配置使其易于使用。随着本地 AI 代理的普及，这类沙箱系统将成为基础设施的关键组成部分。
