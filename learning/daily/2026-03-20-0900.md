## 技术扫描（32 项）

### AI Testing / Software Testing（10 项）

1. **TDAD: Test-Driven Agentic Development** (arXiv:2603.17973)
   - **来源：** arXiv (cs.SE)
   - **领域：** Software Testing / AI Coding
   - **技术介绍：** 提出测试驱动代理开发框架，通过基于图的依赖影响分析减少 AI 编码代理的代码回归。核心创新是将传统 TDD 原则适配到代理开发场景：代理生成代码前先生成测试用例，利用代码依赖图进行影响分析，识别潜在回归风险点。提交至 ACM AIware 2026，工具论文 7 页含 3 表 1 图 1 算法。
   - **应用场景：** AI 辅助编程、代码生成质量保障、持续集成
   - **为什么值得关注：** 首个将 TDD 系统化适配到 AI 代理开发的框架，解决代码回归痛点

2. **VeriGrey: Greybox Agent Validation** (arXiv:2603.17639)
   - **来源：** arXiv (cs.AI)
   - **领域：** AI Testing / Validation
   - **技术介绍：** 提出灰盒代理验证框架，介于黑盒测试和白盒验证之间。利用部分内部状态信息进行更高效的代理行为验证。作者包括 Marcel Böhhme 和 Abhik Roychoudhury 等形式化验证专家。框架支持部分可观测场景下的代理正确性验证，平衡验证精度和开销。
   - **应用场景：** 代理系统验证、安全关键代理部署、合规审计
   - **为什么值得关注：** 填补黑盒/白盒验证之间的空白，实用性强

3. **FailureMem: A Failure-Aware Multimodal Framework for Autonomous Software Repair** (arXiv:2603.17826)
   - **来源：** arXiv (cs.SE)
   - **领域：** Software Testing / Auto-Repair
   - **技术介绍：** 失败感知多模态自主软件修复框架。整合执行轨迹、错误日志、代码上下文等多模态失败信息到统一记忆表示，用于指导自动修复。框架包含失败模式分类器、多模态记忆编码器和修复策略生成器三个核心组件。
   - **应用场景：** 自动化调试、程序修复、持续维护
   - **为什么值得关注：** 多模态失败记忆提升修复准确率，减少重复失败

4. **scicode-lint: Detecting Methodology Bugs in Scientific Python Code with LLM-Generated Patterns** (arXiv:2603.17893)
   - **来源：** arXiv (cs.SE)
   - **领域：** Software Testing / Scientific Computing
   - **技术介绍：** 使用 LLM 生成的模式检测科学 Python 代码中的方法论缺陷。针对科学计算代码特有的问题（如数据泄露、统计检验误用、随机种子管理不当）生成检测规则。填补科学代码质量保障工具空白。
   - **应用场景：** 科研代码审查、可复现性保障、学术出版前检查
   - **为什么值得关注：** 首个针对科学代码方法论缺陷的自动化检测工具

5. **CodeScout: An Effective Recipe for Reinforcement Learning of Code Search Agents** (arXiv:2603.17829)
   - **来源：** arXiv (cs.SE)
   - **领域：** Software Testing / Code Search
   - **技术介绍：** 代码搜索代理的强化学习有效方案。12 位作者包括 Graham Neubig。提出 CodeScout 框架，通过 RL 训练代码搜索代理，优化搜索策略和结果排序。在多个代码库基准上验证有效性。
   - **应用场景：** 代码复用、漏洞模式搜索、测试用例生成
   - **为什么值得关注：** CMU 团队工作，RL 赋能代码搜索新方向

6. **Toward Scalable Automated Repository-Level Datasets for Software Vulnerability Detection** (arXiv:2603.17974)
   - **来源：** arXiv (cs.SE)
   - **领域：** Software Testing / Security
   - **技术介绍：** 面向软件漏洞检测的可扩展自动化仓库级数据集构建方法。解决现有漏洞数据集规模小、标注质量低、代表性不足的问题。提出自动化漏洞挖掘和标注流程，支持大规模数据集生成。

7. **Maestro: Painless E2E Automation for Mobile and Web** (GitHub: mobile-dev-inc/Maestro)

   - **为什么值得关注：** E2E 测试领域新兴热门工具，今日 trending

8. **AgentFactory: A Self-Evolving Framework Through Executable Subagent Accumulation and Reuse** (arXiv:2603.18000)

   - **领域：** AI Testing / Agent Systems

9. **RPMS: Enhancing LLM-Based Embodied Planning through Rule-Augmented Memory Synergy** (arXiv:2603.17831)
