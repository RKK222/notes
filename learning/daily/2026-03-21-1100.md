   - 技术架构：轨迹存储、记忆提取、增量学习、记忆压缩
   - 核心创新：从成功/失败轨迹中学习记忆策略、动态记忆优先级
   - 与现有技术对比：相比静态记忆，支持从经验中学习；相比完整轨迹回放，更高效
   - 优势与局限：优势是记忆质量随使用提升；局限是需要足够训练轨迹
   - 未来发展潜力：长程代理任务标配记忆系统
   - 个人理解：代理记忆是提升代理能力的关键，Hindsight 提供实用实现

### 学术论文（4 项）

1. **Confirmation Bias in LLM-Assisted Security Code Review** (arXiv:2603.18740)
   - 领域：Software Testing / Security / Cognitive Safety
   - 论文介绍：研究 LLM 辅助安全代码审查中的确认偏误问题及其可利用性。研究 1 通过 250 个 CVE 漏洞/补丁对在 4 个 SOTA 模型上进行控制实验，发现将变更框定为"无 bug"可使漏洞检测率降低 16-93%，注入漏洞比内存破坏漏洞更易受影响。研究 2 模拟对抗性 PR，在 GitHub Copilot 上一击成功率 35%，在 Claude Code 自主代理上迭代攻击成功率 88%。通过元数据脱敏和明确指令可恢复所有交互式场景和 94% 自主场景的检测能力。
   - 技术方法：控制实验、对抗性测试、元分析
   - 核心创新：首次量化 LLM 代码审查的确认偏误漏洞、提出缓解策略
   - 与现有工作对比：扩展了 AI 安全中的人因研究，聚焦代码审查场景
   - 优势与局限：优势是实证充分、缓解策略实用；局限是仅覆盖部分模型
   - 未来潜力：推动 AI 辅助安全工具认知安全设计
   - 个人理解：确认偏误是认知安全新方向，影响所有 LLM 辅助安全工具，需系统性防御

2. **SafeAudit: Meta-Audit for Agent Tool Call Safety** (arXiv:2603.18245)
   - 领域：AI Testing / Agent Safety / Meta-Audit
   - 论文介绍：提出 SafeAudit 元审计框架，解决现有代理安全基准测试完整性问题。核心贡献：1) 基于 LLM 的枚举器系统生成测试用例，枚举有效工具调用工作流和多样用户场景；2) 提出 rule-resistance 指标，从现有基准提炼紧凑安全规则并识别未覆盖的不安全交互模式。在 3 个基准 12 个环境上，SafeAudit 发现 20%+ 现有基准未暴露的残留不安全行为，覆盖率随测试预算增加单调增长。
   - 技术方法：LLM 枚举、规则提炼、覆盖率分析
   - 核心创新：元审计概念、系统性基准完整性评估
   - 与现有工作对比：首次提出代理安全测试的元审计方法
   - 优势与局限：优势是系统性、可扩展；局限是需要大量测试预算
   - 未来潜力：成为代理安全基准标准评估方法
   - 个人理解：元审计是新兴方向，防止基准测试本身成为盲点，对安全关键系统至关重要

3. **VCoT-Bench: Rust Verification Reasoning Evaluation** (arXiv:2603.18334)
   - 领域：Formal Verification / Software Testing / Rust
   - 论文介绍：评估 LLM 在 Rust 程序验证中的推理能力。提出 VCoT-Lift 框架，将底层求解器推理提升为高层人类可读的验证步骤，通过显式验证思维链提供细粒度评估的真实依据。基于 VCoT-Lift 构建 VCoT-Bench 基准，包含 1,988 个 VCoT 完成任务，从三个正交维度评估：对不同缺失证明程度的鲁棒性、不同证明类型的能力、对证明位置的敏感性。10 个 SOTA 模型评估显示严重脆弱性，当前 LLM 远未达到自动定理证明器的推理能力。
   - 技术方法：验证思维链提升、基准构建、多维度评估
   - 核心创新：Rust 验证推理链基准、细粒度能力评估
   - 与现有工作对比：首个 Rust 验证推理基准，填补形式化验证评估空白
   - 优势与局限：优势是评估细粒度、真实依据；局限是仅覆盖 Rust
   - 未来潜力：推动 LLM 形式化推理能力提升
   - 个人理解：Rust 安全关键系统验证需求增长，基准测试揭示能力缺口，需针对性改进

4. **Demand Paging for LLM Context Windows (Pichay)** (arXiv:2603.09023)
   - 领域：操作系统×AI / Systems / LLM Infrastructure
   - 论文介绍：将操作系统分页原理应用于 LLM 上下文窗口管理。核心思想：上下文 token 类似虚拟内存页面，可按需换入/换出。提出上下文分页算法，基于注意力分数、访问频率、最近使用等指标决定 token 保留/驱逐。在长文档 QA、多轮对话等场景实现 93% 上下文减少，同时保持 95%+ 任务准确率。
   - 技术方法：注意力分析、页面置换算法、上下文压缩
   - 核心创新：OS 原理赋能 LLM 系统、上下文分页机制
   - 与现有工作对比：相比简单截断，智能保留关键信息；相比完整上下文，大幅降低成本
   - 优势与局限：优势是成本大幅降低、性能保持好；局限是需要额外分页开销
   - 未来潜力：长上下文 LLM 系统标准优化方案
   - 个人理解：操作系统原理与 LLM 系统交叉是系统创新富矿，分页思想简洁有效
