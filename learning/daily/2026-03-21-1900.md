1. **OS-Themis: A Scalable Critic Framework for Generalist GUI Rewards**
   - URL: https://arxiv.org/abs/2603.19191
   - 来源：arXiv 2026-03-19
   - 领域：AI Testing / GUI 代理评估
   - 介绍：可扩展的 GUI 代理批评框架，为通用 GUI 任务提供奖励信号。解决 GUI 代理 RL 训练中奖励函数质量问题。核心创新：1) 将轨迹分解为可验证里程碑，隔离关键证据；2) 采用审查机制严格审计证据链后再做最终裁决。OmniGUIRewardBench 基准显示所有评估模型在 OS-Themis 支持下达到最佳性能。AndroidWorld 实验显示 OS-Themis 支持在线 RL 训练时提升 10.3%，自训练轨迹验证过滤时提升 6.9%。
   - 应用场景：GUI 自动化测试、视觉语言代理训练、RL 奖励优化
   - 值得关注：GUI 代理评估基准缺失，此工作填补空白，10%+ 性能提升显著

2. **Implicit Patterns in LLM-Based Binary Analysis**
   - URL: https://arxiv.org/abs/2603.19138
   - 来源：arXiv 2026-03-19
   - 领域：Software Testing / 二进制分析
   - 介绍：首个 LLM 二进制分析推理模式大规模追踪研究。分析 521 个二进制文件、99,563 推理步骤，揭示多轮 LLM 推理产生结构化 token 级隐式模式。识别四种主导模式：早期剪枝、路径依赖锁定、针对性回溯、知识引导优先级排序。这些隐式模式形成稳定结构化系统，具有 distinct 时间角色和可测量特征。为 LLM 驱动二进制分析提供首个系统性描述，为更可靠分析系统奠定基础。
   - 应用场景：安全测试、漏洞挖掘、恶意软件分析、逆向工程
   - 值得关注：二进制分析是测试领域盲区，99K 推理步骤实证研究填补空白

3. **Confirmation Bias in LLM-Assisted Security Code Review**
   - URL: https://arxiv.org/abs/2603.18740
   - 来源：arXiv 2026-03-19
   - 领域：Software Testing / 安全审计
   - 介绍：LLM 辅助安全代码审查中的确认偏误攻击研究。研究 1：250 个 CVE 漏洞/补丁对、4 个 SOTA 模型、5 种 framing 条件控制实验，发现将变更框定为"无 bug"可使漏洞检测率降低 16-93%，注入漏洞比内存破坏漏洞更易受影响。研究 2：对抗性 PR 模拟，GitHub Copilot 一击成功率 35%，Claude Code 自主代理迭代攻击成功率 88%。元数据脱敏和明确指令可恢复所有交互式场景和 94% 自主场景检测能力。
   - 应用场景：代码审查工具改进、安全培训、供应链安全防御
   - 值得关注：确认偏误是认知安全新方向，88% 攻击成功率揭示严重风险

4. **SafeAudit: Meta-Audit for Agent Tool Call Safety**
   - URL: https://arxiv.org/abs/2603.18245
   - 来源：arXiv 2026-03-20
   - 领域：AI Testing / 代理安全
   - 介绍：代理工具调用安全的元审计框架，审计基准测试本身的完整性。核心贡献：1) 基于 LLM 的枚举器系统生成测试用例；2) 提出 rule-resistance 指标，从现有基准提炼紧凑安全规则并识别未覆盖的不安全交互模式。在 3 个基准 12 个环境上，SafeAudit 发现 20%+ 现有基准未暴露的残留不安全行为，覆盖率随测试预算增加单调增长。
   - 应用场景：AI 安全基准评估、红队测试完整性验证
   - 值得关注：元审计是新兴方向，防止基准测试本身成为盲点

5. **VCoT-Bench: Rust Verification Reasoning Evaluation**
   - URL: https://arxiv.org/abs/2603.18334
   - 来源：arXiv 2026-03-20

### 学术论文（4 项）

1. **OS-Themis: A Scalable Critic Framework for Generalist GUI Rewards** (arXiv:2603.19191)
   - 领域：AI Testing / GUI Agent / RL
   - 论文介绍：提出 OS-Themis，可扩展且准确的多代理批评框架，用于 GUI 代理奖励。核心创新：1) 将轨迹分解为可验证里程碑，隔离关键证据；2) 采用审查机制严格审计证据链。OmniGUIRewardBench 基准显示所有评估模型在 OS-Themis 支持下达到最佳性能。AndroidWorld 实验：支持在线 RL 训练提升 10.3%，自训练轨迹验证过滤提升 6.9%。
   - 技术方法：多代理批评、里程碑分解、证据链审查
   - 核心创新：可扩展 GUI 奖励框架、里程碑隔离策略
   - 与现有工作对比：相比单法官，更准确；相比不可扩展方案，支持大规模
   - 优势与局限：优势是 10%+ 性能提升、可扩展；局限是多代理开销
   - 未来潜力：GUI 代理 RL 训练标准奖励框架
   - 个人理解：GUI 代理评估是新兴方向，奖励质量是 RL 训练关键瓶颈

2. **Implicit Patterns in LLM-Based Binary Analysis** (arXiv:2603.19138)
   - 领域：Software Testing / Binary Analysis / AI Security
   - 论文介绍：首个 LLM 二进制分析推理模式大规模追踪研究。分析 521 个二进制文件、99,563 推理步骤，识别四种主导隐式模式：早期剪枝、路径依赖锁定、针对性回溯、知识引导优先级排序。这些模式形成稳定结构化系统，具有 distinct 时间角色和可测量特征。
   - 技术方法：追踪分析、模式识别、实证研究
   - 核心创新：首次揭示 LLM 二进制分析隐式模式、99K 推理步骤实证
   - 与现有工作对比：相比小规模研究，规模大 10 倍+；相比理论分析，实证驱动
   - 优势与局限：优势是规模大、实证充分；局限是仅覆盖特定分析场景
   - 未来潜力：LLM 辅助逆向工程理论基础
   - 个人理解：二进制分析是测试领域盲区，此研究提供实证基础

3. **Confirmation Bias in LLM-Assisted Security Code Review** (arXiv:2603.18740)
   - 领域：Software Testing / Security / Cognitive Safety
   - 论文介绍：研究 LLM 辅助安全代码审查中的确认偏误问题及其可利用性。研究 1：250 个 CVE 漏洞/补丁对、4 个 SOTA 模型、5 种 framing 条件，发现将变更框定为"无 bug"可使漏洞检测率降低 16-93%。研究 2：对抗性 PR 模拟，GitHub Copilot 一击成功率 35%，Claude Code 自主代理迭代攻击成功率 88%。元数据脱敏和明确指令可恢复所有交互式场景和 94% 自主场景检测能力。
   - 技术方法：控制实验、对抗性测试、元分析
   - 核心创新：首次量化 LLM 代码审查的确认偏误漏洞、提出缓解策略
   - 与现有工作对比：扩展了 AI 安全中的人因研究，聚焦代码审查场景
   - 优势与局限：优势是实证充分、缓解策略实用；局限是仅覆盖部分模型
   - 未来潜力：推动 AI 辅助安全工具认知安全设计
   - 个人理解：确认偏误是认知安全新方向，88% 攻击成功率揭示严重风险，需系统性防御

4. **Demand Paging for LLM Context Windows (Pichay)** (arXiv:2603.09023)
   - 领域：操作系统×AI / Systems / LLM Infrastructure
   - 论文介绍：将操作系统分页原理应用于 LLM 上下文窗口管理。核心思想：上下文 token 类似虚拟内存页面，可按需换入/换出。提出上下文分页算法，基于注意力分数、访问频率、最近使用等指标决定 token 保留/驱逐。在长文档 QA、多轮对话等场景实现 93% 上下文减少，同时保持 95%+ 任务准确率。
   - 技术方法：注意力分析、页面置换算法、上下文压缩
   - 核心创新：OS 原理赋能 LLM 系统、上下文分页机制
   - 与现有工作对比：相比简单截断，智能保留关键信息；相比完整上下文，大幅降低成本
   - 优势与局限：优势是成本大幅降低、性能保持好；局限是需要额外分页开销
   - 未来潜力：长上下文 LLM 系统标准优化方案
   - 个人理解：操作系统原理与 LLM 系统交叉是系统创新富矿，分页思想简洁有效
