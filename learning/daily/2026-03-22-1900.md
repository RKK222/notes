#### 1. pytest-dev/pytest-ai（AI 测试生成）

**技术架构：**
- 基于 pytest 插件系统，钩子函数拦截测试发现过程
- LLM 后端可配置（支持 OpenAI、Anthropic、本地模型）
- 测试生成流水线：代码解析 → 上下文提取 → 提示构建 → 生成 → 验证 → 优化

**核心创新：**
- 官方 pytest 插件，生态兼容性好
- 支持增量生成，基于已有测试补充新场景
- 生成过程可追溯，每个测试用例附带生成理由

**与现有技术对比：**
- 相比 GitHub Copilot：专注测试生成，覆盖率更高
- 相比 EvoSuite：支持 Python 生态，LLM 泛化能力强
- 相比手动编写：效率提升 10 倍，但需人工审核

**优势与局限：**
- 优势：官方背书、易用、集成好
- 局限：复杂业务逻辑测试生成质量待提升

**未来发展潜力：** ⭐⭐⭐⭐

**个人技术理解：**

- 联合优化：最大化多模态模型的预测错误
- 评估指标：多模态鲁棒性分数（MRS）

**核心算法：**
- 梯度-based 跨模态扰动生成
- 约束优化确保扰动不可感知
- 多目标优化平衡攻击强度和隐蔽性

**关键实现机制：**
- 支持 CLIP、BLIP、Flamingo 等主流 VLM
- 自动化攻击流水线
- 可视化攻击效果

**与现有技术对比：**
- 相比单模态攻击：揭示跨模态脆弱性
- 相比黑盒攻击：效率更高，可解释
- 相比传统对抗测试：针对多模态特性优化

**优势与局限：**
- 优势：首个多模态对抗测试框架、实用性强
- 局限：计算成本高、需要模型梯度

**未来发展潜力：** ⭐⭐⭐⭐⭐

- 扩展方向：防御方法、鲁棒训练

**个人技术理解：**
这篇论文揭示了一个重要问题：单模态鲁棒性不足以保证多模态安全。随着多模态 AI 普及，这类测试将成为刚需。

#### 2. Benchmarking LLM Self-Correction Abilities Across Domains（LLM 自校正基准）

**技术架构：**
- 基准数据集：4 领域 × 5 难度 × 1000 样本
- 评估协议：初始生成 → 自校正 → 最终评估
- 指标：校正成功率、校正质量、过度校正率

**核心发现：**
- 自校正能力高度依赖任务类型
- 数学和代码任务自校正效果最好
- 事实问答自校正可能导致"过度自信错误"

**关键实现机制：**
- 自动化评估流水线
- 人工标注验证集
- 跨模型对比分析

**与现有技术对比：**
- 相比单一任务基准：覆盖更广，结论更可靠
- 相比人工评估：规模大，可重复
- 相比厂商自测：独立第三方，更客观

**优势与局限：**
- 优势：系统性、跨领域、可重复
- 局限：评估成本高、需要人工验证

**未来发展潜力：** ⭐⭐⭐⭐
- 预测：LLM 能力评估标准基准
- 扩展方向：更多任务类型、更多模型

**个人技术理解：**
