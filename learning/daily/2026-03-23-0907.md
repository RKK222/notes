1. **SpecOps: A Fully Automated AI Agent Testing Framework in Real-World GUI Environments** ⭐深度研究
   - URL: https://arxiv.org/abs/2603.10268
   - 来源：arXiv (ICSE 2026 接收)
   - 领域：AI Testing / GUI 测试
   - 介绍：SpecOps 是首个面向真实世界 GUI 环境的全自动 AI 代理测试框架。核心创新在于将代理测试形式化为状态空间搜索问题，通过 GUI 元素识别、状态抽象、动作序列生成实现端到端测试自动化。框架支持跨平台（Web、移动、桌面）GUI 测试，内置断言生成、回归检测、视觉差异分析能力。实验表明，SpecOps 在 15 个真实应用上达到 87% 的测试覆盖率，比现有方法提升 34%。
   - 应用场景：GUI 应用自动化测试、跨平台测试统一、AI 代理行为验证
   - 值得关注：ICSE 2026 接收，GUI 测试领域里程碑工作

2. **TDAD: Test-Driven Agentic Development - Reducing Code Regressions in AI Coding Agents via Graph-Based Impact Analysis** ⭐深度研究
   - URL: https://arxiv.org/abs/2603.17973
   - 来源：arXiv (ACM AIWare 2026 提交)
   - 领域：AI Testing / 测试驱动开发
   - 介绍：TDAD 提出测试驱动的代理开发范式，通过基于图的依赖影响分析减少 AI 编码代理的代码回归。核心方法包括：(1) 代码变更图建模，(2) 测试用例影响范围分析，(3) 最小测试集选择策略。TDAD 集成到 AI 编码工作流中，在代理生成代码后自动识别受影响测试并执行，回归检测准确率 94%，测试执行开销降低 67%。
   - 应用场景：AI 辅助开发质量保证、持续集成测试优化、代码回归预防
   - 值得关注：测试驱动开发与 AI 编码代理的创新融合

3. **Automated Self-Testing as a Quality Gate: Evidence-Driven Release Management for LLM Applications**
   - URL: https://arxiv.org/abs/2603.15676
   - 来源：arXiv (EMSE 投稿中)
   - 领域：Software Testing / LLM 应用
   - 介绍：针对 LLM 应用非确定性输出和模型演化特性，提出自动化自测试作为质量门限的发布管理框架。核心创新在于建立证据驱动的发布决策机制，通过持续自测试收集模型行为证据，基于统计显著性判断是否可以发布。框架包含测试用例生成、输出一致性检测、性能回归监控三大模块。
   - 应用场景：LLM 应用发布管理、模型更新验证、质量门限自动化
   - 值得关注：LLM 应用发布管理的系统性解决方案

4. **Test-Driven AI Agent Definition (TDAD): Compiling Tool-Using Agents from Behavioral Specifications**
   - URL: https://arxiv.org/abs/2603.08806
   - 来源：arXiv
   - 领域：AI Testing / 代理开发

2. **多跳推理**: 需要跨文档、跨段落推理
