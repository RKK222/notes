### 学术论文

#### 1. DAIRA: 动态分析增强的问题解决代理
- **地址**: https://arxiv.org/abs/2603.22048
- **介绍**: 提出 Test Tracing-Driven 方法论，将动态分析原生嵌入代理推理循环。轻量监控器提取运行时变量变异和调用栈，合成为结构化语义报告。在 Gemini 3 Flash Preview 驱动下，SWE-bench Verified 达 79.4% SOTA。
- **技术架构**: 动态分析模块 → 轻量监控器 → 结构化语义报告 → Agent 推理循环
- **核心算法**: Test Tracing-Driven 方法论，将盲目猜测转为证据驱动推导
- **与现有技术对比**: 超越所有现有基线，推理费用降低 10%，token 消耗降低 25%
- **优势**: 解决复杂隐式类型退化和多态控制流 bug
- **局限**: 依赖 Gemini 3 Flash Preview 模型
- **未来潜力**: 动态分析 + Agent 推理的范式可推广到更多领域

#### 2. Emerge: 大模型实现等价验证
- **地址**: https://arxiv.org/abs/2603.21851
- **介绍**: 基于 e-graph 检查大模型跨框架实现等价性。自动推断候选关系、按需合成重写规则。通过 SMT 检查和约束感知随机测试验证规则。目标 HuggingFace Transformers 和 vLLM。
- **技术架构**: e-graph 表示 + 执行值推断 + 按需规则合成 + SMT/随机测试验证
- **核心创新**: 首个自动化大模型跨框架等价验证系统
- **与现有技术对比**: 无需手工编写重写规则
- **优势**: 检测 10/13 已知 bug + 8 个新 bug
- **局限**: 当前仅针对推理计算图
- **个人理解**: 大模型部署中"同一模型不同实现"的正确性验证是被严重忽视的问题

#### 3. MCP 客户端提示注入安全

- **介绍**: OWASP Top 10 for LLM 将提示注入列为第一大漏洞。本文首次系统分析 7 个 MCP 客户端的工具投毒安全态势。

- **评估维度**: 静态验证、参数可见性、注入检测、用户警告、执行沙箱、审计日志
- **个人理解**: MCP 正成为 Agent 工具调用的事实标准，其安全性直接决定 Agent 生态的可信度

#### 4. AI 编码工具工程缺陷实证研究
- **地址**: https://arxiv.org/abs/2603.20847
- **介绍**: 首个针对 Claude Code/Codex/Gemini CLI 的系统性 bug 研究，分析 3.8K+ 公开报告的 bug。
- **关键发现**: 67% 功能性 bug、36.9% API/集成/配置错误、18.3% API 错误症状
- **与现有技术对比**: 首次从 MSR 视角系统分析 AI 编码工具
- **未来潜力**: 为可靠 AI 编码助手设计提供实证基础

#### 5. AER: 自主 AI 代理推理溯源
- **地址**: https://arxiv.org/abs/2603.21692
- **介绍**: 提出 Agent Execution Record 结构化推理溯源原语，将意图/观察/推理作为一等可查询字段。支持群体行为分析、推理模式挖掘、反事实回归测试。
- **核心创新**: 从"计算状态持久化"到"推理溯源"的范式区分

## 技术趋势

1. **Agent 编排平台爆发**：superpowers（110K 星）、everything-claude-code（105K 星）、deer-flow（43K 星）、ruflo（25K 星）在同一周爆发性增长，标志着 Agent 编排从实验走向生产。技能框架 > 提示工程的范式转换正在发生。

2. **AI 编码工具质量与安全成焦点**：DAIRA 在 SWE-bench 创 79.4% SOTA、MCP 安全分析揭示严重漏洞、3.8K bug 实证研究，三篇论文从不同角度定义了 AI 编码工具的质量治理方向。动态分析 + 安全审计 + 工程缺陷分类三管齐下。

3. **多代理系统运维基础设施成形**：EAGER（多代理失败管理）、RuntimeSlicer（统一运行时状态）、AER（推理溯源）三篇 FSE/顶会论文同时出现，标志着从"构建多代理系统"到"运维多代理系统"的转变。
