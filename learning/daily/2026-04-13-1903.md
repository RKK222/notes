## 技术扫描（30 项）

### AI Testing / Software Testing（10 项）

1. **Knowledge Conflicts from Evolving APIs in Code Generation**
   - 来源：arXiv:2604.09515
   - URL：https://arxiv.org/abs/2604.09515
   - 介绍：研究 LLM 在 API 演化场景下的知识冲突问题。构建 270 个真实 API 更新基准（8 个 Python 库），评估 11 个模型。发现无完整文档时 LLM 生成代码可执行率仅 42.55%，结构化文档 + 大模型规模提升至 66.36%，自反思策略额外提升 11%。
   - 价值：揭示 RAG 在 API 演化场景的局限性，为进化感知基准设计提供依据

2. **Do AI Coding Agents Log Like Humans? An Empirical Study**
   - 来源：arXiv:2604.09409
   - URL：https://arxiv.org/abs/2604.09409
   - 介绍：分析 4550 个代理 PR（81 个仓库），发现代理变更日志频率比人类低 58.4%，但密度更高。显式日志指令罕见（4.7%）且无效（67% 不遵守）。人类承担 72.5% 的日志修复工作，成为"沉默的清洁工"。
   - 价值：揭示 AI 代理在非功能需求上的双重失败，建议采用确定性护栏

3. **The AI Codebase Maturity Model: From Assisted Coding to Self-Sustaining Systems**
   - 来源：arXiv:2604.09388
   - URL：https://arxiv.org/abs/2604.09388
   - 介绍：提出 AI 代码库成熟度模型（ACMM），5 级框架描述从 AI 辅助编码到自维持系统的演进。基于 4 个月 KubeStellar Console 实战经验（63 个 CI/CD 工作流、91% 覆盖率、<30 分钟 bug 修复）。核心发现：智能不在模型本身，而在 surrounding 基础设施。
   - 价值：首个 AI 驱动开发系统化框架，测试是解锁下一级的关键

4. **Vulnerability Detection with Interprocedural Context in Multiple Languages**
   - 来源：arXiv:2604.08417
   - URL：https://arxiv.org/abs/2604.08417
   - 介绍：研究 LLM 在跨过程依赖漏洞检测中的效果。评估 4 个现代 LLM（Claude Haiku 4.5、GPT-4.1 Mini、GPT-5 Mini、Gemini 3 Flash）在 509 个漏洞上的表现。Gemini 3 Flash 性价比最优（F1≥0.978，成本$0.50-0.58），Claude Haiku 4.5 正确识别并解释 93.6% 案例。
   - 价值：跨语言漏洞检测新基准，指导 AI 辅助安全分析工具设计

5. **Security Concerns in Generative AI Coding Assistants: Insights from Online Discussions**
   - 来源：arXiv:2604.08352
   - URL：https://arxiv.org/abs/2604.08352
   - 介绍：分析 Stack Overflow、Reddit、Hacker News 上关于 GitHub Copilot 安全讨论。识别 4 大关注领域：数据泄露、代码许可、对抗攻击（提示注入）、不安全代码建议。采用 BERTopic 聚类 + 主题分析。
   - 价值：开发者视角的 GenAI 安全担忧全景图

6. **Multi-Objective Optimization of Agent Skills for Software Engineering**
   - 来源：arXiv:2604.09297
   - URL：https://arxiv.org/abs/2604.09297
   - 介绍：SkillMOO 框架，使用 LLM 提议编辑 + NSGA-II 选择自动进化技能包。在 3 个 SkillsBench 任务上，通过率提升最高 131%，成本降低 32%。模式分析显示剪枝和替换是主要改进驱动。
   - 价值：技能包自动优化新范式，避免手动调优

7. **Test-Oriented Programming: rethinking coding for the GenAI era**
   - 来源：arXiv:2604.08102
   - URL：https://arxiv.org/abs/2604.08102
   - 介绍：提出测试导向编程（TOP）新范式。开发者只检查基于自然语言规格生成的测试代码，生产代码完全委托给 LLM。开发概念验证工具并在两个 LLM 上生成小型命令行程序验证可行性。
   - 价值：GenAI 时代编程范式革新，抽象层次提升

8. **Multimodal Graph Alignment for Bug Localization in Automated Program Repair**
   - 来源：arXiv:2604.08089
   - URL：https://arxiv.org/abs/2604.08089
   - 介绍：GALA 框架，解决多模态 APR 中 GUI 截图与代码组件的 disconnect 问题。通过图像 UI 图构建、文件级对齐、函数级对齐、补丁生成四阶段，实现视觉 - 代码精确映射。在 SWE-bench Multimodal 上达到 SOTA。
   - 价值：多模态 Bug 定位新突破， hierarchical 结构对齐

9. **Quantifying the Contribution of Oracle Information Signals on SWE Agents**
   - 来源：arXiv:2604.07789
   - URL：https://arxiv.org/abs/2604.07789
   - 介绍：Oracle-SWE 方法，隔离并提取 SWE 基准中的 oracle 信息信号（复现测试、回归测试、编辑位置、执行上下文、API 使用），量化各信号对代理性能的贡献。指导自主编码系统研究优先级。
   - 价值：SWE 代理性能归因分析，指导研究方向

10. **Empirical Evaluation of Taxonomic Trace Links: A Case Study**
    - 来源：arXiv:2604.08207
    - URL：https://arxiv.org/abs/2604.08207
    - 介绍：在 Ericsson 工业场景评估 TTL（分类追溯链接）方法。建立 463 个业务用例、64 个测试用例、277 个 ISO 需求之间的追溯链接。从业者认为 TTL 在一个场景中有用，在另一个场景中效果较差。
    - 价值：工业级追溯性实践评估，分类方法补充传统追溯
