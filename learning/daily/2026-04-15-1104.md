### GitHub 项目（4 项）

#### 1. RePAIR: Interactive Machine Unlearning Framework
- **URL**: https://github.com/repair-unlearning/repair (拟)
- **项目介绍**: RePAIR 是首个交互式机器遗忘框架，使用户能够通过自然语言指令 LLM 遗忘特定知识。传统机器遗忘需要模型服务提供商介入，进行重新训练和 curated retain 数据集准备。RePAIR 实现用户驱动、设备上的模型编辑，无需重新训练。
- **技术架构**: 三模型架构：(1) Watchdog 模型检测遗忘意图；(2) Surgeon 模型生成修复程序；(3) Patient 模型自主更新参数。核心 STAMP 技术通过伪逆更新操纵 MLP 激活，重定向到拒绝子空间。
- **核心算法**: STAMP (Steering Through Activation Manipulation with PseudoInverse) - 训练免费、单样本遗忘方法。低秩变体将计算复杂度从 O(d^3) 降至 O(r^3 + r^2 * d)。
- **关键实现**: 闭式伪逆更新，无需梯度下降；设备上执行，~3x 加速于训练基线；支持有害知识抑制、错误信息纠正、个人数据擦除。
- **与现有技术对比**: 相比提供者中心方法（需要 MSP 介入、重新训练），RePAIR 实现用户中心、即时遗忘。相比其他模型编辑方法（如知识编辑、参数编辑），RePAIR 支持自然语言接口和单样本操作。
- **优势与局限**: 优势：用户友好、设备上执行、计算高效、保持模型效用（Acc_r 达 84.47）。局限：目前仅支持 MLP 层，需扩展到注意力层；单样本遗忘可能不如多样本方法彻底。
- **未来发展潜力**: 扩展到多模态基础模型；支持批量遗忘；与隐私法规（如 GDPR 被遗忘权）集成；成为 LLM 标准功能。
- **个人技术理解**: 这是机器遗忘领域的范式转变——从提供者中心到用户中心，从离线批处理到在线即时。STAMP 的伪逆方法巧妙利用线性代数性质，避免昂贵的优化过程。这是 LLM 可控性和透明度的重要一步。

#### 2. Hermes-Agent: Continuous Learning Agent
- **URL**: https://github.com/NousResearch/hermes-agent
- **项目介绍**: Hermes-Agent 是持续学习代理框架，通过技能图谱和元学习架构实现代理随用户共同成长。日增 6485 stars，展现 AGI 基础设施潜力。
- **技术架构**: 技能图谱（Skill Graph）存储已学技能及其依赖关系；元学习架构（Meta-Learning Architecture）快速适应新任务；经验回放缓冲区防止灾难性遗忘。
- **核心算法**: 技能发现算法自动识别新技能；技能组合算法将原子技能组合成复杂技能；元学习优化器快速适应新分布。
- **关键实现**: 技能图谱使用图神经网络编码技能关系；元学习采用 MAML 变体；经验回放使用优先级采样。
- **与现有技术对比**: 相比静态代理（固定技能集），Hermes 持续成长；相比传统持续学习（易灾难性遗忘），Hermes 通过技能图谱和回放有效保留旧技能。
- **优势与局限**: 优势：持续成长、技能可解释、快速适应。局限：技能图谱维护开销；元学习需要大量任务分布。
- **未来发展潜力**: 成为个人 AI 助手基础设施；支持跨代理技能共享；与人类教师协作学习。
- **个人技术理解**: Hermes 代表代理进化的正确方向——不是单一巨型模型，而是可组合、可成长的技能生态系统。技能图谱是关键创新，使代理能够显式表示和管理其能力。

#### 3. Archon: AI Programming Harness

- **项目介绍**: Archon 是开源 AI 编程 harness 构建器，解决 AI 编程非确定性问题。通过测试驱动流程、审计日志和多模型共识，实现可靠的 AI 辅助编程。
- **技术架构**: 测试驱动层（自动生成和执行测试）；审计日志层（记录所有 AI 决策和代码变更）；多模型共识层（多个模型投票决定最佳方案）。

- **优势与局限**: 优势：可靠性高、可审计、多模型冗余。局限：多模型推理成本高；测试生成可能不完整。

#### 4. EdgeLLM v3: On-Device Inference Engine
- **URL**: https://github.com/edge-llm/edge-llm-v3
- **项目介绍**: EdgeLLM v3 是端侧推理引擎，支持 13B 模型在手机上实现 12 token/s 推理速度。通过动态专家路由和端云协同，实现高效边缘 AI。

- **核心算法**: 量化感知训练保持精度；动态跳过算法基于输入难度预测；专家路由算法优化负载均衡。
- **关键实现**: iOS/Android 原生支持；内存优化至 5.2GB（13B 模型）；功耗优化，支持长时间推理。
- **与现有技术对比**: 相比云端推理，EdgeLLM 提供低延迟、隐私保护；相比其他端侧引擎（如 MLC-LLM、llama.cpp），EdgeLLM 支持更大模型和更高速度。
- **优势与局限**: 优势：高性能、低功耗、隐私保护。局限：大模型仍需高内存设备；端云协同需要网络连接。

- **个人技术理解**: EdgeLLM v3 代表端侧 AI 的成熟——13B 模型在手机上实时运行曾是幻想，现在成为现实。动态专家路由是关键创新，使 MoE 架构在端侧可行。

#### 1. AISafetyBenchExplorer: AI Safety Benchmark Catalogue
- **URL**: https://arxiv.org/abs/2604.12875
- **论文介绍**: 本论文呈现 AISafetyBenchExplorer，一个 195 个 AI 安全基准的结构化目录（2018-2026 年）。通过多表模式记录基准级元数据、指标级定义、基准论文元数据和仓库活动。核心发现：基准增长超过测量标准化，领域碎片化而非稀缺是主要失败模式。
- **技术架构**: 多表模式：(1) 基准级元数据（名称、年份、任务类型）；(2) 指标级定义（准确率、F1、安全分数）；(3) 论文元数据（作者、机构、发表 venue）；(4) 仓库活动（stars、commits、最后更新）。
- **核心算法**: 复杂度分类算法将基准分为 Popular/Medium/Niche；指标标准化算法统一不同基准的评分方式；趋势分析算法识别研究热点转移。
- **关键实现**: 195 个基准手动标注；94/195 为中等复杂度，仅 7 个为 Popular；165/195 仅英语；137/195 GitHub 仓库停滞；96/195 HuggingFace 数据集停滞。
- **与现有技术对比**: 相比简单基准列表，AISafetyBenchExplorer 提供深度元分析和可追溯性；相比其他目录，本工作重点在指标标准化和基准治理。
- **优势与局限**: 优势：全面覆盖、结构化元数据、支持元分析。局限：手动标注工作量大；需要持续维护。
- **未来发展潜力**: 成为 AI 安全评估标准参考；支持基准选择和比较；促进测量标准化。
- **个人技术理解**: 这篇论文揭示 AI 安全评估的核心问题——不是缺少基准，而是缺少共享测量语言。195 个基准看似丰富，但指标定义、聚合规则、威胁模型各不相同，导致结果不可比。这是领域成熟的必经阶段，AISafetyBenchExplorer 是重要一步。

#### 2. PAL: Personal Adaptive Learner
- **URL**: https://arxiv.org/abs/2604.13017
- **论文介绍**: PAL 是 AI 驱动教育平台，将讲座视频转换为交互式学习体验。现有教育平台局限于静态适应（预定义测验、统一进度、通用反馈），PAL 实现实时、个体化支持。通过多模态内容分析和自适应决策，PAL 动态生成问题、调整难度、生成个性化总结。
- **技术架构**: 多模态内容分析（视频、音频、文本）；自适应决策引擎（根据学习者响应调整）；个性化总结生成（根据兴趣定制示例）。
- **核心算法**: 问题生成算法基于讲座内容自动生成不同难度问题；难度调整算法根据学习者表现动态调整；总结生成算法提取关键概念并定制示例。
- **关键实现**: 实时处理讲座视频；问题难度分级（基础/中级/高级）；兴趣建模基于学习者历史交互。
- **与现有技术对比**: 相比静态教育平台（固定内容、统一进度），PAL 实现实时适应；相比其他自适应系统（仅调整进度），PAL 调整内容、难度、示例。
- **优势与局限**: 优势：实时适应、个性化强、多模态支持。局限：需要高质量讲座视频；兴趣建模可能不准确。
- **未来发展潜力**: 成为在线教育标准平台；支持 K12 到高等教育；与 VR/AR 集成。
- **个人技术理解**: PAL 代表 AI 教育的正确方向——从"一刀切"到"一人一策"。核心创新是实时适应，不是事后调整。多模态分析使系统理解讲座内容，自适应决策使系统理解学习者，两者结合实现真正个性化。

#### 3. CCS: Cycle-Consistent Search
- **URL**: https://arxiv.org/abs/2604.12967
- **论文介绍**: CCS 是无黄金监督的搜索代理训练框架，受无监督机器翻译和图像转换的循环一致性技术启发。核心假设：最优搜索轨迹是问题意图的无损编码，应能准确重构原始问题。通过信息瓶颈（排除最终响应、NER 掩码）防止信息泄漏。
- **技术架构**: 搜索代理（执行搜索）；重构代理（根据轨迹重构问题）；奖励计算（比较重构与原始问题）。
- **核心算法**: 循环一致性损失衡量重构质量；信息瓶颈防止浅层线索利用；策略优化算法更新搜索代理。
- **关键实现**: NER 掩码搜索查询防止实体泄漏；排除最终响应防止答案泄漏；在 QA 基准上验证。
- **与现有技术对比**: 相比监督方法（需要黄金答案），CCS 无需黄金监督；相比其他无监督方法，CCS 利用循环一致性提供密集奖励信号。
- **优势与局限**: 优势：无需黄金监督、可扩展、性能相当监督基线。局限：重构质量影响训练效果；信息瓶颈可能过度限制。
- **未来发展潜力**: 成为搜索代理标准训练方法；扩展到其他信息检索任务；与主动学习结合。
- **个人技术理解**: CCS 的核心洞见是：好的搜索轨迹应该"编码"问题意图，因此可以"解码"回问题。这是自监督学习在搜索领域的巧妙应用。信息瓶颈设计精妙，防止代理走捷径。

#### 4. MISID: Multimodal Multi-turn Intent Recognition Dataset
- **URL**: https://arxiv.org/abs/2604.12700
- **论文介绍**: MISID 是多模态、多轮、多参与者意图识别基准，源自高利害社交策略游戏（如狼人杀、Among Us）。现有意图识别数据集聚焦单话语或简单对话，MISID 填补复杂战略交互空白。两层多维度标注方案支持长上下文话语分析和基于证据的因果追踪。
- **技术架构**: 数据收集（游戏录像转录）；标注系统（两层多维度）；评估框架（多模态 LLM 基准测试）。
- **核心算法**: FRACTAM 基线框架："解耦 - 锚定 - 推理"范式。解耦提取纯单模态事实表示；锚定使用两阶段检索进行长范围事实锚定；推理构建显式跨模态证据链。
- **关键实现**: 多参与者对话标注；意图层次结构（表面意图/隐藏意图）；因果链标注（证据→推理→意图）。
- **与现有技术对比**: 相比简单意图数据集（单轮、单模态），MISID 支持复杂战略交互；相比其他多轮数据集，MISID 增加多模态和多参与者维度。
- **优势与局限**: 优势：复杂场景、多模态、细粒度标注。局限：数据收集成本高；标注主观性强。
- **未来发展潜力**: 成为意图识别标准基准；支持欺骗检测、谈判分析；扩展到真实场景（商务谈判、外交）。
- **个人技术理解**: MISID 揭示 MLLM 的关键缺陷：文本优先视觉幻觉（过度依赖文本，忽视视觉）；跨模态协同受损（无法有效融合多模态信息）；因果链能力有限（无法追踪长程因果关系）。FRACTAM 的"解耦 - 锚定 - 推理"是有效对策，但根本问题仍需解决。
