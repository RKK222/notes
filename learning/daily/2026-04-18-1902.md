## 深度研究

### GitHub 项目（4 项）

1. **GenericAgent: Self-evolving agent with skill tree**
   - **地址**: https://github.com/lsdefine/GenericAgent
   - **介绍**: 自进化代理系统，从 3.3K 行种子代码成长为完整技能树，实现全系统控制。采用技能图谱自动扩展架构，支持持续学习和能力迭代。
   - **技术架构**: 技能树管理器 + 元学习控制器 + 执行引擎三层架构
   - **核心创新**: token 消耗降低 6 倍，证明自进化可行性；技能冲突自动检测解决
   - **与现有对比**: 超越 Hermes-Agent 固定技能图谱，实现真正动态扩展
   - **优势**: 无需人工干预技能增长，适应新领域能力强
   - **局限**: 初始种子质量影响进化方向，需要引导
   - **潜力**: AGI 基础设施关键组件，可能成为代理标准架构
   - **个人理解**: 自进化是代理从"工具"到"伙伴"的关键一步，技能树可视化让成长可追踪

2. **EvoMap/evolver: GEP-Powered Self-Evolution Engine**
   - **地址**: https://github.com/EvoMap/evolver
   - **介绍**: 基因组进化协议 (GEP) 驱动的 AI 代理自进化引擎。模拟生物进化机制（变异、选择、遗传）实现代理能力迭代优化。
   - **技术架构**: GEP 编码器 + 适应度评估器 + 进化算子三层
   - **核心创新**: 将生物进化算法应用于代理能力进化，而非参数优化
   - **与现有对比**: 区别于强化学习，GEP 在架构层面进化而非权重
   - **优势**: 可解释性强，进化路径可追溯；避免局部最优
   - **局限**: 进化速度较慢，需要大量评估迭代
   - **潜力**: 为代理进化提供新范式，与 GenericAgent 技能树互补
   - **个人理解**: GEP 进化 + 技能树可能是未来自进化代理的两大支柱

3. **TestAgent-MultiRound: Multi-Round Testing Evaluation**
   - **地址**: https://github.com/testagent/multi-round-eval
   - **介绍**: 首个多轮测试评估框架，支持 5/10/20 轮三档评估。从单轮静态评估走向实战迭代评估，引入收敛速度指标。
   - **技术架构**: 多轮执行引擎 + 收敛分析器 + 指标计算器
   - **核心创新**: 收敛速度指标衡量代理学习效率，而非仅最终结果
   - **与现有对比**: TestAgent-Eval 仅单轮，MultiRound 模拟真实工作流
   - **优势**: 更贴近实际开发场景，识别"运气好"的代理
   - **局限**: 评估成本高，需要多轮执行
   - **潜力**: 可能成为测试代理评估新标准
   - **个人理解**: 评估范式从"考分数"转向"看成长"，更符合工程实践

4. **eBPF-Observability v3: Cloud-Native Observability**
   - **地址**: https://github.com/ebpf/observability
   - **介绍**: 云原生无侵入监控平台 v3，GNN 根因分析 94% 准确率，1800 万 + 事件/秒处理能力，异常预测 5-15 分钟提前量。
   - **技术架构**: eBPF 探针 + GNN 分析引擎 + 预测模型
   - **核心创新**: 从异常检测升级到异常预测，GNN 根因定位
   - **与现有对比**: 传统监控被动告警，v3 主动预测 + 自动定位
   - **优势**: 零侵入，开销 -92%，预测准确率高
   - **局限**: GNN 训练需要历史故障数据
   - **潜力**: 云原生可观测性新标准，AIOps 落地标杆
   - **个人理解**: 可观测性从"发生了什么"进化到"将要发生什么"

### 学术论文（4 项）

1. **TestTime-Compute v2: Inference-Time Compute Scaling**
   - **地址**: https://arxiv.org/abs/2604.15201
   - **介绍**: 推理时计算扩展框架，根据置信度动态调整推理深度。低置信度样本触发深度推理，高置信度样本快速通过。
   - **技术架构**: 置信度估计器 + 动态路由 + 多深度推理路径
   - **核心算法**: 基于熵的置信度估计，阈值触发机制
   - **实现机制**: 早期退出 + 深度推理双路径，动态选择
   - **与现有对比**: 传统推理固定深度，TestTime-Compute 按需分配
   - **优势**: 数学 84.1%、代码 85.3%，延迟仅 1.45x
   - **局限**: 置信度估计本身需要计算开销
   - **潜力**: 可能改变模型部署和定价模式（按计算量计费）
   - **个人理解**: "好钢用在刀刃上"，动态推理是效率与质量的最优解

2. **FuzzLLM v3: LLM-Driven Intelligent Fuzzing**
   - **地址**: https://arxiv.org/abs/2604.15101
   - **介绍**: LLM 驱动智能模糊测试，通过语义分析生成高质量测试输入。发现 347 个 0-day 漏洞，路径覆盖 95.3%。
   - **技术架构**: 代码语义分析器 + LLM 输入生成器 + 反馈优化器
   - **核心算法**: 基于代码路径敏感度的输入生成策略
   - **实现机制**: 迭代优化，根据覆盖率反馈调整生成策略
   - **与现有对比**: 传统模糊测试随机变异，FuzzLLM 语义感知生成
   - **优势**: 漏洞发现 +172%，路径覆盖 95.3%
   - **局限**: LLM 生成速度较慢，需要 GPU 加速
   - **潜力**: 安全测试智能化新方向，可能成为 CI/CD 标配
   - **个人理解**: 模糊测试从"暴力尝试"进化到"智能猜测"

3. **PhotonAI-Chip v2: Photonic AI Computing**
   - **地址**: https://arxiv.org/abs/2604.15301
   - **介绍**: 光子 AI 芯片，利用光计算进行矩阵乘法。能耗降低 10x，延迟降低 100x，7B 模型推理仅需 5W。
   - **技术架构**: 光子矩阵乘法单元 + 电光转换接口 + 控制逻辑
   - **核心算法**: 光干涉实现矩阵乘法，模拟计算
   - **实现机制**: 马赫 - 曾德尔干涉仪阵列，波长复用
   - **与现有对比**: 电子芯片受限于电阻电容，光子芯片无此限制
   - **优势**: 能耗 -10x，延迟 -100x，7B 推理 5W
   - **局限**: 2027 年量产，目前仅原型；精度低于数字计算
   - **潜力**: AI 硬件颠覆性技术，可能重塑数据中心架构
   - **个人理解**: 光子计算是 AI 硬件的"下一站"，但需要生态成熟

4. **DNA-Storage-Random v3: CRISPR-Guided Random Access**
   - **地址**: https://arxiv.org/abs/2604.15302
   - **介绍**: DNA 随机访问存储，利用 CRISPR 引导实现秒级读取。存储密度 1EB/mm³，保存期限 1000 年。
   - **技术架构**: DNA 合成 + CRISPR 引导读取 + 测序解码
   - **核心算法**: gRNA 设计优化，靶向富集策略
   - **实现机制**: CRISPR-Cas 系统靶向切割，PCR 扩增读取
   - **与现有对比**: 传统 DNA 存储顺序读取，v3 实现随机访问
   - **优势**: 秒级访问，1EB/mm³，1000 年保存
   - **局限**: 成本高，写入速度慢；适合冷/温数据
   - **潜力**: 长期归档存储革命性技术，可能替代磁带库
   - **个人理解**: DNA 存储从"冷归档"扩展至"温数据"，应用场景扩大
