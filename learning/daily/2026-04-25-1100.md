### AI Testing / Software Testing（10 项）

1. **DIVERT: Diversity-Guided User Simulation for Agent Evaluation** (arXiv:2604.21480)
   - 地址：https://arxiv.org/abs/2604.21480
   - 领域：AI Testing / Agent Evaluation
   - 介绍：针对 LLM 代理评估的线性 Monte Carlo 方法效率低下问题，提出快照+分支的覆盖引导用户模拟框架。在关键决策点捕获完整的代理-环境状态并从中恢复执行，实现共享对话前缀复用。从每个分叉点使用多样性诱导的用户响应进行分支，定向探索替代交互路径。发现比标准线性滚动协议更多的失败/令牌，同时扩展发现失败的任务集。
   - 价值：Agent 评估从"线性模拟"进化到"覆盖引导+快照复用"，效率和质量双提升

2. **Evaluating AI Meeting Summaries with Reusable Cross-Domain Pipeline** (arXiv:2604.21345)
   - 地址：https://arxiv.org/abs/2604.21345
   - 领域：AI Testing / LLM Evaluation
   - 介绍：可复用的生成式 AI 应用评估管道，实例化为 AI 会议摘要评估。分离可复用编排与任务特定语义，五阶段：源摄取、结构化参考构建、候选生成、结构化评分、报告。基准测试 114 个会议，340 会议-模型对，680 评判运行。gpt-4.1-mini 最高平均准确率 0.583，gpt-5.1 最高完整性 0.886 和覆盖率 0.942。
   - 价值：评估管道从"一次性脚本"进化到"可复用 typed artifact"

3. **Probabilistic Verification of Neural Networks via Efficient Probabilistic Hull Generation** (arXiv:2604.21556)
   - 地址：https://arxiv.org/abs/2604.21556
   - 领域：Software Testing / Formal Verification
   - 介绍：神经网络概率验证框架，计算安全概率的保证范围。三个创新：回归树状态空间细分产生概率壳、边界感知采样识别输入空间安全边界、概率优先级迭代精炼计算保证范围。ACAS Xu 和火箭着陆控制器基准评估，明显优于 SOTA。
   - 价值：神经网络安全验证从"确定性边界"进化到"概率保证范围"

4. **SemanticAgent: Semantics-Aware Framework for Text-to-SQL Data Synthesis** (arXiv:2604.21414)

#### 2. DIVERT: Diversity-Guided User Simulation for Agent Evaluation (arXiv:2604.21480)

**地址：** https://arxiv.org/abs/2604.21480
**来源：** arXiv
**类型：** Paper

**论文介绍（200-400 字）：**
DIVERT 针对 LLM 代理评估的效率问题，提出快照+分支的覆盖引导用户模拟框架。当前评估协议依赖线性 Monte Carlo 滚动完整代理-用户对话来估计成功率，但这种方法计算效率低下——重复生成相同的前缀，且经常无法发现由罕见用户行为引起的深层失败模式。DIVERT 在关键决策点捕获完整的代理-环境状态并从中恢复执行，实现共享对话前缀复用。从每个分叉点使用多样性诱导的用户响应进行分支，定向探索替代交互路径。

**技术架构：**
- 快照捕获：在决策点保存完整代理-环境状态
- 分支生成：多样性诱导的用户响应
- 覆盖引导：优先探索未充分测试的轨迹
- 复用引擎：共享前缀避免重复计算

**核心算法/系统设计：**
- 决策点检测算法
- 多样性度量（语义距离）

| 覆盖引导 | ✅ | ❌ | 部分 |
| 多样性保证 | ✅ | ❌ | ❌ |
| 效率 | 高 | 低 | 中 |

**优势与局限：**
- 优势：效率显著提升、发现更多失败模式
- 局限：快照存储开销、分支爆炸风险

**未来发展潜力：**

**个人技术理解：**

#### 3. On the Generalization Ability of Transformers in Symbolic Reasoning (arXiv:2604.21632)

**地址：** https://arxiv.org/abs/2604.21632
**来源：** arXiv
**类型：** Paper

**论文介绍（200-400 字）：**
这篇论文研究了 decoder-only Transformer 在抽象符号推理中的泛化能力，特别是解决命题逻辑推理问题。前人工作表明模型无法泛化到训练期间未观察到的变量名，原因之一是难以复制/生成未见 token。本文发现 unembedding collapse 是关键机制：未见 token 的 unembedding（最后一层权重）在训练中坍缩到几乎相同的向量，使得区分多个未见变量变得困难。提出结合小架构变更（促进复制）、数据多样性和冻结/重置 (un)embedding 的技术组合，实现未见 token 泛化。在 Gemma 3 系列（含 99 个保留 token）中观察到相同的 unembedding 坍缩现象。

**技术架构：**
- 理论分析：unembedding collapse 的数学描述
- 实验验证：命题逻辑推理基准
- 解决方案：架构变更+数据多样性+冻结/重置
- 实证研究：Gemma 3 系列分析

**核心算法/系统设计：**
- Unembedding 坍检测算法
- 主动遗忘（active forgetting）机制
- 冻结/重置策略

**与现有技术对比：**
| 方法 | 泛化能力 | 理论解释 | 可扩展性 |
|------|---------|---------|---------|
| 标准训练 | 弱 | 无 | 中 |
| 主动遗忘 | 中 | 弱 | 中 |
| 本文方法 | 强 | 强 | 高 |

**优势与局限：**
- 优势：机制解释清晰、实证验证充分
- 局限：仅针对符号推理、需要架构变更

**未来发展潜力：**
符号推理泛化是 Transformer 能力边界的核心问题。本文的机制解释为未来的架构设计提供了理论基础。

**个人技术理解：**
这篇论文的价值在于提供了"为什么 Transformer 在符号推理中泛化困难"的机制解释——unembedding collapse。这与深度学习理论化方向（arXiv:2604.21691）形成呼应，是理论化的具体案例。Gemma 3 的 99 个保留 token 的实证发现尤其重要，表明这一问题不仅存在于合成实验中，也存在于真实模型中。
