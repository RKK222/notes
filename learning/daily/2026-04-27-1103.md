#### 1. AgentSearchBench: A Benchmark for AI Agent Search in the Wild
- **地址**：https://arxiv.org/abs/2604.22436
- **来源**：arXiv 2604.22436 / GitHub: https://github.com/Bingo-W/AgentSearchBench
- **领域**：AI Testing / Agent 搜索评估
- **技术介绍**：随着 AI Agent 生态系统快速增长，为给定任务识别合适 Agent 成为新挑战。与传统工具不同，Agent 能力通常是组合性和执行依赖的，难以从文本描述评估。现有研究假设功能明确定义、候选池受控或仅可执行任务查询。AgentSearchBench 从近 10,000 个跨多个提供商的真实 Agent 构建大规模基准，将 Agent 搜索形式化为检索和重排序问题，使用执行感知的性能信号评估相关性。实验揭示语义相似度与实际 Agent 性能之间存在持续差距，暴露了基于描述检索和重排序方法的局限性。轻量级行为信号（包括执行感知探测）可显著提升排名质量。
- **应用场景**：Agent 发现与选择、多 Agent 系统编排、Agent 市场评估
- **为什么值得关注**：首个针对真实世界 Agent 搜索的大规模基准，揭示语义相似度与执行性能的关键差距，为 Agent 评估基础设施提供新范式

#### 2. NL2VC-60: From Natural Language to Verified Code
- **地址**：https://arxiv.org/abs/2604.22601
- **来源**：arXiv 2604.22601 (cs.SE)
- **领域**：AI Testing / 形式化验证
- **技术介绍**：LLM 在自动化软件工程中有前景，但其正确性保证常被错误代码或幻觉代码破坏。为确保模型诚实，形式化验证要求 LLM 合成实现逻辑连同形式规格说明，然后由数学验证器证明正确。但从非正式自然语言到精确形式规格的转换仍然困难。本文提供 NL2VC-60 数据集：60 个复杂算法问题集合。在 11 个随机选择的问题集上评估 7 个开源 LLM，使用分层提示策略：无上下文提示、提供结构锚点的签名提示、利用 Dafny 验证器迭代反馈的自修复提示。为解决空验证问题（模型用琐碎规格满足验证器），集成 uDebug 平台确保功能验证。结果显示无上下文提示导致近乎普遍失败，但结构签名和迭代自修复带来戏剧性性能逆转：Gemma 4-31B 达到 90.91% 验证成功率，GPT-OSS 120B 从零提升到 81.82%。
- **应用场景**：高保证软件开发、形式化验证自动化、安全关键代码生成
- **为什么值得关注**：证明形式化验证对开源 LLM 现已可行，LLM 可作为有效学徒合成复杂注释

#### 3. When Does LLM Self-Correction Help? A Control-Theoretic Markov Diagnostic
- **地址**：https://arxiv.org/abs/2604.22273
- **来源**：arXiv 2604.22273
- **领域**：AI Testing / 自校正诊断
- **技术介绍**：迭代自校正在 Agent LLM 系统中广泛使用，但重复 refinement 何时帮助何时伤害仍不清楚。本文将自校正构建为控制论反馈回路，同一语言模型同时作为控制器和被控对象，使用 {Correct, Incorrect} 上的双状态 Markov 模型操作化简单部署诊断：仅当 ECR/EIR > Acc/(1-Acc) 时迭代。在此视角下，EIR 充当稳定裕度，提示充当轻量级控制器设计。在 7 个模型和 3 个数据集（GSM8K、MATH、StrategyQA）上评估，发现接近零的 EIR 阈值（≤0.5%）清晰分离有益与有害的自校正。仅 o3-mini（+3.4pp，EIR=0%）、Claude Opus 4.6（+0.6pp，EIR~0.2%）和 o4-mini（+/-0pp）保持非退化；GPT-5 退化 -1.8pp。verify-first 提示消融提供因果证据：在 GPT-4o-mini 上将 EIR 从 2% 降至 0%，将 -6.2pp 退化转为 +0.2pp（配对 McNemar p<10^-4）。
- **应用场景**：Agent 自校正策略优化、推理链质量控制、迭代 refinement 部署
- **为什么值得关注**：首次用控制理论框架诊断 LLM 自校正，提供可测量的部署决策准则，而非默认行为

#### 4. Rethinking Publication: A Certification Framework for AI-Enabled Research
- **地址**：https://arxiv.org/abs/2604.22026
- **来源**：arXiv 2604.22026
- **领域**：AI Testing / 研究质量保证
- **技术介绍**：AI 研究流水线现在产生越来越多的可发表学术产出，但出版系统建立在普遍人类作者的假设上，缺乏评估自动化流水线产生知识的 principled 方法。本文提出双层认证框架，将知识质量评估与人类贡献分级分离。框架将贡献分级为：Category A（流水线可达）、Category B（需要人类在可识别阶段的指导）、Category C（超出当前流水线在公式化阶段的触及范围）。引入基准插槽用于完全披露的自动化研究，作为透明发表轨道和评审判断校准工具。干运行验证显示框架可适当认证知识，同时容忍不可归因不确定性。
- **应用场景**：AI 生成研究质量评估、学术出版认证、自动化科学质量保证
- **为什么值得关注**：首次系统解决 AI 生成研究的质量保证问题，为测试 AI 科学产出提供框架

#### 6. Read the Paper, Write the Code: Agentic Reproduction of Social-Science Results
- **地址**：https://arxiv.org/abs/2604.21965
- **来源**：arXiv 2604.21965
- **领域**：AI Testing / 可重复性测试
- **技术介绍**：扩展 Agent 可重复性研究范围：仅给定论文方法描述和原始数据，能否重现结果？开发 Agent 重现系统，从论文提取结构化方法描述，在严格信息隔离下运行重新实现（Agent 从不看原始代码、结果或论文），启用确定性、单元格级比较。错误归因步骤追踪差异通过系统链识别根本原因。在 48 篇论文上评估 4 个 Agent 支架和 4 个 LLM：Agent 可 largely 恢复已发表结果，但性能在模型、支架和论文间差异显著。根本原因分析显示失败既来自 Agent 错误也来自论文本身的不充分说明。

- **为什么值得关注**：严格信息隔离下的可重复性测试，区分 Agent 错误与论文说明不足

#### 7. Rethinking XAI Evaluation: A Human-Centered Audit of Shapley Benchmarks

- **来源**：arXiv 2604.22662
- **领域**：AI Testing / XAI 评估

- **应用场景**：XAI 评估基准、高风险 AI 决策系统测试
- **为什么值得关注**：首次大规模人类中心 XAI 评估，揭示定量指标与人类效用的根本错位

#### 12. Rethinking Math Reasoning Evaluation: LLM-as-a-Judge Beyond Symbolic Rigidity

#### 13. QuantClaw: Precision Where It Matters for OpenClaw
- **地址**：https://arxiv.org/abs/2604.22577
- **来源**：arXiv 2604.22577
- **领域**：AI / Agent 推理优化
- **技术介绍**：自主 Agent 系统因长上下文输入和多轮推理引入显著效率挑战。量化是降低成本和延迟的标准方法，但其对 Agent 性能的影响在真实场景中仍不清楚。本文分析 QuantClaw，一个即插即用的精度路由插件，根据任务特征动态分配精度。轻量级任务路由到低成本配置，同时为高需求工作负载保留更高精度。实验显示在 GLM-5（FP8 基线）上实现高达 21.4% 成本节省和 15.7% 延迟降低。
- **应用场景**：Agent 推理成本优化、精度动态分配
- **为什么值得关注**：首次将精度作为动态资源在 Agent 系统中处理

#### 14. Actively Evaluating Collective Intelligence of Agent Society via Probing Agents
- **地址**：https://arxiv.org/abs/2604.22452
- **来源**：arXiv 2604.22452
- **领域**：AI / 多 Agent 系统
- **技术介绍**：集体智能指群体实现超越任何个体成员的能力。随着 LLM Agent 扩展到数百万人口规模，关键问题：集体智能是否自发从规模中涌现？本文在 MoltBook 平台上进行首次实证评估，该平台托管超过 200 万 Agent。引入 Superminds Test，分层框架使用受控探测 Agent 在三个层级探测社会级智能：联合推理、信息综合和基础交互。实验揭示集体智能的明显缺失：社会在复杂推理任务上无法超越个体前沿模型，很少综合分布式信息，经常连琐碎协调任务都失败。平台级分析显示交互仍然浅层，线程很少超过单次回复，大多数响应是通用或离题的。
- **应用场景**：多 Agent 社会评估、集体智能研究
- **为什么值得关注**：首个 200 万 Agent 规模的集体智能实证研究，证伪"规模=集体智能"假设

#### 15. From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company
- **地址**：https://arxiv.org/abs/2604.22446
- **来源**：arXiv 2604.22446
- **领域**：AI / 多 Agent 组织
- **技术介绍**：个体 Agent 能力通过模块化技能和工具集成快速发展，但多 Agent 系统仍受限于固定团队结构、紧密耦合协调逻辑和会话绑定学习。本文引入 OneManCompany (OMC)，将多 Agent 系统提升到组织层面。OMC 将技能、工具和运行时配置封装为可移植 Agent 身份（称为 Talents），通过类型化组织接口编排，抽象异构后端。社区驱动的 Talent Market 实现按需招聘，使组织在执期间动态关闭能力差距和重新配置。组织决策通过 Explore-Execute-Review (E²R) 树搜索操作化，在单个分层循环中统一规划、执行和评估。在 PRDBench 上 OMC 实现 84.67% 成功率，超越 SOTA 15.48 个百分点。
- **应用场景**：多 Agent 组织编排、动态团队构建、企业级 Agent 系统
- **为什么值得关注**：将多 Agent 系统从静态预配置管道转变为自组织和自改进的 AI 组织

#### 16. Introducing Background Temperature to Characterise Hidden Randomness in LLMs
- **地址**：https://arxiv.org/abs/2604.22411

#### 1. Actively Evaluating Collective Intelligence of Agent Society via Probing Agents
- **地址**：https://arxiv.org/abs/2604.22452
- **论文介绍（200-400 字）**：本文在 MoltBook 平台（托管超过 200 万 Agent）上进行首次大规模集体智能实证评估。引入 Superminds Test，分层框架使用受控探测 Agent 在三个层级探测社会级智能：联合推理、信息综合和基础交互。实验揭示集体智能的明显缺失：社会在复杂推理任务上无法超越个体前沿模型，很少综合分布式信息，经常连琐碎协调任务都失败。平台级分析显示交互仍然浅层，线程很少超过单次回复，大多数响应是通用或离题的。这些结果表明集体智能不会仅从规模中涌现，当前 Agent 社会的主要限制是极其稀疏和浅层的交互，阻止 Agent 交换信息和构建彼此输出。
- **技术架构**：探测 Agent 架构——注入受控 Agent 到真实 Agent 社会，观察社会响应。三层评估：(1) 联合推理——多 Agent 协作解决复杂问题；(2) 信息综合——分布式信息整合能力；(3) 基础交互——基本协调任务。
- **核心算法或系统设计**：探测 Agent 设计是关键创新——需要既融入社会又不被识别为外来者。评估指标包括任务完成率、信息传递保真度、协调效率。
- **关键实现机制**：大规模平台（200 万 Agent）提供真实社会环境，探测 Agent 提供受控实验条件。
- **与现有技术对比**：相比之前的多 Agent 模拟环境（如 SWE-bench、GAIA），本文在真实大规模社会平台上评估，生态效度更高。
- **优势与局限**：优势：真实大规模数据，证伪"规模=集体智能"的流行假设。局限：MoltBook 平台特性可能影响泛化性，探测方法可能干扰自然行为。
- **未来发展潜力**：为多 Agent 系统设计提供关键洞察——需要设计促进深度交互的机制，而非简单扩大规模。

#### 2. When Does LLM Self-Correction Help? A Control-Theoretic Markov Diagnostic
- **地址**：https://arxiv.org/abs/2604.22273
- **论文介绍（200-400 字）**：本文首次用控制理论框架诊断 LLM 自校正行为。将自校正构建为控制论反馈回路，同一语言模型同时作为控制器和被控对象。使用 {Correct, Incorrect} 上的双状态 Markov 模型操作化部署诊断：仅当 ECR/EIR > Acc/(1-Acc) 时迭代。在 7 个模型和 3 个数据集上评估，发现接近零的 EIR 阈值（≤0.5%）清晰分离有益与有害的自校正。仅 o3-mini、Claude Opus 4.6 和 o4-mini 保持非退化；GPT-5 退化 -1.8pp。verify-first 提示消融提供因果证据：在 GPT-4o-mini 上将 EIR 从 2% 降至 0%，将 -6.2pp 退化转为 +0.2pp。
- **技术架构**：控制论框架——将 LLM 自校正建模为反馈控制系统。控制器（LLM 评估自身输出）+ 被控对象（LLM 生成输出）。EIR（错误引入率）充当稳定裕度。
- **核心算法或系统设计**：双状态 Markov 模型 + ECR/EIR 比率诊断。verify-first 提示作为轻量级控制器设计。
- **关键实现机制**：通过提示工程调整 EIR，证明自校正行为可通过设计控制。
- **与现有技术对比**：相比之前的自校正研究（多为经验性），本文提供理论框架和可测量准则。
- **优势与局限**：优势：理论严谨，提供可操作的部署决策准则。局限：仅评估数学和策略任务，未覆盖编码和开放域任务。
- **未来发展潜力**：为 Agent 自校正策略提供理论基础，可能影响 Agent 架构设计。

#### 3. From Natural Language to Verified Code: NL2VC-60
- **地址**：https://arxiv.org/abs/2604.22601
- **论文介绍（200-400 字）**：本文解决 LLM 生成代码的正确性保证问题。提供 NL2VC-60 数据集（60 个复杂算法问题），在 7 个开源 LLM 上评估分层提示策略：无上下文提示、签名提示（结构锚点）、自修复提示（Dafny 验证器迭代反馈）。为解决空验证问题，集成 uDebug 平台确保功能验证。结果显示无上下文提示导致近乎普遍失败，但结构签名和迭代自修复带来戏剧性性能逆转：Gemma 4-31B 达到 90.91% 验证成功率，GPT-OSS 120B 从零提升到 81.82%。
- **技术架构**：三层提示策略 + Dafny 形式化验证器 + uDebug 功能验证。迭代反馈循环：LLM 生成代码和规格 → Dafny 验证 → 反馈错误信息 → LLM 修复。
- **核心算法或系统设计**：自修复机制是关键——验证器反馈指导 LLM 迭代改进，而非一次性生成。
- **关键实现机制**：Dafny 验证器提供形式化正确性保证，uDebug 防止空验证。
- **与现有技术对比**：相比之前的代码生成评估（仅运行测试），本文提供形式化正确性保证。相比纯形式化方法，本文利用 LLM 的生成能力。
- **优势与局限**：优势：证明形式化验证对开源 LLM 现已可行。局限：仅评估算法问题，未覆盖工程级代码。
- **未来发展潜力**：为高保证软件开发提供新范式，LLM + 形式化验证可能成为安全关键代码的标准方法。

#### 4. From Skills to Talent: OneManCompany (OMC)
- **地址**：https://arxiv.org/abs/2604.22446
- **论文介绍（200-400 字）**：本文提出 OneManCompany (OMC)，将多 Agent 系统提升到组织层面。OMC 将技能、工具和运行时配置封装为可移植 Agent 身份（Talents），通过类型化组织接口编排。社区驱动的 Talent Market 实现按需招聘，使组织在执期间动态关闭能力差距。组织决策通过 Explore-Execute-Review (E²R) 树搜索操作化，在单个分层循环中统一规划、执行和评估，提供终止和无死锁的形式保证。在 PRDBench 上 OMC 实现 84.67% 成功率，超越 SOTA 15.48 个百分点。
- **技术架构**：三层架构——(1) Talent 层：可移植 Agent 身份封装技能和工具；(2) 组织接口层：类型化接口抽象异构后端；(3) E²R 决策层：分层树搜索统一规划、执行和评估。
- **核心算法或系统设计**：E²R 树搜索是核心——探索（任务分解）→ 执行（Talent 分配）→ 审查（结果聚合和反馈），提供形式化终止保证。
- **关键实现机制**：Talent Market 实现动态能力匹配，E²R 提供决策闭环。
- **与现有技术对比**：相比之前的多 Agent 框架（固定团队结构），OMC 实现动态组织重构。相比 AutoGen（对话式编排），OMC 提供形式化保证。
- **优势与局限**：优势：形式化保证、动态重构、超越 SOTA 15.48pp。局限：33 页 13 图的复杂框架，实现成本高。
- **未来发展潜力**：为多 Agent 系统提供组织级抽象，可能成为企业级 Agent 系统的标准架构。

#### 5. Introducing Background Temperature: Hidden Randomness in LLMs
- **地址**：https://arxiv.org/abs/2604.22411
- **论文介绍（200-400 字）**：本文正式化 LLM 的隐式随机性问题。即使在使用温度 T=0 解码时，LLM 也可能对相同输入产生不同输出。Thinking Machines Lab 最近的工作突出实现级别的不确定性来源（batch-size 变化、kernel 非不变性、浮点非结合性）。本文引入背景温度 T_bg 概念——由实现依赖的扰动过程引起的有效温度。提供清晰定义，展示 T_bg 如何与推理环境 I 控制的随机扰动相关，并提出通过理想参考系统的等效温度 T_n(I) 估计 T_bg 的经验协议。在主要 LLM 提供商的代表性池上运行试点实验。发表于 TMLR 2026。
- **技术架构**：理论框架——将实现级扰动建模为随机过程，定义背景温度 T_bg 为等效热力学温度。经验协议——通过参考系统估计 T_bg。
- **核心算法或系统设计**：等效温度估计协议——在参考系统上运行相同输入，找到产生相似输出分布的温度值。
- **关键实现机制**：将工程问题（非确定性）转化为可测量的理论概念（背景温度）。
- **与现有技术对比**：相比之前的非确定性研究（描述性），本文提供正式化理论和可测量协议。
- **优势与局限**：优势：理论创新，为可重复性提供基础。局限：试点实验规模有限，需要更大规模验证。
- **未来发展潜力**：可能影响 LLM 评估基准设计、部署确定性保障和监管合规。
