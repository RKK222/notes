#### 3. Dirac: OSS Agent Topped TerminalBench on Gemini-3-flash-preview
- **地址**：https://github.com/dirac-run/dirac
- **来源**：Hacker News #25（293 分，118 评论）
- **领域**：AI Testing / 终端代理 / 编码代理
- **技术介绍**：Dirac 是一个开源终端代理，在 TerminalBench 上以 Gemini-3-flash-preview 达到 SOTA 性能。核心技术包括：(1) Hash-Anchored edits — 优化的文件编辑算法，结合 Myers diff 实现单 token 精度编辑；(2) AST 驱动上下文管理 — 利用语言抽象语法树决定上下文获取，完全避免大文件读取；(3) 批量操作 — 同时执行大量读/编辑操作；(4) 动态代码执行 — 模型可编写 bash/python/perl 脚本完成任务；(5) 机会主义上下文更新 — 预判模型下一步需要的信息。HN 讨论特别关注 AST 在代码审查中的应用潜力。
- **应用场景**：终端自动化、编码代理上下文管理、文件编辑优化
- **为什么值得关注**：AST 驱动的上下文管理是编码代理效率突破的关键方向，Token 消耗大幅降低

#### 4. QuantClaw: Precision Where It Matters for Agent Systems
- **地址**：https://arxiv.org/abs/2604.22577
- **来源**：arXiv 2604.22577
- **领域**：AI Testing / 代理系统量化
- **技术介绍**：QuantClaw 是一个即插即用的精度路由插件，根据任务特征动态分配精度。轻量任务路由到低成本配置，复杂任务保持高精度。在 GLM-5 (FP8 基线) 上实现成本降低 21.4%、延迟降低 15.7%，同时保持或提升任务性能。核心洞察：代理系统中精度需求高度任务依赖，将精度视为动态资源而非固定配置。
- **应用场景**：代理系统成本优化、动态精度路由、FP8/混合精度推理
- **为什么值得关注**：将量化从静态配置变为动态资源分配，对 AI 编码代理经济性有直接影响

#### 8. CRAFT: Clustered Regression for Adaptive Filtering of Training Data
- **地址**：https://arxiv.org/abs/2604.22693
- **来源**：arXiv 2604.22565 (cs.CL)
- **领域**：AI Testing / 训练数据筛选
- **技术介绍**：CRAFT 是一种向量无关的训练数据选择方法，用于从大规模语料库中选择高质量子集进行微调。两阶段选择：(1) 通过 k-means 聚类按比例分配预算匹配验证源分布；(2) 在每个源聚类内选择目标嵌入最小化条件期望距离的训练对。在 3300 万 NLLB 句子对上选择并微调 mBART via LoRA，达到 43.34 BLEU，超越 TSDS 2.13 分，选择速度快 40 倍。CPU 上 TF-IDF 向量化整个管道 <1 分钟。
- **应用场景**：AI 编码代理训练数据优化、高效微调
- **为什么值得关注**：为 AI 编码代理的训练数据筛选提供了高效方法

#### 12. UAE: Aligning Dense Retrievers with LLM Utility via Distillation
- **地址**：https://arxiv.org/abs/2604.22722
- **来源**：arXiv 2604.22722 (cs.IR)
- **领域**：AI / RAG / 检索
- **技术介绍**：UAE (Utility-Aligned Embeddings) 将检索公式化为分布匹配问题，训练 bi-encoder 模仿从 perplexity 减少导出的效用分布，使用 Utility-Modulated InfoNCE 目标。在 QASPER 基准上，Recall@1 提升 30.59%、MAP 提升 30.16%、Token F1 提升 17.3%（对比 BGE-Base）。比高效 LLM 重排序方法快 180 倍，同时保持竞争性能。
- **应用场景**：RAG 系统检索优化、Agent 记忆检索
- **为什么值得关注**：将 LLM 效用信号直接注入嵌入空间，无需测试时 LLM 推理

### GitHub 项目

#### 1. Dirac: OSS Agent Topped TerminalBench
**地址**：https://github.com/dirac-run/dirac

**项目介绍（200-400 字）**：
Dirac 是一个开源终端代理，在 TerminalBench 上以 Gemini-3-flash-preview 达到 SOTA 性能。其核心技术创新在于彻底重构了编码代理的上下文管理和文件编辑范式。传统编码代理（如 Claude Code、Cursor）依赖全文读取和正则匹配编辑，而 Dirac 采用 AST（抽象语法树）驱动的方法——利用编程语言的语法结构来决定哪些代码片段需要放入上下文，完全避免了大文件读取的开销。在文件编辑方面，Dirac 实现了 Hash-Anchored edits 算法，结合 Myers diff 实现单 token 精度的文件编辑，大幅减少编辑操作的 token 消耗。批量操作机制允许模型同时发出多个读/编辑请求，而非传统的一问一答模式。此外，Dirac 允许模型在运行时编写和执行 bash/python/perl 脚本，将复杂任务分解为脚本执行而非逐行编码。机会主义上下文更新机制预判模型下一步可能需要的信息，提前放入上下文。这些创新的组合使 Dirac 在 TerminalBench 上超越了多个闭源代理。

**技术架构**：
- AST 上下文管理器：解析代码文件语法树，按需提取相关代码片段
- Hash-Anchored 编辑器：Myers diff + 哈希锚点，单 token 精度编辑
- 批量操作引擎：并发执行多个读/编辑请求
- 动态脚本执行器：模型运行时编写和执行脚本
- 机会主义上下文缓存：预判并预加载可能需要的上下文

**核心算法**：
- Hash-Anchored edits：通过哈希锚点定位编辑位置，结合 Myers diff 计算最小编辑集
- AST 上下文选择：基于语法树节点依赖关系选择上下文片段

**与现有技术对比**：
- vs Claude Code：Dirac 的 AST 驱动上下文管理避免了全文读取，token 消耗显著降低
- vs 8v：Dirac 专注于终端代理场景，8v 专注于跨代理工具抽象
- vs SWE-agent：Dirac 的批量操作和动态脚本执行提供了更高的效率

**优势与局限**：
- 优势：AST 驱动方法在大型代码库中 token 效率极高；批量操作减少 API 调用次数
- 局限：AST 解析需要语言特定实现；对非结构化文本文件支持有限

**未来发展潜力**：
AST 驱动的上下文管理是编码代理效率突破的关键方向。如果这一方法能扩展到更多编程语言，将显著降低 AI 编码代理的运行成本。

**个人技术理解**：
Dirac 的核心洞察是"不要读取整个文件，只读取模型需要的部分"。这一看似简单的原则在 AST 的支撑下变得可行。与 8v 的工具抽象思路不同，Dirac 选择了上下文管理的优化路径——两条路线互补，共同推动 AI 编码代理经济性。

#### 2. QuantClaw: Dynamic Precision Routing for Agent Systems
**地址**：https://arxiv.org/abs/2604.22577

**项目介绍**：
QuantClaw 是一个即插即用的精度路由插件，用于自主代理系统（以 OpenClaw 为案例研究）。传统量化方案为整个系统选择单一精度配置，而 QuantClaw 的核心创新是将精度视为动态资源——根据任务特征实时分配不同精度级别。轻量任务（如简单文本处理、格式转换）路由到低成本配置（如 INT4），复杂任务（如逻辑推理、代码生成）保持高精度（如 FP16/FP8）。在 GLM-5 (FP8 基线) 上实验显示：成本降低 21.4%、延迟降低 15.7%，同时保持或提升任务性能。这一方法挑战了"代理系统需要统一高精度"的假设，证明精度需求高度任务依赖。

**技术架构**：
- 任务分类器：实时分析任务复杂度和精度需求
- 精度路由器：根据分类结果分配最优精度配置
- 性能监控器：跟踪路由决策的实际效果并持续优化

**核心算法**：
- 动态精度分配：基于任务特征的实时精度选择
- 成本-性能权衡：在精度和成本之间寻找最优平衡点

**与现有技术对比**：
- vs TurboQuant（昨日覆盖）：TurboQuant 是量化原理教程，QuantClaw 是运行时精度路由系统
- vs 静态量化：QuantClaw 将量化从静态配置变为动态资源分配

**优势与局限**：
- 优势：显著降低代理系统运行成本；即插即用，无需修改代理逻辑
- 局限：任务分类器的准确性直接影响路由效果；对混合精度任务的分类可能不准确

**未来发展潜力**：
动态精度路由可能成为代理系统的标准组件。随着更多代理框架支持混合精度推理，这一方法的应用范围将不断扩大。

**个人技术理解**：
QuantClaw 的核心洞察与 8v 的工具抽象殊途同归——都在追求代理系统的经济性。8v 通过减少工具调用的 token 消耗，QuantClaw 通过动态精度分配降低推理成本。两者结合可能实现代理系统成本的量级降低。

#### 1. Sound Agentic Science Requires Adversarial Experiments
**地址**：https://arxiv.org/abs/2604.22080
**会议**：ICLR 2026 Workshop on Agents in the Wild

**论文介绍**：
LLM 代理正在被快速采用用于科学数据分析，但这同时加速了一个熟悉的失败模式——快速生产可信、可无限修订的分析。假设空间变成候选声明，通过选择性分析优化为可发表的正结果。与软件不同，科学知识不是通过代码的迭代积累和事后统计支持来验证的。流畅的解释或单个数据集上的显著结果不是验证。缺失的证据是负空间——本可以证伪该声明的实验从未运行或从未发表。论文提出"证伪优先"标准：代理不应主要用于构建最有说服力的叙述，而应主动搜索声明可能失败的方式。

**技术架构**：
- 证伪优先框架：代理首先搜索证伪路径而非证实路径
- 负空间分析：系统性地探索未运行的实验和未发表的负面结果
- 对抗性实验设计：为每个正面声明设计对应的对抗性实验

**核心算法**：
- 证伪搜索：从声明出发搜索可能的失败路径
- 负空间枚举：系统生成可能证伪声明的实验设计

**与现有技术对比**：
- vs 传统科学方法：传统方法依赖人类研究者设计实验，代理方法可以自动化证伪搜索
- vs DryRUN（昨日覆盖）：DryRUN 消除公共测试依赖，本文消除证实偏见

**优势与局限**：
- 优势：为 AI 辅助科学研究提供了严谨的方法论框架
- 局限：证伪搜索的计算成本可能高于证实搜索

**未来发展潜力**：
这一方法论可能成为 AI 辅助科学研究的标配——从"验证正确"到"主动证伪"的范式转变。

**个人技术理解**：
这篇论文的核心价值不在于技术方案，而在于方法论反思。它揭示了 AI 代理在科学研究中的一个根本性风险：代理可以无限生成"看起来正确"的分析，但这些分析可能建立在选择性证据之上。"证伪优先"原则对 AI 编码代理测试同样适用——测试不应只验证"能工作"，更应主动搜索"何时失败"。

**地址**：https://arxiv.org/abs/2604.22722

**论文介绍**：
密集向量检索是 RAG 的实际 backbone，但相似度搜索可能遭受精度限制。相反，利用 LLM 重排序的效用导向方法通常实现更优性能但计算上 prohibitive 且容易受到 perplexity 估计的噪声影响。UAE 将检索公式化为分布匹配问题，训练 bi-encoder 模仿从 perplexity 减少导出的效用分布，使用 Utility-Modulated InfoNCE 目标。这一方法将分级效用信号直接注入嵌入空间，无需测试时 LLM 推理。在 QASPER 基准上，Recall@1 提升 30.59%、MAP 提升 30.16%、Token F1 提升 17.3%（对比 BGE-Base）。比高效 LLM 重排序方法快 180 倍。

**技术架构**：
- Utility-Modulated InfoNCE：将 LLM perplexity 减少作为效用信号
- Bi-encoder 训练：模仿 LLM 重排序的效用分布
- 无测试时 LLM 推理：训练后检索无需 LLM

**核心算法**：
- 效用分布蒸馏：从 LLM perplexity 减少推导效用分布
- 分布匹配训练：bi-encoder 输出匹配效用分布

**与现有技术对比**：
- vs BGE-Base：Recall@1 +30.59%，显著提升
- vs LLM 重排序：快 180 倍，性能竞争

**优势与局限**：
- 优势：无需测试时 LLM 推理，检索速度极快
- 局限：需要 LLM 生成训练信号，训练阶段仍有 LLM 成本

**未来发展潜力**：
这一方法可能成为 RAG 系统的标准检索方案——在检索精度和速度之间找到最优平衡。

**个人技术理解**：
UAE 的核心创新是将 LLM 的"理解能力"蒸馏到轻量级 bi-encoder 中。这类似于 QuantClaw 的思路——将高成本能力迁移到低成本组件。对 AI 编码代理的意义：Agent 记忆检索可以受益于这一方法，在保持检索质量的同时大幅降低推理成本。

## 新领域收获

### 1. 生物识别数据安全（Mercor 语音泄露事件）
4TB 语音样本从 40,000 名 AI 承包商处泄露，揭示生物识别数据的核心脆弱性——"你不能轮换你的声音"。这对 AI 训练数据供应链安全有深远影响：生物识别数据的收集、存储和使用需要全新的安全框架。

### 2. XAI 评估中定量指标与人类效用的错位
Shapley 基准的人类中心审计揭示：标准定量指标（稀疏性、忠实度）与人类感知清晰度和决策效用脱钩。解释一致增加决策信心但不改善客观性能——在高 stakes 设置中发出自动化偏见风险信号。这对 AI 编码代理的可解释性评估有直接启示。

### 3. 质量多样性算法在机器人操作中的应用
QDTraj 使用质量多样性算法生成多样化机器人轨迹基元，每任务 704 条不同轨迹。这一方法可能迁移到 AI 编码代理的测试场景生成——通过质量多样性算法生成多样化的测试用例。
