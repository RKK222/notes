#### 1. AutoMat: Can Coding Agents Reproduce Findings in Computational Materials Science?
- **地址**：https://arxiv.org/abs/2605.00803
- **来源**：arXiv (cs.SE → cs.AI cross-list)
- **领域**：AI Testing / 科学编码代理
- **技术介绍**：LLM 编码代理在软件工程基准上表现强劲，但能否迁移到计算科学工作流尚不清楚。AutoMat 基准评估 LLM 代理复现计算材料科学论文声称的能力，提出三个挑战：恢复欠指定计算流程、导航领域特定程序、在科学主张上下文中解释结果。该基准首次系统性地测试编码代理在科学领域的泛化能力，揭示软件工程成功 ≠ 科学工作流成功。
- **应用场景**：科学计算自动化、研究可复现性
- **为什么值得关注**：挑战"编码代理万能"假设，科学工作流的领域知识门槛远高于普通软件工程

#### 2. CLOZEMASTER: Fuzzing Rust Compiler by Harnessing LLMs for Infilling Masked Real Programs
- **地址**：https://arxiv.org/abs/2605.00413
- **来源**：arXiv (cs.SE) / ICSE 2025
- **领域**：AI Testing / 编译器模糊测试
- **技术介绍**：Rust 编译器测试因复杂语法和严格类型系统而困难。CLOZEMASTER 提出 clozeMask 策略：从历史 issue 报告提取测试代码，识别并掩码特定结构代码片段，用 LLM 填充掩码部分合成新测试程序。结合历史 bug 触发能力和 LLM 生成能力。已发现 27 个 rustc/mrustc 确认 Bug（10 个已修复），在代码覆盖率和有效性上超越现有模糊测试器。
- **应用场景**：Rust 编译器测试、类型安全语言模糊测试
- **为什么值得关注**：ICSE 2025 接受，历史 bug 引导 + LLM 填充的混合策略在编译器测试中展现强大效果

#### 3. SocialBias-Bench + Fairness Monitor Agent (FMA)
- **地址**：https://arxiv.org/abs/2605.00382
- **来源**：arXiv (cs.SE)
- **领域**：AI Testing / 代码生成公平性
- **技术介绍**：LLM 生成代码的社会偏见长期被忽视。SocialBias-Bench 包含 343 个真实编码任务覆盖 7 个人口维度，发现所有模型 Code Bias Score 高达 60.58%。标准 prompt 干预（CoT、fairness persona）反而放大偏见。FMA 作为模块化组件插入现有管道，分析任务描述确定哪些属性应被考虑或限制，通过迭代审查检测和纠正违规，无需可执行测试套件。降低偏见 65.1%，功能正确性从 75.80% 提升至 83.97%。
- **应用场景**：代码生成公平性保障、AI 辅助开发伦理
- **为什么值得关注**：首次系统量化 LLM 代码生成的社会偏见，FMA 无需修改现有管道即可显著改善

#### 4. RECRL: Requirement-Aware Curriculum Reinforcement Learning for Code Generation
- **地址**：https://arxiv.org/abs/2605.00433
- **来源**：arXiv (cs.SE)
- **领域**：AI Testing / 代码生成 RL
- **技术介绍**：现有课程 RL 方法存在需求难度感知不对齐、需求难度优化缺失、课程采样次优等问题。RECRL 从软件工程需求工程角度，自动感知模型特定需求难度，优化挑战性需求提高训练数据利用率，采用自适应课程采样构建难度平滑变化的训练批次。在 5 个 SOTA LLM 和 5 个基准上，Pass@1 平均提升 1.23%-5.62%。
- **应用场景**：LLM 代码生成后训练、代码生成质量优化
- **为什么值得关注**：将需求工程思想引入 RL 课程学习，需求难度感知是关键创新

#### 5. GeoContra: From Fluent GIS Code to Verifiable Spatial Analysis
- **地址**：https://arxiv.org/abs/2605.00782
- **来源**：arXiv (cs.SE)
- **领域**：AI Testing / GIS 代码验证
- **技术介绍**：LLM 生成 GIS 脚本流畅但很少强制执行地理规则。GeoContra 将每个任务表示为可执行地理空间合约（自然语言问题、schema、CRS 元数据、期望输出、空间谓词、拓扑、指标），生成程序经过静态规则检查、运行时验证和语义验证，违规反馈到有界修复循环。在 7,079 个真实地理空间任务上，DeepSeek-V4 从 47.6% 提升到 77.5%，11 个开源模型平均提升 26.6%。
- **应用场景**：GIS 代码生成质量保障、空间分析自动化
- **为什么值得关注**：将合约式设计引入地理空间代码生成，捕获负旅行时间、CRS 违规等隐蔽错误

#### 6. EnCoDe: Energy Estimation of Source Code At Design-Time
- **地址**：https://arxiv.org/abs/2605.00504
- **来源**：arXiv (cs.SE) / EASE 2026
- **领域**：AI Testing / 能源效率测试
- **技术介绍**：现有 profiling 工具仅在运行时工作且粒度粗糙。EnCoDe 提出 PowerLens 测量方法实现亚毫秒级代码块能量读数，对 18,000+ Python 程序的代码块进行实证研究，发现能量消耗与静态代码特征（结构、复杂度、密度、上下文）之间的线性和非线性关系。ML 模型实现 R²=0.75 回归和 80.6% 分类准确率识别能量热点。
- **应用场景**：绿色软件工程、设计时能源优化
- **为什么值得关注**：EASE 2026 接受，首次实现设计时细粒度能量估计，无需执行即可定位低效代码

#### 7. Q-ARE: Query Based API Recommendation Evaluation Dataset
- **地址**：https://arxiv.org/abs/2605.00472
- **来源**：arXiv (cs.SE) / EQUISA 2026
- **领域**：AI Testing / API 推荐评估
- **技术介绍**：构建基于 GitHub 开源 Java 项目的 Q-ARE 数据集，递归展开多级调用将层次调用结构统一为 API 推荐目标集。引入两个新指标：API Call Depth（查询方法与目标 API 的调用距离）和 Invocation Density（调用链中与目标 API 相关的代码行比例）。评估发现性能随 API Call Depth 增加和调用密度降低显著下降。
- **应用场景**：API 推荐系统评估、代码补全质量
- **为什么值得关注**：EQUISA 2026 接受，新指标揭示现有方法在多级调用结构上的根本局限

#### 8. Issue-Commit Linking with LLM-Assisted Retrieval

- **来源**：arXiv (cs.SE)

#### 13. TUR-DPO: Topology- and Uncertainty-Aware DPO
- **地址**：https://arxiv.org/abs/2605.00224
- **来源**：arXiv (cs.AI) / ICML 2026
- **领域**：AI / 对齐优化
- **技术介绍**：DPO 将偏好视为扁平的赢家 vs 输家信号，对脆弱思维链产生的噪声偏好敏感。TUR-DPO 通过轻量推理拓扑 elicitation 和语义忠实度、效用、拓扑质量的不确定性校准信号，将"答案如何推导"纳入对齐目标。可学习奖励分解为这些信号，纳入不确定性加权 DPO 目标，保持 RL-free。在 7-8B 模型上提升 judge win-rate、忠实度和校准。
- **应用场景**：LLM 对齐、推理质量优化
- **为什么值得关注**：ICML 2026 录用，首次将推理路径拓扑纳入 DPO 目标，不只是"说什么"而是"怎么说"

#### 14. LOCA: Local Causal Explanations for Jailbreak Success
- **地址**：https://arxiv.org/abs/2605.00123
- **来源**：arXiv (cs.AI)
- **领域**：AI / 安全可解释性
- **技术介绍**：现有 jailbreak 研究全局解释所有攻击为减少有害性或增强拒绝概念，但不同策略可能通过不同中间概念成功。LOCA 给出局部因果解释——为什么这个特定 jailbreak 成功？通过识别最小可解释中间表示变化集来因果诱导模型拒绝。在 Gemma 和 Llama 上平均仅需 6 个变化诱导拒绝（prior work 20+ 变化仍失败）。
- **应用场景**：LLM 安全分析、jailbreak 防御
- **为什么值得关注**：从全局解释转向局部解释，机械可解释安全分析的重要一步

#### 15. GUI-SD: On-Policy Self-Distillation for GUI Grounding
- **地址**：https://arxiv.org/abs/2605.00642
- **来源**：arXiv (cs.AI)
- **领域**：AI / GUI Agent
- **技术介绍**：GUI grounding 将自然语言指令映射到目标元素视觉坐标。现有 GRPO 方法依赖昂贵多次 rollout 且在困难样本上信号稀疏。GUI-SD 是首个针对 GUI grounding 的 OPSD 框架：为教师构建视觉增强特权上下文（目标边界框 + 高斯软掩码），采用熵引导蒸馏根据数字显著性和教师信心自适应加权 token。在 6 个基准上持续超越 GRPO。
- **应用场景**：GUI Agent 训练、桌面自动化
- **为什么值得关注**：单 rollout 密集监督替代多 rollout，大幅降低 GUI Agent 训练成本

#### 16. AEM: Adaptive Entropy Modulation for Multi-Turn Agentic RL
- **地址**：https://arxiv.org/abs/2605.00425
- **来源**：arXiv (cs.AI)
- **领域**：AI / Agent RL
- **技术介绍**：多轮 Agent RL 中稀疏结果奖励难以分配单步信用。AEM 是无监督信用分配方法，在 RL 训练中自适应调制熵动力学实现更有效探索-利用权衡。理论上将熵分析从 token 级别提升到响应级别，证明自然梯度下熵漂移由优势函数和相对响应惊奇度的乘积内在控制。导出实用代理重塑训练动力学。
- **应用场景**：多轮 Agent 训练、RL 探索策略
- **为什么值得关注**：无监督信用分配，无需过程奖励模型或辅助自监督信号

#### 17. TokenArena: Continuous Benchmark Unifying Energy and Cognition
- **地址**：https://arxiv.org/abs/2605.00300
- **来源**：arXiv (cs.AI)
- **领域**：AI / 推理基准
- **技术介绍**：现有推理基准在模型和提供商级别比较，但实际部署决策在端点级别做出。TokenArena 在端点粒度测量推理（输出速度、TTFT、工作负载混合价格、有效上下文、端点质量），合成三个综合指标：每正确回答焦耳数、每正确回答美元数、端点保真度。78 个端点 12 个模型家族，同模型在不同端点准确率差异达 12.5 分，能耗差异 6.2 倍。
- **应用场景**：AI 推理基础设施选型、能耗优化
- **为什么值得关注**：端点粒度基准揭示部署决策的关键维度，能耗/价格/质量三位一体

#### 18. IVLR: Interleaved Vision-Language Reasoning for Robot Manipulation
- **地址**：https://arxiv.org/abs/2605.00438
- **来源**：arXiv (cs.AI)
- **领域**：AI / 机器人
- **技术介绍**：长程机器人操作需要逻辑连贯且几何 grounded 的计划。IVLR 框架围绕 trace 表示构建，交替文本子目标和视觉关键帧覆盖完整任务地平线。测试时单个原生多模态 transformer 自生成全局语义-几何 trace，缓存并条件化闭环动作解码器。LIBERO 平均成功率 95.5%（LIBERO-Long 92.4%），消融显示两种模态缺一不可。
- **应用场景**：长程机器人操作、具身智能
- **为什么值得关注**：文本+视觉交错推理 trace 显著提升长程操作成功率

#### 19. Hamiltonian World Models
- **地址**：https://arxiv.org/abs/2605.00412
- **来源**：arXiv (cs.AI)
- **领域**：AI / 世界模型

### 学术论文

#### 4. TUR-DPO: 拓扑+不确定性感知 DPO (ICML 2026)

**地址**：https://arxiv.org/abs/2605.00224

**论文介绍**：TUR-DPO 是 ICML 2026 录用的对齐优化方法。传统 DPO 将偏好视为扁平的赢家 vs 输家信号，对脆弱思维链产生的噪声偏好敏感。TUR-DPO 通过 eliciting 轻量推理拓扑，结合语义忠实度、效用和拓扑质量生成校准的不确定性信号。可学习奖励分解为这些信号，纳入不确定性加权 DPO 目标。保持 RL-free 和训练简单性，在 7-8B 模型上提升 judge win-rate、忠实度和校准。

**技术架构**：(1) 推理拓扑 elicitation——提取推理步骤的结构关系；(2) 多信号融合——语义忠实度 + 效用 + 拓扑质量；(3) 不确定性校准——生成加权信号；(4) 不确定性加权 DPO 目标。

**核心算法**：不确定性信号 u = f(semantic_faithfulness, utility, topology_quality)，奖励 r = g(u) 分解为多信号，DPO 目标加权为 L_DPO weighted = σ(r_w - r_l) / u。

**与现有技术对比**：相比标准 DPO（仅考虑赢家/输家），TUR-DPO 考虑推理路径质量；相比 PPO（在线 rollout），TUR-DPO 保持离线简单性。

**优势与局限**：优势是 RL-free 且考虑推理质量；局限是拓扑 elicitation 增加额外计算。

**个人理解**：TUR-DPO 代表了对齐优化的新方向——从"说什么"到"怎么说"。推理路径质量纳入对齐目标，有助于训练更可靠的推理模型。

#### 5. Hamiltonian World Models: 物理原生世界模型

**地址**：https://arxiv.org/abs/2605.00412

**论文介绍**：当前世界模型研究被 2D 视频生成、3D 场景重建、JEPA 潜在模型三条路线分割，难以提供物理可靠、动作可控、长程稳定的预测。论文提出 Hamiltonian World Models 物理 grounded 视角：将观测编码为结构化潜在相空间，通过 Hamiltonian 启发动力学（控制项、耗散项、残差项）演化潜在状态，解码预测轨迹用于规划。讨论 Hamiltonian 结构如何改善可解释性、数据效率和长程稳定性。

**技术架构**：(1) 观测 → 潜在相空间编码；(2) Hamiltonian 动力学演化（H(q,p) + 控制 + 耗散 + 残差）；(3) 潜在轨迹 → 未来观测解码；(4) 轨迹用于规划。

**核心创新**：首次将 Hamiltonian 力学框架引入世界模型，利用物理守恒律约束预测，解决长程预测漂移问题。

**优势与局限**：优势是物理 grounded 提供长程稳定性；局限是实际机器人场景中的摩擦、接触、非保守力和可变形物体处理困难。

**个人理解**：Hamiltonian World Models 代表了世界模型从"数据驱动生成"向"物理约束预测"的范式转变。物理守恒律嵌入是解决长程预测稳定性的关键方向。

#### 6. LOCA: 局部因果 Jailbreak 解释

**地址**：https://arxiv.org/abs/2605.00123

**论文介绍**：现有 jailbreak 研究全局解释所有攻击为减少有害性或增强拒绝概念，但不同策略可能通过不同中间概念成功。LOCA 给出局部因果解释——为什么这个特定 jailbreak 成功？通过识别最小可解释中间表示变化集来因果诱导模型拒绝。在 Gemma 和 Llama 上平均仅需 6 个变化诱导拒绝（prior work 20+ 变化仍失败）。

**技术架构**：(1) 成功 jailbreak 和原始请求的中间表示对比；(2) 因果干预识别最小变化集；(3) 干预验证诱导拒绝。

**核心算法**：基于因果推断的局部解释——对每个中间表示维度进行干预测试，找到因果诱导拒绝的最小变化集合。

**与现有技术对比**：相比全局解释（所有攻击共享同一解释），LOCA 提供每个攻击的个性化局部解释。平均 6 个变化 vs 20+ 变化，效率显著提升。

**优势与局限**：优势是机械可解释、高效；局限是仅适用于中间表示可访问的开源模型。

**个人理解**：LOCA 代表了安全分析从"黑盒测试"向"机械解释"的转变。局部因果解释有助于理解 jailbreak 的微观机制，为防御设计提供指导。

## 技术趋势

1. **Agent 评估从通用向领域分化**：AutoMat 揭示编码代理在科学工作流中的泛化能力 ≠ 软件工程能力。未来 Agent 评估将更多按领域划分（科学、金融、医疗等），而非通用基准。

2. **推理质量纳入对齐目标**：TUR-DPO 首次将推理路径拓扑纳入 DPO 目标，代表对齐优化从"结果正确"向"过程可靠"演进。这对 Agent 推理可靠性有深远影响。

3. **端点粒度 AI 基础设施评估**：TokenArena 揭示同模型在不同端点性能差异巨大（准确率 12.5 分、能耗 6.2 倍）。企业 AI 选型将从"选模型"转向"选端点"。

4. **物理约束世界模型**：Hamiltonian World Models 代表世界模型从数据驱动向物理约束转变，解决长程预测稳定性的根本问题。

5. **编译器测试的混合范式**：CLOZEMASTER 展示历史数据引导 + LLM 生成的混合策略在编译器测试中的高效性，可推广到其他类型安全语言。
