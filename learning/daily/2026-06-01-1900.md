#### 1. TraceGraph: Shared Decision Landscapes for Agent Trajectories
- **地址**：https://arxiv.org/abs/2605.31308
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 轨迹分析
- **技术介绍**：将多模型 Agent 交互轨迹转化为共享决策图谱的框架。首先从合并的 rollouts 构建可观察动作-状态图，然后叠加结果导向的"生产核心"和"陷阱区域"，每个 rollout 用三个事件总结：Access、Trap 暴露、Repair。在 5 个基准 split 上揭示聚合分数隐藏的导航差异。基于此提出 SWE-bench 陷阱感知恢复流水线：aruntime 检测器匹配历史陷阱区域状态，轻量级延续策略从相同前缀评估。在触发状态上，最佳单因素策略将官方解决率从 40.4% 提升至 43.5%。
- **应用场景**：Agent 基准测试诊断、SWE-bench 等代码修复任务的失败恢复
- **为什么值得关注**：首次提供 Agent 基准测试的过程级诊断工具，从"通过率"升级到"轨迹导航分析"

#### 2. GLIDE: Industrializing Prediction-Powered Inference for Agentic Evaluation
- **地址**：https://arxiv.org/abs/2605.31278
- **来源**：arXiv (cs.AI) / ICML 2026 Workshop
- **领域**：AI Testing / 评估基础设施
- **技术介绍**：统一 Prediction-Powered Inference (PPI) 估计器的开源 Python 库。PPI 结合低成本 LLM-as-judge 代理和高成本人工标注，产生无偏估计和有效置信区间。GLIDE 统一了 PPI++、Stratified PPI、Predict-Then-Debias 及其分层变体、Active Statistical Inference 等 SOTA 估计器，以及 uniform、stratified、active、cost-optimal 等采样器，提供 scipy 风格 API。附带可复现的 Monte Carlo 验证套件、基于实证的方法选择决策树，以及 Agent 评估案例研究展示在等价精度下的显著标注节省。
- **应用场景**：Agent 系统评估、GenAI 输出质量测量
- **为什么值得关注**：解决 Agent 评估中"人工标注太贵 vs LLM-as-judge 有偏"的核心困境，ICML 2026 Workshop 接收

#### 3. COLLEAGUE.SKILL: Automated AI Skill Generation via Expert Knowledge Distillation
- **地址**：https://arxiv.org/abs/2605.31264 (GitHub: colleague.skill)
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 技能框架
- **技术介绍**：从异构痕迹（聊天记录、文档、代码等）自动蒸馏为可检查、可纠正、可部署的 Agent 技能包。生成版本化技能包包含两条协调轨道：能力轨道（实践、心智模型、决策启发式）和有界行为轨道（沟通风格、交互规则、纠正历史）。技能包可检查、调用、通过自然语言反馈更新、回滚、跨 Agent 主机安装、可选准备受控分发。公开仓库约 18.5K GitHub stars，画廊列出 215 个技能来自 165 位贡献者，累计 100K+ stars。
- **应用场景**：专家知识编码为 Agent 技能、个性化 Agent 构建
- **为什么值得关注**：18.5K stars 表明社区需求旺盛，将"人"的 expertise 转化为可移植、可纠正的技能包

#### 4. LLM-FACETS: Privacy-Preserving Framework for LLM Transparency Evaluation
- **地址**：https://arxiv.org/abs/2605.31167
- **来源**：arXiv (cs.AI) / ACM Journal on Responsible Computing
- **领域**：AI Testing / LLM 透明度评估
- **技术介绍**：面向非技术从业者的开源 LLM 评估框架，浏览器可访问界面和插件架构。围绕三类从业者画像（技术专家、领域专家、合规官）构建，镜像 EU AI Act 和 NIST AI RMF 的利益相关方分类。确定性指标（BLEU/ROUGE/BERTScore）完全在自托管服务器内运行无外发；LLM-judge 指标显式联系外部 API，用户保留凭证控制。通过三种机制实现透明度：token 级 log-prob 可视化（认知不确定性）、多 judge 共识（缓解 judge 偏差）、RAG Triad 指标（Faithfulness/Answer Relevance/Context Relevance 检测幻觉）。
- **应用场景**：企业 LLM 合规审计、负责任 AI 部署
- **为什么值得关注**：将 LLM 透明度评估从编程门槛降低到浏览器操作，直接对齐 EU AI Act 合规需求

#### 5. R+R: Reassessing Java Security API Misuse in Current LLMs
- **地址**：https://arxiv.org/abs/2605.31135
- **来源**：arXiv (cs.CR) / cs.SE
- **领域**：AI Testing / 代码安全
- **技术介绍**：对 Mousavi et al. 2024 年 Java 安全 API 滥用研究的复制和扩展。聚焦 GPT-5.5（前沿闭源编码模型）和 Llama-3.3-70B-Instruct（强开源模型）两个设置。结果显示虽然新 LLM 在 Java 安全 API 使用上表现更好，但问题尚未消除。外部安全知识显著改善结果，但效果因模型而异：Llama-3.3-70B 最受益于安全代码示例，GPT-5.5 通过显式滥用模式消除了所有检测到的安全 API 滥用。开发者指南知识变得更加有效。
- **应用场景**：LLM 代码安全审计、Java 安全 API 使用指导
- **为什么值得关注**：首次系统评估最新 LLM（GPT-5.5）在 Java 安全 API 上的表现，RAG 知识效果因模型能力而异

#### 6. Automating Formal Verification with RL and Recursive Inference (Dafny/Lean)
- **地址**：https://arxiv.org/abs/2605.30914
- **来源**：arXiv (cs.LG) / Master Thesis
- **领域**：AI Testing / 形式化验证
- **技术介绍**：硕士论文，研究验证器环境如何通过 RLVR（Group Relative Policy Optimization）和验证器引导推理搜索改进 LLM 生成验证程序和证明。Dafny 实验：GRPO 将验证奖励从 2.2% 提升至 58.1%，但发现"规格黑客"现象（模型利用弱规格而非实现预期解决方案）。过滤后多轮 RLVR 将验证通过率从 9.7% 提升至 31.1%。Lean 实验：验证器引导推理脚手架将证明生成视为结构化搜索，完整脚手架将 VeriCoding 通过率从 46.2% 提升至 69.2%，在 VERINA 数据集上解决 7 个先前未解决的任務。引入 Dalek-Bench（Rust curve25519-dalek 验证项目派生的仓库级 Lean 基准）。
- **应用场景**：形式化验证自动化、可验证代码生成
- **为什么值得关注**：系统展示 RLVR 在形式化验证中的潜力和局限（规格黑客），Dalek-Bench 填补仓库级验证基准空白

#### 7. FASR: Automated Identification of Unsafe Control Actions in STPA
- **地址**：https://arxiv.org/abs/2605.30697
- **来源**：arXiv (cs.CR) / cs.SE
- **领域**：AI Testing / 安全分析
- **技术介绍**：STPA（系统理论过程分析）是成熟的安全关键系统危害分析技术，但自动化支持有限。FASR 工具利用模型工程和形式化方法辅助人类分析师高效准确地执行 STPA，实现不安全控制动作（UCA）的自动完整识别。利用鲁棒性分析将 UCA 识别为控制器动作的不期望偏差。在航空电子系统刹车系统控制单元（BSCU）案例研究中演示，9 人用户研究显示大多数参与者认为工具对识别 UCA 有用。
- **应用场景**：安全关键系统危害分析、航空电子安全
- **为什么值得关注**：将形式化方法引入 STPA 这一传统手工流程，填补安全分析自动化工具空白

#### 9. Combinatorial Synthesis: Scaling Code RLVR via Atomic Decomposition

#### 13. LinTree: Improving LLM Reasoning with Explicitly Structured Search Histories
- **地址**：https://arxiv.org/abs/2605.31492
- **来源**：arXiv (cs.AI)
- **领域**：AI / LLM 推理
- **技术介绍**：研究 LLM 如何解决推理问题——生成探索和部分修订的中间轨迹，可视为线性化搜索树。在 Blocks World、网格导航和 Sokoban 三个受控推理环境中，比较轨迹条件推理策略与仅观察当前局部状态的 LLM 启发式最佳优先搜索。发现仅凭搜索历史访问不足以可靠超越启发式搜索，原因是 LLM 推理轨迹中搜索树仅隐式表示，回溯时不明确标识哪个早期状态被重新访问。添加简单父指针显式表示线性树（LinTree）结构后，任务性能和搜索效率均优于隐式推理模型和 LLM 启发式引导搜索。
- **应用场景**：LLM 推理优化、复杂任务求解
- **为什么值得关注**：揭示"搜索历史本身不够，必须显式表示树结构"这一关键发现，对 Agent 推理架构设计有直接指导意义

#### 14. HypoAgent: Agentic Framework for Interactive Abductive Hypothesis Generation

#### 15. SCALE: Self-Improving Web Agent via Cognitive-Aware Exploration

- **来源**：arXiv (cs.AI)

- **技术介绍**：利用三个对抗角色（Selector、Predictor、Judger）自主发现 Agent 局限性并通过环境探索扩展认知边界。提出 SCALE-Hop 图探索策略促进全局规划并帮助 Agent 避免局部探索陷阱。构建 SCALE-20k 数据集，从 19 个真实网站收集，包含多样化任务类型和结构化演示。实验显示显著提升多个 MLLM 在各种 Web 环境中的性能和泛化能力。

#### 16. CoSee: Diagnosing Failure Modes of Shared-State Collaboration in Visual Agents
- **地址**：https://arxiv.org/abs/2605.31354
- **来源**：arXiv (cs.AI)
- **领域**：AI / 多模态 Agent 诊断
- **技术介绍**：研究弱学习者（4B-8B 模型）协作推理中共享工作内存的失败模式。引入 CoSee 审计框架，形式化 read-write-verify 循环以追踪文档视觉问答中的信息流。在多页、图表和 Web 基准上发现反直觉退化： naive 共享工作空间通常放大幻觉而非解决。识别两种主导失败模式：噪声强化（未扎根笔记被重用为证据）和策略崩溃（添加上下文将模型推向未充分指定的短格式答案）。增加计算量可能与性能负相关，除非有显式验证。
- **应用场景**：多模态 Agent 架构设计、模块化视觉推理系统
- **为什么值得关注**：揭示"更多计算 ≠ 更好性能"的反直觉发现，为资源受限 Agent 设计提供诊断工具

#### 17. GraphARC: Comprehensive Benchmark for Graph-Based Abstract Reasoning
- **地址**：https://arxiv.org/abs/2605.31031
- **来源**：arXiv (cs.AI) / KDD 2026

#### 2. TraceGraph: Shared Decision Landscapes for Agent Trajectories

**地址**：https://arxiv.org/abs/2605.31308

**项目介绍（300 字）**：
TraceGraph 代表 Agent 基准测试评估范式的根本转变——从聚合分数（通过率/奖励分数）到过程级诊断（轨迹导航分析）。

传统 Agent 评估将每个 rollout 简化为 pass/fail，忽略了执行过程中的丰富信息。TraceGraph 从多模型 pooled rollouts 构建可观察动作-状态图，叠加结果导向的"生产核心"和"陷阱区域"，每个 rollout 用三个事件总结：Access（访问了哪些状态）、Trap 暴露（是否进入陷阱区域）、Repair（是否从陷阱恢复）。

在 SWE-bench 上的应用尤为引人注目：基于 TraceGraph 的陷阱感知恢复流水线，aruntime 检测器匹配历史陷阱区域状态，轻量级延续策略从相同前缀评估。在触发状态上，最佳单因素策略将官方解决率从 40.4% 提升至 43.5%，在公共触发实例上从 41.0% 提升至 44.8%。

**技术架构**：
- 轨迹收集层：从多模型 Agent 收集 pooled rollouts
- 图谱构建层：构建动作-状态图，不引入模型身份
- 区域标注层：叠加 outcome-informed 生产核心和陷阱区域
- 事件摘要层：每个 rollout 提取 Access/Trap/Repair 三事件
- 恢复应用层：基于陷阱检测的 runtime 恢复策略

**核心创新**：
1. 首次将多模型 Agent 轨迹统一为共享决策图谱
2. "陷阱区域"概念揭示基准测试中隐藏的困难区域
3. 从诊断到修复的闭环：TraceGraph 不仅诊断还能指导改进

**优势与局限**：
- 优势：提供过程级诊断而非仅聚合分数，可直接指导改进
- 局限：需要多模型 pooled rollouts 才能构建有意义的图谱，单模型场景受限

**未来发展潜力**：
如果扩展到更多基准（如 WebArena、AgentBench），TraceGraph 可能成为 Agent 基准测试的标准诊断工具。

#### 3. GLIDE: Industrializing Prediction-Powered Inference for Agentic Evaluation

**地址**：https://arxiv.org/abs/2605.31278 (GitHub: EmertonData/glide)

**论文介绍（350 字）**：
GLIDE 解决 Agent 评估中的核心困境：人工标注成本高但 LLM-as-judge 有偏。Prediction-Powered Inference (PPI) 提供了一条中间道路——结合低成本 LLM 代理和高成本人工标注，产生无偏估计和有效置信区间。但 PPI 的各种方法分散在论文中，实现不完整。

GLIDE 统一了 SOTA PPI 估计器（PPI++、Stratified PPI、Predict-Then-Debias 及分层变体、Active Statistical Inference）和采样器（uniform、stratified、active、cost-optimal），提供 scipy 风格 API 专用于均值估计。

论文的关键贡献不仅是库的实现，还包括：
1. 可复现的 Monte Carlo 验证套件——确保每种估计器的统计正确性
2. 基于实证的方法选择决策树——帮助用户根据数据特征选择最优方法
3. Agent 评估案例研究——展示在等价精度下的显著标注节省

ICML 2026 Workshop on Agentic Uncertainty 接收，表明该方法在 Agent 评估社区获得认可。

**技术架构**：
- 估计器层：统一 5+ 种 PPI 估计器实现
- 采样器层：4 种采样策略（uniform/stratified/active/cost-optimal）
- API 层：scipy 风格接口，专用于均值估计
- 验证层：Monte Carlo 验证套件 + 方法选择决策树

**核心算法**：
PPI 的核心思想是用大量低成本代理估计（LLM-as-judge）+ 少量高成本真实标注（人工）来产生无偏估计。GLIDE 的关键创新是将多种 PPI 变体统一实现并提供实证指导。

**与现有技术对比**：
- vs 纯人工标注：成本降低 10-100x，精度相当
- vs 纯 LLM-as-judge：消除偏差，提供有效置信区间
- vs 分散的 PPI 实现：统一接口 + 验证套件 + 方法选择指导

**优势与局限**：
- 优势：开源、统计正确性可验证、方法选择有实证指导
- 局限：PPI 本身假设 LLM 代理与真实标注存在相关性，当相关性极弱时效果有限

**未来发展潜力**：
如果扩展支持比例估计、回归估计和假设检验，GLIDE 可能成为 Agent 评估的统计基础设施。

#### 4. LinTree: Improving LLM Reasoning with Explicitly Structured Search Histories

**地址**：https://arxiv.org/abs/2605.31492

**论文介绍（300 字）**：
LinTree 揭示了一个关于 LLM 推理的关键发现：搜索历史本身不够，必须显式表示树结构。

论文在三个受控推理环境（Blocks World、网格导航、Sokoban）中系统比较了两种策略：(1) 轨迹条件推理策略（LLM 访问完整搜索历史）和 (2) LLM 启发式最佳优先搜索（仅观察当前局部状态）。反直觉地发现，仅凭搜索历史访问不足以可靠超越启发式搜索。

深入分析揭示原因：LLM 推理轨迹中搜索树仅隐式表示。当模型回溯或切换分支时，轨迹不明确标识哪个早期状态被重新访问。这导致 LLM 无法有效利用历史信息。

解决方案：添加简单父指针显式表示线性树（LinTree）结构。这使 LLM 能够明确知道"我从哪里回溯"和"我在探索哪个分支"。实验显示 LinTree 在任务性能和搜索效率上均优于隐式推理模型和 LLM 启发式引导搜索。

**核心发现**：
1. 搜索历史 ≠ 有效推理：隐式历史无法被 LLM 有效利用
2. 显式树结构是关键：父指针使历史信息变得可用
3. 对 Agent 架构的启示：Agent 的记忆系统需要显式结构而非仅追加文本

**与现有技术对比**：
- vs Tree of Thoughts：LinTree 在推理过程中显式维护树结构，而非事后构建
- vs ReAct：ReAct 是线性轨迹，LinTree 是结构化树

**优势与局限**：
- 优势：简单有效（仅添加父指针），在多个推理环境上验证
- 局限：仅在受控推理环境验证，复杂开放域任务效果未知

**未来发展潜力**：
LinTree 的思想可推广到 Agent 记忆系统设计——Agent 的长期记忆不应是简单的对话历史追加，而应是结构化的决策树。

#### 5. CHARM: Multimodal JEPA for Semantic Time-Series Embeddings (ICML 2026)

**地址**：https://arxiv.org/abs/2605.31580

**论文介绍（300 字）**：
CHARM 是首个将 JEPA（联合嵌入预测架构，LeCun 推崇的 AI 架构）成功扩展到多变量时间序列的工作，ICML 2026 接收。

JEPA 的核心思想是在表征空间（而非原始数据空间）进行预测，避免像素级生成的信息浪费。CHARM 将这一思想引入时间序列：Channel-Aware Representation Model 将通道级文本描述纳入 Transformer 编码器（对通道顺序等变），使用 JEPA 目标训练——在潜在空间预测未来通道状态，而非预测原始传感器值。

关键设计：
1. 通道感知架构：文本描述作为通道标识符，支持跨数据集泛化
2. JEPA 目标：潜在空间预测鼓励对传感器噪声的鲁棒性
3. 描述感知门控：学习的通道间关系提供可解释性

仅用线性探针即在异常检测、分类、短期/长期预测上达到强性能。JEPA 目标和条件架构是性能驱动因素，文本描述主要作为通道标识符。

**核心创新**：
1. 首次将 JEPA 扩展到时间序列领域
2. 通道级文本描述实现跨数据集泛化
3. JEPA 在时间序列上的优势：比生成式方法更鲁棒、更高效

**与现有技术对比**：
- vs 时间序列 Transformer（如 TimesFM）：JEPA 在潜在空间预测，避免生成式浪费
- vs 对比学习（如 TimeMAE）：JEPA 的预测目标比对比目标提供更丰富的信号

**优势与局限**：
- 优势：ICML 2026 接收，通用时间序列表征新方向
- 局限：仅在特定时间序列任务验证，泛化到其他领域（如金融）效果未知

**未来发展潜力**：
如果扩展到更多时间序列领域（金融、医疗、交通），CHARM 可能成为通用时间序列基础模型。

## 技术趋势

1. **Agent 评估基础设施化**：GLIDE（PPI 估计器统一库）和 TraceGraph（轨迹决策图谱）同时出现，标志着 Agent 评估从"手工跑基准"走向"标准化基础设施"。PPI 方法解决标注成本与偏差的 trade-off，TraceGraph 提供过程级诊断而非仅聚合分数。

2. **专家知识→Agent 技能蒸馏**：COLLEAGUE.SKILL（18.5K stars）代表"将人类 expertise 编码为 Agent 技能"的新范式。技能包可检查、可纠正、可部署，解决了传统 prompt 工程的不可见性和不可维护性问题。

3. **形式化验证 × RL 融合加速**：Dafny RLVR（硕士论文）、Value Function 即 Supermartingale 证书、ASP-based RL 抽象——三条独立工作同时出现，表明形式化验证与强化学习的融合正在加速，为"RL 学到的策略是否有形式保证"提供理论答案。

4. **JEPA 架构扩展**：CHARM 将 JEPA 从视觉扩展到时间序列，ICML 2026 接收。JEPA 在潜在空间预测而非原始数据空间生成的理念正在获得验证，可能成为通用表征学习的新范式。

5. **Agent 推理结构显式化**：LinTree 揭示"搜索历史本身不够，必须显式表示树结构"。这对 Agent 记忆系统设计有直接启示——Agent 的长期记忆不应是简单的对话历史追加，而应是结构化的决策树。

## 新领域收获

1. **时间序列 JEPA 表征学习（CHARM）**：ICML 2026 接收，首次将 JEPA 扩展到多变量时间序列。通道级文本描述 + 潜在空间预测，仅用线性探针即达到强性能。代表通用时间序列表征的新方向，未来可扩展到金融、医疗等领域。
