#### 1. MAVEN: Improving Generalization in Agentic Tool Calling
- **地址**：https://arxiv.org/abs/2605.30738
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 工具调用
- **技术介绍**：针对 Agent 工具调用跨域泛化问题，提出轻量级符号推理脚手架 MAVEN（Modular Agentic Verification and Execution Network），实现结构化分解、自适应工具编排和中间验证。在 BFCL v3、TauBench、Tau2Bench、AceBench 等基准上评估，并引入 MAVEN-Bench 压力测试基准（多步数学和物理推理，含显式验证和对抗性任务组合）。MAVEN-Bench 揭示了"部分推理质量"与"端到端任务成功"之间的巨大差距；在 GPT-OSS-120b 上从 48% 提升至 71% 准确率，同时保持与前沿闭源模型竞争力，成本比约 1/10。
- **应用场景**：Agent 工具调用能力评估、跨域泛化测试
- **为什么值得关注**：首次提出跨域工具调用压力测试基准，揭示 Agent 组合推理能力的系统性不足

#### 2. AdaCoM: Adaptive Context Management for Long-Horizon Tasks
- **地址**：https://arxiv.org/abs/2605.30785
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 上下文管理
- **技术介绍**：针对 LLM Agent 长期任务中上下文累积导致的长上下文退化和推理失败，训练外部 LLM 管理冻结 Agent 的上下文，通过灵活修改动作和端到端 RL 优化。在网页搜索和深度研究基准上显著提升性能。发现 Fidelity-Reliability Trade-off：高 ReAct 性能 Agent 受益于高保真上下文保留，低性能 Agent 需要更积极的压缩以保持在可靠推理区间。跨 Agent 迁移实验表明，相似能力的 Agent 间迁移效果最佳。
- **应用场景**：Agent 长上下文管理、闭源 Agent 优化
- **为什么值得关注**：不修改 Agent 本身即可优化上下文管理，对闭源 Agent 有直接价值

#### 3. Harness Updating Is Not Harness Benefit: Disentangling Evolution Capabilities
- **地址**：https://arxiv.org/abs/2605.30621
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 自进化
- **技术介绍**：首次解耦 LLM Agent 的两种 harness 自进化能力：(i) harness-updating（从执行证据产生有用持久 harness 更新的能力）和 (ii) harness-benefit（从更新后的 harness 中受益的能力）。分析揭示：harness-updating 在基础能力上呈平坦分布（各模型差异不大），但 harness-benefit 随基础能力显著增长。这意味着强模型不仅更善于利用 harness 更新，而且更新本身也更有价值。
- **应用场景**：Agent 自进化能力评估、harness 设计指导
- **为什么值得关注**：首次区分两种自进化能力，对 Agent 自改进研究有重要方法论意义

#### 4. PReMISE: Policy Rubrics as Measurement Specifications for LLM Judges
- **地址**：https://arxiv.org/abs/2605.30803
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / LLM 评估
- **技术介绍**：将可复用 rubric 视为测量规格，改变 rubric 即改变固定 judge 诱导的响应质量测量。PReMISE 框架给定成对人类偏好数据，(i) 发现策略级 rubric 集合，(ii) 沿四个维度审计任何 rubric 集：结构充分性、可靠性、偏好拟合、对抗鲁棒性。发现没有原始 rubric 源同时满足可靠、偏好预测和对抗鲁棒；高评分者一致性不意味着低可利用性。PReMISE 是唯一在适用性、特异性和有效维度上同时获得非平凡分数的 rubric 源。偏好排名选择将 judge 准确率从 65.0% 提升至 68.6%。
- **应用场景**：LLM-as-judge rubric 设计、评估系统审计
- **为什么值得关注**：首次将 rubric 视为可审计的测量规格，对 LLM 评估的可信度有直接价值

#### 5. SLAT: Segment-Level Adaptive Trimming for Efficient CoT Reasoning
- **地址**：https://arxiv.org/abs/2605.30832
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 推理优化
- **技术介绍**：针对 LLM 生成推理链中的结构性冗余（overthinking），证明效率损失集中在高概率低边际效用段。推导了正确性-长度权衡目标下的段次优性理论特征，提出 SLAT（Segment-Level Adaptive Trimming）RL 框架，基于此标准选择性抑制冗余段。在标准基准上建立优越的准确性-效率帕累托前沿，推理长度减少 50% 同时保持竞争力准确率。
- **应用场景**：CoT 推理效率优化、降低推理成本
- **为什么值得关注**：理论驱动的段级裁剪，比 token 级均匀惩罚更精准

#### 6. EHRBench: Automated EHR-based Benchmark for Clinical Decision Making
- **地址**：https://arxiv.org/abs/2605.30637
- **来源**：arXiv (cs.AI) / KDD 2026 Oral
- **领域**：AI Testing / 医疗 AI 基准
- **技术介绍**：针对 LLM 临床决策可靠性评估，提出 EHR-LLM-KB 交互流水线构建自动化可靠基准。专用 LLM 自动将 encounter-level EHR 轨迹转换为结构化模板并确定性实例化为 QA 项，同时应用系统性 KB 验证和丰富以过滤幻觉或模糊关系。构建近 100 万（960,067）QA 项，涵盖诊断、治疗和预后三个核心推理任务。基准 30+ 代表性 LLM，揭示一致的能力趋势和可操作的差距。
- **应用场景**：医疗 LLM 评估、临床决策支持系统验证
- **为什么值得关注**：KDD 2026 Oral，近百万级 EHR 基准，医疗 AI 评估基础设施

#### 7. DecomposeR: Planner-Centric RL for Deep Research
- **地址**：https://arxiv.org/abs/2605.30824
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 深度研究 Agent
- **技术介绍**：将研究计划表示为类型化有向无环图（DAG），使规划显式化、结构化和可奖励。两阶段训练：规划器 RL 先学习图结构和查询分解，回答者 RL 再学习分支级执行和最终综合。通过给显式规划器 token 和结构化组件分配奖励而非扁平轨迹，实现更细粒度优化。DecomposeR-8B 在流行长形式基准上超越强开源基线 5.1-8.0 分。
- **应用场景**：深度研究 Agent 训练、规划-执行解耦
- **为什么值得关注**：首次将 DAG 结构引入研究规划 RL 训练，解决信用分配模糊问题

#### 8. LALS: Latent Association Leaning Score for VLM Bias Detection
- **地址**：https://arxiv.org/abs/2605.31556
- **来源**：arXiv (cs.CV)
- **领域**：AI Testing / VLM 偏见检测
- **技术介绍**：针对歧义输入（如穿全套防护服的工人）下 VLM 性别偏见问题，引入 LALS（Latent Association Leaning Score）零样本指标，将视觉 token 激活投影到模型文本嵌入空间以测量每个 token 和层的概念关联。在 15 个职业、800+ 性别歧义图像和 4 个 VLM 上发现：内部表示和输出系统性地解耦——模型内部编码女性关联但输出男性。层分析揭示不对称过滤器：男性信号端到端放大，女性信号在中层达到峰值并在生成前被抑制。
- **应用场景**：VLM 偏见审计、对齐效果评估
- **为什么值得关注**：揭示 VLM 内部编码与外部输出的系统性差异，对偏见检测有方法论贡献

#### 9. BilliardPhys-Bench: Benchmarking Physical Reasoning of MLLMs
- **地址**：https://arxiv.org/abs/2605.30900
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 物理推理基准
- **技术介绍**：针对 MLLM 直观物理推理弱点，提出台球合成环境基准。程序化引擎生成带摩擦和弹性碰撞的随机场景。测试三种能力：(1) 预测球-球碰撞，(2) 推理墙壁反弹，(3) 估计运动停止后的最终球位。评估 GPT、Claude、Gemini、Qwen 系列。发现"静止偏差"失败模式：当正确物理结果更难推断时，模型倾向于预测无交互。
- **应用场景**：MLLM 物理推理评估、视觉动力学理解
- **为什么值得关注**：首次系统测试 MLLM 物理推理，揭示系统性失败模式

#### 10. COMPASS: Cognitive MCTS-Guided Process Alignment for Search Agents
- **地址**：https://arxiv.org/abs/2605.30838
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 安全对齐
- **技术介绍**：针对 LLM 搜索 Agent 的检索诱导安全退化（有害意图分解为看似无害的子查询导致不安全结果），提出 COMPASS 框架。集成认知树探索（CTE）高效合成隐蔽攻击轨迹，以及内省逐步对齐（ISA）隔离风险中间动作以进行细粒度过程监督。在保持通用效用的同时实现有利的安全-效用权衡，所需训练数据大幅减少。
- **应用场景**：搜索 Agent 安全对齐、多步交互安全审计
- **为什么值得关注**：首次将 MCTS 引入 Agent 安全对齐，解决稀疏安全信号问题

#### 11. GRiD: Generating Graph-like Rules for KG Reasoning via Diffusion
- **地址**：https://arxiv.org/abs/2605.30747
- **来源**：arXiv (cs.AI) / KDD 2026
- **领域**：AI / 知识图谱推理
- **技术介绍**：针对现有规则挖掘方法主要关注简单链式规则而忽略图状结构（循环和分支）的问题，提出 GRiD 框架将图状规则发现重构为以目标关系为条件的离散生成过程。两阶段训练：(i) 监督预训练从 KG 元图采样的子图中捕获结构先验，(ii) RL 通过策略梯度优化直接以非可微规则质量指标引导。在 6 个基准数据集上实现竞争性能，图状规则与链式规则互补。
- **应用场景**：知识图谱推理、关系模式发现
- **为什么值得关注**：KDD 2026，首次将扩散模型引入图状规则挖掘

#### 12. ReuseRL: Skill Reuse as Compression in Agentic RL
- **地址**：https://arxiv.org/abs/2605.31509
- **来源**：arXiv (cs.LG)
- **领域**：AI / RL 技能复用
- **技术介绍**：针对 LLM Agent RL 训练中学习的脆弱特定任务捷径，假设当成功轨迹在结构上可压缩（分解为少量可复用抽象模式）时 Agent 泛化更好。引入 ReuseRL，将 Agent RL 建立在最小描述长度（MDL）原则基础上。从成功轨迹中提取共享技能字典，并在 RL 目标中增加分段成本，显式惩罚编码差的特立独行行为。证明压缩惩罚的 PAC-Bayes 泛化界。在 ALFWorld、TextWorld-Cooking、Countdown-Stepwise 上超越 vanilla GRPO。
- **应用场景**：Agent RL 泛化提升、技能抽象
- **为什么值得关注**：首次将 MDL 原则引入 Agent RL，提供理论保证

#### 13. Feedback Distillation for Lean Theorem Proving
- **地址**：https://arxiv.org/abs/2605.30861
- **来源**：arXiv (cs.AI)
- **领域**：AI / 形式化推理
- **技术介绍**：针对 GRPO 在推理模型后训练中的稀疏奖励、有限探索和模式崩溃问题，提出 Feedback Distillation：模型在 token 级别匹配自身在 LLM 生成的特权反馈条件下的分布。提供 token 级监督并可注入外部知识。在 Lean4 定理证明中，Feedback Distillation 比 GRPO 保持更多轨迹多样性，产生更高策略熵和更好的 pass@k 缩放。两种方法互补：从 Feedback Distillation checkpoint 初始化 GRPO 超越任一方法单独使用。
- **应用场景**：定理证明模型训练、复杂推理后训练
- **为什么值得关注**：为复杂推理的后训练提供新方向

#### 14. UniScale: Unified Inference Scaling via Online Joint Optimization
- **地址**：https://arxiv.org/abs/2605.30898
- **来源**：arXiv (cs.AI) / ICML 2026
- **领域**：AI / 推理优化
- **技术介绍**：针对模型路由（粗粒度离散性能变化）和测试时缩放 TTS（容量天花板和递减回报）的解耦设计局限，提出统一推理缩放 UIS，在单一优化空间中统一模型路由和 TTS。UniScale 将自适应 UIS 建模为上下文多臂老虎机问题，通过 LinUCB 学习推理策略。结合效率感知学习和成本建模确保在高维动作空间上的稳定可扩展优化。在多样动态推理场景中提供细粒度且一致更优的质量-成本权衡。
- **应用场景**：LLM 推理成本优化、动态模型路由
- **为什么值得关注**：ICML 2026，首次统一模型路由和 TTS

#### 15. VecLinking: Vector Linking via Cross-Model Local Isometric Consistency
- **地址**：https://arxiv.org/abs/2605.31100
- **来源**：arXiv (cs.AI) / ICML 2026
- **领域**：AI / 向量数据库互操作
- **技术介绍**：研究向量链接问题：给定两个不同黑盒编码器在部分重叠数据集上产生的嵌入云，仅使用向量恢复跨模型对象对应关系。实证和理论上证明独立训练的对比编码器表现出局部几何一致性：短距离近似保留（至多缩放因子），长距离因模型特定失真而不保留。提出迭代基于参考的几何嵌入哈希，从极小种子配对锚点恢复向量链接。在多个基准和嵌入模型对上展示准确鲁棒的链接。
- **应用场景**：向量数据库集成、跨模型聚类
- **为什么值得关注**：ICML 2026，解决多向量数据库互操作的核心问题

#### 16. RayDer: Scalable Self-Supervised Novel View Synthesis

- **应用场景**：3D 场景重建、新视角合成
- **为什么值得关注**：将自监督 NVS 转化为单模型缩放问题，展示清晰缩放规律

#### 17. Positional vs Symbolic Attention Heads: Learning Dynamics and RoPE Geometry

- **为什么值得关注**：首次定量分离位置和符号机制的鲁棒性差异

#### 18. TunerDiT: Training-free Progressive Steering for Multi-Event Video
- **地址**：https://arxiv.org/abs/2605.31590
- **来源**：arXiv (cs.CV)

- 相比 Agent 微调方案，MAVEN 是推理时脚手架，成本更低

**优势与局限**：
- 优势：无需训练、跨域泛化、成本效益高（1/10 成本比）
- 局限：验证模块可能增加推理延迟、对复杂多步任务的分解质量依赖任务特征

**未来发展潜力**：
- Agent 工具调用泛化是核心挑战，MAVEN 的验证中心方法有广泛适用性
- 可扩展到更多工具类型（API、数据库、物理设备）
- 可结合 RL 训练实现自适应验证策略

**个人技术理解**：
MAVEN 揭示了一个关键洞察：Agent 的部分推理质量与端到端成功之间存在巨大差距。这意味着当前 Agent 评估过于关注最终结果，而忽略了中间过程的正确性。MAVEN 的符号推理脚手架提供了一种低成本提升 Agent 可靠性的方法——不需要训练新模型，只需在推理时添加验证层。这对闭源 Agent 尤其有价值，因为无法修改其内部参数。MAVEN-Bench 的引入也为 Agent 工具调用评估提供了新的压力测试标准。

#### 2. PReMISE: Policy Rubrics as Measurement Specifications

**标题**：PReMISE — 将策略级 Rubric 视为 LLM Judge 的测量规格

**地址**：https://arxiv.org/abs/2605.30803

**项目或论文介绍**（200-400 字）：
PReMISE 将可复用 rubric 视为测量规格——改变 rubric 即改变固定 judge 诱导的响应质量测量。当前 LLM-as-judge 评估高度依赖 rubric 质量，但 rubric 设计缺乏系统方法论。PReMISE 框架给定成对人类偏好数据，(i) 发现策略级 rubric 集合，(ii) 沿四个维度审计任何 rubric 集：结构充分性、可靠性、偏好拟合、对抗鲁棒性。发现没有原始 rubric 源同时满足可靠、偏好预测和对抗鲁棒；高评分者一致性不意味着低可利用性。PReMISE 是唯一在适用性、特异性和有效维度上同时获得非平凡分数的 rubric 源。偏好排名选择将 judge 准确率从 65.0% 提升至 68.6%，可靠性约束精炼将利用响应获得高分的比率从 46.4% 降至 36.0%。

**技术架构**：
- Rubric 发现模块（从人类偏好数据中发现策略级 rubric）
- 四维审计框架（结构充分性、可靠性、偏好拟合、对抗鲁棒性）
- Rubric 修复操作（偏好排名选择、可靠性约束精炼）

**核心算法或系统设计**：
- 策略级 rubric 发现算法（聚类+优化）
- 对抗鲁棒性评估（生成利用响应测试 rubric 脆弱性）
- 多维度联合优化（平衡适用性、特异性、有效维度）

**关键实现机制**：
- 成对人类偏好数据建模
- Rubric 效用函数定义
- 对抗样本生成和测试

**与现有技术对比**：
- 相比手工设计 rubric，PReMISE 提供数据驱动的发现方法
- 相比单一维度评估，PReMISE 提供四维审计框架
- 相比静态 rubric，PReMISE 支持自适应修复

**优势与局限**：
- 优势：系统化 rubric 设计方法、四维审计全面、修复操作有效
- 局限：依赖成对人类偏好数据、计算成本较高

**未来发展潜力**：
- LLM 评估基础设施的核心组件
- 可扩展到多模态评估、Agent 评估等场景
- 可结合自动化 rubric 生成和持续审计

**个人技术理解**：
PReMISE 的核心贡献是将 rubric 从"提示词工程"提升到"测量规格"的层面。这一视角转换对 LLM 评估的可信度有深远影响——如果 rubric 是测量规格，那么它应该像科学测量工具一样被审计和校准。PReMISE 的四维审计框架（结构充分性、可靠性、偏好拟合、对抗鲁棒性）为 rubric 质量提供了系统评估标准。特别值得注意的是"高评分者一致性不意味着低可利用性"这一发现——这意味着当前许多 LLM 评估可能看似可靠，实则存在系统性脆弱性。

#### 3. AdaCoM: Adaptive Context Management for Long-Horizon Tasks

**标题**：AdaCoM — 长时域任务的自适应上下文管理

**地址**：https://arxiv.org/abs/2605.30785

**项目或论文介绍**（200-400 字）：
AdaCoM 针对 LLM Agent 长期任务中上下文累积导致的长上下文退化和推理失败问题，训练外部 LLM 管理冻结 Agent 的上下文。与需要训练 Agent 本身的上下文管理方法不同，AdaCoM 通过灵活修改动作和端到端 RL 优化，在不修改 Agent 内部参数的情况下提升性能。在网页搜索和深度研究基准上显著提升。发现 Fidelity-Reliability Trade-off：高 ReAct 性能 Agent 受益于高保真上下文保留，低性能 Agent 需要更积极的压缩以保持在可靠推理区间。跨 Agent 迁移实验表明，相似能力的 Agent 间迁移效果最佳。

**技术架构**：
- 外部上下文管理器（独立于 Agent 的 LLM）
- 灵活修改动作集（摘要、截断、重组、删除）
- 端到端 RL 训练（以 Agent 最终任务表现为奖励信号）

**核心算法或系统设计**：
- Fidelity-Reliability Trade-off 理论
- 基于 Agent 能力的自适应压缩策略
- 跨 Agent 迁移学习

**关键实现机制**：
- 上下文重要性评分
- 任务约束和进度保持
- 陈旧内容修剪

**与现有技术对比**：
- 相比 Agent 内置上下文管理，AdaCoM 不修改 Agent 本身
- 相比固定策略（如简单截断），AdaCoM 是自适应的
- 相比需要训练 Agent 的方法，AdaCoM 适用于闭源 Agent

**优势与局限**：
- 优势：适用于闭源 Agent、自适应压缩、跨 Agent 迁移
- 局限：外部管理器增加额外推理开销、压缩可能丢失关键信息

**未来发展潜力**：
- Agent 上下文管理是刚需，AdaCoM 的外部管理范式有广泛适用性
- 可扩展到多 Agent 协作场景
- 可结合长期记忆管理实现完整上下文生命周期管理

**个人技术理解**：
AdaCoM 的 Fidelity-Reliability Trade-off 是一个重要理论发现——不同能力的 Agent 对上下文保真度的需求不同。这意味着不存在"一刀切"的上下文管理策略，必须根据 Agent 能力自适应调整。这对 Agent 系统设计有直接指导意义：强 Agent 应该保留更多原始上下文，弱 Agent 应该更积极地压缩。此外，AdaCoM 的外部管理范式（不修改 Agent 本身）使其成为闭源 Agent 优化的实用方案。

## 技术趋势

1. **Agent 工具调用泛化成为核心挑战**：MAVEN 揭示"部分推理质量 vs 端到端成功"的巨大差距，表明当前 Agent 评估过于关注最终结果而忽略中间过程正确性。符号推理脚手架（无需训练）是低成本提升 Agent 可靠性的有效路径。

3. **推理效率优化从 token 级走向段级和系统级**：SLAT（段级自适应裁剪）减少 50% 推理长度，UniScale（统一推理缩放）统一模型路由和 TTS，AdaCoM（外部上下文管理）揭示 Fidelity-Reliability Trade-off。推理效率优化正从粗粒度走向细粒度、从单一维度走向系统级。
