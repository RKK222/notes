### AI Testing / Software Testing（10 项）

#### 1. CNN-CodeBERT 三分类凭证泄露检测
- **地址**：https://arxiv.org/abs/2605.31520
- **来源**：arXiv (cs.SE) — ICSME 2026 Accepted
- **领域**：AI Testing / 代码安全
- **技术介绍**：针对凭证泄露检测中假阳性率高的问题，提出三分类框架（真实凭证/占位符凭证/弱凭证），结合 CodeBERT 语义理解和字符级模式识别。在 9,426 样本、10 种编程语言的新数据集上评估，MCC 0.86，macro F1 0.90，真实凭证召回率 93%、精确率 89%。相比字符级方法，占位符/弱凭证检测从 54% 提升到 81% F1，高严重性告警减少 33%（373→250），10 种语言中 9 种 F1 > 0.80（留一语言评估）。
- **应用场景**：代码仓库凭证泄露自动化检测、CI/CD 安全门控
- **为什么值得关注**：首次将凭证检测从二分类提升到三分类，显著降低误报同时保持安全覆盖

#### 2. R+R: Java 安全 API 滥用复制研究
- **地址**：https://arxiv.org/abs/2605.31135
- **来源**：arXiv (cs.CR/cs.SE) — Preprint
- **领域**：AI Testing / 代码安全
- **技术介绍**：对 Mousavi et al. 2024 年 Java 安全 API 滥用研究的复制与扩展，聚焦 JCA 和 JSSE API。评估 GPT-5.5（前沿闭源）和 Llama-3.3-70B-Instruct（强开源）。发现：新模型表现更好但问题未消除；外部安全知识显著提升效果但模型依赖性强。Llama-3.3 最有效的是安全代码示例；GPT-5.5 中显式滥用模式消除了所有检测到的安全 API 滥用（有效程序中），尽管部分输出仍因编译错误无效。开发者指南知识对 GPT-5.5 效果大幅提升。
- **应用场景**：LLM 生成代码安全评估、RAG 安全知识注入策略
- **为什么值得关注**：首次用 GPT-5.5 复现 Java 安全 API 滥用问题，揭示 RAG 知识效果因模型而异

#### 3. RLVR 小模型代码生成验证反馈
- **地址**：https://arxiv.org/abs/2605.30478
- **来源**：arXiv (cs.SE) — AINL-2026 Accepted
- **领域**：AI Testing / 代码生成
- **技术介绍**：对 Qwen3-0.6B 和 Llama3.2-1B 使用 RLVR（可验证奖励强化学习）在 MBPP 基准上进行 LoRA 微调。比较单元测试奖励、静态分析（Ruff linter）奖励和组合奖励，以及 GRPO 和 GSPO 优化变体。组合奖励下 pass@1 提升最高 13 个百分点。发现：仅静态分析惩罚会诱导策略偏向更短完成（减少 lint 错误但不改善功能正确性）；组合奖励缓解退化。RLVR 效果对奖励设计和优化粒度高度敏感。
- **应用场景**：小模型代码生成能力增强、RLVR 奖励设计指南
- **为什么值得关注**：首次系统评估 RLVR 在 1B 以下模型的代码生成效果，揭示奖励设计敏感性

#### 4. 神经多样性在敏捷团队中的障碍
- **地址**：https://arxiv.org/abs/2605.30555
- **来源**：arXiv (cs.SE)
- **领域**：Software Testing / 团队工程
- **技术介绍**：探索敏捷软件开发中团队协作质量与神经多样性包容的关系。混合方法：Reddit/LinkedIn 网页内容分析 + 11 次半结构化专家访谈（德国组织神经多样性网络）。发现：团队实践高度碎片化，依赖个体适应而非共享标准；敏捷实践和支持工具可促进神经多样性参与，但刚性结构、刻板印象和一刀切方法常破坏包容性。组织意识和定制调整仍不足。
- **应用场景**：敏捷团队多样性管理、AI 辅助团队适配
- **为什么值得关注**：首次系统研究神经多样性在敏捷团队中的包容障碍，对 AI 辅助团队工具有启示

#### 11. ASP 抽象 RL（ICLP 2026）
- **地址**：https://arxiv.org/abs/2605.31444
- **来源**：arXiv (cs.AI) — ICLP 2026 Accepted (TPLP)
- **领域**：AI / 神经符号推理
- **技术介绍**：将 CARCASS 关系强化学习框架从 Prolog 迁移到 Answer-Set Programming (ASP)。ASP 是比 Prolog 更丰富的完全声明式建模语言。在 Blocks World 和 Minigrid 两个领域评估，结果表明 CARCASS + ASP 为 RL 抽象构建提供了有前景的方法，尤其在有领域知识可用时。

#### 12. CDAF 上下文依赖论证（FLoC 2026）
- **地址**：https://arxiv.org/abs/2605.31581
- **来源**：arXiv (cs.AI) — FLoC 2026 LAMAS&SR Workshop
- **领域**：AI / 形式化推理
- **技术介绍**：扩展 Dung 论证理论，引入上下文依赖论证框架（CDAF）。 defeat 函数根据上下文决定哪些攻击成功。通过相关性集合 ρ 和优先级 π 派生 defeat 函数。在示例中，目标论证在全相关性注入优先级下被拒绝，但在部分激活下被接受，其中一种 VAF 受众无法镜像。定义 ACTIVATION-MANIPULATION 决策问题并记录基线复杂度界限。
- **应用场景**：多 Agent 论证系统、策略性推理
- **为什么值得关注**：扩展 Dung 理论到上下文依赖场景，为 Agent 策略性论证提供形式化基础

#### 14. SCALE 自改进 Web Agent

#### 15. CoSee 共享状态协作失败模式审计
- **地址**：https://arxiv.org/abs/2605.31354
- **来源**：arXiv (cs.AI)
- **领域**：AI / 多模态 Agent
- **技术介绍**：研究弱学习者（4B-8B 模型）共享工作内存协作推理的失败模式。引入 CoSee 审计框架形式化 read-write-verify 循环追踪文档视觉 QA 中的信息流。发现反直觉退化：朴素共享工作空间常放大幻觉而非解决。识别两种主导失败模式：噪声强化（无根据笔记被重用为证据）和策略崩溃（添加上下文使模型偏向未充分指定的简短回答）。成本-精度 Pareto 前沿显示增加计算可能与性能负相关。
- **应用场景**：多模态 Agent 协作架构设计、资源受限 Agent 优化
- **为什么值得关注**："更多计算≠更好性能"的反直觉发现，对 Agent 架构设计有重要启示

#### 17. TraceGraph 共享决策图谱

## 深度研究

### GitHub 项目

#### 1. CNN-CodeBERT 三分类凭证泄露检测（ICSME 2026）

**项目介绍**：
凭证泄露是代码仓库安全的重大威胁，2024 年超过 2380 万密钥暴露。现有检测工具因刚性模式匹配和二元分类方案导致高误报率，无法区分真实凭证与占位符/弱凭证。本研究提出三分类框架，将占位符/弱凭证建模为独立类别，结合 CodeBERT 语义理解和字符级模式识别。在 9,426 样本、10 种编程语言的新数据集上，MCC 0.86，macro F1 0.90。真实凭证召回率 93%、精确率 89%。相比字符级方法，占位符/弱凭证检测从 54% 提升到 81% F1，高严重性告警减少 33%。留一语言评估中 10 种语言中 9 种 F1 > 0.80。

**技术架构**：
- CodeBERT 预训练模型提取代码语义特征
- 字符级 CNN 捕获模式匹配特征
- 三分类头（真实凭证/占位符/弱凭证）
- 留一语言交叉验证评估跨语言泛化

**核心创新**：
- 首次将凭证检测从二元分类提升到三分类
- 语义+字符双路特征融合
- 显著降低误报同时保持安全覆盖

**与现有技术对比**：
| 维度 | 传统模式匹配 | 二元分类 | 三分类框架 |
|------|-------------|---------|-----------|
| 误报率 | 高 | 中 | 低 |
| 占位符检测 | 无法区分 | 误报 | 81% F1 |
| 跨语言泛化 | 差 | 中 | 9/10 语言 F1>0.80 |

**优势与局限**：
- 优势：显著降低误报，跨语言泛化强
- 局限：需要 CodeBERT 推理，资源消耗较高；数据集仅 10 种语言

**未来发展潜力**：
- 扩展到更多编程语言
- 集成到 CI/CD 管道实现实时检测
- 结合上下文感知减少假阳性

**个人技术理解**：
三分类框架的核心洞察是"不是所有看起来像凭证的都是凭证"。将占位符/弱凭证作为独立类别而非简单过滤，既降低了误报又保持了安全覆盖。这种思路可以推广到其他代码安全检测场景。

#### 2. R+R: Java 安全 API 滥用复制研究

**项目介绍**：
对 Mousavi et al. 2024 年 Java 安全 API 滥用研究的复制与扩展。聚焦 JCA 和 JSSE API，评估 GPT-5.5 和 Llama-3.3-70B-Instruct。发现新模型表现更好但问题未消除；外部安全知识显著提升效果但模型依赖性强。Llama-3.3 最有效的是安全代码示例；GPT-5.5 中显式滥用模式消除了所有检测到的安全 API 滥用。

- RAG 增强生成 + 安全检测

**核心创新**：

**未来发展潜力**：
- 扩展到更多安全 API 类别

**个人技术理解**：
这项研究揭示了一个重要趋势：前沿模型（GPT-5.5）在安全 API 使用上显著优于早期模型，但问题仍未消除。更重要的是，RAG 知识的效果高度依赖模型能力——对弱模型有效的知识类型对强模型可能不是最优。这对 AI 辅助代码安全工具有直接指导意义。

### 学术论文

#### 3. ASP 抽象 RL（ICLP 2026）

**论文介绍**：
将 CARCASS 关系强化学习框架从 Prolog 迁移到 Answer-Set Programming (ASP)。CARCASS 由 Martijn van Otterlo 提出，利用逻辑表示在一阶域中建模 MDP。ASP 是比 Prolog 更丰富的完全声明式建模语言。在 Blocks World 和 Minigrid 两个领域评估，结果表明 CARCASS + ASP 为 RL 抽象构建提供了有前景的方法，尤其在有领域知识可用时。

**技术架构**：
- CARCASS 框架：关系表示 MDP
- ASP 实现：完全声明式建模
- 领域知识编码：对象、关系、动作抽象
- 评估：Blocks World（符号推理）+ Minigrid（视觉-动作）

**核心算法**：
- ASP 规则定义状态抽象
- 抽象 MDP 推导
- 策略学习与抽象反馈

**与现有技术对比**：
| 维度 | Prolog CARCASS | ASP CARCASS | 传统 RL |
|------|---------------|-------------|---------|
| 表达能力 | 中等 | 高 | 低 |
| 声明性 | 部分 | 完全 | 无 |
| 领域知识集成 | 中等 | 高 | 低 |

**优势与局限**：
- 优势：完全声明式，领域知识集成能力强
- 局限：ASP 求解器性能可能成为瓶颈

**未来发展潜力**：
- 扩展到更大规模领域
- 结合深度学习实现神经符号 RL
- 应用于实际机器人控制

**个人技术理解**：
ASP 的完全声明式特性使其比 Prolog 更适合 RL 抽象建模。声明式规则可以直接编码领域知识，避免了 Prolog 中常见的顺序依赖性。这对于需要强领域知识的 RL 应用（如机器人控制）有重要价值。

#### 4. CDAF 上下文依赖论证（FLoC 2026）

**论文介绍**：
扩展 Dung 论证理论，引入上下文依赖论证框架（CDAF）。defeat 函数根据上下文决定哪些攻击成功。通过相关性集合 ρ 和优先级 π 派生 defeat 函数。在示例中，目标论证在全相关性注入优先级下被拒绝，但在部分激活下被接受。定义 ACTIVATION-MANIPULATION 决策问题并记录基线复杂度界限。

**技术架构**：
- CDAF：扩展 Dung 框架
- defeat 函数：上下文依赖
- 相关性集合 ρ：Agent 动作空间
- 优先级 π：参数化 defeat 关系

**核心算法**：
- ACTIVATION-MANIPULATION 决策问题
- 基线复杂度分析
- 部分 vs 全相关性对比

**优势与局限**：
- 优势：为策略性论证提供形式化基础
- 局限：紧界限和多 Agent 变体留作未来工作

**未来发展潜力**：
- 多 Agent 变体分析
- 应用于实际多 Agent 系统
- 结合 LLM 实现自动化论证

**个人技术理解**：
CDAF 的核心创新是"论证的成功与否取决于上下文"。这打破了传统 Dung 框架中 defeat 关系固定的假设，为 Agent 策略性操纵论证结果提供了形式化工具。这对多 Agent 协商和安全分析有重要意义。

#### 5. SCALE 自改进 Web Agent

**论文介绍**：
提出 SCALE（Self-Cognitive-Aware Learning and Exploration），利用三个对抗角色（Selector、Predictor、Judger）自主发现 Agent 局限并通过环境探索扩展认知边界。提出 SCALE-Hop 图探索策略促进全局规划避免局部陷阱。构建 SCALE-20k 数据集（19 个真实网站）。显著提升多个 MLLM 在各种 Web 环境中的性能和泛化能力。

**技术架构**：
- Selector：选择探索目标
- Predictor：预测探索结果
- Judger：评估探索价值
- SCALE-Hop：图探索策略

**核心创新**：
- 无需专家轨迹的自主自改进
- 三对抗角色协作
- SCALE-20k 数据集

**优势与局限**：
- 优势：真正自主，无需专家数据
- 局限：探索成本可能较高

**未来发展潜力**：
- 扩展到更多 Web 环境
- 结合强化学习优化探索策略
- 应用于实际自动化任务

**个人技术理解**：
SCALE 的核心价值在于"无需专家轨迹"。当前大多数 Web Agent 依赖大量专家演示数据，SCALE 通过对抗角色实现自主探索，大幅降低了数据需求。这对 Web Agent 的实际部署有重要意义。
