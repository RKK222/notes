### 学术论文

#### 1. HarnessFix: From Failed Trajectories to Reliable LLM Agents

**地址**: https://arxiv.org/abs/2606.06324

**论文介绍**:
本文提出 HarnessFix，一个轨迹引导的框架，用于诊断 Agent 失败并修复 Harness 缺陷。现有自改进 Agent 和自动 Harness 进化方法主要通过运行时监督、提示优化、工作流搜索或基于最终结果的 Harness 修改来改进 Agent。然而，它们往往无法诊断失败轨迹中的责任证据位置，也无法确定哪层 Harness 导致不可靠行为，导致修改范围宽泛、间接或作用域不佳。HarnessFix 通过细粒度分层诊断解决这一问题。

**技术架构**:
- 轨迹引导的失败诊断：从执行轨迹中提取失败证据
- 分层 Harness 诊断：执行环境/工具接口/上下文/编排/可观测性/验证/治理
- 针对性修复建议：基于诊断结果生成精确修复
- 自动化验证：修复后自动验证效果

**核心算法**:
- 失败轨迹分析算法：定位失败证据
- 分层责任归因：确定问题所在 Harness 层
- 修复建议生成：基于问题类型生成针对性修复

**与现有技术对比**:
- 相比 SWE-bench 评估（只看结果），HarnessFix 诊断过程
- 相比 AutoGen/CrewAI 框架（固定架构），HarnessFix 支持动态修复
- 相比传统调试（人工），HarnessFix 自动化诊断和修复

**优势与局限**:
- 优势：首次将 Agent 失败归因到具体 Harness 层；自动化程度高
- 局限：依赖轨迹日志质量；修复建议需要人工审核

**发展潜力**:
可能成为 Agent 开发调试的标准工具，显著提升 Agent 可靠性。

#### 2. TLA-Prover: Verifiable TLA+ Specification Synthesis

**地址**: https://arxiv.org/abs/2606.06133

**论文介绍**:
TLA+ 是用于验证分布式系统和安全关键协议的形式化规格语言。LLM 经常产生语义上无法通过 TLC 模型检查器的 TLA+ 规格。本文提出 TLA-Prover，一个 20B 参数的 TLA+ 规格合成模型。训练结合 SFT（在已验证示例上）和基于修复的 GRPO（模型学习修复被拒规格）。还有从相同数据训练的 DPO 变体。在 25 个 LLM 中，最佳公开基线仅 26.6% 语法解析和 8.6% 语义模型检查。

**技术架构**:
- 20B 参数专用模型
- 三阶段训练：SFT → GRPO（自修复） → DPO
- TLC 模型检查器集成验证
- 修复循环：模型学习修复自己的错误规格

**核心算法**:
- GRPO（Group Relative Policy Optimization）：基于修复的强化学习
- 自修复机制：模型从 TLC 反馈中学习
- DPO 偏好优化：从相同数据训练

**与现有技术对比**:
- 相比通用 LLM（最佳 8.6% 语义正确），TLA-Prover 显著提升
- 相比手动编写 TLA+，大幅降低门槛
- 相比其他形式化方法（Lean/Coq），TLA+ 更侧重分布式系统

**优势与局限**:
- 优势：专用模型性能显著提升；自修复机制创新
- 局限：20B 参数规模较大；TLA+ 学习曲线陡峭

**发展潜力**:
可能成为分布式系统形式化验证的标配工具，降低形式化方法使用门槛。

#### 3. Scaffold, Not Vocabulary? Popperian Code-Generation Skill 控制研究

**地址**: https://arxiv.org/abs/2606.06454

**论文介绍**:
LLM 越来越多地编写、审查和判断代码，快速增长的实践为它们配备提示"技能"，要求模型像科学家一样推理。一个突出例子告诉模型充当 Popperian 证伪主义者，这些技能据报道能改进生成的代码。但这些增益几乎总是从 LLM-as-a-Judge 读取，这是一个有文档记录的位置/自偏好/风格偏差的工具。本文问：如果看起来有帮助，增益来自技能 Popperian 内容还是任何脚手架施加的结构？

**技术架构**:
- 预注册两阶段消融实验设计
- 三个控制组：长度匹配安慰剂 / 标签仅脚手架 / Popperian 内容
- 控制 LLM-as-a-Judge 偏差
- 统计显著性检验

**核心发现**:
- 分离脚手架结构与内容价值的实验设计
- 揭示"技能"提升可能来自结构而非内容
- 对 Agent 技能生态的评估方法论贡献

**与现有技术对比**:
- 相比以往技能评估（依赖 LLM Judge），本文使用控制实验
- 相比其他代码生成研究，本文关注"为什么有效"而非"是否有效"
- 对 mattpocock/skills 等技能生态的方法论挑战

**优势与局限**:
- 优势：严谨的实验设计；预注册减少偏差
- 局限：仅测试 Popperian 技能；需要更多技能类型验证

**发展潜力**:
可能推动 Agent 技能评估从"黑盒测试"走向"控制实验"，提升技能生态质量。

## 技术趋势

1. **Agent Harness 诊断与修复成为新刚需**：HarnessFix 首次将 Agent 失败归因到具体 Harness 层，标志着 Agent 开发从"能用"走向"可调试"。随着 Agent 框架（AutoGen/CrewAI/OpenClaw 等）普及，Harness 可靠性成为关键瓶颈。

2. **代码生成评估从"功能正确"扩展到"多维质量"**：SmellBench（可维护性）、Readability via RepE（可读性）、Asuka-Bench（模糊意图多轮交互）代表评估从单一 Pass@k 向多维质量扩展。代码生成进入"质量治理"时代。

3. **形式化验证规模化加速**：TLA-Prover（20B TLA+ 专用模型）和 Can LLMs Write Correct TLA+?（30 模型系统评估）代表形式化验证从手工走向自动化。GRPO 自修复机制是关键技术突破。
