### AI Testing / Software Testing（8 项）

1. **SmellBench: Towards Fine-Grained Evaluation of Code Agents on Refactoring Tasks** (arXiv 2606.05574, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05574
   - 领域：AI Testing / 代码重构质量
   - 技术介绍：SmellBench 是一个可扩展的代码重构基准，主动向真实仓库的干净代码片段注入代码异味。设计允许生成可控、高质量、多样化的重构用例，附带人工编写的 ground truth。包含 294 个用例，覆盖 7 种流行异味类型、3 个难度级别、2 种指令设置、7 个真实仓库。设计 3 个评估维度：功能正确性、定位能力、重构质量评估。实验：Qwen Code + Claude Sonnet 4.5 最佳组合仅得 50.34 分。分析揭示差距源于对局部代码异味的关注和对跨文件理解的缺乏。
   - 应用场景：Code Agent 重构能力评估、代码质量长期维护性测试
   - 为什么值得关注：现有基准主要评估功能正确性而非长期可维护性。SmellBench 填补了"Agent 生成的代码是否会产生技术债务"这一关键评估空白。

2. **Asuka-Bench: Benchmarking Code Agents on Underspecified User Intent and Multi-Round Refinement** (arXiv 2606.05920, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05920
   - 领域：AI Testing / 多轮代码生成
   - 技术介绍：Asuka-Bench 配对 underspecified 用户意图与多轮精炼，基于浏览器渲染行为。每个任务通过闭环解决：Code Agent 生成 Web 项目，UI Agent 在部署站点上执行测试用例，User LLM 将评估结果转化为自然语言反馈用于下一轮。包含 50 个 Web 任务、784 个评估标准、2402 个预期结果。基准 8 个 LLM 跨 2 个 Agent 框架。结果清晰分离模型：加权 Task Pass Rate 差异达 38 个百分点，模型在从反馈修复能力上也有显著差异。即使最强模型三轮后也只完成 52% 的项目。
   - 应用场景：Web 开发多轮 Agent 评估、用户意图渐进式理解
   - 为什么值得关注：真实 Web 开发中用户很少一开始就写出完整规范。Asuka-Bench 模拟了"看到中间结果后才明确需求"的真实场景。

3. **TLA-Prover: Verifiable TLA+ Specification Synthesis via Preference-Optimized LoRA** (arXiv 2606.06133, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06133
   - 领域：形式化验证 / TLA+ 规格合成
   - 技术介绍：TLA+ 是用于验证分布式系统和安全关键协议的形式规格语言。LLM 经常产生语义上无法通过 TLC 模型检查器的 TLA+ 规格。25 个 LLM 中最佳公开基线为 26.6% 语法解析和 8.6% 语义模型检查。TLA-Prover 是一个 20B 参数模型，训练结合 SFT（在已验证示例上）和基于修复的 GRPO。在 GRPO 阶段模型学习修复自己被拒绝的规格。TLC 直接提供奖励信号，无学习奖励模型。四级评分：Bronze（解析）、Silver（无警告）、Gold（通过 TLC）、Diamond（正确性属性被自动改变后 TLC 仍能检测违规）。TLA-Prover 在保持 30 问题基准上达到 30% pass@1 Gold 和 Diamond，约为未调优基线 8.6% 的 3.5 倍。
   - 应用场景：分布式系统形式化验证、安全关键协议规格生成
   - 为什么值得关注：TLA+ 在 Amazon/Microsoft 等工业界广泛使用，但编写正确规格需要专业知识和时间。TLA-Prover 将形式化验证的门槛大幅降低。

4. **Metamorphic Testing with the Rashomon Set: Explanation Faithfulness in ML** (arXiv 2606.06056, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06056
   - 领域：ML 测试 / 解释可信度
   - 技术介绍：多个 ML 模型可在同一任务上达到接近等效的预测性能，但提供不同的基于特征的解释。这被称为 Rashomon 效应，引发了"哪些解释是可信的"问题。提出基于 metamorphic testing 的框架，在不要求 ground-truth 标签的情况下评估解释的忠实度，通过探索后验解释方法的归因特征重要性。五个 metamorphic 关系形式化了模型行为与特征归因之间的预期一致性属性。应用于两个表格回归数据集和两个后验解释器（SHAP 和 LIME）。
   - 应用场景：ML 解释可信度评估、XAI 质量保障
   - 为什么值得关注：Rashomon 效应揭示了"性能相同的模型可能给出完全不同的解释"这一根本问题。Metamorphic testing 提供了一种无需 ground truth 的解决方案。

5. **Microskill Architecture: A Modular Skill-Driven Framework for AI-Native Code Generation** (arXiv 2606.05720, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05720
   - 领域：Agent 架构 / 知识封装
   - 技术介绍：将微服务设计理念应用于知识封装而非服务分解。将知识分区为原子、范围精确的技能胶囊，动态路由器仅选择语义相关的胶囊用于任务。将上下文分配建模为在 token 预算约束下对语义相关性的约束优化。实证案例研究：企业内容管理系统 15 个复杂功能，MicroSkill 将 token 消耗减少 90% 以上，首次编译成功率几乎翻倍，完全消除架构违规，并通过自学习机制自主提取和注册 7 个新技能胶囊。
   - 应用场景：AI 原生代码生成系统、大规模项目知识管理
   - 为什么值得关注：与 COLLEAGUE.SKILL 形成互补——MicroSkill 关注"如何高效路由知识"，COLLEAGUE.SKILL 关注"如何蒸馏专家知识"。

6. **Human Oversight and Overload: Two Hidden Burdens of AI-Assisted SE** (arXiv 2606.05770, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05770
   - 领域：SE 研究 / AI 辅助工程人类负担
   - 技术介绍：刻画两个常被忽视的负担：(1) 对 AI 生成工件持续的人类监督和检查需求；(2) 软件工程师从 AI 工具接收大量建议而产生的认知过载。人类监督不是可选的——工程师必须审查、验证、有时重新处理 AI 产生的内容。同时，AI 建议、提示和可能解决方案的洪流可能让开发者精神紧张。
   - 应用场景：AI 辅助软件工程的人因工程研究

7. **Enhancing SE Through Closed-Loop Memory Optimization** (arXiv 2606.05651, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05651
   - 领域：Agent 记忆 / SE 效率
   - 技术介绍：SE Agent 本质上是 episodic 的——它们无法跨任务保留、精炼和重用经验，反复从零构建上下文并重现类似错误。引入闭框架用于 SE Agent 记忆增强。将记忆效用建立在"已验证的下游影响"基础上，建立效用作为任务无关的评估基准和无标注的优化信号。单轮和跨轮记忆增强评估显示，成功率高提升 5.25%，解决效率提升 4.63%，计算成本降低 ≥9.79%。
   - 应用场景：SE Agent 跨任务记忆优化
   - 为什么值得关注：首次将"记忆效用"定义为可优化的信号而非启发式策略，为 Agent 记忆系统提供了评估基准和优化方向。

8. **The End of Software Engineering: How AI Agents Are Restructuring the Software Paradigm** (arXiv 2606.05608, cs.SE)
   - 来源：https://arxiv.org/abs/2606.05608
   - 领域：SE 范式 / Agent-as-a-Service

   - 应用场景：SE 范式转型理论框架
   - 为什么值得关注：从第一性原理分析复杂度扩展，论证"代码从决策逻辑载体变为临时工具"的范式转变，对行业有深远影响。

### AI / LLM 技术（12 项）

9. **SARDI: Self-Augmenting Retrieval for Diffusion Language Models** (arXiv 2606.06474, cs.AI)
   - 来源：https://arxiv.org/abs/2606.06474
   - 领域：RAG / 扩散语言模型
   - 技术介绍：离散扩散语言模型通过并行迭代去噪整个响应来生成文本。在每一步，它们为每个掩码位置预测 tentative tokens，将自信的预测提交到输出并丢弃不自信的。发现被丢弃的 tokens 实际上是检索增强生成的有用前瞻信号——即使低置信度 tokens 也经常在地去噪轨迹中早期突出显著实体，使在输出最终确定前检索更强证据成为可能。SARDI 利用这些前瞻 tokens 在去噪过程中引导检索。SARDI 无需训练、与检索器无关，适用于任何具有推理能力的离散扩散语言模型。在五个多跳 QA 基准上，SARDI 以高达 8x 的吞吐率超越当前无训练扩散和自回归检索基线。
   - 应用场景：扩散模型 RAG、多跳问答
   - 为什么值得关注：首次将扩散模型的去噪过程中的"丢弃信息"转化为检索信号，将通常被视为"噪声"的内容变为有用信息。

10. **NF-CoT: Latent Reasoning with Normalizing Flows** (arXiv 2606.06447, cs.AI)
    - 来源：https://arxiv.org/abs/2606.06447
    - 领域：潜推理 / 归一化流
    - 技术介绍：LLM 通常通过生成显式 CoT 来改善推理，证明中间计算的重要性。但文本 CoT 强制计算通过离散的、串行的、面向通信的 token 流进行。潜推理提供更高带宽的替代方案，在连续状态中进行中间计算再提交到文本。但现有潜推理方法往往牺牲了使 CoT 有效的关键优势：原生左到右生成、概率采样、KV-cache 解码兼容性、可处理的似然估计。NF-CoT 在 LLM 主干内实例化 TARFlow 风格归一化流，定义从显式 CoT 蒸馏的紧凑连续思维上的可处理概率模型。连续思维位置由 NF head 生成，文本位置由标准 LM head 在同一因果流中生成。在代码生成基准上，NF-CoT 在显著降低中间推理成本的同时提高了通过率。
    - 应用场景：高效推理、代码生成
    - 为什么值得关注：解决了潜推理与 KV-cache 解码兼容性的核心矛盾，使潜推理可以像显式 CoT 一样使用标准解码机制。

11. **RREDCoT: Segment-Level Reward Redistribution for Reasoning Models** (arXiv 2606.06475, cs.AI)
    - 来源：https://arxiv.org/abs/2606.06475
    - 领域：RL / 推理模型奖励分配
    - 技术介绍：推理语言模型的最新进展由 RL 微调驱动，通常依赖 GRPO 算法。最终答案只能在 CoT 轨迹完成后验证和分配奖励，造成延迟奖励问题。GRPO 对应标准 RL 中的蒙特卡洛方法，以高方差著称。RREDCoT 利用模型自身来近似最优奖励重新分配，无需额外生成。调查了与 MC 采样和几种归因方法相比的优势，分析了 CoT 轨迹分割和状态值估计等重新分配构建的关键方面。
    - 应用场景：推理模型 RL 训练、CoT 奖励分配
    - 为什么值得关注：将奖励分配从"轨迹级"细化到"段级"，且无需额外生成——用模型自身近似最优分配，大幅降低计算开销。

12. **CLSA: You Only Index Once - Cross-Layer Sparse Attention with Shared Routing** (arXiv 2606.06467, cs.AI)
    - 来源：https://arxiv.org/abs/2606.06467
    - 领域：稀疏注意力 / 推理优化
    - 技术介绍：现代 LLM 的长上下文推理受解码效率约束，尤其在推理密集型场景中。现有稀疏注意力方法面临效率-质量权衡。结构化块稀疏方法通常提供更强加速但质量损失明显，而 token 稀疏方法通常更准确但端到端加速有限。CLSA 构建在 YOCO 等 KV 共享架构之上，核心思想是不仅在跨解码器层共享 KV 缓存，还共享路由索引。单个索引器一次性计算 token 级 top-k 选择并在各层重用结果索引，同时保留 token 稀疏注意力的细粒度选择性并分摊路由开销。在 128K 上下文下实现高达 7.6x 解码加速和 17.1x 整体吞吐提升。
    - 应用场景：长上下文 LLM 推理、KV 共享架构优化
    - 为什么值得关注：与 RedKnot 和 Tangram 形成完整的 KV/注意力优化栈——CLSA 解决"路由重复计算"问题，RedKnot 解决"头级 KV 管理"，Tangram 解决"非均匀 KV 压缩"。

13. **PC Layer: Polynomial Weight Preconditioning for LLM Pre-Training** (arXiv 2606.06470, cs.AI)
    - 来源：https://arxiv.org/abs/2606.06470
    - 领域：LLM 训练 / 优化
    - 技术介绍：提出 PC 层——通过多项式预条件器的权重参数化，确保 LLM 训练期间稳定的权重条件。PC 模块通过低阶多项式预条件重塑权重矩阵的奇异值谱。训练后可将预条件权重合并回原始架构，无推理开销。在 Llama-1B 预训练中展示了相对于标准 Transformer 的优势，适用于 AdamW 和 Muon 优化器。理论上证明了均匀约束每层奇异值确保几何收敛到全局最小值。
    - 应用场景：LLM 预训练优化
    - 为什么值得关注：从优化角度解决权重矩阵条件数问题，训练后可合并回原始架构，零推理开销。

14. **DoPr: Double Preconditioning - Optimization for Test-Time Performance, not Validation Loss** (arXiv 2606.06418, cs.LG)
    - 来源：https://arxiv.org/abs/2606.06418
    - 领域：优化 / 测试时性能
    - 技术介绍：许多深度学习应用通过一步预测损失训练神经网络，但通过沿自身预测滚动来部署网络。这导致"测试时反馈"(TTF) 现象——训练/验证损失与下游指标之间的失配随任务长度增长。DoPr 结合梯度预条件（如 Adam/Muon）与激活预条件（如 KFAC），为 TTF 场景量身定制。激活预条件的添加产生即插即用的干预，在一系列 TTF 设置中提高下游模型性能。有趣的是，这些测试时性能提升并不一致伴随验证损失改善。
    - 应用场景：自回归语言建模、流生成模型、机器人策略学习
    - 为什么值得关注：揭示了"验证损失改善 ≠ 测试时性能改善"这一关键洞察，为 TTF 设置提供了新的优化设计轴。

15. **Agent Recusal Signal: Will the Agent Recuse Itself?** (arXiv 2606.06460, cs.AI)
    - 来源：https://arxiv.org/abs/2606.06460
    - 领域：Agent 治理 / 访问控制
    - 技术介绍：随着自主 LLM Agent 越来越多地持有真实凭证并无人类监督地操作基础设施，操作者没有标准方式告诉 Agent 某个资源是禁区。访问控制要么让 Agent 进入（有有效凭证）要么硬失败（与任何客户端无法区分）。提出第三种模式：轻量级的、发布的带内拒绝信号——Recusal Signal——服务器通过协议的现有通道（SSH banner、PostgreSQL NOTICE）发出，请求连接的自动化 Agent 自愿退出。这是合作治理控制，是 robots.txt 对实时访问的类比。在试点中（SSH；GPT-4o/GPT-4o-mini；Claude Code），信号干净地诱导退出——存在时 100% 退出 vs 无信号对照 100% 任务完成。
    - 应用场景：Agent 访问治理、基础设施安全
    - 为什么值得关注：提出了"自愿退出"这一全新的 Agent 治理范式，不是安全边界而是合作信号，对 Agent 基础设施安全有重要意义。

16. **TailLoR: Protecting Principal Components in Parameter-Efficient Continual Learning** (arXiv 2606.06494, cs.LG)
    - 来源：https://arxiv.org/abs/2606.06494
    - 领域：持续学习 / 参数高效微调

#### 1. Sandboxes - Self-Hosted Dev Sandboxes for Coding Agents
- **地址**：https://github.com/tastyeffectco/sandboxes
- **Stars**：449（快速增长中）
- **项目介绍**：Sandboxes 是一个自托管开发沙箱系统，专为编码 Agent 和 SaaS 工厂设计。一行命令即可启动带预览 URL 的完整开发环境，无需 Kubernetes。核心定位是"Agent 的代码执行沙箱"——为 AI 编码代理提供隔离、可预览、可销毁的执行环境。在 Agent-as-a-Service 范式下，每个 Agent 会话需要一个完整的开发环境来生成、测试和部署代码。Sandboxes 填补了"Agent 代码执行基础设施"这一空白。

- **未来发展潜力**：随着 Agent 编码的普及，沙箱基础设施将成为刚需。Sandboxes 的定位恰好处于这一趋势的核心。

#### 1. SARDI: Self-Augmenting Retrieval for Diffusion Language Models
- **地址**：https://arxiv.org/abs/2606.06474
- **论文介绍**（200-400 字）：SARDI 首次将扩散语言模型的去噪过程中的"丢弃 token"转化为检索信号。离散扩散模型在每一步为每个掩码位置预测 tentative tokens——自信的提交输出，不自信的丢弃。传统观点认为丢弃的 tokens 是"噪声"或"错误"，但 SARDI 发现这些低置信度 tokens 实际上经常在地去噪轨迹的早期阶段就突出显著实体。SARDI 利用这些前瞻 tokens 在去噪过程中引导检索，使模型能够在输出最终确定前检索更强证据。关键创新在于将扩散模型的去噪轨迹视为一个渐进的信息揭示过程，而非简单的"正确/错误"二元决策。SARDI 无需训练、与检索器无关，适用于任何具有推理能力的离散扩散语言模型。在五个多跳 QA 基准上，SARDI 以高达 8x 的吞吐率超越当前无训练扩散和自回归检索基线。这一发现对 RAG 系统有深远影响——扩散模型的去噪过程本身就是一个"渐进式信息检索"的线索源。
- **技术架构**：扩散模型去噪 → 丢弃 token 提取 → 前瞻信号识别 → 动态检索引导 → 增强证据注入
- **核心算法**：(1) 丢弃 token 前瞻提取——从去噪轨迹中提取被拒绝的 tentative tokens；(2) 显著性评分——评估丢弃 token 的信息价值；(3) 动态检索——在去噪过程中实时检索增强证据；(4) 证据注入——将检索结果融入后续去噪步骤。
- **与现有技术对比**：传统 RAG 在生成前一次性检索，SARDI 在去噪过程中渐进式检索。自回归模型的 RAG 是单向的（检索→生成），SARDI 是双向的（去噪↔检索）。
- **优势与局限**：优势是 8x 吞吐率、无需训练、与检索器无关；局限是仅适用于离散扩散模型，对连续扩散模型不直接适用。
- **未来发展潜力**：扩散语言模型正在成为自回归模型的有力竞争者。SARDI 为扩散模型的 RAG 提供了全新范式，可能推动扩散模型在需要外部知识的任务上的表现。

#### 2. NF-CoT: Latent Reasoning with Normalizing Flows
- **地址**：https://arxiv.org/abs/2606.06447
- **论文介绍**（200-400 字）：NF-CoT 解决了潜推理与 KV-cache 解码兼容性之间的核心矛盾。现有潜推理方法通常在连续状态中进行中间计算再提交到文本，但往往牺牲了使显式 CoT 有效的关键优势：原生左到右生成、概率采样、KV-cache 解码兼容性、可处理的似然估计。NF-CoT 在 LLM 主干内实例化 TARFlow 风格归一化流，定义从显式 CoT 蒸馏的紧凑连续思维上的可处理概率模型。连续思维位置由 NF head 生成，文本位置由标准 LM head 在同一因果流中生成。这一设计提供了连续思维的精确似然、启用原始 KV cache 的概率左到右解码、并支持潜推理空间中的直接策略梯度优化。关键洞察是：潜推理不需要放弃自回归解码的便利性——归一化流可以在同一因果流中同时建模连续思维和离散文本。在代码生成基准上，NF-CoT 在显著降低中间推理成本的同时提高了通过率。
- **技术架构**：LLM 主干 → NF head（连续思维）+ LM head（文本）→ 统一因果流 → KV-cache 兼容解码
- **核心算法**：(1) TARFlow 归一化流实例化——在 LLM 内部定义连续思维的概率模型；(2) 双 head 生成——NF head 生成连续思维，LM head 生成文本；(3) 因果流统一——连续思维和文本在同一因果流中生成；(4) 策略梯度优化——在潜推理空间直接优化。
- **与现有技术对比**：传统潜推理方法需要特殊的解码机制（如并行生成或特殊的注意力掩码），NF-CoT 使用标准 KV-cache 解码。这使得潜推理可以无缝集成到现有推理框架中。
- **优势与局限**：优势是 KV-cache 兼容、精确似然、策略梯度优化；局限是归一化流的计算开销和训练复杂度。
- **未来发展潜力**：潜推理是推理效率优化的关键方向。NF-CoT 使潜推理可以与现有推理基础设施无缝集成，可能推动潜推理的广泛应用。

#### 3. SmellBench: Fine-Grained Evaluation of Code Agents on Refactoring
- **地址**：https://arxiv.org/abs/2606.05574
- **论文介绍**（200-400 字）：SmellBench 是首个系统性评估 Code Agent 代码重构能力的基准。现有基准主要评估功能正确性（Agent 能否完成任务），但忽视了长期可维护性（Agent 生成的代码是否会产生技术债务）。SmellBench 主动向真实仓库的干净代码注入代码异味，生成可控、高质量、多样化的重构用例。包含 294 个用例，覆盖 7 种流行异味类型（如 Long Method、Large Class、Feature Envy 等）、3 个难度级别、2 种指令设置、7 个真实仓库。三个评估维度：功能正确性（重构后代码是否仍能通过测试）、定位能力（Agent 能否准确定位异味位置）、重构质量评估（重构后的代码是否真正消除了异味）。实验结果令人担忧——最佳组合 Qwen Code + Claude Sonnet 4.5 仅得 50.34 分。分析揭示差距源于 Agent 对局部代码异味的关注和对跨文件理解的缺乏。这意味着当前最强的 Code Agent 在重构复杂代码库时仍然存在根本性局限。
- **技术架构**：代码异味注入 → 重构用例生成 → 三维度评估（功能/定位/质量）
- **核心算法**：(1) 可控异味注入——在干净代码中精确注入特定类型的代码异味；(2) 人工 ground truth——每个用例附带人工编写的正确重构方案；(3) 三维度评估——功能正确性、定位能力、重构质量。
- **与现有技术对比**：SWE-bench 评估"修复 bug"，SmellBench 评估"消除技术债务"。两者互补——一个关注功能正确性，一个关注代码质量。
- **优势与局限**：优势是可控、高质量、多样化；局限是仅覆盖 7 种异味类型，尚未覆盖所有常见异味。
- **未来发展潜力**：随着 Code Agent 的普及，"Agent 生成的代码质量"将成为关键问题。SmellBench 为这一评估提供了基础设施，可能推动 Code Agent 在代码质量方面的改进。

## 技术趋势

1. **Agent 执行基础设施从"工具调用"到"完整沙箱"**：Sandboxes（449 stars）和 Ghidra RPC 代表 Agent 基础设施从简单的"工具调用"扩展到"完整执行环境"。Agent 不仅需要 API 调用能力，还需要隔离的、可预览的、可销毁的执行沙箱。这是 Agent-as-a-Service 范式的必然需求。

2. **推理优化从"显式 CoT"到"潜推理+归一化流"**：NF-CoT 将归一化流引入潜推理，解决了潜推理与 KV-cache 解码的兼容性问题。结合 ReLAT 的闭环潜推理，推理优化正在从"鼓励短轨迹"升级到"验证潜推理保真度+保持解码兼容性"。

3. **扩散模型 RAG 从"一次性检索"到"渐进式检索"**：SARDI 利用扩散模型去噪过程中的丢弃 token 作为检索前瞻信号，实现渐进式检索。这为扩散语言模型在需要外部知识的任务上提供了全新的 RAG 范式。

## 新领域收获

1. **无线 JEPA 基础模型**（LatentWave）：JEPA 架构从视觉扩展到无线通信领域。per-channel patch embedding 使其能处理可变天线配置，在 RF 信号分类、5G NR 定位、波束预测等任务上展现迁移能力。这是一个全新的 JEPA 应用领域。

2. **音频理解+生成统一分词器**（F3-Tokenizer）：解决音频 AI 中长期存在的"理解 vs 生成"表征鸿沟。连续自编码器适合生成但理解结构弱，自监督编码器适合理解但不能直接解码。F3-Tokenizer 通过噪声正则化瓶颈+潜侧表示编码器实现统一。

3. **Agent 自愿退出信号**（Recusal Signal）：提出"robots.txt 对实时访问的类比"——服务器通过协议内 deny 信号（SSH banner、PostgreSQL NOTICE）请求 Agent 自愿退出。100% 退出率（有信号）vs 100% 任务完成（无信号）。这是 Agent 治理的全新范式。
