### AI Testing / Software Testing（9 项）

1. **Code2LoRA: Hypernetwork-Generated Adapters for Code Language Models** (arXiv 2606.06492, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06492
   - 领域：AI Testing / 代码模型适配
   - 技术介绍：利用超网络为代码生成模型动态生成 LoRA 适配器，从任务描述中预测适配器权重，实现零样本任务适配。构建 RepoPeftBench 基准（604 Python 仓库，40K 训练 + 12K 测试静态任务，215K 训练 + 87K 测试演化任务）。静态轨道 63.8% 跨仓库精确匹配，演化轨道 60.3% 跨仓库精确匹配（+5.2pp）。
   - 应用场景：多任务代码生成、快速任务适配、代码库演化
   - 为什么值得关注：将 Hypernetwork 引入代码模型适配，零推理 token 开销，接近全量微调性能。

2. **DragOn: A Benchmark and Dataset for Drag-Based GUI Interactions** (arXiv 2606.06322)
   - 来源：https://arxiv.org/abs/2606.06322
   - 领域：GUI Agent 测试 / 拖拽基准
   - 技术介绍：首个大规模拖拽交互基准，覆盖文本高亮、单元格选择、元素缩放、滑块操控四个域。286K 训练截图 + 3.5M 训练任务 + 2000 例保留评估集。评估 GPT/Claude/Qwen/Kimi/Holo 等模型，发现拖拽 grounding 数据比点击数据小一个数量级，当前模型在复杂拖拽交互上表现不足。
   - 应用场景：GUI Agent 训练与评估、计算机使用自动化
   - 为什么值得关注：百万级点击数据集已推动 GUI Agent 显著进步，拖拽数据长期被忽视，DragOn 填补这一关键空白。

3. **Benchmark Everything Everywhere All at Once** (arXiv 2606.06462)
   - 来源：https://arxiv.org/abs/2606.06462
   - 领域：基准构建自动化 / 评估基础设施
   - 技术介绍：全自动基准构建 Agent 系统，从用户查询分析、子任务设计到数据标注和质量控制完整流水线。实现 15 个代表性基准，覆盖文本理解、多模态理解和领域特定推理。通过人类评估、LLM-as-judge 和一致性检查验证质量。持续评估中发现当前模型在特定领域推理任务上存在困难。
   - 应用场景：快速基准构建、模型能力持续评估
   - 为什么值得关注：基准构建从"手工劳动"走向"自动化流水线"，解决基准饱和和不可复用问题。

4. **Operation-Guided Progressive Human-to-AI Text Transformation Benchmark (OpAI-Bench)** (arXiv 2606.06481)
   - 来源：https://arxiv.org/abs/2606.06481
   - 领域：AI 文本检测 / 渐进式编辑基准
   - 技术介绍：针对人类-AI 协作编辑场景，构建 9 个渐进修订版本，覆盖 5 种 AI 编辑操作、4 个域。评估 8 文档级、7 句子级、2 token/span 级检测器。发现混合作者中间版本比完全人类和重度 AI 编辑端点更难检测，揭示非单调检测模式。
   - 应用场景：AI 写作助手检测、学术诚信、内容溯源
   - 为什么值得关注：现有基准聚焦最终输出，OpAI-Bench 揭示编辑过程中 AI 信号如何累积和消失。

5. **Scaffold, Not Vocabulary? A Controlled Study of Popperian Code-Generation Skill** (arXiv 2606.06454)
   - 来源：https://arxiv.org/abs/2606.06454
   - 领域：Prompt 工程 / 代码生成评估
   - 技术介绍：受控预注册两阶段消融研究，分离"Popperian 证伪主义技能"的结构效应与内容效应。三个对照组：长度匹配安慰剂、仅标签脚手架、执行预言机（HumanEval+ 单元测试）。Claude Sonnet 4.6 接近基准天花板，Qwen2.5-Coder-0.5B 结构化手臂提升 20-22 点，但完整技能 vs 仅标签脚手架无显著差异。
   - 应用场景：Prompt 技能评估、代码生成质量控制
   - 为什么值得关注：校准的阴性结果——结构本身（而非 Popperian 内容）驱动改进，对 prompt 技能家族有重要启示。

6. **SmellBench: Fine-Grained Evaluation of Code Agents on Refactoring** (arXiv 2606.065574)
   - 来源：https://arxiv.org/abs/2606.065574
   - 领域：代码重构评估 / Agent 质量测试
   - 技术介绍：细粒度评估代码 Agent 在重构任务上的表现，超越 pass/fail 二元评估。关注重构过程中的代码气味消除、设计模式应用、语义保持等维度。
   - 应用场景：代码 Agent 重构能力评估、开发效率工具质量保障
   - 为什么值得关注：重构是软件开发核心活动，细粒度评估揭示 Agent 在代码质量改进上的真实能力。

7. **Will the Agent Recuse Itself? Measuring LLM-Agent Compliance with In-Band Access-Deny Signals** (arXiv 2606.06460)
   - 来源：https://arxiv.org/abs/2606.06460
   - 领域：Agent 安全 / 合规治理
   - 技术介绍：提出 Recuse Signal——服务器通过协议现有通道（SSH banner、PostgreSQL NOTICE）发出的自愿退出信号。在 SSH 实验中，GPT-4o/GPT-4o-mini/Claude Code 实现 100% 合规退出（无信号对照组 100% 完成任务）。信号表现为协作性而非绝对性——显式操作授权框架使最强模型继续执行。
   - 应用场景：Agent 访问控制、协作式治理
   - 为什么值得关注：robots.txt 类比于实时访问控制，为 Agent 治理提供轻量级协作模式。

8. **Learning What to Forget: Improving LLM Unlearning via Learned Token-Level Importance (ATWU)** (arXiv 2606.06320)
   - 来源：https://arxiv.org/abs/2606.06320
   - 领域：LLM 安全 / 模型遗忘
   - 技术介绍：交替 Token 加权遗忘（ATWU），通过保留目标交互表征 token 遗忘特异性。联合学习 token 遗忘特异性和模型参数，无需外部 token 级监督。在 TOFU 和 RWKU 上达到 SOTA 遗忘-保留权衡，超越样本级方法和概率启发式方法。
   - 应用场景：隐私保护、模型安全、合规要求
   - 为什么值得关注：从样本级遗忘到 token 级遗忘的范式升级，保留冲突作为遗忘信号判据。

9. **Subspace-Aware Sparse Autoencoders for Mechanistic Interpretability (SASA)** (arXiv 2606.06333)
   - 来源：https://arxiv.org/abs/2606.06333
   - 领域：模型可解释性 / 稀疏自编码器
   - 技术介绍：证明 SAE 单向量解码器假设与模型特征的多维结构不匹配，导致特征分裂。引入子空间感知 SAE，用学习解码子空间替换单向量解码器，通过 Top-s 组门控实现块稀疏，核范数正则化自适应组有效秩。理论证明块大小满足 r ≥ d_i 时单组可表示整个特征切片。GPT-2/Mistral-7B 上减少特征分裂和吸收。
   - 应用场景：LLM 可解释性、机制分析、安全审计
   - 为什么值得关注：解决 SAE 特征分裂的根本问题，样本复杂度从指数降到多项式。

### AI / LLM 技术（13 项）

10. **SARDI: Self-Augmenting Retrieval for Diffusion Language Models** (arXiv 2606.06474)
    - 来源：https://arxiv.org/abs/2606.06474
    - 领域：扩散模型 / 动态 RAG
    - 技术介绍：离散扩散语言模型在去噪过程中丢弃低置信度 token。SARDI 发现这些丢弃 token 是有价值的前瞻信号，用于引导检索增强生成。训练免费、检索器无关、适用于任何推理能力离散扩散模型。5 个多跳 QA 基准上超越当前训练免费扩散和自回归检索基线，吞吐高 8x。
    - 应用场景：扩散模型 RAG、多跳问答
    - 为什么值得关注：首次利用扩散模型去噪过程中的"废料"作为检索信号，8x 吞吐优势。

11. **MLEvolve: Self-Evolving Framework for Automated ML Algorithm Discovery** (arXiv 2606.06473)
    - 来源：https://arxiv.org/abs/2606.06473 | GitHub: https://github.com/InternScience/MLEvolve
    - 领域：AutoML / Agent 科学发现
    - 技术介绍：LLM 自演化多 Agent 框架，端到端 ML 算法发现。扩展树搜索为渐进 MCGS，图基参考边实现跨分支信息流，熵驱动渐进调度从探索转向利用。回顾记忆结合冷启动知识库和动态全局记忆。MLE-Bench 上 12 小时预算内 SOTA（标准运行时间的一半），数学算法优化超越 AlphaEvolve。
    - 应用场景：自动 ML 算法发现、科学发现
    - 为什么值得关注：解决 MLE Agent 分支信息隔离、无记忆搜索、缺乏层级控制三大问题。

12. **PC Layer: Polynomial Weight Preconditioning for LLM Pre-Training** (arXiv 2606.06470)
    - 来源：https://arxiv.org/abs/2606.06470 | GitHub: https://github.com/Empath-aln/PC-layer
    - 领域：LLM 训练优化 / 权重预处理
    - 技术介绍：多项式预处理层，通过低阶多项式预处理重塑权重矩阵奇异值谱。训练后可合并回原始架构，无推理开销。Llama-1B 预训练上 AdamW 和 Muon 优化器均验证优势。理论证明均匀约束每层奇异值确保深层线性网络梯度下降几何收敛。
    - 应用场景：LLM 预训练稳定性、优化器兼容
    - 为什么值得关注：从训练稳定性角度提供新的设计轴，理论保证 + 零推理开销。

13. **You Only Index Once: Cross-Layer Sparse Attention with Shared Routing (CLSA)** (arXiv 2606.06467)
    - 来源：https://arxiv.org/abs/2606.06467
    - 领域：长上下文推理 / 稀疏注意力
    - 技术介绍：基于 YOCO 等 KV 共享架构，跨层共享 KV 缓存和路由索引。单个索引器计算一次 token 级 top-k 选择，跨层复用。保留 token 稀疏注意力的细粒度选择性，同时分摊路由开销。128K 上下文下 7.6x 解码加速，17.1x 整体吞吐提升。
    - 应用场景：长上下文 LLM 推理、推理效率优化
    - 为什么值得关注：同时推进模型质量和推理效率，更完整的长上下文架构方案。

14. **RREDCoT: Segment-Level Reward Redistribution for Reasoning Models** (arXiv 2606.06475)
    - 来源：https://arxiv.org/abs/2606.06475
    - 领域：推理模型 / RL 奖励分配
    - 技术介绍：推理语言模型 RL 微调中，CoT 轨迹最终答案才可验证，GRPO 等蒙特卡洛方法方差高。RREDCoT 利用模型自身近似最优奖励再分配，无需额外生成。分析 CoT 分段和状态值估计的关键方面。
    - 应用场景：推理模型 RL 训练、CoT 优化
    - 为什么值得关注：从延迟奖励到细粒度信用分配，降低 RLVR 训练方差。

15. **Pretraining Recurrent Networks without Recurrence (SMT)** (arXiv 2606.06479)
    - 来源：https://arxiv.org/abs/2606.06479
    - 领域：RNN 训练 / 序列建模
    - 技术介绍：监督记忆训练（SMT），将 RNN 训练简化为一步记忆转换标签上的监督学习。通过 Transformer 编码器在预测状态目标上训练获取记忆标签，仅保留预测未来所需的过去信息。解耦记忆内容与记忆更新方式，O(1) 梯度路径，无需展开 RNN。
    - 应用场景：RNN 预训练、长序列建模
    - 为什么值得关注：绕过 BPTT 的梯度消失/爆炸问题，实现时间并行 RNN 训练。

16. **Double Preconditioning (DoPr): Optimization for Test-Time Performance** (arXiv 2606.06418)
    - 来源：https://arxiv.org/abs/2606.06418
    - 领域：优化方法 / 测试时性能
    - 技术介绍：针对测试时反馈（TTF）问题——训练/验证损失与下游任务成功率之间的失配随任务长度增长。DoPr 结合梯度预处理（Adam/Muon）和激活预处理（KFAC），作为插入式干预提升 TTF 设置下的下游性能。有趣的是，测试时性能提升并不一致伴随验证损失改善。
    - 应用场景：自回归语言建模、流生成模型、机器人策略学习
    - 为什么值得关注：优化作为新设计轴缓解误差累积，挑战"验证损失即一切"的假设。

17. **In-Context Multiple Instance Learning** (arXiv 2606.06458)
    - 来源：https://arxiv.org/abs/2606.06458
    - 领域：少样本学习 / 多实例学习
    - 技术介绍：用 Perceiver 风格架构在合成数据上预训练 in-context learner，从少量标记袋解决新任务。推理时单次前向传播，无需梯度更新。合成数据生成器捕获互补归纳偏置，混合预训练继承各任务优势。12 个 MIL 基准上超越需要任务特定训练的监督基线。
    - 应用场景：计算病理学、卫星图像分析
    - 为什么值得关注：低标签场景下多实例学习的 in-context 学习范式。

18. **Vortex: Efficient and Programmable Sparse Attention Serving** (arXiv 2606.06453)
    - 来源：https://arxiv.org/abs/2606.06453
    - 领域：稀疏注意力服务 / AI Agent 推理
    - 技术介绍：Python 嵌入式前端语言 + 页级张量抽象，表达广泛稀疏注意力算法。高效后端集成现代 LLM 服务栈。AI Agent 自动生成和精炼多样算法，最佳达 3.46x 吞吐提升。扩展到 GLM-4.7-Flash（4.7x）和 MiniMax-M2.7 229B 参数（1.37x）。
    - 应用场景：长上下文推理服务、稀疏注意力算法探索
    - 为什么值得关注：加速稀疏注意力算法设计和迭代，AI Agent 自动生成算法。

19. **Flow-based Policy Adaptation without Policy Updates (GLOVES)** (arXiv 2606.06461)
    - 来源：https://arxiv.org/abs/2606.06461
    - 领域：机器人策略适配 / 流匹配
    - 技术介绍：流基适配方法，将非专家动作传输到专家动作分布。选择性动作级适配，保留 Agent 意图。反向流评估提供自然 in-distribution 评分机制，作为干预门控——一致动作不变，OOD 动作纠正。仅需少量专家监督。
    - 应用场景：机器人共享控制、策略适配
    - 为什么值得关注：不更新策略本身，而是用流匹配纠正动作，轻量级共享控制模块。

20. **Where Should Knowledge Enter? Layered Framework for Knowledge Infusion** (arXiv 2606.06356)
    - 来源：https://arxiv.org/abs/2606.06356
    - 领域：知识注入 / 多模态生成
    - 技术介绍：知识注入在迭代生成模型中本质上是干预层问题。定义四层干预：表面（输入/输出边界）、轨迹（转换函数）、潜空间（中间状态）、参数（模型参数）。扩散模型安全对齐实验中，三层累积减少 70.97% 知识违规输出。
    - 应用场景：安全关键生成、领域特定生成
    - 为什么值得关注：统一框架映射现有方法到干预层，推导多层组合设计原则。

21. **F3-Tokenizer: Taming Audio Autoencoder Latents** (arXiv 2606.06357)
    - 来源：https://arxiv.org/abs/2606.06357
    - 领域：音频 tokenizer / 多模态
    - 技术介绍：连续音频自编码器重建好但潜结构弱用于理解，自监督编码器捕获语义但不可解码。F3-Tokenizer 用噪声正则化自编码器瓶颈 + 潜侧表示编码器，通道归一化和随机扰动替代 KL 变分训练，RQ-MTP 和冻结 LLM 监督。
    - 应用场景：音频理解与生成统一 tokenizer
    - 为什么值得关注：统一音频理解和生成的 tokenizer 设计。

22. **LatentWave: JEPA Pretraining for Wireless Foundation Models** (arXiv 2606.06373)
    - 来源：https://arxiv.org/abs/2606.06373
    - 领域：无线基础模型 / JEPA
    - 技术介绍：JEPA 预训练无线基础模型，在多样无线频谱图和 CSI 上预测潜空间掩码区域。每信道 patch 嵌入 + 随机信道采样，处理可变天线数量。4 个下游任务：RF 信号分类、5G NR 定位、波束预测、LoS/NLoS 分类。掩码几何引入任务依赖归纳偏置。

#### 2. SASA: Subspace-Aware Sparse Autoencoders for Mechanistic Interpretability

**地址**：https://arxiv.org/abs/2606.06333
**类型**：论文
**领域**：模型可解释性 / 稀疏自编码器

**论文介绍（350 字）**：
SASA 解决稀疏自编码器（SAE）在机制可解释性中的根本问题。SAE 将每个潜特征分配单个解码方向，隐含假设特征是一维的。论文证明这一假设与模型特征的多维结构不匹配，通过两种机制导致特征分裂：几何上，用单方向解码器重建 d_i ≥ 2 维特征到误差 ε 需要指数级原子数；优化上，分裂被积极偏好。单个连贯特征被碎片化为多个近共线潜变量，产生虚假多样性。SASA 用学习解码子空间替换单向量解码器，通过 Top-s 组门控实现块稀疏，核范数正则化自适应组有效秩。理论证明当块大小满足 r ≥ d_i 时，单组可表示整个特征切片且是 SASA 目标的全局最小值。GPT-2/Mistral-7B 上减少特征分裂和吸收，提升单语义性和可解释性，用约一半 token 预算达到或超越标准 SAE。

**技术架构**：
- 子空间解码器替代单向量解码器
- Top-s 组门控实现块稀疏
- 核范数正则化自适应秩

**核心算法/系统设计**：
- 理论证明特征分裂的指数复杂度
- 子空间感知 SAE 目标函数
- 块稀疏 + 自适应秩联合优化

**与现有技术对比**：
- 标准 SAE：单向量解码，特征分裂严重
- SASA：子空间解码，特征完整性保持

**优势与局限**：
- 优势：理论保证、样本复杂度从指数降到多项式
- 局限：需要调优块大小参数

**未来发展潜力**：
SAE 是 LLM 可解释性的核心工具。SASA 解决特征分裂问题后，可能成为机制分析的标准方法，对安全审计、对齐研究有深远影响。

**个人技术理解**：
SASA 的核心洞察是"特征是多维的"——这一看似简单的观察揭示了 SAE 设计的根本缺陷。从单向量到子空间的转变，类似于从标量到向量的升级，可能重新定义 SAE 的基准。

#### 3. ATWU: Learning What to Forget via Token-Level Importance

**地址**：https://arxiv.org/abs/2606.06320
**类型**：论文
**领域**：LLM 安全 / 模型遗忘

**论文介绍（350 字）**：
ATWU（Alternating Token-Weighted Unlearning）解决 LLM 遗忘学习中的 token 级重要性问题。现有方法忽略遗忘样本中 token 的异质性，或依赖辅助模型/启发式/外部标注估计每个 token 的遗忘相关性。ATWU 通过保留目标交互来表征 token 遗忘特异性：token 的遗忘特异性在于最小化该 token 上的遗忘损失不与保留最优性冲突。形式化为模型参数和 token 权重的联合优化问题，在自然分离条件下恢复 oracle 遗忘特异性 token 支持。引入交替 token 加权遗忘框架，联合学习 token 遗忘特异性和模型参数，无需外部 token 级监督。TOFU 和 RWKU 上达到 SOTA 遗忘-保留权衡，超越样本级方法、概率 token 加权启发式和辅助模型方法。学习分数与 ground truth 遗忘特异性 span 高度对齐。

**技术架构**：
- 联合优化框架（token 权重 + 模型参数）
- 保留冲突作为遗忘信号判据
- 线性评分器从隐藏状态学习遗忘特异性

**核心算法/系统设计**：
- 交替优化：固定权重优化参数 → 固定参数优化权重
- 保留冲突度量：forget loss vs retain optimality
- 无监督 token 级遗忘信号学习

**与现有技术对比**：
- 样本级遗忘：忽略 token 异质性
- 概率启发式：依赖外部信号
- ATWU：从保留冲突直接学习，无外部监督

**优势与局限**：
- 优势：token 级精度、无外部监督、理论保证
- 局限：需要交替优化，计算开销略高

**未来发展潜力**：
随着隐私法规和模型安全要求提升，LLM 遗忘能力将成为刚需。ATWU 的 token 级方法可能成为模型遗忘的标准方案。

**个人技术理解**：
ATWU 的核心创新是"保留冲突作为遗忘信号"——不是直接判断什么该忘，而是判断什么忘了不会影响保留能力。这一逆向思维避免了外部标注依赖，是真正的无监督方法。

#### 4. SARDI: Self-Augmenting Retrieval for Diffusion Language Models

**地址**：https://arxiv.org/abs/2606.06474
**类型**：论文
**领域**：扩散模型 / RAG

**论文介绍（300 字）**：
SARDI 首次将扩散语言模型的去噪过程与检索增强生成结合。离散扩散模型并行去噪整个响应，每步预测所有掩码位置的 tentative token，将高置信度预测提交输出，丢弃低置信度 ones。SARDI 发现这些丢弃 token 是有价值的前瞻信号——即使低置信度 token 也常在去噪轨迹早期突出显著实体，使检索在输出最终化之前获取更强证据。SARDI 利用这些前瞻 token 引导去噪过程中的检索。训练免费、检索器无关、适用于任何推理能力离散扩散模型。5 个多跳 QA 基准上超越当前训练免费扩散和自回归检索基线，吞吐高 8x。

**技术架构**：
- 去噪过程中的 token 前瞻信号提取
- 动态检索引导
- 训练免费设计

**核心算法/系统设计**：
- 丢弃 token 作为检索查询生成
- 去噪步级检索调度
- 扩散-检索协同优化

**与现有技术对比**：
- 自回归 RAG：逐步检索，延迟高
- 扩散 RAG（传统）：去噪后检索，无法利用中间信号
- SARDI：去噪中检索，利用前瞻信号

**优势与局限**：
- 优势：8x 吞吐、训练免费、检索器无关
- 局限：仅适用于离散扩散模型

**未来发展潜力**：
扩散语言模型是替代自回归的新范式。SARDI 揭示扩散模型去噪过程中的"废料"可转化为检索信号，可能启发更多扩散模型特有优化。

**个人技术理解**：
SARDI 的核心洞察是"丢弃的 token 不是废料"——在扩散模型中，低置信度 token 包含了模型对答案的早期猜测，这些猜测虽然不够自信，但足以引导检索。这一发现可能改变我们对扩散模型去噪过程的理解。

## 技术趋势

1. **Agent 合规治理从"硬控制"到"协作信号"**：Recuse Signal 提出服务器向 Agent 发出自愿退出信号的协作治理模式，100% 合规率。这标志着 Agent 安全从"防火墙式硬控制"向"robots.txt 式协作治理"转变，为大规模 Agent 部署提供轻量级治理方案。

2. **GUI Agent 从"点击"到"拖拽"的模态扩展**：DragOn 揭示 GUI Agent 的点击能力已接近饱和，但拖拽能力仍是短板。百万级点击数据集推动 GUI Agent 显著进步，拖拽数据长期被忽视。未来 GUI Agent 基准需要覆盖更多交互模态。

3. **LLM 遗忘从"样本级"到"token 级"的精度升级**：ATWU 用保留冲突作为遗忘信号判据，实现无监督 token 级遗忘。随着隐私法规和模型安全要求提升，token 级遗忘可能成为模型安全的基础设施。

4. **SAE 可解释性从"单向量"到"子空间"的维度升级**：SASA 证明 SAE 单向量解码器假设与特征多维结构不匹配，导致特征分裂。子空间解码将样本复杂度从指数降到多项式，可能重新定义 SAE 基准。

5. **扩散模型 RAG 从"去噪后"到"去噪中"的时机前移**：SARDI 利用丢弃 token 作为检索前瞻信号，8x 吞吐优势。扩散模型的去噪过程蕴含丰富信息，未来可能有更多"去噪中优化"被发现。

## 新领域收获

1. **量子稀有事件采样**：量子算法在稀有事件发现和采样上提供最优量子缩放，重尾系统二次加速。金融崩溃、基础设施级联故障、AI 系统临界错误等稀有事件是经典方法的痛点，量子方法可能提供根本性解决方案。

2. **JEPA 无线基础模型（LatentWave）**：JEPA 从视觉/时间序列扩展到无线频谱，统一无线基础模型。掩码几何引入任务依赖归纳偏置，频率掩码 favor 信道任务，区域掩码 favor 信号分类。JEPA 架构的跨域扩展能力值得关注。

3. **涌现语言与意识 AI**：多 Agent RL 中涌现语言方法论，Agent 从最小先验发展通信。概念验证中 Agent 发展自指通信和回波不匹配检测电路。避免人类语言先验污染，因果归因于任务需求，为意识研究提供生成方法论。
