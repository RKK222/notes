## 技术扫描（32 项）

### AI Testing / Software Testing（9 项）

1. **MAGE: Memory as Agent-Guided Exploration** (arXiv 2606.06090)
   - 来源：https://arxiv.org/abs/2606.06090
   - 领域：Agent 记忆 / 执行状态管理
   - 技术介绍：提出 MAGE（Memory as Agent-Guided Exploration），将记忆从语义检索系统重构为主动执行状态管理器。记忆存储在分层状态树中，Agent 从根到当前路径派生状态，结合子目标摘要、近期轨迹和先前分支提示。四个耦合操作维护树：Grow 记录新轨迹、Compress 摘要完成子目标、Maintain 验证状态一致性、Prune 移除无效分支。解决 RAG 系统碎片化决策轨迹、混合有效和错误轨迹的问题。
   - 应用场景：长周期 Agent 任务、多步推理、错误隔离
   - 为什么值得关注：从"语义组织"到"执行状态管理"的范式转变，对 Agent 架构有根本性影响。

2. **TokenMizer: Graph-Structured Session Memory** (arXiv 2606.06337)
   - 来源：https://arxiv.org/abs/2606.06337
   - 领域：Agent 记忆 / 图结构会话管理
   - 技术介绍：开源代理系统，将 LLM 会话历史建模为类型化知识图谱。14 种节点类型 + 7 种边类型。混合提取管道增量填充图，三层检查点系统序列化为紧凑恢复块。8 层压缩管道减少上下文开销，语义缓存降低重复查询延迟。21 个会话基准测试，恢复块平均 78 tokens（42-124），比基线小 2x，决策回忆率 +9-17pp。
   - 应用场景：长会话 LLM 上下文管理、会话恢复
   - 为什么值得关注：保留"为什么"而不仅是"说了什么"，对 Agent 可恢复性有重要意义。

3. **MemGate: Trustworthy Memory Search for Personal AI Agents** (arXiv 2606.06054)
   - 来源：https://arxiv.org/abs/2606.06054
   - 领域：Agent 安全 / 记忆可信搜索
   - 技术介绍：将记忆搜索视为个人 AI Agent 的信任边界。评估 A-Mem、Mem0、MemOS 和 OpenClaw 等框架，发现长期记忆不仅是工具层，更是可重塑 Agent 行为解释的持久控制通道。提出 MemGate——9M 参数、35.1MB 体量的记忆门控插件，插入向量存储和 LLM 之间，将相似性搜索转换为任务条件记忆准入。
   - 应用场景：个人 Agent 安全、记忆注入防御
   - 为什么值得关注：轻量级即插即用方案，无需修改 LLM 或重写记忆数据库。

4. **MRAgent: Graph Memory with Active Reconstruction** (arXiv 2606.06036, ICML 2026)
   - 来源：https://arxiv.org/abs/2606.06036
   - 领域：Agent 记忆 / 关联图重建
   - 技术介绍：Cue-Tag-Content 关联记忆图 + 主动重建机制。标签作为语义桥梁连接细粒度线索到记忆内容。LLM 推理直接整合到记忆访问中，Agent 基于累积证据迭代探索和修剪检索路径。LoCoMo 和 LongMemEval 基准提升最高 23%。
   - 应用场景：长周期记忆推理、动态检索
   - 为什么值得关注：ICML 2026 录用，"记忆是重建而非检索"的核心理念。

5. **RBI-Eval: When Should Memory Stay Silent** (arXiv 2606.06055)
   - 来源：https://arxiv.org/abs/2606.06055
   - 领域：Agent 记忆 / 安全边界评估
   - 技术介绍：控制测量研究，比较模型在有/无敏感记忆访问下的行为差异。四个基座 LLM + 四种记忆访问设置。发现 GPT-5.4-mini 敏感记忆整合降低 8.9-26.6%，而 Claude-Sonnet-4.6/DeepSeek-V4-Flash/Qwen3.5-9B 降低 51.1-82.9%。检索系统减少暴露但不消除整合。
   - 应用场景：个人 Agent 隐私安全、记忆使用边界
   - 为什么值得关注：首次系统测量"何时应该不使用记忆"，揭示模型间巨大差异。

6. **The Self-Correction Illusion: LLMs Correct Others but Not Themselves** (arXiv 2606.05976)
   - 来源：https://arxiv.org/abs/2606.05976
   - 领域：LLM 评估 / 自我修正
   - 技术介绍：13 模型域 × 7 模型家族 × 3 领域实验。错误声明字节完全相同，仅改变 chat-template role 标签（self thought/user/tool/system memory）。从 self thought 改为外部 role 提升显式修正率 23-93pp，13/13 中 10 个达到 p<0.001。自我修正失败不是认知缺陷，而是 chat-template artifact。
   - 应用场景：LLM 自我修正、Agent 推理质量控制
   - 为什么值得关注：颠覆性发现——自我修正是角色标签问题而非能力问题。

7. **CoRe-3: Framing, Judging, Steering Competency Model** (arXiv 2026.05983)
   - 来源：https://arxiv.org/abs/2606.05983
   - 领域：AI 教育 / 能力评估
   - 技术介绍：将生产性 AI 使用分解为三个可评估技能：Framing（调用 AI 前定义模糊任务）、Judging（评估输出错误和隐含假设）、Steering（迭代重定向模型）。区分生成前 Framing 和生成后 Steering，Judging 作为中间门控。CoReasoningLab 开放平台。
   - 应用场景：AI 素养教育、人机协作评估
   - 为什么值得关注：首次将 AI 协作能力分解为可独立诊断的三技能模型。

8. **Appropriate Reliance on Set-Valued AI Advice** (arXiv 2606.06081)
   - 来源：https://arxiv.org/abs/2606.06081
   - 领域：人机协作 / AI 依赖评估
   - 技术介绍：首个形式化框架测量对集合值 AI 建议（离散集合或连续区间）的适当依赖。分类任务引入正确依赖率指标，回归任务引入 AI 依赖量和质量指标。在顺序 judge-advisor 范式下覆盖分类和回归。
   - 应用场景：AI 辅助决策、不确定性传达
   - 为什么值得关注：超越点预测，首次处理集合值建议的依赖度量。

9. **Benchmark Everything Everywhere All at Once** (arXiv 2606.06462)
   - 来源：https://arxiv.org/abs/2606.06462
   - 领域：基准构建自动化 / 评估基础设施
   - 技术介绍：全自动基准构建 Agent 系统，从用户查询分析、子任务设计到数据标注和质量控制完整流水线。15 个代表性基准，覆盖文本理解、多模态理解和领域特定推理。持续评估发现当前模型在特定领域推理任务上存在困难。
   - 应用场景：快速基准构建、模型能力持续评估
   - 为什么值得关注：基准构建从"手工劳动"走向"自动化流水线"。

#### 1. turbovec: Vector Index Built on TurboQuant

**地址**：https://github.com/RyanCodrai/turbovec

**项目介绍**（~300 字）：
turbovec 是一个基于 Google Research TurboQuant 算法的 Rust 向量索引库，提供 Python 绑定。TurboQuant 是一种数据无关量化器，匹配 Shannon 下界上的失真，无需码本训练和独立训练阶段。这使得 turbovec 支持在线注入——添加向量即被索引，无需重建。核心优势：10M 文档 corpus 从 float32 的 31GB 压缩到仅 4GB，搜索速度超越 FAISS IndexPQFastScan 12-20%（ARM NEON）或匹配/超越（x86 AVX-512BW）。支持搜索时过滤（allowlist/bitmask），内核直接 honor 过滤条件，无过度获取和 recall 损失。纯本地部署，无托管服务，适合隐私敏感的 RAG 栈。提供 IdMapIndex 支持稳定 ID 和 O(1) 删除。与主流框架（LangChain、LlamaIndex 等）的向量存储 API 兼容。

**技术架构**：
- 底层：Rust 实现，手写 NEON (ARM) 和 AVX-512BW (x86) SIMD 内核
- 量化：TurboQuant 数据无关量化，无需训练，bit_width 可配置（默认 4-bit）
- 索引：TurboQuantIndex（简单追加）+ IdMapIndex（稳定 ID + 删除支持）
- 过滤：SIMD 内核内 32-vector 块粒度过滤，空块短路跳过 LUT 查找

**核心算法**：
TurboQuant 是一种 data-oblivious quantizer，不依赖数据分布训练码本，而是通过数学构造匹配 Shannon 下界。这意味着：(1) 无需预训练阶段，(2) 新增向量无需重建索引，(3) 量化失真理论最优。

**与现有技术对比**：
- vs FAISS：内存效率 8x（4-bit vs float32），搜索速度 1.12-1.2x，且支持在线增量
- vs HNSW：HNSW 需要 O(n) 内存且不支持高效删除，turbovec 支持 O(1) 删除
- vs PQ：PQ 需要训练阶段，turbovec 无需训练

**优势与局限**：
- 优势：极致内存效率、在线增量、搜索时过滤、纯本地、SIMD 优化
- 局限：4-bit 量化精度有限（适合召回阶段而非精确排序）、不支持近似最近邻的 HNSW 级别精度

**未来发展潜力**：
随着 RAG 系统规模持续增长，turbovec 的内存效率和搜索速度优势将愈发重要。可能扩展方向：支持更高 bit_width、多模态向量、分布式部署。

**个人技术理解**：
turbovec 代表了向量检索基础设施的一个重要趋势——从"精度优先"转向"效率优先"。在 RAG 系统中，召回阶段不需要精确排序，4-bit 量化 + TurboQuant 的理论最优失真恰好满足需求。Google Research 的算法 + Rust 的工程实现 = 生产级向量索引。

### 学术论文

#### 1. The Self-Correction Illusion: LLMs Correct Others but Not Themselves

**地址**：https://arxiv.org/abs/2606.05976

**论文介绍**（~350 字）：
这篇论文揭示了一个颠覆性发现：LLM 的自我修正困难不是能力缺陷，而是 chat-template artifact。实验设计极为精巧——错误声明字节完全相同（SHA-256 验证），仅改变其包装的 chat-template role 标签（self thought / user message / tool response / system memory block）。在 13 个模型域（7 个模型家族 × 3 个领域）上，n=30 配对任务/域。结果：从 self thought 改为外部 role 提升显式修正率 23-93 个百分点，13/13 中 10 个达到 p<0.001。效应是不对称的、可机制分解的、跨域鲁棒的。进一步实验设计了仅依赖 prompt 结构改变的干预方案（无需训练、无需模型修改），最强 role 标签因领域而异：math 上 system memory 主导，logical deduction 上 user message 主导。

**技术架构**：
- 实验设计：字节级控制（SHA-256 验证错误声明相同），仅改变 role 标签
- 评估：13 模型域 × 7 模型家族 × 3 领域
- 干预：prompt-structure-only，无训练无修改

**核心发现**：
1. 自我修正失败是 chat-template artifact，不是认知缺陷
2. 效应大小惊人（23-93pp），且高度统计显著
3. 不同领域最优 role 标签不同（domain-dependent）
4. 可设计零成本干预方案

**与现有技术对比**：
现有自我修正研究假设 LLM 有内在自我修正能力缺陷，本文证明这是角色标签造成的假象。

**优势与局限**：
- 优势：实验设计严谨、发现颠覆性、零成本干预方案
- 局限：仅测试了 7 个模型家族、3 个领域

**未来发展潜力**：
这一发现对 Agent 推理质量控制有深远影响。如果自我修正可以通过简单的 role 标签改变实现，那么 Agent 架构设计需要重新思考推理循环的组织方式。

**个人技术理解**：
这是本周最有价值的发现之一。它提醒我们：很多"能力缺陷"可能是"表示缺陷"。在 Agent 测试和评估中，需要警惕这类 artifact——模型可能具备能力，但当前的交互方式阻止了能力的展现。

#### 2. MAGE: Memory as Agent-Guided Exploration

**地址**：https://arxiv.org/abs/2606.06090

**论文介绍**（~300 字）：
MAGE 提出将记忆从语义检索系统重构为主动执行状态管理器。核心论点：RAG 和 Agent 记忆系统按语义相似性组织历史，这与执行状态依赖不匹配——碎片化决策轨迹、混合有效和错误轨迹、阻碍连贯状态重建和错误隔离。MAGE 将交互存储在分层状态树中，Agent 从活跃根到当前路径派生状态，结合子目标摘要、近期轨迹和先前分支提示。四个耦合操作：Grow（记录新轨迹）、Compress（摘要完成子目标）、Maintain（验证状态一致性）、Prune（移除无效分支）。

**技术架构**：
- 数据结构：分层状态树（而非扁平向量存储）
- 状态派生：根到当前路径 + 子目标摘要 + 近期轨迹 + 分支提示
- 四操作：Grow / Compress / Maintain / Prune

**核心创新**：
从"语义组织"到"执行状态管理"的范式转变。记忆不再是"存储和检索"，而是"主动管理执行状态"。

**与现有技术对比**：
- vs RAG：RAG 按语义相似性检索，MAGE 按执行状态路径检索
- vs 图记忆（TokenMizer/MRAgent）：图记忆关注关联关系，MAGE 关注执行状态
- vs 传统 Agent 记忆：传统记忆是被动存储，MAGE 是主动管理

**优势与局限**：
- 优势：错误隔离、状态重建、长期一致性
- 局限：状态树管理复杂度、需要 Agent 主动参与

**未来发展潜力**：
执行状态管理范式可能成为长周期 Agent 任务的标准架构。与 TokenMizer 的图结构会话记忆互补——一个关注执行状态，一个关注会话结构。

**个人技术理解**：
MAGE 代表 Agent 记忆系统的第三次范式演进：第一次是 RAG（语义检索），第二次是图记忆（关联重建），第三次是执行状态管理（主动控制）。这反映了 Agent 从"问答系统"向"执行系统"的转变。

#### 3. RBI-Eval: When Should Memory Stay Silent

**地址**：https://arxiv.org/abs/2606.06055

**论文介绍**（~250 字）：
RBI-Eval 是首个系统测量"何时应该不使用记忆"的研究。构建探针集比较模型在有/无敏感记忆访问下的行为差异。四个基座 LLM + 四种记忆访问设置（全上下文暴露 + 三种检索系统）。关键发现：GPT-5.4-mini 敏感记忆整合降低 8.9-26.6%，而 Claude-Sonnet-4.6/DeepSeek-V4-Flash/Qwen3.5-9B 降低 51.1-82.9%。控制实验确认效应特定于敏感内容而非一般个性化。检索系统减少暴露但不消除整合——一旦敏感记忆到达生成器，整合就会发生。

**核心发现**：
1. 模型间记忆安全行为差异巨大（GPT vs 其他 8x 差距）
2. 检索系统不能替代生成时决策
3. 安全个性化需要检索和生成双时记忆感知决策

**个人技术理解**：
这篇论文与 MemGate 论文形成互补——RBI-Eval 测量问题，MemGate 提供解决方案。两者共同指向：记忆安全需要多层防御，检索层和生成层都需要记忆感知决策。
