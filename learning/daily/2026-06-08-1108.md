### AI Testing / Software Testing（10 项）

1. **AARRI-Bench: Act As a Real Research Intern** (arXiv 2606.07462)
   - 来源：https://arxiv.org/abs/2606.07462
   - 领域：Agent 评估 / 研究生命周期
   - 技术介绍：AARR（Act As a Real Researcher）基准系列的首个基准 AARRI-Bench，评估 Agent 在细粒度研究场景中是否能模拟人类研究人员的专业性、彻底性和细致推理。与现有评估宏观执行能力的基准不同，AARR 聚焦于领域敏感性、研究伦理和微妙科学判断。实验显示最佳配置（Mini-SWE-Agent + Claude Opus 4.7）仅达到 68.3% 成功率，经常忽略人类研究人员能轻易察觉的微妙但关键的细节。
   - 应用场景：Agent 研究能力评估、科研自动化
   - 为什么值得关注：首次系统评估 Agent 的"研究员级"能力，揭示 Frontier Agent 与人类研究者的关键差距。

2. **Lean4Agent: 形式化建模与验证 Agent 工作流** (arXiv 2606.06523)
   - 来源：https://arxiv.org/abs/2606.06523
   - 领域：Agent 验证 / 形式化方法
   - 技术介绍：首次使用 Lean4 依赖类型形式语言来建模和验证 Agent 行为。构建 FormalAgentLib——可扩展的 Lean4 库，在明确假设下形式化建模 Agent 工作流的语义一致性，并通过轨迹定位执行时失败。进一步开发 LeanEvolve，利用形式化验证结果修订工作流以提升能力。在 SWE-Bench-Verified 和 ELAIP-Bench 上，验证通过的工作流比失败的平均高出 11.94%，LeanEvolve 进一步提升 SWE 性能 7.47%。
   - 应用场景：Agent 工作流验证、SWE Agent 可靠性保障
   - 为什么值得关注：开创性地用依赖类型形式语言验证 Agent 行为，为 Agent 可靠性提供数学保证。

3. **Attack Selection in Agentic AI Control Evaluations** (arXiv 2606.06529)
   - 来源：https://arxiv.org/abs/2606.06529
   - 领域：Agent 安全 / 控制评估
   - 技术介绍：研究策略性攻击选择在 Agent AI 控制评估中的影响。当前控制评估通常假设无差别攻击者，但现实中攻击者会策略性地选择何时攻击。将攻击决策分解为 start policy（何时开始攻击）和 stop policy（何时中止攻击）。在 BashArena 和 LinuxArena 两个 Agent 环境中，start policy 在 1% 审计预算下使安全降低 20pp，stop policy 在 LinuxArena 上降低 28pp。
   - 应用场景：AI 控制安全评估、红队测试
   - 为什么值得关注：揭示现有 AI 控制评估可能过于乐观，建议未来评估必须包含攻击选择策略。

4. **Socratic-SWE: 自我进化编码 Agent** (arXiv 2606.07412, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07412
   - 领域：Agent 自我进化 / 软件工程
   - 技术介绍：闭环自我进化框架，将 Agent 的历史求解轨迹蒸馏为结构化 Agent 技能，总结反复失败和有效修复模式。这些技能指导在真实仓库中生成针对性修复任务。候选任务通过执行验证和 solver-gradient 对齐奖励评分。在 SWE-bench Verified 上三轮迭代后达到 50.40%，在 SWE-bench Lite/Pro 和 Terminal-Bench 2.0 上一致超越基线。
   - 应用场景：SWE Agent 自动改进、代码修复自我进化
   - 为什么值得关注：将求解轨迹从奖励信号升级为技能蒸馏源，实现真正的自我进化闭环。

5. **TRUST: 不确定性对齐的工具调用 RL** (arXiv 2606.06976)
   - 来源：https://arxiv.org/abs/2606.06976
   - 领域：Agent 工具调用 / RL
   - 技术介绍：LLM Agent 的工具使用决策常次优（不支持的工具调用和幻觉直接响应）。现有方法通过推理时纠正或基于结果的结构化检查表改进，但忽略了决策的不确定性特征。发现决策导向 RL 倾向于削弱正确和错误动作之间的不确定性分离，导致过度自信的失误。TRUST 将不确定性量化纳入奖励设计作为维持不确定性分离的排斥力，并标注轻量级关键轮次注释用于多轮轨迹统一后训练。
   - 应用场景：Agent 工具调用优化、多轮决策可靠性
   - 为什么值得关注：首次将不确定性量化系统性地融入 Agent 工具调用 RL 奖励设计。

6. **StainFlow: 实体染色追踪 GUI Agent 过程奖励** (arXiv 2606.07027)
   - 来源：https://arxiv.org/abs/2606.07027
   - 领域：GUI Agent / 过程奖励模型
   - 技术介绍：针对 GUI Agent 长周期随机数字环境中 RL 奖励稀疏问题。全局里程碑分解主观且单一，固定局部判断窗口可能遗漏长距离关键证据。受网络流分析中染色追踪机制启发，提出实体染色流过程奖励模型。全局实体染色追踪模块提取视觉可验证任务实体，追踪其染色浓度和状态沿轨迹演化，客观分离任务阶段。局部染色证据链接提高局部验证准确性。
   - 应用场景：GUI Agent 训练、桌面自动化
   - 为什么值得关注：从网络流分析借鉴染色追踪思想，为 GUI Agent 提供客观的过程奖励信号。

8. **Declarative Skills for AI Agents in Tool-Use Workflows** (arXiv 2606.06923)
   - 来源：https://arxiv.org/abs/2606.06923
   - 领域：Agent 技能 / 工具使用
   - 技术介绍：比较三种 Agent 编排机制：DeclarativeAgent（自然语言技能文件）、ImperativeAgent（程序化状态机）、无脚手架基线。在去中心化部分可观察 MDP 中形式化为策略类，分析信息论和结构特性。在 5 个语言模型和 2 种检索制度下测试。发现检索质量是主导瓶颈——证据不完整时所有 Agent 显著退化；高质量检索下声明式技能一致提高程序任务准确性。
   - 应用场景：Agent 技能编排、客服工作流
   - 为什么值得关注：首次系统比较声明式 vs 命令式 Agent 编排，揭示检索质量是关键瓶颈。

9. **RWSA: 技能创建的工作流分解框架** (arXiv 2606.06893)
   - 来源：https://arxiv.org/abs/2606.06893
   - 领域：Agent 技能 / 自动构建
   - 技术介绍：从异构交互证据（演示、Agent 轨迹、工具追踪、执行日志）自动构建 Agent 技能。引入 RWSA 工作流导向中间表示，将技能分解为 Workflow 结构、执行 Semantics 和运行时 Attachments，捕获任务分解、控制流、验证、安全、回滚和状态管理。W2S 框架分段追踪、诱导局部技能草稿、对齐共享结构、协调分支、压缩冗余。70 个技能实验显示行为回放一致性提升 10.5%。

10. **Advanced Memory for Task-solving Agents** (arXiv 2606.06787)

    - 应用场景：长周期 Agent 任务、多轮交互
    - 为什么值得关注：统一三种记忆类型的双层架构，结合多 Agent 自动管理。

### AI / LLM 技术（13 项）

11. **How AI Agents Reshape Knowledge Work** (arXiv 2606.07489)
    - 来源：https://arxiv.org/abs/2606.07489
    - 领域：Agent 生产力 / 知识工作
    - 技术介绍：基于 Perplexity Search 和 Computer 产品的生产数据，研究 AI Agent 如何加速和重塑知识工作。Computer 每用户会话执行 26 分钟自主工作（vs Search 的 33 秒），将后续查询分布转向验证和扩展等高层工作。每查询不满率降低 55%。匹配任务完成时间从 269 分钟降至 36 分钟，时间和成本分别降低 87% 和 94%。Computer 查询更常跨越职业边界、需要高阶认知、解锁 Search 用户几乎不使用的活动。
    - 应用场景：AI Agent 生产力评估、知识工作自动化
    - 为什么值得关注：首个基于真实生产数据的 Agent 知识工作影响研究，量化 87-94% 效率提升。

12. **Online Pandora's Box for LLM Cascading** (arXiv 2606.07392)
    - 来源：https://arxiv.org/abs/2606.07392
    - 领域：LLM 路由 / 在线学习
    - 技术介绍：受 LLM 级联启发，提出在线上下文 Pandora's Box 模型，自适应查询和选择 LLM API。决策者观察请求上下文，顺序查询 API，每个查询揭示生成输出并产生（输出依赖）成本，然后选择部署一个输出。直接建模保留索引而非估计完整条件输出分布，结合 GMM 类型估计和 UCB 风格置信界。在正则条件下证明达到维度依赖的 Õ(√T) 累积遗憾。
    - 应用场景：多 LLM 路由、成本优化
    - 为什么值得关注：为 LLM 级联提供理论保证的在线决策框架。

13. **DyCon: 动态推理控制** (arXiv 2606.07108, ICML 2026)
    - 来源：https://arxiv.org/abs/2606.07108
    - 领域：LLM 推理效率 / 过思考缓解
    - 技术介绍：大推理模型通过迭代反思和探索显著提升性能，但存在"过思考"低效问题。现有方法依赖静态难度估计或需要任务特定训练。实证发现任务难度在推理过程中动态演化，且线性编码在 LRM 的步骤级嵌入中。DyCon 利用潜在步骤级表示显式建模演化任务难度，动态控制推理深度。在 4B-32B 四个模型和 12 个基准上显著减少冗余步骤而不牺牲准确性。
    - 应用场景：推理效率优化、过思考缓解
    - 为什么值得关注：ICML 2026 录用，训练-free 动态推理控制，发现难度在线性嵌入中的编码。

14. **No-CoT Task-Completion Time Horizons** (arXiv 2606.07157)
    - 来源：https://arxiv.org/abs/2606.07157
    - 领域：LLM 安全 / 无 CoT 推理
    - 技术介绍：许多前沿 AI 模型安全依赖监控其 CoT 推理。如果模型能够在内部执行足够复杂的推理而不使用显式思维令牌，将破坏这种监督。测量前沿模型在无 CoT 情况下的推理能力，跨越 43 个基准、30,000+ 问题，涵盖数学、编码、谜题、因果关系、心理理论和策略推理。
    - 应用场景：AI 安全监控、无 CoT 推理检测
    - 为什么值得关注：直接挑战当前 AI 安全监控范式，评估模型绕过 CoT 监督的能力。

15. **Glassbox AI via Probabilistic Mediation** (arXiv 2606.07113)
    - 来源：https://arxiv.org/abs/2606.07113
    - 领域：AI 可解释性 / 可审计推理
    - 技术介绍：现有可解释性方法主要是后验的，提供不稳定、不可争论的说明。提出 Glassbox Framework——贝叶斯网络作为生成模型的透明前摄式调解层。贝叶斯网络在推理前编码领域知识、因果假设和概率依赖，实现可审计推理追踪、不确定性量化和可争论输出。在福利资格场景中验证架构，识别语义对齐、动态模型构建、概率基础和人治理等基础挑战。
    - 应用场景：高风险 AI 决策、可审计推理
    - 为什么值得关注：从后验解释到先验概率调解的范式转变，为 AI 问责制提供原则性路径。

16. **OpenSkill: Open-World 自我进化** (arXiv 2606.06741)
    - 来源：https://arxiv.org/abs/2606.06741
    - 领域：Agent 自我进化 / 开放世界学习
    - 技术介绍：研究开放世界自我进化——Agent 必须从零开始构建技能和验证信号，仅使用开放世界资源而无目标任务监督。从文档、仓库和网络获取基础知识和验证锚点，合成为可迁移技能，并在自建的虚拟任务上精炼技能。三个基准和两个目标 Agent 上达到最佳自动化通过率，技能可在模型间迁移而无需模型特定适配。
    - 应用场景：开放世界 Agent 部署、无监督技能学习
    - 为什么值得关注：首次研究无目标监督的开放世界自我进化，技能可跨模型迁移。

17. **PTD-PO: 特权辅导蒸馏策略优化** (arXiv 2606.07000)
    - 来源：https://arxiv.org/abs/2606.07000
    - 领域：多模态推理 / RLVR
    - 技术介绍：RLVR 增强 LVLM 推理能力，但可验证奖励稀疏，失败回滚缺乏 token 级监督。外部教师方法计算开销大，答案条件微调可能暴露答案级信息。PTD-PO 构建结构化特权提示（空间注意力引导 + 中间文本推理步骤），通过上下文学习产生逐步 token 分布监督。学生仍在原始无答案上下文中优化，失败回滚与提示增强参考模型在 token 分布级别对齐。引入 Top-K Jensen-Shannon 散度目标聚焦信息 token 概率。
    - 应用场景：多模态推理优化、LVLM 后训练
    - 为什么值得关注：在不暴露答案的前提下提供密集指导，缓解熵崩溃。

18. **EP-HUBO: 量子启发证据选择** (arXiv 2606.06941)
    - 来源：https://arxiv.org/abs/2606.06941
    - 领域：法律推理 / 组合优化
    - 技术介绍：LLM 在专业证据密集型领域（如法律）仍然脆弱。多数投票返回最流行答案而不考虑证据强度。将 CoT 推理片段选择转化为显式组合优化问题，允许有支撑但少数假设覆盖噪声多数。EP-HUBO 用小本地模型生成多条 CoT 追踪，解析为每个假设的证据池，用质量导出权重（相关性、特异性、区分度）求解高阶无约束二元优化。在两个法律基准上评估，使用经典硬件模拟退火和 Dirac-3 光子熵量子机器。
    - 应用场景：法律推理、证据密集型决策

### GitHub 项目

#### 1. Socratic-SWE: 自我进化编码 Agent

**地址**：https://arxiv.org/abs/2606.07412

**项目介绍（300 字）**：
Socratic-SWE 是一个闭环自我进化框架，专门针对 LLM 驱动的软件工程 Agent。现有合成数据方法通过固定突变或 bug 注入程序创建任务，生成的数据分布与 Agent 自身弱点和训练进展无关。Socratic-SWE 的创新在于将 Agent 的历史求解轨迹从单纯的奖励信号升级为结构化技能蒸馏源——总结反复失败模式和有效修复策略。这些蒸馏出的技能指导在真实仓库中生成针对性修复任务，候选任务通过执行验证和 solver-gradient 对齐奖励双重筛选，确保任务既可验证又对改进求解器有用。更新后的求解器产生新轨迹，使任务课程在连续轮次中自适应演化。在 SWE-bench Verified 上三轮迭代后达到 50.40%，在 SWE-bench Lite、SWE-bench Pro 和 Terminal-Bench 2.0 上一致超越相同计算预算下的自我进化基线。

**技术架构**：
- 轨迹收集层：记录 Agent 求解过程中的完整交互轨迹
- 技能蒸馏层：从轨迹中提取失败模式和修复策略，构建结构化技能
- 任务生成层：基于技能指导在真实仓库中生成针对性修复任务
- 验证评分层：执行验证 + solver-gradient 对齐奖励
- 闭环演化层：更新求解器 → 新轨迹 → 新技能 → 新任务

**核心算法**：
- 轨迹到技能的结构化蒸馏：不只是摘要，而是提取可执行的模式
- Solver-gradient 对齐奖励：确保生成的任务对当前求解器有信息量
- 执行验证：确保任务的真实可解性

**与现有技术对比**：
| 维度 | 传统合成数据 | Socratic-SWE |
|------|------------|-------------|
| 数据来源 | 固定突变/bug 注入 | Agent 自身求解轨迹 |
| 任务针对性 | 与 Agent 能力无关 | 针对 Agent 弱点 |
| 演化机制 | 静态分布 | 自适应课程 |
| SWE-bench Verified | ~45% | 50.40% |

**优势与局限**：
- 优势：真正的自我进化闭环，无需外部标注数据，任务针对性强
- 局限：依赖 Agent 初始能力，三轮迭代计算成本高

**未来发展潜力**：
- 扩展到多 Agent 协作场景
- 结合 OpenSkill 的开放世界学习
- 应用于更广泛的软件工程任务

**个人技术理解**：
Socratic-SWE 代表了 Agent 自我进化的正确方向——从"被动接收合成数据"到"主动从自身经验学习"。轨迹蒸馏为技能是关键创新，将执行记录从日志升级为可执行知识。

#### 2. Lean4Agent: 形式化验证 Agent 工作流

**地址**：https://arxiv.org/abs/2606.06523

**项目介绍（300 字）**：
Lean4Agent 是首个使用 Lean4 依赖类型形式语言来建模和验证 Agent 行为的框架。LLM Agent 的多步骤工作流缺乏形式化方法指定、验证和调试，这类似于数学中自然语言的歧义性推动了形式语言的发展。FormalAgentLib 是可扩展的 Lean4 库，在明确假设下形式化建模 Agent 工作流的语义一致性，并通过轨迹定位执行时失败。LeanEvolve 进一步利用形式化验证结果修订工作流以提升能力。在 SWE-Bench-Verified 和 ELAIP-Bench 上，验证通过的工作流比失败的平均高出 11.94%，LeanEvolve 进一步提升 SWE 性能 7.47%。

**技术架构**：
- 形式化建模层：用 Lean4 定义 Agent 工作流的语义
- 验证层：检查语义一致性和执行轨迹合法性
- 演化层：基于验证失败修订工作流
- 实验层：在 SWE-Bench 和 ELAIP-Bench 上评估

**核心算法**：
- 依赖类型工作流建模：将 Agent 行为映射为形式化类型
- 轨迹验证：检测执行时失败并定位根因
- 工作流修订：基于验证结果自动改进工作流

**优势与局限**：
- 优势：为 Agent 行为提供数学保证，验证通过的工作流显著优于未验证的
- 局限：需要形式化建模专业知识，扩展到大规模工作流有挑战

**个人技术理解**：
这是 Agent 可靠性领域的重要突破。用形式化方法验证 Agent 行为，类似于用类型系统验证程序正确性。FormalAgentLib 可能成为 Agent 可靠性的"标准库"。

#### 3. DyCon: 动态推理控制（ICML 2026）

**地址**：https://arxiv.org/abs/2606.07108

**项目介绍（250 字）**：
大推理模型通过迭代反思和探索显著提升性能，但存在"过思考"低效问题。DyCon 发现任务难度在推理过程中动态演化，且线性编码在 LRM 的步骤级嵌入中。基于这一洞察，提出训练-free 框架利用潜在步骤级表示显式建模演化任务难度，动态控制推理深度。在 4B-32B 四个模型和 12 个基准（数学推理、通用问答、编码任务）上显著减少冗余步骤而不牺牲准确性或泛化能力。

**核心技术**：
- 难度动态演化发现：任务难度在线性嵌入中编码
- 训练-free 控制：无需额外训练，直接利用步骤级嵌入
- 动态深度调节：根据实时难度调整推理步数

**优势**：训练-free，即插即用，跨模型和任务通用

### 学术论文

#### 1. Attack Selection in Agentic AI Control Evaluations

**地址**：https://arxiv.org/abs/2606.06529

**论文介绍（250 字）**：
研究策略性攻击选择在 Agent AI 控制评估中的影响。当前控制评估假设无差别攻击者，但现实中攻击者会策略性选择何时攻击。将攻击决策分解为 start policy（何时开始）和 stop policy（何时中止）。在 BashArena 和 LinuxArena 中，start policy 在 1% 审计预算下使安全降低 20pp，stop policy 在 LinuxArena 上降低 28pp。这些减少应解释为攻击选择效果的上界。建议未来评估、系统卡片和安全案例必须包含攻击选择以产生更现实的安全估计。

**核心发现**：
- 策略性攻击选择使安全评估下降 20-28pp
- 现有 AI 控制评估可能严重高估安全性
- 攻击选择是独立于攻击能力的新维度

**个人技术理解**：
这篇论文对 AI 安全社区有重要警示意义。如果现有控制评估不考虑攻击选择，那么报告的"安全水平"可能严重偏离现实。建议所有 AI 控制基准更新以包含策略性攻击者模型。

#### 2. Glassbox AI via Probabilistic Mediation

**地址**：https://arxiv.org/abs/2606.07113

**论文介绍（250 字）**：
LLM 快速成为公共管理、法律推理和医疗等高风险机构环境中的基础设施组件，不透明性不仅不便而且在机构和法律上不可接受。现有可解释性方法主要是后验的，提供不稳定、不可争论的说明。提出 Glassbox Framework——贝叶斯网络作为生成模型的透明前摄式调解层。贝叶斯网络在推理前编码领域知识、因果假设和概率依赖，实现可审计推理追踪、不确定性量化和可争论输出。在福利资格场景中验证架构。

**核心创新**：
- 从后验解释到先验概率调解的范式转变
- 贝叶斯网络作为透明调解层
- 可审计、可量化不确定性、可争论

**个人技术理解**：
Glassbox AI 代表了一个重要的架构方向。在高风险决策场景中，后验解释远远不够——需要在推理之前就嵌入可审计的结构。贝叶斯网络的前摄式调解可能是实现 AI 问责制的可行路径。

#### 3. No-CoT Task-Completion Time Horizons

**地址**：https://arxiv.org/abs/2606.07157

**论文介绍（250 字）**：
许多前沿 AI 模型安全依赖监控其 CoT 推理。如果模型能够在内部执行足够复杂的推理而不使用显式思维令牌，将破坏这种监督。测量前沿模型在无 CoT 情况下跨越 43 个基准、30,000+ 问题的推理能力，涵盖数学、编码、谜题、因果关系、心理理论和策略推理。评估模型在"无思维"模式下的任务完成时间范围。

**核心关注**：
- 无 CoT 推理能力评估
- 对当前 AI 安全监控范式的挑战
- 43 个基准的大规模测试

**个人技术理解**：
这篇论文直接挑战当前 AI 安全的核心假设——监控 CoT 足以保证安全。如果模型能在无 CoT 情况下执行复杂推理，现有的安全监控框架需要根本性重构。

## 技术趋势

1. **Agent 自我进化从合成数据到经验学习**：Socratic-SWE 和 OpenSkill 代表两个互补方向——前者从自身求解轨迹蒸馏技能，后者从开放世界构建技能。Agent 不再依赖外部合成数据，而是从自身经验或开放世界资源中学习。

2. **AI 安全评估从被动到策略性**：Attack Selection 论文揭示现有 AI 控制评估可能严重高估安全性。未来安全评估必须包含策略性攻击者模型，这对 AI 控制框架设计有深远影响。

3. **形式化方法进入 Agent 领域**：Lean4Agent 首次用依赖类型形式语言验证 Agent 行为，标志着形式化验证从传统软件扩展到 AI Agent。这为 Agent 可靠性提供了数学保证的基础。

4. **Glassbox AI 架构兴起**：从后验解释到先验概率调解，贝叶斯网络作为透明调解层为高风险 AI 决策提供可审计性。这是 AI 问责制的重要架构方向。

5. **推理效率优化进入动态阶段**：DyCon 发现任务难度在线性嵌入中编码，实现训练-free 动态推理控制。推理效率从静态压缩进入动态调节阶段。

## 新领域收获

1. **GPU SAT 求解（AFSAT）**：将 SAT 求解从 CPU 传统领域扩展到 GPU 大规模并行，利用 JAX 实现近线性多加速器扩展。组合优化问题可能在 GPU 上获得数量级加速。

2. **物理 AI 备份反射（AEGIS）**：为机器人操作提供"备份反射"机制——用轻量级探针检测高风险步骤并切换到强策略。杠杆是时机而非计算量，为物理 AI 安全提供新思路。

3. **量子提议 + 经典认证的自动驾驶（CARVE-Q）**：量子计算仅用于搜索最优修复方案，安全认证完全由经典验证器完成。这种"量子提议、经典认证"模式可能是量子 AI 最实用的落地路径。
