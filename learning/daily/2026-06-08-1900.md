### AI Testing / Software Testing（9 项）

1. **Sim-to-Real Gap of Foundation Model Agents: Unified MDP Perspective** (arXiv 2606.07017)
   - 来源：https://arxiv.org/abs/2606.07017
   - 领域：Agent 评估 / Sim-to-Real
   - 技术介绍：将 Foundation Model Agent 的训练与评估差距形式化为经典 Sim-to-Real 问题，围绕 MDP 四要素（Observation、Action、Transition、Reward）构建统一框架。提出多语言工具调用等具体示例，展示观察空间差距如何导致语义正确但操作无效的行为。倡导采用域随机化等成熟方案，推动标准化压力测试基准。
   - 应用场景：Agent 鲁棒性评估、Sim-to-Real 迁移
   - 为什么值得关注：为 Agent 鲁棒性问题提供统一词汇表和标准化测试框架，推动从"新颖现象"到"经典问题"的范式转变。

2. **Declarative Skills for AI Agents in Knowledge-Grounded Tool-Use** (arXiv 2606.06923)
   - 来源：https://arxiv.org/abs/2606.06923
   - 领域：Agent 技能 / 工具使用
   - 技术介绍：比较三种 Agent 编排机制：DeclarativeAgent（自然语言技能文件）、ImperativeAgent（程序化状态机）、Baseline（无脚手架）。在去中心化部分可观察 MDP 框架下形式化分析三类 Agent 的信息论和结构属性。在 5 个 LLM + 2 种检索设置上实证测试。发现检索质量是主导瓶颈，DeclarativeAgent 在高质量检索下表现最佳。
   - 应用场景：客服 Agent、知识密集型工具使用
   - 为什么值得关注：首次在结构化框架下比较声明式 vs 命令式 Agent 编排，对 Agent 架构选择有指导意义。

3. **RWSA: Skill Creation via Routing-Workflow-Semantics-Attachments Decomposition** (arXiv 2606.06893)
   - 来源：https://arxiv.org/abs/2606.06893
   - 领域：Agent 技能 / 轨迹到技能
   - 技术介绍：自动从异构交互证据（演示、轨迹、工具调用、执行日志）构建 Agent 技能。引入 RWSA 中间表示，将技能分解为 Workflow 结构、Semantics 语义和 Attachments 运行时附件。W2S 框架实现轨迹分段、局部技能草稿诱导、共享结构对齐、分支协调和冗余压缩。70 个技能实验，行为重放一致性提升 10.5%。
   - 应用场景：Agent 技能自动生成、轨迹蒸馏
   - 为什么值得关注：从"简单摘要"到"结构化分解"的技能构建范式转变。

4. **Advanced Memory for Task-solving Agents** (arXiv 2606.06787)
   - 来源：https://arxiv.org/abs/2606.06787
   - 领域：Agent 记忆 / 语义+情景+程序记忆
   - 技术介绍：统一自动记忆框架，集成语义、情景和程序记忆，双层设计（短期 + 长期存储）。多 Agent 架构（Actor、Memory、Critic）实现自动记忆生成、奖励标注和自适应检索。长期记忆通过奖励评估、合并和修剪管理，确保可扩展性和持续改进。
   - 应用场景：长周期多轮任务、Agent 知识复用
   - 为什么值得关注：三种记忆类型的统一框架，对 Agent 长期任务能力有系统性提升。

5. **Online Pandora's Box for Contextual LLM Cascading** (arXiv 2606.07392)
   - 来源：https://arxiv.org/abs/2606.07392
   - 领域：LLM 路由 / 成本优化
   - 技术介绍：提出在线上下文 Pandora's Box 模型，自适应查询和选择 LLM API。决策者顺序查询 API，每次查询揭示生成输出并产生（输出依赖）成本。直接建模保留索引（reservation index），开发查询阶段学习方法。结合 GMM 类型估计和 Weitzman 策略的上下文保留索引函数参数化结构。
   - 应用场景：多 LLM 路由、成本-质量权衡
   - 为什么值得关注：为 LLM Cascading 提供理论保证的自适应选择框架。

6. **AARRI-Bench: Act As a Real Research Intern** (arXiv 2606.07462)
   - 来源：https://arxiv.org/abs/2606.07462
   - 领域：Agent 评估 / 研究生命周期
   - 技术介绍：AARR 基准系列首个基准 AARRI-Bench，评估 Agent 在细粒度研究场景中模拟人类研究人员专业性、彻底性和细致推理的能力。最佳配置（Mini-SWE-Agent + Claude Opus 4.7）仅达 68.3% 成功率，经常忽略人类研究人员能轻易察觉的微妙细节。
   - 应用场景：Agent 研究能力评估、科研自动化
   - 为什么值得关注：首次系统评估 Agent 的"研究员级"能力，揭示 Frontier Agent 与人类研究者的关键差距。

7. **Lean4Agent: Formal Modeling and Verification for Agent Workflow** (arXiv 2606.06523)
   - 来源：https://arxiv.org/abs/2606.06523
   - 领域：Agent 验证 / 形式化方法
   - 技术介绍：首次使用 Lean4 依赖类型形式语言建模和验证 Agent 行为。构建 FormalAgentLib 可扩展库，形式化建模 Agent 工作流语义一致性，轨迹定位执行时失败。LeanEvolve 利用验证结果修订工作流，SWE-Bench-Verified 性能提升 7.47%。
   - 应用场景：Agent 工作流验证、SWE Agent 可靠性保障
   - 为什么值得关注：开创性地用依赖类型形式语言验证 Agent 行为。

8. **Attack Selection in Agentic AI Control Evaluations** (arXiv 2606.06529)
   - 来源：https://arxiv.org/abs/2606.06529
   - 领域：Agent 安全 / 控制评估
   - 技术介绍：策略性攻击选择在 Agent AI 控制评估中的影响。将攻击决策分解为 start policy（何时开始攻击）和 stop policy（何时中止攻击）。BashArena 和 LinuxArena 环境中，start policy 在 1% 审计预算下使安全降低 20pp，stop policy 降低 28pp。
   - 应用场景：AI 控制安全评估、红队测试
   - 为什么值得关注：揭示现有 AI 控制评估可能过于乐观。

9. **Socratic-SWE: Self-Evolving Coding Agents via Trace-Derived Agent Skills** (arXiv 2606.07412)
   - 来源：https://arxiv.org/abs/2606.07412
   - 领域：Agent 自我进化 / 软件工程
   - 技术介绍：闭环自我进化框架，将 Agent 历史求解轨迹蒸馏为结构化 Agent 技能，总结反复失败和有效修复模式。技能指导在真实仓库中生成针对性修复任务。SWE-bench Verified 三轮迭代后达 50.40%。
   - 应用场景：SWE Agent 自动改进、代码修复自我进化
   - 为什么值得关注：将求解轨迹从奖励信号升级为技能蒸馏源。

### AI / LLM 技术（13 项）

10. **How AI Agents Reshape Knowledge Work** (arXiv 2606.07489, Perplexity)
    - 来源：https://arxiv.org/abs/2606.07489
    - 领域：Agent 生产力 / 知识工作
    - 技术介绍：基于 Perplexity Search 和 Computer 产品生产数据，研究 AI Agent 从对话助手到自主执行者的转变。Computer 每会话执行 26 分钟自主工作 vs Search 33 秒。完成时间从 269 分钟降至 36 分钟，时间和成本降低 87% 和 94%。每查询不满率降低 55%。
    - 应用场景：知识工作自动化、Agent 生产力评估
    - 为什么值得关注：首个基于生产数据的 Agent 自主性实证研究。

    - 技术介绍：估计 Frontier AI 模型在无 CoT 情况下的任务完成时间范围。对模型自我时间感知能力的系统评估。

    - 为什么值得关注：探索 LLM 对任务复杂度的内在时间感知。

12. **PTD-PO: Privileged Tutoring Distillation for Multimodal Policy Optimization** (arXiv 2606.07000)

    - 技术介绍：特权辅导蒸馏策略优化框架，为 RLVR 提供密集指导而不暴露答案给学生策略。从空间注意力指导和中间文本推理步骤构建结构化特权提示，通过上下文学习产生逐词分布监督。学生仍在无答案条件下优化。
    - 应用场景：多模态推理训练、LVLM 后训练

13. **EP-HUBO: Quantum-Inspired Trace-Augmented Evidence Selection** (arXiv 2606.06941)
    - 来源：https://arxiv.org/abs/2606.06941
    - 领域：LLM 推理 / 证据选择
    - 技术介绍：将 CoT 推理片段选择视为显式组合优化问题。生成多个 CoT 轨迹，解析为每个假设的证据池，用质量导出权重求解高阶无约束二元优化。在法律文书推理基准上评估，使支持充分但少数假设覆盖噪声多数。
    - 应用场景：法律推理、证据密集型决策
    - 为什么值得关注：超越多数投票，显式优化证据质量。

14. **How reliable are LLMs when it comes to playing dice?** (arXiv 2606.07515)
    - 来源：https://arxiv.org/abs/2606.07515
    - 领域：LLM 评估 / 概率推理
    - 技术介绍：通过离散概率问题控制基准测试评估 8 个 SOTA 模型。标准问题平均准确率 0.96，反直觉问题仅 0.59。替换为伪装变体后性能下降 20%+，嵌入误导性建议后下降 34%。无模型免疫。
    - 应用场景：LLM 概率推理能力评估
    - 为什么值得关注：揭示 LLM 并非真正的概率推理器，尽管在高级数学问题上成功。

15. **MemDreamer: Hierarchical Graph Memory for Long Video Understanding** (arXiv 2606.07512)
    - 来源：https://arxiv.org/abs/2606.07512
    - 领域：长视频理解 / 图记忆
    - 技术介绍：解耦感知和推理，将长视频理解转变为 Agent 探索过程。增量流式视频构建分层图记忆（三层架构），推理模型通过 Observation-Reason-Action 循环导航层次、搜索节点、遍历逻辑边。推理上下文窗口仅 2% 全量，精度提升 12.5 点。
    - 应用场景：长视频理解、多模态推理
    - 为什么值得关注：首次将 Agent 检索引入长视频理解，大幅降低推理成本。

16. **A Comprehensive Anatomy of Human and DeepSeek-R1 Mathematical Reasoning** (arXiv 2606.07410)
    - 来源：https://arxiv.org/abs/2606.07410
    - 领域：LLM 推理 / 数学推理
    - 技术介绍：全面比较模型和人类推理（AIME 2025 全部 30 题），标注 10,247 个推理步骤为 5 类（Analysis、Inference、Branch、Backtrace、Reflection）。发现人类解法保持分析与演绎的紧凑交替，而 DeepSeek-R1 频繁重访中间结果、执行浅层验证循环，称为"拓扑模仿"。
    - 应用场景：LLM 推理质量评估、数学推理分析
    - 为什么值得关注：首次系统对比 LLM 和人类推理的拓扑结构差异。

17. **SETA: Sparse Subspace-to-Expert Sharing for Continual Learning** (arXiv 2606.07500)
    - 来源：https://arxiv.org/abs/2606.07500
    - 领域：持续学习 / 灾难性遗忘
    - 技术介绍：通过自适应稀疏子空间分解解决可塑性-稳定性困境。将知识分离为唯一专家（隔离任务特定模式）和共享专家（捕获通用特征）。通过自适应弹性锚定和路由感知正则化维护结构，统一门控网络自动检索正确专家组合。
    - 应用场景：LLM 持续学习、多任务适配
    - 为什么值得关注：从"均匀参数更新"到"稀疏专家分离"的持续学习范式。

### 学术论文

#### 1. MemDreamer: Hierarchical Graph Memory for Long Video Understanding

**论文地址：** https://arxiv.org/abs/2606.07512

**论文介绍（350 字）：**
MemDreamer 解决长视频理解的核挑战：处理全长度视觉序列导致 Token 爆炸和注意力稀释。论文提出解耦感知和推理的架构，将长视频理解转变为 Agent 探索过程。

**核心架构：**
1. **分层图记忆（Hierarchical Graph Memory）**：三层自顶向下架构
   - 基础层：捕获时空和因果关系的底层图
   - 语义抽象层：自底向上的语义聚合
   - 推理导航层：Agent 驱动的层次导航
2. **Agent 检索机制**：Observation-Reason-Action 循环
   - 导航层次结构
   - 搜索节点
   - 遍历逻辑边
3. **增量流式构建**：视频增量流式输入，图记忆逐步构建

**关键数据：**
- 推理上下文窗口仅 2% 全量上下文
- 精度提升 12.5 点绝对增益
- 与人类专家差距仅 3.7 分
- 四个主流基准 SOTA

**技术架构：**
```
视频流 → 增量感知 → 分层图构建 → Agent 检索 → 推理输出
         (2% context)    (graph)      (ORA loop)
```

**与现有技术对比：**
- vs 全量视频处理：Token 消耗降低 98%
- vs 传统 RAG：图结构保留时空和因果关系，非简单向量检索
- vs 其他长视频方法：Agent 检索 vs 静态摘要

**优势与局限：**
- 优势：大幅降低推理成本、保留结构信息、SOTA 性能
- 局限：图构建质量依赖感知模型、Agent 检索增加延迟

**未来发展潜力：**
分层图记忆 + Agent 检索可能成为长上下文理解的标准架构，扩展到长文档、长音频等领域。

**个人技术理解：**
MemDreamer 的核心洞察是"感知和推理应该解耦"。传统方法试图在一次推理中处理所有信息，而 MemDreamer 让感知系统构建结构化记忆，推理系统按需检索。这种架构与人类认知过程高度一致：我们先感知世界构建心理模型，然后在需要时回忆相关信息进行推理。

#### 2. A Comprehensive Anatomy of Human and DeepSeek-R1 Mathematical Reasoning

**论文地址：** https://arxiv.org/abs/2606.07410

**论文介绍（350 字）：**
这篇论文对 DeepSeek-R1 的数学推理能力进行了迄今为止最系统的解剖分析。研究团队将 AIME 2025 全部 30 题的 10,247 个推理步骤标注为 5 类功能角色：Analysis（分析）、Inference（推断）、Branch（分支）、Backtrace（回溯）、Reflection（反思），然后系统比较人类和 DeepSeek-R1 的推理拓扑结构。

**核心发现：**
1. **拓扑模仿（Topological Mimicry）**：DeepSeek-R1 重现推理的表面形式，但缺乏功能角色。人类解法保持分析与演绎的紧凑交替，而 R1 频繁重访中间结果、执行浅层且通常不必要的验证、在局部检查中循环而无有意义的逻辑进展。
2. **真正推理的两个信号**：
   - 成功轨迹表现出分支和回溯的稳定使用，失败轨迹要么过度使用要么不足
   - 反思仅在特定位置有效（非任意位置）
3. **"Aha 时刻"的本质**：R1 的"Aha 时刻"更多是搜索策略的结果，而非真正的洞察

**技术方法：**
- 穷尽标注 10,247 个推理步骤
- 5 类功能角色编码
- 拓扑结构对比分析
- 成功/失败轨迹对比

**与现有技术对比：**
- vs 传统基准测试：从"答案正确性"到"推理过程质量"
- vs CoT 分析：首次系统对比 LLM 和人类推理的拓扑差异
- vs 其他推理分析：最细粒度的步骤级功能标注

**优势与局限：**
- 优势：最系统的推理过程分析、大规模手工标注、清晰的分类框架
- 局限：仅分析 DeepSeek-R1、仅数学推理领域

**未来发展潜力：**
推理拓扑分析可能成为 LLM 推理质量评估的新标准，从"答案是否正确"进化到"推理是否合理"。

**个人技术理解：**
"拓扑模仿"这个概念非常深刻。它揭示了 LLM 推理的本质：它们学习的是推理的"形状"而非推理的"逻辑"。这解释了为什么 LLM 在简单问题上表现良好（形状匹配），但在需要真正洞察的问题上失败（形状不匹配）。对 Agent Testing 的启示：评估 Agent 不应只看最终输出，还需要分析推理过程的拓扑结构。

#### 3. Sim-to-Real Gap of Foundation Model Agents: A Unified MDP Perspective

**论文地址：** https://arxiv.org/abs/2606.07017

**论文介绍（300 字）：**
这篇论文将 Foundation Model Agent 的训练与评估差距形式化为经典 Sim-to-Real 问题，围绕 MDP 四要素（Observation、Action、Transition、Reward）构建统一框架。

**核心论点：**
机器人学和经典控制已有成熟的 Sim-to-Real 框架，但 Foundation Model 社区将 Agent 鲁棒性视为全新现象。论文主张翻译经典差异到 Foundation Model 领域。

**四要素映射：**
1. **Observation Gap**：多语言工具调用示例展示观察空间差距如何导致操作无效
2. **Action Gap**：Agent 动作空间与真实环境的对齐问题
3. **Transition Gap**：环境动态建模偏差
4. **Reward Gap**：奖励函数设计偏差

**研究议程：**
- 统一词汇表
- 标准化压力测试基准
- 域随机化等成熟方案采纳
- 新一代高可信 Agent

**个人技术理解：**
这篇论文的价值不在于提出新技术，而在于提供统一框架。将 Agent 鲁棒性问题从"新颖现象"重新定义为"经典 Sim-to-Real 问题"，可以直接借用机器人学 20 年的研究成果。这对 Agent Testing 有重大意义：我们可以借用机器人学的成熟测试方法（如域随机化测试）来评估 Agent 鲁棒性。
