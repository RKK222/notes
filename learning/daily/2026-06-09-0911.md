1. **FrontierCode: Raising the Bar from Correctness to Quality** (Cognition AI Blog, 2026-06-08)
   - 来源：https://cognition.ai/blog/frontier-code
   - 领域：Agent 代码质量评估 / 基准测试
   - 技术介绍：Cognition AI 推出 FrontierCode 基准，首次测量代码"可合并性"而非仅正确性。20+ 世界顶级开源维护者每人花费 40+ 小时构建真实编码任务，定义各自仓库中的"可合并"标准。采用单元测试、评分 rubric 和新型验证器的集成评分方法。包含三个嵌套子集：Extended（150 任务）、Main（100 任务）、Diamond（50 最难任务）。最佳模型 Claude Opus 4.8 在 Diamond 上仅得 13.4% 分数，GPT-5.5 得 6.3% 但 token 消耗少 4 倍。QC 管道使假阳性率比 SWE-Bench Pro 低 81%。

### 新领域探索（12 项）

23. **How AI Agents Reshape Knowledge Work** (arXiv 2606.07489, Perplexity)
    - 来源：https://arxiv.org/abs/2606.07489
    - 领域：Agent 生产力研究 / 知识工作
    - 技术介绍：基于 Perplexity Search 和 Computer 产品的生产数据，研究 AI Agent 如何加速和重塑知识工作。Computer 每用户会话执行 26 分钟自主工作（vs Search 的 33 秒）。匹配任务完成时间从 269 分钟降至 36 分钟，时间和成本分别降低 87% 和 94%。Computer 改变用户尝试的工作范围：更多跨职业边界、需要高阶认知、形式为复合任务。
    - 应用场景：知识工作自动化、Agent 生产力评估
    - 为什么值得关注：首次用真实生产数据量化 Agent 对知识工作的影响，87% 时间降低是里程碑式数据。

    - 应用场景：策略评估、人机交互博弈

## 技术趋势

1. **Agent 代码质量评估从正确性转向可合并性**：FrontierCode 代表 Agent 代码生成评估的范式转变——从"代码能运行"到"代码值得合并"。这反映 Agent 代码生成正在从实验性工具向生产级工具演进，评估标准必须跟上。

2. **推理速度突破 1000 TPS 开启"速度即智能"范式**：MiMo-V2.5-Pro-UltraSpeed 在 1T 模型上突破 1000 TPS，使模型能在同一墙钟时间内并行运行数十条推理路径。速度本身开始转化为智能，可能改变 AI 应用设计范式。

3. **Agent 基础设施从模型层竞争转向 harness 层竞争**：ECC（210K stars）、supermemory（26K stars）、impeccable（36K stars）等项目的快速增长反映 Agent 生态正在从"模型能力"竞争转向"harness 质量"竞争。当模型能力趋同时，基础设施质量将成为差异化关键。

4. **端侧 AI 框架推动 AI 能力系统级集成**：Apple Core AI 框架代表端侧 AI 从"应用级集成"向"系统级框架"演进。AI 能力将成为操作系统的一部分，而非独立应用。

5. **AI 安全评估需要重构无 CoT 推理假设**：No-CoT 推理能力测量揭示现有安全框架可能过于依赖 CoT 监控。如果模型能在内部进行复杂推理而不需要显式 CoT，安全策略需要根本性重构。
