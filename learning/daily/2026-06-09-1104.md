### AI Testing / Software Testing（12 项）

1. **Socratic-SWE: Self-Evolving Coding Agents via Trace-Derived Agent Skills** (arXiv:2606.07412, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07412
   - 领域：Agent 自我进化 / SWE Agent 训练
   - 技术介绍：提出闭环自我进化框架，复用 Agent 历史求解轨迹作为训练信号。不同于将轨迹仅用于奖励计算，Socratic-SWE 将其蒸馏为结构化 Agent 技能，总结重复失败和有效修复模式。这些技能指导真实仓库中针对性修复任务的生成。候选任务通过执行验证和求解器梯度对齐奖励评分，确保任务既可验证又能提升求解器。更新后的求解器产生新轨迹，使任务课程在连续轮次中自适应。在 SWE-bench Verified、Lite、Pro 和 Terminal-Bench 2.0 上持续超越自我进化基线，三轮迭代后 SWE-bench Verified 达到 50.40%。
   - 应用场景：SWE Agent 自我训练、代码修复技能积累
   - 为什么值得关注：首次证明求解轨迹可作为自我进化 SWE Agent 的可扩展基础，从"求解器"升级为"自我进化的求解器"。

2. **SWE-Explore: Benchmarking How Coding Agents Explore Repositories** (arXiv:2606.07297, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07297
   - 领域：Agent 仓库探索 / 代码定位评估
   - 技术介绍：SWE-bench 等仓库级基准将编码任务视为整体二元预测问题，忽视细粒度 Agent 能力。SWE-Explore 隔离评估仓库探索能力——给定仓库和问题，要求探索者在固定行数预算内返回相关代码区域的排序列表。覆盖 848 个问题、10 种编程语言、203 个开源仓库。沿覆盖率、排序和上下文效率三个维度评估探索能力，证明这些指标强烈跟踪下游修复行为。发现 Agent 探索者明显优于经典检索方法，但文件级定位仍有巨大改进空间。
   - 应用场景：Agent 代码理解能力评估、仓库级编码基准
   - 为什么值得关注：揭示 Agent 在"理解代码在哪里"这一基础能力上的差距，为 SWE Agent 提供细粒度诊断工具。

3. **QBugLM: Agentic Benchmarking Framework for LLM-based Quantum Software Debugging** (arXiv:2606.07314, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07314
   - 领域：量子软件调试 / LLM Agent 评估
   - 技术介绍：量子软件 Bug 通常产生静默错误输出而非显式错误，传统技术难以检测和修复。QBugLM 是多 Agent 框架，自动化量子软件调试管线——从分类驱动的 Bug 注入到 LLM 检测和修复，再到仿真验证，支持框架无关的 OpenQASM 3.0 程序。使用 Claude 4.6 Sonnet 和 Qwen3 Coder Next 进行基准测试，发现迭代反馈至关重要——单次重试将 Pass@1 从低于 25% 提升到高于 80%。更简单的结构化提示在固定资源约束下甚至优于 CoT 和 ReAct。
   - 应用场景：量子程序调试、LLM 量子代码能力评估
   - 为什么值得关注：首次系统 benchmark LLM 调试量子程序的能力，揭示量子软件调试与传统软件的根本差异。

4. **AKIRA: Understanding and Recommending API Replacement Refactorings** (arXiv:2606.06912, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06912
   - 领域：代码重构 / API 替换推荐
   - 技术介绍：API 替换重构（用 API 调用替换自定义逻辑）研究不足。通过对 6 个 Java 开源项目的 166,299 次提交进行实证研究，识别 366 个验证实例。提出 AKIRA 框架，结合模式确定性启发式与重构感知知识库。在人工标注数据集上达到 90% 召回率和 88% 精确率，在外部 RETIWA 数据集上将召回率从 21% 提升到 81%、精确率从 40% 提升到 78%。
   - 应用场景：代码质量维护、自动重构推荐
   - 为什么值得关注：将静态模式匹配与语义推理结合，显著提升复杂 API 替换重构的自动化推荐能力。

5. **LLM Agent-Assisted Reverse Engineering with Quantitative Readability Metrics** (arXiv:2606.06838, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06838
   - 领域：逆向工程 / 代码可读性
   - 技术介绍：自动反编译器产生功能正确但可读性差的 C 代码。提出三阶段研究演进：Phase 1（Ghidra MCP 工具驱动）因缺乏定量指导而效果不一致；Phase 2（结构相似度验证）发现 Agent 会优化指标但产生可读性更差的代码。贡献是 QRS（Quantitative Readability Score）框架——结合结构相似度门控与三个独立可读性子指标（词法惊喜度、结构简洁性、惯用质量）。QRS 引导的改进使 LLM Agent 在不牺牲正确性的前提下提升可读性。
   - 应用场景：二进制逆向、代码可读性自动化改进
   - 为什么值得关注：揭示 Agent 优化度量时的意外行为，提供定量可读性框架指导 Agent 改进。

6. **Empirical Study on AI-usage in GitHub Repositories** (arXiv:2606.06843, cs.SE)
   - 来源：https://arxiv.org/abs/2606.06843
   - 领域：AI 辅助开发 / 代码注释分析
   - 技术介绍：分析 35,361 条明确引用 AI 使用的 GitHub 代码注释及其关联代码块。开放编码 500 条唯一注释推导 AI 辅助开发活动分类法，然后使用两个 LLM 分类器标注完整数据集。分析 12,996 条后续提交消息研究 AI 辅助代码的演化。发现开发者主要将 LLM 用于代码实现，其次是代码增强、调试、文档和测试。后续提交频繁涉及重构和清理、功能集成扩展和 Bug 修复，表明人类在适应 AI 辅助代码方面的持续监督。AI 引用注释从直接代码生成转向知识和概念支持。
   - 应用场景：AI 辅助开发趋势分析、开发者行为研究
   - 为什么值得关注：大规模实证研究揭示 AI 工具如何真正嵌入开发工作流，从代码生成工具演变为协作支持机制。

7. **Agentic Very Much! Adoption of Coding Agent in New GitHub Projects** (arXiv:2606.07448, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07448
   - 领域：编码 Agent 采用率 / GitHub 趋势
   - 技术介绍：延续前序研究，分析新创建的 GitHub 项目中编码 Agent 的采用情况。发现新样本中编码 Agent 采用率是前次研究的 2 倍以上，采用强度也显著更高——AI 辅助提交比例明显上升，尽管有强烈迹象表明并未检测到全部采用。
   - 应用场景：AI 编码工具采用趋势追踪
   - 为什么值得关注：编码 Agent 在新项目中的采用率翻倍增长，反映 AI 辅助编码正从实验性工具变为主流开发方式。

8. **DoD AI Acquisition Readiness Assessment** (arXiv:2606.07393, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07393
   - 领域：AI 采购政策 / 国防软件获取
   - 技术介绍：评估美国国防部软件获取路径（SWP）是否足以应对 AI 获取的独特需求。通过场景分析追踪一个假设 AI 项目通过关键 SWP 规划活动，发现治理栈为迭代交付和 AI 测试提供了可行基础，但核心指导中存在反复的"可操作性"问题——AI 特定控制（数据溯源、生命周期管理、人工监督）分散在补充文件中而非嵌入 SWP 执行的项目级机制中。建议增加 AI 支持子路径和针对性工件改进。
   - 应用场景：AI 系统采购政策、国防 AI 治理
   - 为什么值得关注：揭示 AI 系统采购在政策到执行层面的关键缺口，对 AI 安全治理有广泛参考价值。

9. **MetaConfigurator: AI-Assisted RDF Authoring from JSON Data** (arXiv:2606.07094, cs.SE)
   - 来源：https://arxiv.org/abs/2606.07094
   - 领域：语义 Web / AI 辅助数据转换
   - 技术介绍：扩展开源 JSON Schema 编辑器 MetaConfigurator，使研究人员能使用 AI 辅助 RML 映射将现有 JSON/YAML/CSV 数据转换为 RDF。支持三元组精炼、SPARQL 查询执行、知识图谱可视化和 RDF 序列化导出。通过本体感知 IRI 自动补全、JSON-LD 文本视图与 RDF 三元组表之间的双向同步、以及从自然语言提示生成 SPARQL 查询来支持工作流。使用金属有机框架（MOF）合成实验数据演示。
   - 应用场景：科学数据语义化、AI 辅助知识图谱构建
   - 为什么值得关注：将传统结构化数据管理与语义 Web 技术桥接，通过 AI 辅助降低技术门槛。

10. **ArkTrans: Porting Declarative UI to HarmonyOS** (arXiv:2606.07085, cs.SE)

    - 领域：UI 跨平台迁移 / LLM 代码翻译

    - 应用场景：跨平台 UI 迁移、鸿蒙生态适配
    - 为什么值得关注：首次系统解决声明式 UI 跨平台翻译问题，对鸿蒙生态建设有直接价值。

11. **StainFlow: Entity-Stain Tracking for GUI Agent Process Rewards** (arXiv:2606.07027, cs.AI)
    - 来源：https://arxiv.org/abs/2606.07027
    - 领域：GUI Agent / 过程奖励模型
    - 技术介绍：RL 用于提升 GUI Agent 在长周期随机数字环境中的表现，但轨迹级成功反馈过于稀疏。提出 StainFlow——实体污渍流过程奖励模型。全局实体污渍追踪模块提取视觉可验证任务实体，追踪其污渍浓度和状态沿轨迹的演化，使任务阶段由实体证据流变化客观分离。局部污渍证据链接模块通过多帧证据聚合提升局部验证精度。灵感来自网络流分析中的污渍追踪机制。
    - 应用场景：GUI Agent RL 训练、过程奖励建模
    - 为什么值得关注：将网络流分析中的污渍追踪机制引入 GUI Agent 奖励设计，提供新颖的细粒度反馈方法。

12. **Sim-to-Real Gap of Foundation Model Agents: Unified MDP Perspective** (arXiv:2606.07017, cs.AI, KDD 2026 Blue Sky)

### AI / LLM 技术（13 项）

13. **How AI Agents Reshape Knowledge Work** (arXiv:2606.07489, cs.AI)
    - 来源：https://arxiv.org/abs/2606.07489
    - 领域：AI Agent 生产力 / 知识工作
    - 技术介绍：使用 Perplexity Search 和 Computer 产品的生产数据，研究 AI Agent 从对话助手到自主 Agent 的转变。三个关键发现：(1) Computer 每用户会话执行 26 分钟自主工作（vs Search 的 33 秒），将后续查询转向更高层次工作（验证和扩展），每查询不满率比 Search 低 55%；(2) 在匹配任务上将完成时间从 269 分钟减少到 36 分钟，估计时间和成本分别降低 87% 和 94%；(3) Computer 改变用户尝试的工作范围——查询更频繁跨越职业边界、需要更高阶认知、涉及更广泛专业知识、解锁 Search 使用中基本不存在的工作活动。
    - 应用场景：AI Agent 生产力评估、知识工作自动化
    - 为什么值得关注：首次使用真实生产数据量化 AI Agent 对知识工作的变革性影响，87% 时间降低和 94% 成本降低是里程碑式数据。

14. **AARRI-Bench: Act As a Real Research Intern** (arXiv:2606.07462, cs.AI)
    - 来源：https://arxiv.org/abs/2606.07462
    - 领域：Agent 研究能力评估 / 基准测试
    - 技术介绍：概念化 AARR（Act As a Real Researcher）基准系列，聚焦 Agent 是否能模拟人类研究者的专业性、彻底性和细致推理。提出 AARRI-Bench（研究实习生级），现有基准主要评估宏观执行能力，而 AARR 关注细粒度研究场景。在前沿模型和 Agent 系统上进行广泛实验，发现最佳配置（Mini-SWE-Agent + Claude Opus 4.7）仅达到 68.3% 成功率，经常忽略真实人类研究者显而易见的微妙但关键的细节。
    - 应用场景：Agent 研究能力评估、科研自动化
    - 为什么值得关注：从"能不能执行"到"能不能像真正研究者一样思考"的评估范式转变。

15. **DuMate-DeepResearch: Auditable Multi-Agent Deep Research** (arXiv:2606.07299, cs.AI)
    - 来源：https://arxiv.org/abs/2606.07299
    - 领域：多 Agent 深度研究 / 可审计性
    - 技术介绍：深度研究（DR）作为新 Agent 范式应对复杂开放研究任务，但当前 DR 系统受限于四个相互关联的限制：未指定范围内的长周期规划、单 Agent 分解和调度瓶颈、长文合成中的幻觉风险、有限的过程可审计性。DuMate 框架解耦 Agent Core（任务理解、规划、调度）与可扩展 Tool Ecosystem（检索、证据获取、报告渲染），使每个中间决策和工具调用显式可追踪。基于百度千帆 Agent 工厂构建。
    - 应用场景：深度研究自动化、多 Agent 协作
    - 为什么值得关注：解耦架构使深度研究过程完全可审计，解决当前 DR 系统的核心可信度问题。

16. **Online Pandora's Box for Contextual LLM Cascading** (arXiv:2606.07392, cs.AI)
    - 来源：https://arxiv.org/abs/2606.07392
    - 领域：LLM 级联 / API 选择优化
    - 技术介绍：受 LLM 级联启发，提出在线上下文 Pandora's Box 模型，用于自适应查询和选择 LLM API。决策者观察请求上下文，面临两阶段决策：查询阶段顺序查询 API（每次查询产生输出并产生成本），选择阶段选择生成的输出部署并观察下游奖励。这种输出介导的反馈结构不同于经典模型（打开盒子直接揭示奖励）。直接建模保留指数并开发查询阶段学习方法，结合 GMM 类型估计和 UCB 风格置信界。在 T 周期范围内实现维度依赖的 Õ(√T) 累积遗憾。
    - 应用场景：LLM API 成本优化、模型级联决策
    - 为什么值得关注：为 LLM 级联提供理论保证的在线决策框架，在成本和质量之间实现最优权衡。

17. **Off-Policy Evaluation with Strategic Agents via Local Disclosure** (arXiv:2606.07308, cs.AI)

Socratic-SWE 提出了一种闭环自我进化框架，从根本上改变了 SWE Agent 的训练范式。传统方法依赖固定突变或 Bug 注入程序合成数据，这些分布独立于 Agent 自身的弱点和训练进展。Socratic-SWE 的创新在于将 Agent 的历史求解轨迹从"奖励计算证据"升级为"结构化技能来源"——蒸馏重复失败模式和有效修复模式，生成针对性修复任务。候选任务通过执行验证和求解器梯度对齐奖励评分，确保任务既可通过执行验证又对提升求解器有用。更新后的求解器产生新轨迹，使任务课程在连续轮次中自适应演化。在 SWE-bench Verified、Lite、Pro 和 Terminal-Bench 2.0 四个基准上持续超越自我进化基线，三轮迭代后 SWE-bench Verified 达到 50.40%。

**技术架构**：
- 轨迹收集层：记录 Agent 求解历史轨迹
- 技能蒸馏层：从轨迹中提取结构化 Agent 技能（失败模式+修复模式）
- 任务生成层：基于技能生成针对性修复任务
- 验证评分层：执行验证 + 求解器梯度对齐奖励
- 自适应课程层：基于求解器表现动态调整任务难度

**核心算法**：
- 轨迹→技能蒸馏算法：将非结构化求解轨迹转化为可复用的结构化技能
- 求解器梯度对齐奖励：衡量任务对当前求解器的提升价值
- 执行验证管道：确保生成任务的可执行性和正确性

**关键实现机制**：
- 技能表示：结构化格式存储失败模式和修复模式
- 任务生成：基于技能指导的真实仓库任务生成
- 课程自适应：根据求解器表现动态调整任务分布

**与现有技术对比**：
- 传统合成数据：固定分布，独立于 Agent 弱点 → Socratic-SWE：自适应分布，针对 Agent 弱点
- 轨迹仅用于奖励：信息利用不充分 → Socratic-SWE：轨迹蒸馏为技能，信息利用最大化
- 静态课程：固定任务分布 → Socratic-SWE：动态课程，随求解器演化

**优势与局限**：
- 优势：自我进化闭环、无需外部数据、技能可积累
- 局限：依赖高质量初始轨迹、计算成本较高

**未来发展潜力**：
自我进化范式可能成为 SWE Agent 的标准训练方法，扩展到更多编码任务和编程语言。

**个人技术理解**：
Socratic-SWE 代表了 Agent 训练从"外部数据驱动"到"自我驱动"的范式转变。将求解轨迹从"一次性奖励信号"升级为"可积累技能资产"是关键创新，类似人类从经验中学习的机制。

#### 2. DuMate-DeepResearch: Auditable Multi-Agent Deep Research

**地址**：https://arxiv.org/abs/2606.07299

**项目介绍**（250 字）：
DuMate-DeepResearch 是基于百度千帆 Agent 工厂构建的多 Agent 深度研究框架。当前深度研究系统受限于四个相互关联的限制：未指定范围内的长周期规划、单 Agent 分解和调度瓶颈、长文合成中的幻觉风险、有限的过程可审计性。DuMate 的架构创新在于解耦 Agent Core（处理任务理解、规划和调度）与可扩展 Tool Ecosystem（检索、证据获取、报告渲染），使每个中间决策和工具调用显式可追踪。这种解耦设计使深度研究过程从"黑盒"变为"白盒"，每个决策步骤都有明确的记录和审计轨迹。

**技术架构**：
- Agent Core：任务理解、规划、调度
- Tool Ecosystem：检索、证据获取、报告渲染
- 审计层：每个中间决策和工具调用显式可追踪

**核心算法**：
- 递归搜索：迭代式问题框架化、证据获取、源验证
- Rubric-grounded 推理：基于评分标准的推理过程

**关键实现机制**：
- Core/Ecosystem 解耦：职责分离，独立扩展
- 审计追踪：每步决策可追溯

**与现有技术对比**：
- 单 Agent DR：黑盒、不可审计 → DuMate：白盒、完全可审计
- 不可追踪决策：无法定位错误 → DuMate：每步可追溯

**优势与局限**：
- 优势：完全可审计、模块化扩展、幻觉风险可控
- 局限：架构复杂度增加、多 Agent 协调开销

**未来发展潜力**：

**个人技术理解**：
解耦架构是解决深度研究可信度问题的正确方向。将"能不能做研究"升级为"能不能证明研究过程正确"，是 AI 研究工具从实验性走向生产级的关键一步。

#### 3. AFSAT: GPU-Accelerated SAT Solver

**地址**：https://arxiv.org/abs/2606.06641

**项目介绍**（250 字）：
AFSAT（Accelerated Fourier SAT）是基于连续局部搜索（CLS）的 GPU 加速伪布尔可满足性求解器。它将 FastFourierSAT 的概念验证方法实现为完全工程化的求解器，支持单个问题实例中任意异构混合的对称约束类型和长度。使用 JAX 编译器，AFSAT 利用纯函数组合、自动向量化、自动微分和 JIT 编译，在候选分配批次上执行大规模并行 CLS。通过识别和解决内存延迟和浮点表示的局限性，以及利用自动并行化和紧凑表示，实现显著改善的数值稳定性、运行性能和内存效率。通过定制的离散傅里叶变换实现部分解决浮点固有的表示和稳定性限制。通过 JAX 数组分片实现扩展到多个加速器的近线性吞吐。

**技术架构**：
- JAX 编译器层：纯函数组合、自动向量化、JIT 编译
- 并行 CLS 引擎：大规模并行候选分配评估
- DFT 实现：定制离散傅里叶变换解决浮点限制
- 数组分片：多加速器近线性扩展

**核心算法**：
- 连续局部搜索（CLS）：将 SAT 问题放松为连续优化
- 离散傅里叶变换：解决浮点表示限制
- 数组分片：多 GPU 并行

**关键实现机制**：
- 纯函数组合：确保可并行性
- 自动向量化：最大化 GPU 利用率
- 紧凑表示：减少内存占用

**与现有技术对比**：
- CPU SAT 求解器：串行、扩展性有限 → AFSAT：大规模并行、近线性扩展
- 概念验证 AFSAT：功能不完整 → AFSAT：完全工程化、支持任意约束混合

**优势与局限**：
- 优势：GPU 加速、近线性扩展、数值稳定性改善
- 局限：浮点精度限制、依赖 JAX 生态

**未来发展潜力**：
GPU SAT 求解器在硬件验证、组合优化等领域有广泛应用前景，可能成为 SAT 求解的新标准。

**个人技术理解**：
将 SAT 求解从 CPU 串行扩展到 GPU 大规模并行是组合求解领域的重大突破。JAX 的自动并行化能力使这一扩展变得可行，代表了 AI 加速器向传统求解领域的渗透。

#### 4. A Science of AI Must Study Training Dynamics (ICML 2026 Oral)

**地址**：https://arxiv.org/abs/2606.06533

**论文介绍**（300 字）：
这篇由 Stella Biderman 等人撰写的立场论文（ICML 2026 Oral 接收）提出了一个根本性的方向转变：AI 科学必须从分析训练后的固定模型转向研究训练动态。论文的核心论点是模型不是静态对象——它们是由数据、目标、架构和优化动态塑造的时间演化过程的快照。然而大量 AI 研究将模型视为固定工件，分析训练后的行为而非询问它们为何出现。论文主张 AI 科学应支持 progressively 更强的理解形式：从早期训练信号预测结果、在轨迹出错时干预、最终设计更可靠产生所需属性的训练程序。Scaling laws 使损失预测成为常规；挑战是将此成功扩展到能力、偏见、鲁棒性和安全相关行为。论文从科学史和科学哲学角度阐述了这种理论的要求，审视了机制可解释性、公平性、记忆化和简单性偏见的进展，并识别了具体的开放问题。

**技术架构**：
- 训练动态分析框架：从早期信号预测最终行为
- 干预机制：在训练轨迹出错时进行干预
- 设计理论：设计更可靠产生所需属性的训练程序

**核心算法**：
- 早期训练信号分析：从训练初期数据预测最终能力
- 轨迹干预：在训练过程中修正偏离
- 训练程序设计：从经验到理论的转变

**关键实现机制**：
- Scaling laws 扩展：从损失预测到能力/偏见/鲁棒性预测
- 机制可解释性：理解训练过程中能力的涌现
- 公平性分析：训练动态对公平性的影响

**与现有技术对比**：
- 后验分析：分析训练后模型的行为 → 训练动态：研究行为为何涌现
- 经验调参：基于试错的训练调整 → 理论指导：基于理解的设计
- 固定模型评估：静态能力测量 → 动态过程理解：演化轨迹分析

**优势与局限**：
- 优势：方向性贡献、理论框架、ICML Oral 认可
- 局限：立场论文、具体算法仍在发展中

**未来发展潜力**：
可能成为 AI 基础研究的范式转变——从"模型工程"到"训练科学"。

**个人技术理解**：
这篇论文的核心价值在于方向性——它指出当前 AI 研究的一个系统性盲点：我们花了大量时间分析"模型做了什么"，却很少研究"模型为什么变成这样"。将 AI 研究从"后验解释"转向"训练过程理解"，是建立真正 AI 科学的必经之路。这与 Glassbox AI 架构（从后验解释到先验调解）形成有趣的呼应。

#### 5. Attack Selection in Agentic AI Control Evaluations

**地址**：https://arxiv.org/abs/2606.06529

**论文介绍**（250 字）：
这篇论文揭示了 AI 控制评估中的一个关键缺陷：当前评估通常假设不策略性选择何时攻击的攻击者，而策略性选择攻击时机的攻击者远比 indiscriminately 攻击的攻击者更难捕捉。论文在 BashArena 和 LinuxArena 两个 Agent 设置中研究攻击选择能力，将攻击决策分解为开始策略（何时攻击）和停止策略（何时中止进行中的攻击）。在 1% 审计预算下，开始策略在两个 Arena 上将安全性降低 20pp，停止策略在 BashArena 上降低 20pp、LinuxArena 上降低 28pp。这些降低应解释为攻击选择效应的上界。现有控制评估可能产生过于乐观的安全估计。论文建议未来评估、系统卡和安全案例应包含攻击选择以产生更现实的安全估计。

**技术架构**：
- 开始策略：决定何时启动攻击
- 停止策略：决定何时中止进行中的攻击
- 审计预算约束：有限人工审计资源

**核心算法**：
- 攻击时机选择：策略性决定攻击窗口
- 攻击中止决策：避免被检测的撤退策略

**关键实现机制**：
- 审计预算建模：有限资源下的最优审计策略
- 安全度量：策略性攻击下的安全性评估

**与现有技术对比**：
- 非策略性攻击：indiscriminately 攻击 → 策略性攻击：选择性攻击
- 乐观安全估计：忽略攻击选择 → 现实安全估计：包含策略性攻击者

**优势与局限**：
- 优势：揭示现有评估的系统性偏差、量化攻击选择的影响
- 局限：仅在两个 Arena 上验证、需要更多场景验证

**未来发展潜力**：
可能推动 AI 控制评估框架的重构——从"假设诚实攻击者"到"假设策略性攻击者"。

**个人技术理解**：
这篇论文的发现对 AI 安全领域有深远影响。如果现有 AI 控制评估在策略性攻击下安全性下降 20-28pp，那么当前声称的"安全"可能严重高估。这类似于网络安全中的"红队测试"——必须假设攻击者是有策略的，而非随机攻击。

## 新领域收获

1. **扩散模型引导约束满足搜索（DiBS）**：扩散模型从生成任务扩展到搜索引导——用学习型全局指导增强精确符号求解器。在 Royle 17-clue Sudoku 基准上大幅减少搜索成本。代表学习型方法与精确求解的首次有效融合。

2. **公平性作为对称性操作**：将偏差形式化为对称性破坏操作——分类器在切换敏感属性（保持优点特征固定）时输出应保持不变。通过基于损失的正规化作为对称恢复机制，在四个合成数据集上实现 90%+ 违规减少，准确率成本约 5%。不需要因果图知识，计算轻量，可泛化到任何可定义为位翻转的敏感属性。

3. **策略行为下的离线策略评估（OPE）**：将博弈论引入策略评估——决策主体通过策略性修改协变量来响应决策者策略。通过事后解释披露局部信息揭示主体先策略协变量，缓解策略行为引起的信息损失。代表 AI 决策评估从"被动主体"到"策略性主体"的范式转变。
