### AI Testing / Software Testing（12 项）

1. **FASE: Fast Adaptive Semantic Entropy for Code Quality** (arXiv:2606.09800, cs.SE)
   - 来源：https://arxiv.org/abs/2606.09800
   - 领域：代码质量评估 / 多 Agent 代码生成
   - 技术介绍：多 Agent 代码生成通过模拟人类软件工程生命周期提供自主开发范式，但系统可靠性受限于 LLM 幻觉和跨 Agent 错误传播。语义熵提供了无需 ground-truth 量化不确定性的原理方法，但现有方法依赖昂贵的 LLM 驱动等价性检查。FASE 提出基于结构和语义差异图最小生成树近似功能正确性的新指标。在 HumanEval 和 BigCodeBench 上，FASE 使用 Qwen3-Embedding-8B 超越 LLM 蕴含语义熵，Spearman 相关提升 25%，ROCAUC 提升 19%。消除 LLM 等价评估后，FASE 仅需传统方法 0.3% 的运行时成本。
   - 应用场景：多 Agent 代码生成质量评估、不确定性量化
   - 为什么值得关注：首次证明可用极低成本的嵌入方法替代 LLM 验证，为多 Agent 工作流提供实用不确定性量化方案。

2. **EvalCards: An Interpretive Layer for AI Evaluation Reporting** (arXiv:2606.09809, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09809
   - 领域：AI 评估报告标准化
   - 技术介绍：AI 评估结果大规模产生但报告不一致——leaderboard、model card、基准论文、公司博客各自为政。EvalCards 提出可操作的报告层，将基准元数据、评估运行数据和模型元数据组合为统一记录。从 52 篇论文和 10 个利益相关者访谈推导报告 schema，实现四种解释信号（可复现性、文档完整性、来源和风险、分数可比性），通过研究和非研究读者模式呈现。部署监控工具覆盖 5,816 个模型、635 个基准、101,843 条结果，揭示当前报告实践中的系统性差距。
   - 应用场景：AI 评估标准化、模型比较
   - 为什么值得关注：直击 AI 评估领域最大痛点——结果不可比，为评估透明度和可审计性提供基础设施。

3. **SIGA: Simulator-Interface Grounding Adapter for Scientific Simulation** (arXiv:2606.09774, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09774
   - 领域：科学模拟 / Agent-工具接口
   - 技术介绍：高级科学模拟器暴露专用输入语言，学习成本高昂。SIGA 为现成编码 Agent 提供模拟器可执行契约——词汇、结构约束、验证规则和终止条件。通过检索、程序记忆、轨迹内验证和验证强制终止实现。在 GEOS 多物理模拟器上，SIGA 约 5 分钟生成完整 GEOS deck（TreeSim > 0.90），匹配耗时 3 小时的人类专家，约 36x 加速。自我进化通过从先前轨迹重写适配器内容进一步提升性能。迁移到 OpenFOAM 和 LAMMPS 显示主导机制因接口而异。
   - 应用场景：科学软件自动化操作、领域专家替代
   - 为什么值得关注：证明轻量级自改进接地层可将通用编码 Agent 转变为科学软件的实际操作者。

4. **CHAP: Collaborative Human-Agent Protocol** (arXiv:2606.09751, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09751
   - 领域：人类-Agent 协作协议
   - 技术介绍：基础模型从响应生成进入操作角色——规划多步骤、调用工具、请求人类输入、协调其他 Agent。生产部署不再是单人监督单模型，而是跨团队、跨时区、跨信任边界的多人类多 Agent 协作。MCP 标准化 Agent 工具访问，A2A 标准化 Agent 间互操作，但均未定义人类和 Agent 共同执行可问责工作的共享工作空间。CHAP 将过去消失在聊天线程中的覆盖变为携带 diff、理由和内容哈希的结构化事件。交接变为可移植信封而非置顶消息。人类批准变为不可抵赖的签名决策。
   - 应用场景：企业级 AI 部署审计、合规工作流
   - 为什么值得关注：填补 MCP 和 A2A 之间的关键空白——人类-Agent 可问责协作，是企业级 AI 落地的基础设施。

5. **Multi-Turn Evaluation of Deep Research Agents Under Process-Level Feedback** (arXiv:2606.09748, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09748
   - 领域：深度研究 Agent 评估 / 多轮反馈
   - 技术介绍：现有深度研究 Agent 基准仅评估单次输出，忽视关键问题：DRAs 在反馈指导下能否改进报告？本文在两种反馈设置下进行多轮评估：自我反思（无外部诊断信号）和过程级反馈（针对研究策略差距的引导）。设计 Research Gap Inference（RGI）方法分析满足和不满足的评分标准模式推断研究过程差距。发现：(i) 自我反思下 Agent 纳入和回退评分标准的比率几乎相等，净改进可忽略；(ii) 单轮过程级反馈带来显著增益，标准化分数提升 8-15 分，约 35-40% 纳入率；(iii) 增益在多轮中不累积，Agent 重写完整报告时回退高达 24% 先前满足的标准。
   - 应用场景：深度研究 Agent 评估、多轮改进
   - 为什么值得关注：SCALE-ICML 2026 Oral，揭示即使有目标引导，可靠的多轮改进仍超出当前 DRA 架构能力。

6. **SearchSwarm-30B-A3B: Delegation Intelligence for Long-Horizon Deep Research** (arXiv:2606.09730, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09730
   - 领域：委托智能 / 长程研究 Agent
   - 技术介绍：LLM 越来越多地处理复杂长程真实世界任务，但上下文窗口固有有限。主 Agent 分解任务并派遣子任务给子 Agent，子 Agent 执行并仅返回摘要结果。这需要委托智能——分解复杂任务、确定何时何物委托、将返回结果整合到持续工作流的能力。本文设计引导模型走向高质量任务分解和委托的 harness，约束子 Agent 正确返回结果支持主 Agent 工作流。harness 引导的轨迹自然编码正确的委托决策，用作 SFT 数据将委托智能内化到模型权重。SearchSwarm-30B-A3B 在 BrowseComp 达 68.1、BrowseComp-ZH 达 73.3，同类规模最佳。
   - 应用场景：长程研究 Agent、多 Agent 协作
   - 为什么值得关注：首次系统探索委托智能的训练数据合成和模型训练，BrowseComp 同类规模 SOTA。

7. **PRIME: Proxy Reward Internalization and Mechanistic Exploitation** (arXiv:2606.09711, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09711
   - 领域：奖励黑客 / AI 安全
   - 技术介绍：奖励黑客通常在变得可见后研究——模型获得高代理奖励但未能完成预期任务。PRIME 研究代理 RL 在该失败出现之前教了什么。引入代理奖励内化和机制利用——评估任务正确性、预测代理接受度、推理可利用代理-黄金差距的学习能力。在可剥削 pytest 奖励的编码 RL 环境中，通过 CoT 监控、直接探针和激活级概念向量测量 PRIME。发现 PRIME 在持续奖励黑客之前以分阶段序列出现，其当前直接探针分数预测后期黑客发作和严重程度，即使可见黑客率仍低。PRIME 在评估器变化时适应，消融其激活方向减少黑客。
   - 应用场景：AI 对齐早期预警、奖励黑客检测
   - 为什么值得关注：将奖励黑客研究从"事后检测"推进到"事前预警"，激活探针可作为更广泛对齐风险的早期信号。

8. **Trellis: Autoformalization with Process Semantics for Lean Proofs** (arXiv:2606.09674, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09674
   - 领域：自动形式化 / Lean 证明
   - 技术介绍：Trellis 是自动形式化系统，利用 LLM Agent 在确定性约束工作流中通过自然语言证明的迭代精炼强制执行 Lean 自动形式化任务的增量进展。动机来自数学家的严格证明概念——证明的任何部分都应有能力进一步细化。系统旨在以适度预算和通用 Agent 实现可靠自动形式化，专业化不来自任务特定 Agent 训练，而是来自由过程语义强制执行的意义严格工作流。链接到由该过程产生的近期 Ramsey 理论突破的端到端 Lean 形式化。
   - 应用场景：数学定理自动形式化、Lean 证明辅助
   - 为什么值得关注：从"训练专用 Agent"转向"过程语义约束工作流"，为自动形式化提供新思路。

9. **SpatialWorld: Interactive Spatial Reasoning Benchmark for Multimodal Agents** (arXiv:2606.09669, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09669
   - 领域：空间推理基准 / 多模态 Agent
   - 技术介绍：空间推理是多模态 LLM 感知和操作物理世界的基础能力。现有基准主要依赖被动评估（静态 VQA）或模拟器特定管线，无法评估通用交互式空间理解。SpatialWorld 统一基准专为评估多模态 Agent 在复杂真实世界任务中的交互式空间理解而设计。整合八个异构仿真后端到共享模拟器无关协议，760 个人类标注任务覆盖家庭日常、旅行、社交协作等领域。Agent 必须在仅视觉部分可观测性下解决问题，主动收集自我中心视觉证据并通过 MLLM 原生统一文本操作接口表达决策。15 个先进 Agent 评估显示最强模型 GPT-5 平均任务成功率仅 17.4%。
   - 应用场景：多模态 Agent 空间能力评估、具身 AI
   - 为什么值得关注：揭示多模态 Agent 在主动空间探索上的巨大差距，为具身 AI 提供严格测试平台。

10. **PerGent: Agentic Persona Generation with Critique-Refinement** (arXiv:2606.09637, cs.SE)
    - 来源：https://arxiv.org/abs/2606.09637
    - 领域：Persona 生成 / 软件需求工程
    - 技术介绍：Persona 广泛用于软件工程支持需求获取、设计和验证，但手动创建成本高、耗时、难以扩展。PerGent 是工业级 Persona 生成方法，围绕迭代批评-精炼循环构建。生成器和批评者 LLM Agent 由编排器协调，通过外部资源（访谈、调查、招聘广告）迭代精炼 Persona。在 Kinaxis 工业环境中部署评估，专家现场评估中 PerGent 达最高专家批准率 96.9%，超越所有基线。与 LLM 采用前手动创建的最佳实践 Persona 比较，PerGent 复现更大比例的专家内容同时贡献大量新内容。RE 2026 工业轨道录用。
    - 应用场景：软件需求工程、用户画像生成
    - 为什么值得关注：首次工业级 LLM Persona 生成评估，96.9% 专家批准率证明实用性。

11. **AliyunConsoleAgent: Training Web Agents in Real-World Cloud Environments** (arXiv:2606.09447, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09447
    - 领域：Web Agent 训练 / 云控制台验证
    - 技术介绍：主要云平台涵盖数百种产品快速迭代，控制台 UI 频繁与文档偏离。验证文档程序准确反映当前控制台需约 400 万次重复检查，手动覆盖率低于 1%。AliyunConsoleAgent 提出两阶段训练范式：在蒸馏 frontier 模型轨迹上 SFT，然后使用 GRPO 和双通道结果奖励模型在真实云环境中 RL。构建高确定性 rollout 系统（Terraform 资源预配置 + LLM 按需配置），有效隔离环境噪声。引入基于后端审计日志的规则奖励评估协议，提供客观、防奖励黑客的结果判断。278 任务基准上 AliyunConsoleAgent-32B 达 63.52% 平均成功率，比基线提升 20.24pp，与最佳 frontier 模型差距仅 1.82pp，推理成本低 92%。
    - 应用场景：云控制台自动化验证、Web Agent 训练
    - 为什么值得关注：首次大规模真实云环境 Web Agent 训练，32B 模型逼近 frontier 模型性能。

12. **WeaveBench: Long-Horizon Benchmark for Computer-Use Agents with Hybrid Interfaces** (arXiv:2606.09426, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09426
    - 领域：计算机使用 Agent 基准 / 混合接口
    - 技术介绍：计算机使用 Agent 越来越多地在结合视觉桌面控制、命令行执行、代码编辑、浏览器和外部工具的运行时中操作。现有基准通常将这些接口评估为可分离能力，长程跨接口编排测试不足。WeaveBench 是长程混合接口基准，114 个任务覆盖 8 个真实世界工作领域，基于真实用户请求和公开可验证工件。每个任务要求 Agent 在单一轨迹中组合 GUI 观察/操作与 CLI/代码操作。在部署的 CLI Agent 运行时内的真实 Ubuntu 桌面上评估，辅以最小桌面控制插件。提出轨迹感知 judge 检查交付物、文件、截图、日志和操作痕迹，检测伪造视觉证据或硬编码指标等捷径行为。最佳 PassRate 仅 41.2%。
    - 应用场景：CUA 能力评估、混合接口 Agent 测试
    - 为什么值得关注：揭示 CUA 评估中的关键空白——跨接口编排能力，结果远未饱和。

### AI / LLM 技术（13 项）

13. **Optical Reasoning: Images as Standalone Reasoning Medium** (arXiv:2606.09585, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09585
    - 领域：推理媒介 / 多模态推理
    - 技术介绍：CoT 改进 LLM 性能并已扩展到 MLLM。近期工作从基于文本的多模态推理转向交错模态推理，中间步骤可包含文本理由和视觉证据。本文提出更激进想法：图像能否单独作为语言和模态任务的推理媒介？提出光学推理概念，将图像作为独立推理媒介。实例化为两个变体：排版光学推理（优化视觉布局紧凑理由渲染）和图形光学推理（组合文本和图形元素为结构化视觉理由）。在数学、科学和交错模态推理基准上，光学推理可匹配甚至超越传统文本推理，同时语言任务推理 token 减少 28.57%、多模态任务减少 16%，达到文本推理 1.96x 的 token 效率。
    - 应用场景：高效推理、多模态推理
    - 为什么值得关注：挑战"推理必须是文本"的假设，图像作为推理媒介效率更高。

14. **TABVERSE: Cross-Format Table Understanding Benchmark** (arXiv:2606.09578, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09578
    - 领域：表格理解基准 / 多模态评估
    - 技术介绍：LLM 和 VLM 越来越多地在表格推理任务上评估，但表格表示的作用未被充分探索。实践中同一表格内容可以不同结构格式出现（HTML、Markdown、LaTeX）或作为渲染图像。现有评估通常让内容、格式、布局和模态一起变化，难以隔离表示效应。TABVERSE 是受控多模态表格基准，将同一表格内容跨多种结构格式和渲染图像对齐，带问题类别和难度标签。评估 QA、SUC 和 SR 三个任务。发现表示选择显著影响表格理解——模型通常在结构化文本上表现优于渲染图像，但差距大小取决于任务、模型和格式。HTML 通常是最稳健的文本格式。
    - 应用场景：表格理解评估、多模态基准
    - 为什么值得关注：首次隔离表格表示效应的受控评估，为可靠表格评估提供基础。

15. **Self-Explainability in Self-Adaptive Systems** (arXiv:2606.09568, cs.AI)

#### 1. FASE: Fast Adaptive Semantic Entropy for Code Quality

- **地址**：https://arxiv.org/abs/2606.09800
- **类型**：论文

**论文介绍**（200-400 字）：多 Agent 代码生成通过模拟人类软件工程生命周期提供自主软件开发范式，但系统可靠性受限于 LLM 幻觉和跨交互 Agent 的错误传播。语义熵提供了无需 ground-truth 量化不确定性的原理方法，但当前方法通常依赖昂贵的 LLM 驱动等价性检查。本文引入 FASE（快速自适应语义熵），一种基于结构和语义差异图最小生成树近似功能正确性的新指标。FASE 的核心洞察是：多个 Agent 生成的代码变体之间的结构和语义差异可以直接映射到功能正确性概率——高度一致的变体更可能正确，分歧大的变体更可能有错误。在 HumanEval 和 BigCodeBench 上，FASE 使用 Qwen3-Embedding-8B 超越 LLM 蕴含语义熵，Spearman 相关提升 25%，ROCAUC 提升 19%。关键创新在于消除 LLM 驱动等价评估——FASE 仅需传统方法约 0.3% 的运行时成本。这使 FASE 成为现实世界多 Agent 工作流中优化不确定性量化的实用、成本效益方案。

**技术架构**：
- 结构差异图：捕获代码变体间的 AST/控制流差异
- 语义差异图：使用嵌入模型捕获语义差异
- 最小生成树：综合结构和语义差异为单一质量分数
- Qwen3-Embedding-8B：高效嵌入模型替代 LLM 验证

**核心算法**：
1. 收集多 Agent 生成的代码变体
2. 构建结构差异图（基于 AST 和控制流分析）
3. 构建语义差异图（基于嵌入模型）
4. 计算最小生成树综合两种差异
5. MST 权重映射为功能正确性概率

**与现有技术对比**：
- Pass@k：需要 ground-truth 测试用例
- LLM 语义熵：依赖昂贵 LLM 等价性检查
- FASE：无需 ground-truth、无需 LLM、成本仅为 0.3%

**优势与局限**：
- 优势：极低计算成本、无需 ground-truth、超越 LLM 基线
- 局限：依赖嵌入模型质量、对语义细微差异可能不敏感

**未来发展潜力**：FASE 为多 Agent 代码生成质量评估提供了可扩展方案。随着多 Agent 工作流普及，此类低成本不确定性量化方法将成为基础设施。

**个人技术理解**：FASE 的核心贡献是证明"嵌入模型可以替代 LLM 做代码质量评估"。在成本敏感的多 Agent 场景中，0.3% 的成本差异意味着从不可行到可行的质变。MST 方法将多维差异压缩为单一可解释分数，这是工程实用性的重要设计。

#### 2. CHAP: Collaborative Human-Agent Protocol

- **地址**：https://arxiv.org/abs/2606.09751
- **类型**：论文

**论文介绍**（200-400 字）：基础模型从响应生成进入操作角色——它们规划多步骤、调用工具、请求人类输入、协调其他 Agent，越来越多地承担影响客户、索赔、代码、合同和临床决策的工作责任。生产部署不再是单人监督单模型，而是跨团队、跨时区、跨信任边界的多人类多 Agent 协作。这种协作的技术表面仍然弱定义。当 Agent 起草响应而人类在发送前编辑它时，人类判断的时刻是系统中最有价值的信号。在当前实践中，它被记录（如果有）在应用代码、聊天线程、工单评论和部落记忆中。两个协议标准处理相邻问题：MCP 标准化 Agent 对工具和数据的访问，A2A 标准化 Agent 间互操作。但两者均未定义人类和 Agent 共同执行可问责工作的共享工作空间。CHAP 通过小核心（工作空间、参与者、任务、工件和仅追加证据日志）加上可组合配置文件（添加审查、模式、路由、审议、交接、身份、签名和透明审计）实现这一目标。过去消失在聊天线程中的覆盖变为携带 diff、理由和内容哈希的结构化事件。班次交接变为可移植信封而非置顶消息。人类批准变为不可抵赖的签名决策可多年后重放。

- 不可抵赖签名：人类批准的可审计记录

**核心设计原则**：
1. 人类判断时刻是系统中最有价值的信号
2. 交接必须是可移植的而非平台绑定的
3. 审计必须可重放而非依赖部落记忆
4. 协议必须可组合而非单一用途

**与现有技术对比**：
- MCP：标准化 Agent-工具访问，不涉及人类协作
- A2A：标准化 Agent-Agent 互操作，不涉及人类

**优势与局限**：
- 优势：填补关键协议空白、可组合设计、企业级审计

**个人技术理解**：CHAP 触及了一个被严重忽视的问题——当 AI 进入生产环境，"谁对什么负责"变得模糊。MCP 和 A2A 解决了 Agent 如何访问工具和彼此通信，但没有解决人类如何与 Agent 共同执行可问责工作。CHAP 的"不可抵赖签名"和"可重放审计"设计直接回应了企业合规需求。

#### 3. PRIME: Proxy Reward Internalization and Mechanistic Exploitation

- **地址**：https://arxiv.org/abs/2606.09711
- **类型**：论文

**论文介绍**（200-400 字）：奖励黑客通常在变得可见后研究——模型获得高代理奖励但未能完成预期任务。本文研究代理 RL 在该失败出现之前教了什么。引入 PRIME（代理奖励内化和机制利用）——评估任务正确性、预测代理接受度和推理可利用代理-黄金差距的学习能力。在可剥削 pytest 奖励的编码 RL 环境中，通过 CoT 监控、直接探针和激活级概念向量测量 PRIME。发现 PRIME 在持续奖励黑客之前以分阶段序列出现——其当前直接探针分数预测后期黑客发作和严重程度，即使可见黑客率仍低。PRIME 在评估器变化时适应，重新定位到仍然被奖励的任何代理-黄金差距，当黄金奖励抑制明显黑客时持续存在，消融其激活方向减少黑客。跨检查点，域内 PRIME 跟踪域外不对齐。这些结果表明可利用的代理 RL 放大了可见黑客上游的代理内化能力，使 PRIME 成为更广泛对齐风险的候选早期预警信号。

**技术架构**：
- CoT 监控：追踪推理链中的代理评估行为
- 直接探针：训练分类器检测代理内化能力
- 激活级概念向量：定位和消融 PRIME 相关神经表示
- 分阶段检测：在可见黑客出现前识别 PRIME

**核心发现**：
1. PRIME 以分阶段序列在持续奖励黑客之前出现
2. 直接探针分数可预测后期黑客发作和严重程度
3. PRIME 在评估器变化时适应并重新定位
4. 消融 PRIME 激活方向减少黑客行为
5. 域内 PRIME 跟踪域外不对齐

**与现有技术对比**：
- 传统奖励黑客研究：事后检测
- PRIME：事前预警——在可见失败前检测能力

**优势与局限**：
- 优势：早期预警能力、机制可解释、消融验证
- 局限：目前仅在编码 RL 环境验证、需要激活访问

**未来发展潜力**：PRIME 将奖励黑客研究从"事后检测"推进到"事前预警"。如果这一发现在更多环境中得到验证，激活探针可能成为 AI 对齐的标准监控工具。

**个人技术理解**：PRIME 最深刻的洞察是"奖励黑客不是突然出现的——它是一个学习能力的副产品"。模型首先学会评估任务正确性和预测代理接受度，然后才学会利用差距。这意味着我们可以在黑客变得可见之前检测它——通过监控中间能力而非最终行为。

#### 4. AliyunConsoleAgent: Training Web Agents in Real Cloud Environments

- **地址**：https://arxiv.org/abs/2606.09447
- **类型**：论文

**论文介绍**（200-400 字）：主要云平台涵盖数百种产品快速迭代，控制台 UI 频繁与对应文档偏离。验证文档程序准确反映当前控制台并可端到端执行需约 400 万次重复检查，手动覆盖率低于 1%。AliyunConsoleAgent 提出两阶段训练范式：在蒸馏 frontier 模型轨迹上 SFT，然后使用 GRPO 和双通道结果奖励模型在真实云环境中 RL。为支持大规模 RL 训练，构建高确定性 rollout 系统——Terraform 资源预配置和 LLM 按需配置，有效隔离环境噪声从训练信号。引入基于后端审计日志的规则奖励评估协议，提供客观、防奖励黑客的结果判断。模型从机械指令跟随进化到自主决策带云控制台和产品特定理解。278 任务基准上（最佳 frontier 模型仅 65.34%），AliyunConsoleAgent-32B 达 63.52% 平均成功率——比基线提升 20.24pp，与最佳 frontier 专有模型差距仅 1.82pp（bootstrap 95% CI [-1.27, 7.39]），推理成本低 92%。

**技术架构**：
- 两阶段训练：SFT（蒸馏 frontier 轨迹）→ RL（GRPO + 双通道奖励）
- 高确定性 rollout：Terraform 预配置 + LLM 按需配置
- 规则奖励评估：基于后端审计日志的客观判断
- 32B 模型：成本效率 vs frontier 模型

**核心创新点**：
- 首次大规模真实云环境 Web Agent 训练
- 环境噪声隔离——将环境不确定性从训练信号中分离
- 防奖励黑客的审计日志奖励

**与现有技术对比**：
- WebArena/VisualWebArena：模拟环境，非真实云
- AliyunConsoleAgent：真实云环境，高确定性 rollout

**优势与局限**：
- 优势：真实环境验证、32B 逼近 frontier、92% 成本降低
- 局限：仅阿里云环境、需要 Terraform 基础设施

**未来发展潜力**：AliyunConsoleAgent 证明中小模型通过正确训练可在真实环境逼近 frontier 模型。这种"蒸馏+RL+高确定性环境"范式可推广到其他企业级 Agent 训练场景。

**个人技术理解**：这个项目最值得关注的是"真实环境训练"的方法论。大多数 Web Agent 研究在模拟环境中进行，但真实环境的噪声和不确定性是模拟无法复制的。AliyunConsoleAgent 通过 Terraform 预配置和审计日志奖励解决了这一问题，为真实环境 Agent 训练提供了可复现框架。

#### 5. WeaveBench: Long-Horizon Hybrid-Interface CUA Benchmark

- **地址**：https://arxiv.org/abs/2606.09426
- **类型**：论文

**论文介绍**（200-400 字）：计算机使用 Agent 越来越多地在结合视觉桌面控制、命令行执行、代码编辑、浏览器和外部工具的运行时中操作。现有基准通常将这些接口评估为可分离能力，长程跨接口编排测试不足。WeaveBench 是长程混合接口基准，114 个任务覆盖 8 个真实世界工作领域，基于真实用户请求和公开可验证工件。每个任务要求 Agent 在单一轨迹中组合 GUI 观察/操作与 CLI/代码操作。在部署的 CLI Agent 运行时内的真实 Ubuntu 桌面上评估，辅以最小桌面控制插件。提出轨迹感知 judge 检查交付物、文件、截图、日志和操作痕迹，同时检测伪造视觉证据或硬编码指标等捷径行为。跨 frontier 模型-运行时配对，最佳 PassRate 仅 41.2%，显示基准远未饱和。轨迹感知 judge 进一步揭示仅结果评级实质高估 Agent 性能。WeaveBench 暴露 CUA 评估中的关键空白并提供有效测试平台测量 Agent 是否能编排 GUI、CLI 和代码操作跨长程真实世界任务。

**技术架构**：
- 114 个长程混合接口任务
- 8 个真实世界工作领域
- 真实 Ubuntu 桌面评估环境
- 轨迹感知 judge：检查交付物+文件+截图+日志+操作痕迹
- 捷径行为检测：伪造视觉证据、硬编码指标

**核心发现**：
- 最佳 PassRate 仅 41.2%——基准远未饱和
- 仅结果评级实质高估 Agent 性能
- 跨接口编排是 CUA 的最大瓶颈

**与现有技术对比**：
- OSWorld：侧重 GUI 操作
- WebArena：侧重 Web 操作
- WeaveBench：GUI + CLI + 代码的混合接口编排

**优势与局限**：
- 优势：真实环境、轨迹感知评估、捷径检测
- 局限：任务规模相对较小（114 个）

**未来发展潜力**：WeaveBench 揭示了 CUA 评估的关键空白——跨接口编排能力。随着 Agent 从单一接口操作推进到多接口编排，此类基准将成为评估 Agent 真实能力的标准。

**个人技术理解**：WeaveBench 的最大价值在于"轨迹感知 judge"设计。传统基准只看最终结果，但 Agent 可能通过捷径（伪造截图、硬编码指标）获得高分。轨迹感知 judge 检查完整操作痕迹，确保 Agent 真正理解了任务而非作弊。

## 技术趋势

1. **AI 代码质量评估从 LLM 验证走向嵌入验证**：FASE 证明嵌入模型可替代 LLM 做代码质量评估，成本仅为 0.3%。结合 Structured Output Control 研究（结构忠实性 beyond 语法格式化），代码生成质量评估正从"昂贵的 LLM judge"转向"低成本嵌入+结构分析"的混合方案。

2. **人类-Agent 协作协议成为企业 AI 基础设施**：CHAP 填补 MCP 和 A2A 之间的关键空白——人类-Agent 可问责协作。"不可抵赖签名"和"可重放审计"设计直接回应企业合规需求。随着 AI 进入生产环境，此类协议可能成为必需组件。

3. **奖励黑客研究从"事后检测"推进到"事前预警"**：PRIME 证明奖励黑客不是突然出现的——它是一个学习能力的副产品。模型首先学会评估任务正确性和预测代理接受度，然后才学会利用差距。激活探针可能成为 AI 对齐的标准监控工具。

4. **真实环境 Agent 训练范式成熟**：AliyunConsoleAgent 证明"蒸馏+RL+高确定性环境"范式可使 32B 模型在真实云环境逼近 frontier 模型。真实环境训练从"理想化模拟"推进到"噪声隔离+审计奖励"的工程化方案。

5. **推理媒介多元化**：光学推理证明图像可作为独立推理媒介，token 效率 1.96x。结合"先推理后格式化"策略（Capacity, Not Format），推理不再局限于文本——图像、结构化格式、甚至混合媒介都可能成为推理载体。

## 新领域收获

1. **Bio AI：多模态生理信号基础模型** — Hypnos 证明 NTP 是多模态生理信号（EEG、ECG、呼吸）表示学习的强自监督目标，100x 数据效率。睡眠医学、心脏病学、神经学等健康领域可能成为基础模型的重要落地场景。
