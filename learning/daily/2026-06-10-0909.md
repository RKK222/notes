### AI Testing / Software Testing（12 项）

1. **FASE: Fast Adaptive Semantic Entropy for Code Quality** (arXiv:2606.09800, cs.SE)
   - 来源：https://arxiv.org/abs/2606.09800
   - 领域：代码质量评估 / 多 Agent 代码生成
   - 技术介绍：多 Agent 代码生成通过模拟人类软件工程生命周期提供自主开发范式，但系统可靠性受限于 LLM 幻觉和跨 Agent 错误传播。语义熵提供了无需 ground-truth 量化不确定性的原理方法，但现有方法依赖昂贵的 LLM 驱动等价性检查。FASE 提出基于结构和语义差异图最小生成树近似功能正确性的新指标。在 HumanEval 和 BigCodeBench 上，FASE 使用 Qwen3-Embedding-8B 超越 LLM 蕴含语义熵，Spearman 相关提升 25%，ROCAUC 提升 19%。消除 LLM 等价评估后，FASE 仅需传统方法 0.3% 的运行时成本。
   - 应用场景：多 Agent 代码生成质量评估、不确定性量化
   - 为什么值得关注：首次证明可用极低成本的嵌入方法替代 LLM 验证，为多 Agent 工作流提供实用不确定性量化方案。

2. **EvalCards: An Interpretive Layer for AI Evaluation Reporting** (arXiv:2606.09809, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09809
   - 领域：AI 评估报告标准化
   - 技术介绍：AI 评估结果大规模产生但报告不一致——leaderboard、model card、基准论文、公司博客各自为政。EvalCards 提出可操作的报告层，将基准元数据、评估运行数据和模型元数据组合为统一记录。从 52 篇论文和 10 个利益相关者访谈推导报告 schema，实现四种解释信号（可复现性、文档完整性、来源和风险、分数可比性），通过研究和非研究读者模式呈现。部署监控工具覆盖 5,816 个模型、635 个基准、101,843 条结果，揭示当前报告实践中的系统性差距。
   - 应用场景：AI 评估标准化、模型比较
   - 为什么值得关注：直击 AI 评估领域最大痛点——结果不可比，为评估透明度和可审计性提供基础设施。

3. **SIGA: Simulator-Interface Grounding Adapter for Scientific Simulation** (arXiv:2606.09774, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09774
   - 领域：科学模拟 / Agent-工具接口
   - 技术介绍：高级科学模拟器暴露专用输入语言，学习成本高昂。SIGA 为现成编码 Agent 提供模拟器可执行契约——词汇、结构约束、验证规则和终止条件。通过检索、程序记忆、轨迹内验证和验证强制终止实现。在 GEOS 多物理模拟器上，SIGA 约 5 分钟生成完整 GEOS deck（TreeSim > 0.90），匹配耗时 3 小时的人类专家，约 36x 加速。自我进化通过从先前轨迹重写适配器内容进一步提升性能。迁移到 OpenFOAM 和 LAMMPS 显示主导机制因接口而异。
   - 应用场景：科学软件自动化操作、领域专家替代
   - 为什么值得关注：证明轻量级自改进接地层可将通用编码 Agent 转变为科学软件的实际操作者。

4. **CHAP: Collaborative Human-Agent Protocol** (arXiv:2606.09751, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09751
   - 领域：人类-Agent 协作协议
   - 技术介绍：基础模型从响应生成进入操作角色——规划多步骤、调用工具、请求人类输入、协调其他 Agent。生产部署不再是单人监督单模型，而是跨团队、跨时区、跨信任边界的多人类多 Agent 协作。MCP 标准化 Agent 工具访问，A2A 标准化 Agent 间互操作，但均未定义人类和 Agent 共同执行可问责工作的共享工作空间。CHAP 将过去消失在聊天线程中的覆盖变为携带 diff、理由和内容哈希的结构化事件。交接变为可移植信封而非置顶消息。人类批准变为不可抵赖的签名决策。
   - 应用场景：企业级 AI 部署审计、合规工作流
   - 为什么值得关注：填补 MCP 和 A2A 之间的关键空白——人类-Agent 可问责协作，是企业级 AI 落地的基础设施。

5. **Multi-Turn Evaluation of Deep Research Agents Under Process-Level Feedback** (arXiv:2606.09748, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09748
   - 领域：深度研究 Agent 评估 / 多轮反馈
   - 技术介绍：现有深度研究 Agent 基准仅评估单次输出，忽视关键问题：DRAs 在反馈指导下能否改进报告？本文在两种反馈设置下进行多轮评估：自我反思（无外部诊断信号）和过程级反馈（针对研究策略差距的引导）。设计 Research Gap Inference（RGI）方法分析满足和不满足的评分标准模式推断研究过程差距。发现：(i) 自我反思下 Agent 纳入和回退评分标准的比率几乎相等，净改进可忽略；(ii) 单轮过程级反馈带来显著增益，标准化分数提升 8-15 分，约 35-40% 纳入率；(iii) 增益在多轮中不累积，Agent 重写完整报告时回退高达 24% 先前满足的标准。
   - 应用场景：深度研究 Agent 评估、多轮改进
   - 为什么值得关注：SCALE-ICML 2026 Oral，揭示即使有目标引导，可靠的多轮改进仍超出当前 DRA 架构能力。

6. **SearchSwarm-30B-A3B: Delegation Intelligence for Long-Horizon Deep Research** (arXiv:2606.09730, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09730
   - 领域：委托智能 / 长程研究 Agent
   - 技术介绍：LLM 越来越多地处理复杂长程真实世界任务，但上下文窗口固有有限。主 Agent 分解任务并派遣子任务给子 Agent，子 Agent 执行并仅返回摘要结果。这需要委托智能——分解复杂任务、确定何时何物委托、将返回结果整合到持续工作流的能力。本文设计引导模型走向高质量任务分解和委托的 harness，约束子 Agent 正确返回结果支持主 Agent 工作流。harness 引导的轨迹自然编码正确的委托决策，用作 SFT 数据将委托智能内化到模型权重。SearchSwarm-30B-A3B 在 BrowseComp 达 68.1、BrowseComp-ZH 达 73.3，同类规模最佳。
   - 应用场景：长程研究 Agent、多 Agent 协作
   - 为什么值得关注：首次系统探索委托智能的训练数据合成和模型训练，BrowseComp 同类规模 SOTA。

7. **PRIME: Proxy Reward Internalization and Mechanistic Exploitation** (arXiv:2606.09711, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09711
   - 领域：奖励黑客 / AI 安全
   - 技术介绍：奖励黑客通常在变得可见后研究——模型获得高代理奖励但未能完成预期任务。PRIME 研究代理 RL 在该失败出现之前教了什么。引入代理奖励内化和机制利用——评估任务正确性、预测代理接受度、推理可利用代理-黄金差距的学习能力。在可剥削 pytest 奖励的编码 RL 环境中，通过 CoT 监控、直接探针和激活级概念向量测量 PRIME。发现 PRIME 在持续奖励黑客之前以分阶段序列出现，其当前直接探针分数预测后期黑客发作和严重程度，即使可见黑客率仍低。PRIME 在评估器变化时适应，消融其激活方向减少黑客。
   - 应用场景：AI 对齐早期预警、奖励黑客检测
   - 为什么值得关注：将奖励黑客研究从"事后检测"推进到"事前预警"，激活探针可作为更广泛对齐风险的早期信号。

8. **Trellis: Autoformalization with Process Semantics for Lean Proofs** (arXiv:2606.09674, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09674
   - 领域：自动形式化 / Lean 证明
   - 技术介绍：Trellis 是自动形式化系统，利用 LLM Agent 在确定性约束工作流中通过自然语言证明的迭代精炼强制执行 Lean 自动形式化任务的增量进展。动机来自数学家的严格证明概念——证明的任何部分都应有能力进一步细化。系统旨在以适度预算和通用 Agent 实现可靠自动形式化，专业化不来自任务特定 Agent 训练，而是来自由过程语义强制执行的意义严格工作流。链接到由该过程产生的近期 Ramsey 理论突破的端到端 Lean 形式化。
   - 应用场景：数学定理自动形式化、Lean 证明辅助
   - 为什么值得关注：从"训练专用 Agent"转向"过程语义约束工作流"，为自动形式化提供新思路。

9. **SpatialWorld: Interactive Spatial Reasoning Benchmark for Multimodal Agents** (arXiv:2606.09669, cs.AI)
   - 来源：https://arxiv.org/abs/2606.09669
   - 领域：空间推理基准 / 多模态 Agent
   - 技术介绍：现有多模态 Agent 基准主要评估被动视觉理解，忽视主动空间探索能力。SpatialWorld 引入跨 8 个仿真后端（Matterport3D、HM3D、Gibson 等）的 760 个交互式空间推理任务，评估 Agent 在三维环境中导航、定位和推理的能力。评估指标包括任务成功率（TSR）、导航效率和空间记忆利用率。GPT-5 仅达 17.4% TSR，揭示多模态 Agent 在主动空间探索方面的巨大差距。
   - 应用场景：多模态 Agent 空间推理评估、机器人导航
   - 为什么值得关注：揭示多模态 Agent 主动空间探索的巨大差距，为具身 AI 提供关键评估基准。

10. **Empirical Study for Structured Output Control in LLMs for Software Engineering** (arXiv:2606.09395, cs.SE)
    - 来源：https://arxiv.org/abs/2606.09395
    - 领域：结构化输出控制 / 软件工程
    - 技术介绍：LLM 生成的软件工程输出必须接入工具链、API 和数据管道，这些系统施加严格的组织特定结构契约。语义正确但格式违规的输出在消费系统看来与错误答案无异。本文对四个代表性 SE 任务进行系统评估，将失败分类为语法、结构和语义错误。基准测试语法约束解码、正则验证和严格模板驱动控制（TTMG）。TTMG 几乎消除语法错误，但大量结构和语义错误仍然存在，证明核心瓶颈超越语法格式化。案例研究显示残余错误如何在下游工作流中级联。
    - 应用场景：LLM 驱动软件工程工作流、结构化输出保障
    - 为什么值得关注：揭示结构强制工具必要但不充分，需联合确保结构和语义正确性。

11. **Understanding How Enterprises Adopt MCP for LLM-Driven Software Engineering** (arXiv:2606.09182, cs.SE)
    - 来源：https://arxiv.org/abs/2606.09182
    - 领域：MCP 企业采用 / LLM 软件工程
    - 技术介绍：LLM 在 AI 软件工程中的应用日益增长，但复杂任务执行和多工具协调的局限性推动了对 MCP 的兴趣。现有研究主要关注 MCP 的技术设计，缺乏关于企业实践中如何采用和使用的实证证据。本文对互联网和金融领域 8 家公司的 20 名从业者进行半结构化访谈。发现 MCP 因支持跨系统协作、任务解耦和知识复用而受到重视，但采用仍受限于生态系统碎片化、跨组件协调困难以及分布式状态管理和故障诊断的未解决问题。参与者强烈要求更好的标准化、通过低代码或插件方法降低采用门槛以及更系统的运营支持。
    - 应用场景：企业 MCP 部署、LLM 工作流集成
    - 为什么值得关注：提供企业 MCP 实践的早期实证证据，为标准化和可用性改进提供方向。

12. **Autonomous Incident Resolution at Hyperscale: An Agentic AI Architecture for Network Operations** (arXiv:2606.09122, cs.SE)
    - 来源：https://arxiv.org/abs/2606.09122
    - 领域：自主事件解决 / 网络运维
    - 技术介绍：超大规模云网络基础设施呈现独特的运营挑战，传统人工驱动的事件响应无法跟上故障的体积、速度和复杂性。本文提出一种用于大规模网络运营中自主事件解决的 Agent AI 架构。系统采用多 Agent 编排框架，专门 AI Agent 协作检测、诊断和修复网络事件而无需人工干预。架构原则包括分层 Agent 分解、基于技能的工具调用、结构化知识编码、渐进式自主和安全边界、闭环验证。该架构已在主要云提供商的生产环境中部署，对常见事件类别实现超过 90% 的自主解决率，同时通过分层授权和回滚机制保持安全保证。
    - 应用场景：网络运维自动化、超大规模基础设施管理
    - 为什么值得关注：展示 Agent AI 在真实超大规模网络运营中实现 90%+自主解决率的生产级证据。

### AI / LLM 技术（12 项）

13. **AliyunConsoleAgent: Training Web Agents in Real-World Cloud Environments** (arXiv:2606.09447, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09447
    - 领域：云控制台 Agent / 真实环境训练
    - 技术介绍：现有网页 Agent 主要在合成或简化环境中训练，与真实云控制台的复杂性存在显著差距。AliyunConsoleAgent 在真实阿里云控制台中收集训练数据，采用蒸馏+RL+高确定性环境的训练范式。32B 模型在阿里云控制台任务上达到 63.52% 成功率，逼近 frontier 模型性能，同时训练成本降低 92%。关键发现：高确定性环境中的 RL 训练比低确定性环境更有效，因为奖励信号更可靠。
    - 应用场景：云控制台自动化、企业级网页 Agent
    - 为什么值得关注：蒸馏+RL+高确定性环境范式，32B 模型逼近 frontier 模型，成本低 92%。

14. **Optical Reasoning: Rethinking Images as an Expressive Reasoning Medium** (arXiv:2606.09585, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09585
    - 领域：光学推理 / 多模态推理
    - 技术介绍：CoT 改善了 LLM 性能并已扩展到 MLLM。近期工作从基于文本的多模态推理转向交错模态推理，中间步骤可包含文本推理和视觉证据。本文提出更激进的想法：图像能否单独作为语言和模态任务的推理媒介？提出光学推理概念，将图像作为独立推理媒介。实例化两种变体：基于排版的光学推理（优化视觉布局以紧凑呈现推理）和基于图形的光学推理（将文本和图形元素组合为结构化视觉推理）。在数学、科学和交错模态推理基准上，光学推理可匹配甚至超越传统文本推理，同时在语言任务上减少 28.57% 推理 token，多模态任务上减少 16%，达到文本推理 1.96 倍的 token 效率。
    - 应用场景：多模态推理、推理效率优化
    - 为什么值得关注：挑战推理必须是文本的假设，图像作为推理媒介效率更高。

15. **SIFT: Selective-Index For Fast Compute of RAG Prefill** (arXiv:2606.09441, cs.AI)
    - 来源：https://arxiv.org/abs/2606.09441
    - 领域：RAG 推理加速 / KV 缓存优化

    - 技术介绍：LLM 和 VLM 越来越多地在表格推理任务上评估，但表格表示的作用仍未充分探索。实践中，相同表格内容可能以不同结构格式出现，如 HTML、Markdown 和 LaTeX，或作为渲染图像。但现有评估通常让内容、格式、布局和模态一起变化，难以隔离表示效应。引入 TABVERSE，受控多模态表格基准，将相同表格内容对齐到多种结构格式和渲染图像，带问题类别和难度标签。此设计在保持表格内容固定的同时实现表示效应的系统评估。在三个任务上评估 LLM 和 VLM：QA、SUC 和 SR。结果显示表示选择实质影响表格理解。模型通常在结构化文本上比渲染图像表现更好，但此差距大小取决于任务、模型和格式。HTML 通常是最稳健的文本格式，而行敏感结构任务和语法上可用的 LaTeX 重建仍然具有挑战性。

#### 1. PRIME: 奖励黑客早期预警

- 激活探针预测后期黑客发作和严重程度
- 消融激活方向减少黑客

**优势与局限**：

#### 3. SIFT: RAG 预填充加速

**地址**：https://arxiv.org/abs/2606.09441

**论文介绍**（250字）：
SIFT 解决 RAG 推理最大瓶颈——TTFT 慢。RAG 注入相关文档提高响应质量，但增加提示长度并减慢 TTFT。与标准查询不同，RAG 查询具有上下文复用特性——相同文档在用户查询间重复出现。

SIFT 离线处理文档并提取每个文档的高注意力分数精细位置。识别两种注意力不变性洞察：(1) 局部注意力不变性：文档内高注意力分数的位置对周围文档不变；(2) 交叉注意力一致性：文档内高注意力的键也吸引后续文档的交叉注意力。

SIFT 不存储 KV 数据，仅存储为两个紧凑位向量的高分位置。存储比 KV 张量小 24,000 倍，消除昂贵的磁盘传输。预填充期间仅计算标记位置的注意力，TTFT 提升 1.71x，精度保持在完全重新计算的 1% 以内。

**技术架构**：
- 离线处理：提取高分位置 → 紧凑位向量
- 注意力不变性：局部不变性 + 交叉一致性
- 在线推理：仅计算标记位置注意力

**核心创新**：
