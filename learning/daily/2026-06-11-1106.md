1. **A Five-Plane Reference Architecture for Runtime Governance of Production AI Agents** (arXiv:2606.12320, cs.AI)
   - 来源：https://arxiv.org/abs/2606.12320
   - 领域：Agent 运行时治理 / 企业安全
   - 技术介绍：企业安全建立在数据边界治理上，但生产 AI Agent 溶解了这一假设——Agent 读取上下文、调用工具、修改企业记录系统，风险转移到工作流内部。现有策略引擎评估原子主体的请求时决策，而 Agent 系统需要针对复合主体的有状态评估。本文提出五平面分解架构：推理平面裁决意图，四个执行平面（网络、身份、端点、数据）实现决策。定义六种中断原语泛化 allow/deny，证明四个正确性不变量，演示七个生产 Agent 威胁的阻断。参考实现显示：衰减正确性和证据可重构性在每个试验中成立，裁决在个位数微秒内运行。
   - 应用场景：企业 AI Agent 安全治理、生产部署合规
   - 为什么值得关注：首次为生产 Agent 提供可组合的运行时治理架构，从"数据边界安全"推进到"工作流内安全"。

2. **The Impossibility of Eliciting Latent Knowledge** (arXiv:2606.12268, cs.AI)
   - 来源：https://arxiv.org/abs/2606.12268
   - 领域：AI 安全 / 诚实性 / 理论
   - 技术介绍：ELK（Eliciting Latent Knowledge）问题是训练 AI Agent 诚实报告其对隐藏变量的信念。本文使用因果影响图（CIDs）使 ELK 形式化精确，区分可观察变量和潜变量，定义 Agent 诚实的精确含义，形式化目标泛化错误。证明一个不可能性定理：没有仅依赖 Agent 行为的反馈训练策略能以确定性产生诚实 Agent，即使训练期间反馈完美。开发者可以通过训练期间提供正确反馈来激励 Agent 诚实回答问题，但 Agent 的自然泛化方式是提供人类评估为"真"的答案，而非"诚实"答案。
   - 应用场景：AI 对齐理论、诚实性训练、潜知识 elicitation
   - 为什么值得关注：对 AI 安全有基础性影响——证明基于行为的反馈训练无法保证诚实性，需要新的训练范式。

3. **Rule Taxonomy and Evolution in AI IDEs: A Mining and Survey Study** (arXiv:2606.12231, cs.SE)
   - 来源：https://arxiv.org/abs/2606.12231
   - 领域：AI IDE 规则 / 软件工程实证
   - 技术介绍：AI IDE 引入"Rules"作为新型软件工件，允许开发者将项目特定约束持久注入 LLM 上下文。对 83 个开源项目挖掘 7,310 条规则，建立 5 主类 25 子类的完整分类法。与 99 名从业者调查交叉验证。发现：开发者认为架构约束最重要，但仓库中规则主要是低级工作流和代码格式约束。1,540 次规则进化事件分析显示：规则更新主要由建设性上下文扩展（29.17%）和丰富化（26.59%）驱动。合规评估显示：更新规则后软件工件合规率从 49.14% 提升到 72.13%（+22.99%）。
   - 应用场景：AI IDE 规则设计、Prompt 工程优化、自动化冲突检测
   - 为什么值得关注：首次系统研究 AI IDE 规则的分类、进化和实际影响，为 AI 辅助开发工具设计提供实证基础。

4. **Measuring How Agent Skill Organization Changes Runtime Behavior** (arXiv:2606.11543, cs.AI/cs.SE)
   - 来源：https://arxiv.org/abs/2606.11543
   - 领域：Agent 技能组织 / Progressive Disclosure
   - 技术介绍：Agent Skills 在推理时增强 LLM Agent 的程序知识，但当前基准很少区分 Skill 说什么和如何组织。通过 Progressive Disclosure（简洁根文件按需指向支持资源）与扁平基线对比，在 SkillsBench 82 任务上评估。结果：Progressive Disclosure 在聚合结果之前就改变了运行时行为——每次轨迹中不同 Skill 资源从 1.18 提升到 3.85，有效摄取事件从 1.33 提升到 3.92。在 410 次匹配试验中获得 17 次额外的验证通过试验（+4.1%）。收益因任务而异：当支持资源指导实现、检查或修复时有效，但在依赖精确输出约定、数值阈值或长工件生成管线时较弱。
   - 应用场景：Agent Skill 设计、技能文件组织结构优化
   - 为什么值得关注：首次证明技能组织不只是展示——它改变 Agent 搜索和应用程序知识的方式。

5. **Search Discipline for Long-Horizon Research Agents** (arXiv:2606.11522, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11522
   - 领域：Research Agent / 评估可靠性
   - 技术介绍：自动研究 Agent 在异质空间（区域、切片、队列）上聚合指标评估科学候选，但当科学有效性存在于 disaggregated 结构中时，聚合可能将错误候选排第一—— headline number 改进但底层结构反转。在生态系统模型火情任务中演示：最高分候选和略低的候选在全局分数上噪声范围内，但最高分候选使受保护的北方森林区域崩溃而另一个保持。分离它们的是每个区域的行为而非 headline number。将决策转移到外部控制回路，在 Agent 决定后审计每个候选的 disaggregated 行为。
   - 应用场景：自动研究 Agent 评估、科学发现可靠性
   - 为什么值得关注：揭示聚合指标在科学评估中的系统性风险，需要 disaggregated 审计。

6. **When Do Data-Driven Systems Exhibit the Capability to Infer?** (arXiv:2606.11769, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11769
   - 领域：AI 法规 / 推理能力分级
   - 技术介绍：欧盟 AI Act 是首个全面的 AI 法规，关键区分特征是"推理能力"。但 AI Act 未明确定义推理是什么，导致某些数据驱动系统处于灰色区域（如信用评分系统）。基于统计学习理论开发推理能力分级框架，分析哪些级别构成 AI Act 意义上的充分推理能力。创建两个真实信用评分工作流示例，展示推理何时发生。分析表明：不仅单个模型，整个数据处理工作流都必须考虑；开发中人类专家的参与对推理能力有显著影响。
   - 应用场景：AI 合规评估、EU AI Act 实施
   - 为什么值得关注：为 AI Act 中"推理能力"提供可操作框架，影响信用评分等系统的监管分类。

7. **Multi-Target Adversarial Attacks and Robust Defenses for Continuous Data Summarization** (arXiv:2606.11804, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11804
   - 领域：数据摘要对抗攻击 / 可信 AI
   - 技术介绍：数据摘要是上游组件，决定保留哪些信息并传递给后续学习或决策模块。对摘要过程的对抗扰动可以上游方式损害可信 AI。研究通过 DR-submodular 优化在相似性级别扰动下对连续数据摘要的对抗攻击。将多目标攻击生成为 min-max 问题，一个可允许扰动优化以降级多个目标摘要模型。为缓解此类扰动，将针对混合攻击类型的鲁棒防御公式化为正则化 max-min 问题。开发具有理论保证的近似算法。实验显示攻击在代表性低到中等预算范围内有效，可导致下游任务性能损失。
   - 应用场景：可信 AI 管线、数据摘要安全
   - 为什么值得关注：揭示数据摘要作为上游组件的安全脆弱性，影响整个下游学习管线。

8. **A Local-First, Event-Sourced Memory and Judgment Layer for AI Coding Agents** (arXiv:2606.12329, cs.AI)
   - 来源：https://arxiv.org/abs/2606.12329 / https://github.com/riponcm/projectmem
   - 领域：编码 Agent 记忆 / Memory-as-Governance
   - 技术介绍：AI 编码助手仍然是无状态的——每个新会话重新读取项目文件、重新推导先前决策、可能重复已经失败的调试尝试。重建上下文估计消耗 5,000-20,000 tokens/会话。projectmem 是开源本地优先记忆和判断层，将开发记录为追加纯文本事件日志（问题、尝试、修复、决策、注释），确定性地投影为紧凑 AI 可读摘要通过 MCP 服务。添加确定性预行动门控——在 Agent 重复先前失败修复或编辑已知脆弱文件之前警告。框架为 Memory-as-Governance：记忆不仅回答 Agent 而且作用于其下一个行动。完全离线运行无遥测；不可变日志也作为可重现可审计 AI 辅助开发的来源追踪。通过 10 个项目 207 个日志事件的两个月自研究评估。
   - 应用场景：AI 编码助手记忆管理、可审计开发
   - 为什么值得关注：Memory-as-Governance 新范式——记忆不只是存储，而是主动治理 Agent 行为。

9. **Forecasting Future Behavior as a Learning Task** (arXiv:2606.11445, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11445
   - 领域：LRM 行为预测 / 可解释性替代
   - 技术介绍：对 AI 系统的信任通常锚定在其工作原理的解释上。对大型推理模型（LRMs），单 token 生成的解释方法不自然推广到长轨迹，轨迹本身作为自然语言阅读通常不忠实。提出替代方案：跳过解释步骤，将行为预测作为可学习任务，训练行为预测器在单个推理轨迹上操作以做出通常从解释中寻求的相同预测。预测器的训练数据通过查询 LRM 获得无需人类注释，推理在单个前向传递中完成。在两个任务上实例化：LRM 在重新运行时重复其答案的可能性，以及移除输入部分如何改变其答案。在三个不同推理数据集上评估，发现训练的行为预测器比 GPT-5.4 和 Claude Opus-4.6 作为朴素读者阅读相同轨迹更准确，仅为其推理成本的一小部分。
   - 应用场景：LRM 可预测性评估、行为一致性验证
   - 为什么值得关注：绕过解释步骤直接预测行为，为 LRM 可靠性评估提供新路径。

10. **INFRAMIND: Infrastructure-Aware Multi-Agent Orchestration** (arXiv:2606.11440, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11440
    - 领域：多 Agent 编排 / 基础设施感知
    - 技术介绍：现有多 Agent LLM 编排方法（从暴力集成到学习路由器）基于任务和模型特征选择模型和拓扑，但不考虑服务基础设施的运行时状态。在并发负载下的共享 GPU 集群上，这种基础设施盲导致系统性资源未充分利用：首选模型积累深层请求队列而同样能力的替代品闲置。在多 Agent 管线中，每个查询触发多个顺序模型调用，这些延迟在每个下游步骤中复合。INFRAMIND 使整个多 Agent 栈基础设施感知：基础设施感知规划器根据实时系统负载和剩余预算调节拓扑和角色选择；基础设施感知执行器在每个 Agent 步骤观察每模型队列深度、缓存利用率和响应延迟；预算感知调度器重排每模型队列使紧急请求优先服务。作为分层约束 MDP 并通过 RL 端到端求解。在五个基准上，低负载下比先前基线高 +7.6pp 准确率且延迟低 7 倍，高负载下维持 99.9% SLO 合规（所有基线降至 50% 以下）。
    - 应用场景：多 Agent 系统部署、GPU 集群资源优化
    - 为什么值得关注：首次将基础设施感知引入多 Agent 编排，解决共享 GPU 集群上的系统性资源浪费。

### AI / LLM 技术（12 项）

11. **Architecture-Aware Reinforcement Learning Makes Sliding-Window Attention Competitive in Math Reasoning** (arXiv:2606.11634, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11634
    - 领域：推理效率 / 注意力架构
    - 技术介绍：SWARR（Sliding-Window Attention with Reinforced Adaptation for Math Reasoning）将预训练 SA 模型转换为 SWA 用于数学推理。两阶段：(1) SFT 高效转换 SA→SWA 避免预训练新基础模型；(2) RL 策略适应。发现 SWA 在 SFT 后仍然落后 SA，假设差距部分由数据-架构不匹配引起：大多数 SFT 数据为 SA 模型准备，可能包含 SWA 难以建模的长程依赖。因为 on-policy RL 在 SWA 约束下优化自生成轨迹，可以使轨迹更好地匹配 SWA。实验显示此配方大幅缩小 SWA 和 SA 之间的差距，恢复 SWA 转换中损失的许多准确性同时保持线性复杂度注意力的效率优势。核心贡献：RL 改变仅从转换和 SFT 得出的关于 SWA 在数学推理中可行性的结论。
    - 应用场景：长上下文推理效率优化、SWA 模型适应
    - 为什么值得关注：RL 使 SWA 在数学推理中变得可行，为长上下文推理提供效率-准确性平衡新方案。

12. **Hindsight-Enhanced Reflection from Environment Observations for Agentic Self-Distillation** (arXiv:2606.11559, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11559
    - 领域：Agent 自蒸馏 / 强化学习
    - 技术介绍：RL 通常通过轨迹的最终结果改进多轮 Agent 能力，难以确定每个中间轮的信用分配。最近的 on-policy 自蒸馏方法通过自教师将特权反馈转换为密集 token 级监督提供有希望的替代。但朴素扩展此范式到多轮设置时观察到意外性能退化，归因于特权反馈（如成功轨迹或最终结果）与学生当前决策上下文之间缺乏对齐。HERO 使用下一个环境观察作为局部对齐反馈。每次 rollouts 后，HERO 反思完成的交互将每个观察转换为紧凑轮级诊断，捕获关于原始动作的可操作反馈（如必要性、有效性或失败原因）。在 TauBench 和 WebShop 上，HERO 改进任务成功率并减少不必要轮次。在有限训练轮预算下特别有效，其中成功 rollouts 罕见且 GRPO 提供弱奖励对比信号。
    - 应用场景：多轮 Agent 训练、自蒸馏优化
    - 为什么值得关注：解决自蒸馏在多轮设置中的性能退化问题，用环境观察作为局部对齐反馈。

13. **A Market-of-Claims Code Agent for Financial and Numerical Reasoning** (arXiv:2606.11537, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11537 / https://github.com/UBC-NLP/MoCA-Agent
    - 领域：金融推理 / 多 Agent 证据聚合
    - 技术介绍：金融和表格问答需要比流畅推理更多——答案必须基于支持它们的确切事实、公式、单位、符号和比例。单个误读单元格或错误操作可能静默产生合理但错误的结果。MOCA-Agent 用声明级验证替代自由形式多 Agent 辩论。系统将每个问题分解为类型化原子声明，让专业交易 Agent 买入或卖出这些声明，清除其订单为置信度加权的接受/拒绝决策，从市场支持证据合成可执行 Python 程序。代码感知验证器检查程序的执行、结构一致性和常见金融推理错误，最多一轮市场感知修复。在十个公共基准上实现强性能：FinQA 78.3%、FinanceMath 76.0%、MultiHiertt 71.2%、ESGenius 86.9%、FinChart-Bench 平均 85.6%。
    - 应用场景：金融数值推理、表格推理、ESG 问答
    - 为什么值得关注：在原子声明级别（而非整个答案）聚合证据提高高 stakes 数值推理的鲁棒性。

14. **Let's Reason Recursively for Theory of Mind** (arXiv:2606.11724, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11724
    - 领域：心智理论 / 嵌套信念推理
