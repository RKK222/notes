   - 为什么值得关注：揭示科学结论合成仍是开放挑战，清洁室评估对评估开放域 AI Agent 至关重要。

5. **Self-Gated Clarification for Hierarchical Language Agents** (arXiv:2606.11349, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11349
   - 领域：Agent 澄清机制 / 分层推理
   - 技术介绍：在分层推理中，失败通常源于中间决策点——Agent 在缺乏关键信息时提交了错误分支。提出 ACTION-RATING，将澄清请求放置在 Agent 动作空间内，与导航共享序数尺度，使帮助寻求在中间态可观测。从 Agent 自身评级中涌现出两种信息寻求模式：强制性（无可行分支）和机会性（领先候选仍有残余不确定性）。在 Harmonized Tariff Schedule 分类（30,000 节点分类法）上，信息寻求有效性（ISE）从 50% 提升到 74%。
   - 应用场景：分层 Agent 决策质量提升、知识分类任务
   - 为什么值得关注：首次将帮助寻求建模为 Agent 动作空间的内在组成部分，而非外部不确定性触发器。

   - 应用场景：商业谈判准备、冲突解决、AI 辅助决策
   - 为什么值得关注：证明结构化 LLM 管线可提供可扩展的低成本预调解支持。

7. **Forecasting Future Behavior as a Learning Task** (arXiv:2606.11445, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11445
   - 领域：LLM 行为预测 / 可解释性
   - 技术介绍：绕过解释步骤，将行为预测作为可学习任务。训练行为预测器在单个推理轨迹上做出通常需要解释才能获得的预测。训练数据通过查询 LLM 获得，无需人工标注，推理在单个前向传递中完成。两个任务实例化：LLM 在重新运行时重复答案的可能性，以及移除输入部分如何改变答案。在三个多样化推理数据集上评估，训练的行为预测器比 GPT-5.4 和 Claude Opus-4.6 作为朴素读者更准确，推理成本仅为一小部分。
   - 应用场景：LLM 可靠性评估、推理轨迹分析
   - 为什么值得关注：证明推理轨迹携带关于 LLM 未来行为的信息，超越朴素阅读所能传达的内容。

8. **Multi-Target Adversarial Attacks on Data Summarization** (arXiv:2606.11804, cs.AI)
   - 来源：https://arxiv.org/abs/2606.11804
   - 领域：对抗攻击 / 数据摘要安全
   - 技术介绍：数据摘要是上游组件，决定保留哪些信息并传递给后续学习或决策模块。研究在 DR-子模优化下对连续数据摘要的对抗攻击。证明一类多分辨率图像摘要目标可公式化为非负子模集合函数的多线性扩展，满足 DR-子模性和 m-弱单调性。将多目标攻击生成公式化为 min-max 问题，一个容许的相似性结构扰动优化以降级多个目标摘要模型。提出正则化 max-min 鲁棒防御。在真实数据和受控聚类基准上实验显示攻击在低到中等预算下有效，可导致下游任务性能损失。
   - 应用场景：AI 管线上游安全、数据摘要鲁棒性
   - 为什么值得关注：将对抗攻击研究从下游预测模型推进到上游数据摘要，揭示可信 AI 的供应链风险。

### AI / LLM 技术（13 项）

10. **HORMA - 分层记忆导航** (arXiv:2606.11680, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11680
    - 领域：Agent 记忆 / 分层组织
    - 技术介绍：现有方法依赖有损压缩或相似性检索，无法捕获时间结构和因果依赖。HORMA 将经验组织为文件系统式分层结构，摘要实体链接到原始轨迹，实现高效访问而不丢失详细信息。分解为两阶段：结构化记忆构建（区分信息缺失 vs 误导性/过载上下文导致的失败）和导航式检索（RL 训练的轻量 Agent 选择最小但充分的上下文）。在 ALFWorld、LoCoMo 和 LongMemEval 上，长对话任务中最多仅需基线 22.17% 的 token 使用量。
    - 应用场景：长周期 Agent 任务、上下文预算管理
    - 为什么值得关注：记忆系统从"被动存储"向"主动组织+RL 导航"演进的新范式。

11. **TreeSeeker - 树状深度搜索** (arXiv:2606.11662, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11662
    - 领域：深度搜索 / 试错控制
    - 技术介绍：深度搜索需要 Agent 通过多步网络搜索、浏览、证据比较和综合回答复杂问题。TreeSeeker 将搜索组织为树状状态上的分支-返回搜索，每个分支是子目标的暂定方向。每轮读取所有子目标树，识别活跃目标，使用价值、不确定性和风险的文本 UCB 信号选择利用有希望的分支、探索不确定的替代方案或修剪无益的延续并返回早期分支点。TreeMem 将证据、不确定性、冲突、进展和失败线索附加到产生它们的分支上。在 XBench-DeepSearch、BrowseComp 和 BrowseComp-ZH 上一致超越强开源基线。
    - 应用场景：深度研究 Agent、复杂问题求解
    - 为什么值得关注：显式分支-返回控制补充强推理和工具执行。

12. **MOCA-Agent - 声明市场金融推理** (arXiv:2606.11537, cs.AI)
    - 来源：https://arxiv.org/abs/2606.11537
    - 领域：金融推理 / 多 Agent 协作
    - 技术介绍：用声明级验证替代自由形式多 Agent 辩论。将每个问题分解为类型化原子声明，让专业交易 Agent 买入或卖出这些声明，将订单清算为置信度加权的接受/拒绝决策，从市场支持的证据中综合可执行 Python 程序。代码感知验证器检查执行、结构一致性和常见金融推理错误，最多一轮市场感知修复。十个公开基准上：FinQA 78.3%、FinanceMath 76.0%、MultiHiertt 71.2%、ESGenius 86.9%、FinChart-Bench 平均 85.6%。
    - 应用场景：金融数值推理、表格推理、ESG 问答
    - 为什么值得关注：在原子声明级别（而非整个答案）聚合证据，提高高风险数值推理的鲁棒性。

13. **INFRAMIND - 基础设施感知多 Agent 编排** (arXiv:2606.11440, cs.AI)

- Feedback Agent：性能分析 + 改进建议
- Orchestrator：自动评分 + 反馈注入

**核心机制**：
- 自改进循环：每代自动评分并将结果注入下一代反馈提示
- Harness + Weight 双更新：同时改进代码和模型权重
- 多 Provider 支持：灵活切换底层模型

**优势与局限**：
- 优势：真正的自改进闭环，在多个科学任务上显著超越基线

#### 1. SciConBench - 科学结论合成评估

**地址**：https://arxiv.org/abs/2606.11337

**论文介绍（~300 字）**：
科学 AI Agent 日益检索证据、跨源推理并综合用于重大决策的结论。然而，在健康等高 stakes 领域中，其能力仍不清楚。本文引入 SciConBench，一个 9,110 个问题和来自系统评价的专家撰写结论的大规模活体基准，评估开放域科学结论合成能力。

核心创新在于专家验证的自动化评估管线，将结论分解为原子事实，通过事实精确率和召回率衡量正确性和全面性。为缓解数据泄露，引入 SciConHarness 清洁室评估框架，配备 Agent 受控网络交互确保有效测量。

评估 8 个前沿模型和深度研究 Agent 的结果令人担忧：在清洁室设置下，最佳 Agent 仅达到事实 F1 0.337。清洁室设置持续降低性能相对无约束评估，表明数据泄露膨胀了模型真实合成能力的估计。审计消费者面向 Agent（Google AI Overview、OpenEvidence）发现它们频繁生成不完整且有时矛盾的结论，即使地面真实答案可用。

**技术架构**：
- 基准层：9,110 个问题 + 专家撰写结论
- 评估层：原子事实分解 + 事实精确率/召回率
- 安全层：SciConHarness 清洁室框架
- 审计层：消费者面向 Agent 审计

**核心发现**：
- 事实质量仍然很低（最佳 F1 仅 0.337）
- 数据泄露显著膨胀评估结果
- 消费者面向 Agent 生成不完整/矛盾结论

**优势与局限**：
- 优势：大规模、活体基准、清洁室评估、专家验证
- 局限：仅覆盖健康领域，自动化评估可能遗漏 nuanced 错误

**未来发展**：
揭示可靠科学结论合成仍是开放挑战，清洁室评估对评估开放域 AI Agent 至关重要。

#### 2. INFRAMIND - 基础设施感知多 Agent 编排

**地址**：https://arxiv.org/abs/2606.11440

**论文介绍（~300 字）**：
现有多 Agent LLM 编排方法（从暴力集成到学习路由）基于任务和模型特征选择模型和拓扑。然而，这些方法不考虑服务基础设施的运行时状态。在共享 GPU 集群的并发负载下，这种基础设施盲区导致系统性资源利用不足：首选模型积累深度请求队列，而同样能力的替代方案闲置。在多 Agent 管线中，每个查询触发多个顺序模型调用，这些延迟在每个下游步骤中复合。

INFRAMIND 使整个多 Agent 栈基础设施感知。基础设施感知规划器根据实时系统负载和剩余预算条件化拓扑和角色选择——拥塞时偏向简单图，低负载时偏向丰富图。基础设施感知执行器在每个 Agent 步骤观察每模型队列深度、缓存利用率和响应延迟，决定调用哪个模型和推理多深。预算感知调度器重排序每模型的队列，优先服务紧急请求。

作为层次约束 MDP 并通过 RL 端到端求解。五个基准上，INFRAMIND 在低负载下比先前基线高 +7.6pp 准确率且延迟低 7x，高负载下维持 99.9% SLO 合规（所有基线降至 50% 以下）。

**技术架构**：
- 规划层：基础设施感知拓扑和角色选择
- 执行层：队列深度/缓存/延迟感知的每步路由
- 调度层：预算感知的队列重排序
- 学习层：层次约束 MDP + RL

**核心机制**：
- 实时基础设施信号（队列深度、KV 缓存压力、延迟）纳入编排决策
- 质量-延迟自动权衡学习
- 层次决策（规划→执行→调度）

**优势与局限**：
- 优势：首次将基础设施信号纳入多 Agent 编排，高负载下 SLO 保障
- 局限：需要基础设施可观测性，RL 训练成本

**未来发展**：
代表多 Agent 编排从"任务中心"到"基础设施中心"的范式转变，对生产部署至关重要。

#### 3. HORMA - 分层记忆导航

**地址**：https://arxiv.org/abs/2606.11680

**论文介绍（~300 字）**：
LLM Agent 在长周期任务中挣扎，因为其固有的无状态性，要求所有任务相关信息编码在增长输入上下文中。由此导致的推理质量下降、推理成本增加和延迟升高需要高效工作记忆机制。然而，现有方法依赖有损压缩或相似性检索，无法捕获多步 Agent 任务所需的时间结构和因果依赖。

HORMA（Hierarchical Organize-and-Retrieve Memory Agent）将经验组织为文件系统式分层结构，摘要实体链接到对应的原始轨迹，实现高效访问而不丢失详细信息。工作记忆分解为两阶段：结构化记忆构建和导航式检索。构建模块通过区分信息缺失和误导性/过载上下文导致的失败，迭代精炼经验的组织方式。导航模块通过遍历层次结构检索任务相关上下文——使用 RL 训练的轻量 Agent 选择最小但充分的上下文，减少关键执行路径上的延迟。

在 ALFWorld、LoCoMo 和 LongMemEval 上，HORMA 在约束上下文预算下改进任务性能，长对话任务中最多仅需基线 22.17% 的 token 使用量。相比现有方法，一致实现更好的效率-性能权衡并有效泛化到未见任务。

**技术架构**：
- 构建层：迭代精炼经验组织结构
- 导航层：RL 训练的层次遍历 Agent
- 存储层：文件系统式分层结构
- 检索层：最小充分上下文选择

**核心机制**：
- 组织→检索分离：先组织经验结构，再导航检索
- RL 导航：训练轻量 Agent 选择最小充分上下文
- 失败归因：区分信息缺失 vs 上下文过载

**优势与局限**：
- 优势：显著降低 token 使用（22.17%），保持性能，泛化到未见任务
- 局限：需要 RL 训练，层次结构质量依赖构建模块

**未来发展**：
记忆系统从"被动存储"向"主动组织+RL 导航"演进，代表 Agent 记忆管理的新范式。
