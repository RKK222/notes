## 深度研究

### GitHub 项目

#### 1. SkillSpector - NVIDIA Agent 技能安全扫描器
- **地址**: https://github.com/NVIDIA/SkillSpector
- **介绍**: NVIDIA 开源的 Agent 技能安全扫描工具，在 Hacker News 上引发关注。随着 Agent 技能生态的爆炸式增长（agent-skills 56K+ stars），技能层安全成为 Agent 安全的新前沿。SkillSpector 针对 Agent 技能中的漏洞模式进行自动化扫描，填补了从模型层安全向技能层安全扩展的空白。
- **技术架构**: 基于静态分析和模式匹配的扫描引擎，针对 Agent 技能特有的安全漏洞类型设计检测规则。
- **核心创新**: 将安全扫描从传统的代码/模型层扩展到 Agent 技能层，识别技能定义中的权限滥用、数据泄露等风险。
- **优势与局限**: 与 agent-skills 生态天然互补，但需要持续更新漏洞模式库。
- **发展潜力**: 可能成为 Agent 技能发布前的标准安全检查工具。

#### 2. Paca - 轻量级人机协作项目管理
- **地址**: https://github.com/Paca-AI/paca
- **介绍**: 用 Go 编写的轻量级 Jira 替代方案，核心创新是人类和 AI Agent 作为平等队友共同规划 Sprint 和分配任务。支持自定义视图、字段和 WASM 插件架构。开发团队日常使用，承诺持续维护和完全免费。
- **技术架构**: Go 后端 + WASM 插件系统，支持 Agent 直接参与任务分配和 Sprint 规划。
- **核心创新**: 首次将 AI Agent 定位为与人类平等的协作参与者，而非辅助工具。
- **优势与局限**: 轻量、免费、可扩展，但功能深度尚不及 Jira。
- **发展潜力**: 代表"AI-Native 工具"的新范式——不是给传统工具加 AI 功能，而是从设计上就让人机平等协作。

### 学术论文

#### 1. MaxProof - 群体级测试时扩展数学证明
- **地址**: https://arxiv.org/abs/2606.13473
- **介绍**: MiniMax 团队提出的群体级测试时扩展框架，用于竞争级数学证明。核心创新是训练三种证明导向能力——证明生成、证明验证和批评条件证明修复——使用防御深度生成验证器（低假阳性率）。三种能力合并到单一模型中，通过群体级推理实现测试时扩展。在竞争级数学证明任务上展示出色性能。
- **技术架构**: 三阶段训练（生成→验证→修复），防御深度生成验证器确保低假阳性，群体级推理时多候选并行评估。
- **核心创新**: 将测试时扩展从简单的多采样推进到结构化的群体级协作推理，验证器作为守门员确保质量。
- **与现有技术对比**: 超越传统的 Best-of-N 采样，通过专门的验证和修复能力实现更高效的推理扩展。
- **优势与局限**: 在数学证明领域效果显著，但验证器训练成本高，泛化到其他推理任务需进一步验证。
- **发展潜力**: 群体级推理范式可能成为高可靠性推理任务的标准方法。

#### 2. COM-as-Action - 专业软件操控新范式
- **地址**: https://arxiv.org/abs/2606.13239
- **介绍**: 提出将 Component Object Model (COM) 作为统一可执行抽象，重构专业软件交互为确定性程序合成而非顺序视觉控制。引入 ComCADBench（首个工业 CAD Agent 基准）和 ComActor（自修复 Agent）。实验揭示范式鸿沟：前沿模型在 GUI 交互下接近零成功率，而 COM 基执行显著提升。
- **技术架构**: COM API 作为确定性动作空间，三阶段渐进训练（SFT→几何感知 RL→可行前缀掩码），自修复机制桥接语法正确性与几何精确性。
- **核心创新**: 从"GUI 视觉 grounding"转向"API 程序合成"，解决专业软件中 GUI Agent 的脆弱性和误差累积问题。
- **与现有技术对比**: GUI Agent（Cua/OS World）在专业软件中几乎无效；COM-as-Action 提供确定性接口但依赖软件 COM 支持。
- **优势与局限**: 在支持 COM 的专业软件（AutoCAD, SolidWorks 等）中效果显著，但不支持无 COM 接口的软件。
- **发展潜力**: 可能成为专业软件 Agent 交互的标准范式，推动更多专业软件暴露 COM/可编程接口。

#### 3. The End of Code Review - 编码 Agent 超越人类审查
- **地址**: https://arxiv.org/abs/2606.13175
- **介绍**: Martin Monperrus 提出的激进观点：编码 Agent 已跨越能力阈值，传统人类代码审查不再是软件质量流水线的必要组件。论证基于两个主张：代码审查的每个目标都可由 Agent 以更低成本和更高吞吐量实现；Agent 写代码 + 人类审查的朴素集成是死胡同。
- **技术架构**: 概念性论文，无具体实现。核心论点是 Agent 可以执行更一致的静态分析、测试生成和修复建议。
- **核心创新**: 首次系统性地论证人类代码审查的可替代性，挑战 50 年软件工程实践。
- **与现有技术对比**: 与传统"Agent 辅助代码审查"不同，本文主张完全替代。
- **优势与局限**: 论点激进但有说服力，但忽略了代码审查中的知识传递、团队对齐等社会功能。
- **发展潜力**: 可能引发软件工程实践的重大范式转变，但需要更多实证研究支持。

#### 4. DIG - Oracle 引导的零日漏洞定向生成
- **地址**: https://arxiv.org/abs/2606.13037
- **介绍**: 针对零日漏洞（patch 已发布但未广泛部署）的 PoC 输入生成。关键洞察：补丁通常揭示了触发漏洞的必要前置条件。DIG 使用 LLM 分析补丁并合成显式 Oracle，在高层进行 Oracle 引导的生成器进化，在低层将 Oracle 注入目标程序引导定向模糊测试。在 138 个真实 CVE 上超越 2 个 SOTA Agent 和 10 个 Fuzzer。
- **技术架构**: LLM 补丁分析 → Oracle 合成 → 双层引导（Agent 约束求解 + 定向模糊测试分支距离反馈）。
- **核心创新**: 利用补丁作为"漏洞知识源"，将模糊测试从盲目搜索转为 Oracle 引导的定向搜索。
- **与现有技术对比**: 80 CVE 触发 vs 最佳基线 57（+40%），独占触发 9 个 CVE。
- **优势与局限**: 依赖补丁质量，对模糊或不完整的补丁效果有限。
- **发展潜力**: 可能成为漏洞研究和网络安全防御的标准工具。

## 技术趋势

1. **Agentic PR 质量治理成为新焦点**：今天多项研究（AIDev 拒绝分析、Instructions-as-Code、架构质量因果研究）集中关注 Agent 生成代码的实际质量。46% 的 PR 拒绝率和指令文件效果的矛盾结果表明，Agent 代码生成从"能写"到"写好"仍有巨大差距，需要系统性的质量治理框架。

2. **专业软件 Agent 交互范式转变**：COM-as-Action 提出从 GUI 视觉控制转向 API 程序合成，代表 Agent 从"通用界面操作"向"专业领域深度集成"的演进。这一转变可能重塑 CAD、PLM、ERP 等专业软件的 Agent 生态。

3. **测试时扩展从单模型到群体协作**：MaxProof 展示群体级推理扩展的潜力，从 Best-of-N 到结构化的生成-验证-修复三阶段协作。这一范式可能从数学证明扩展到其他高可靠性推理任务。

## 新领域收获

1. **退役手机低碳计算平台**：Google Research 探索利用退役手机构建低碳计算集群。这一方向将可持续计算与边缘计算结合，可能为 AI 推理提供低成本、低碳的替代方案。

2. **本地编码 Agent 部署生态**：macOS 本地编码 Agent 设置指南在 HN 获得 379 分，反映开发者对本地化、隐私保护的 AI 编码工具的强烈需求。本地 Agent 可能成为企业级 AI 编码的重要部署模式。

3. **群体级测试时扩展**：MaxProof 展示的群体级推理范式超越了传统的单模型推理扩展，通过多候选协作和专门验证器实现更可靠的推理。这一范式可能成为高可靠性 AI 系统的标准架构。
