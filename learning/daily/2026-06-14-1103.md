#### 1. ToolSense - 参数化工具知识审计诊断框架
- **地址**：https://arxiv.org/abs/2606.12451
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / Agent 工具评估
- **技术介绍**：SAP 开源的诊断框架，针对 LLM Agent 在大型工具目录上的参数化工具检索能力进行深度审计。现有基准（如 ToolBench）使用冗长、完全指定的查询和约束解码，无法揭示模型是否真正理解其工具。ToolSense 自动生成三个基准：三级模糊度的真实检索基准（RRB）、MCQ 探测基准和 QA 探测基准。在 ToolBench（~47K 工具）上评估五种参数化模型训练配置后，发现"知识-检索分离"现象：在 RRB 查询上，多个配置相比完全指定的 ToolBench 基准下降 50-64 个百分点，甚至低于嵌入模型基线。此外，尽管检索性能强劲，一些模型在事实探测上的得分接近随机水平。
- **应用场景**：Agent 工具检索能力审计、参数化 vs 嵌入检索对比、工具知识完整性评估
- **为什么值得关注**：首次系统性地揭示 Agent 工具知识中的"知识-检索分离"现象，为 Agent 工具使用能力的真实评估提供诊断工具

#### 2. Arbor - 树搜索作为自主 Agent 认知层
- **地址**：https://arxiv.org/abs/2606.12563
- **来源**：arXiv (cs.AI)
- **领域**：Agent Testing / 自主优化
- **技术介绍**：多 Agent 框架，将结构化树搜索引入自主 Agent 的认知层，适用于大型有状态动作空间。与先前在无状态评估的孤立目标上运行的自主优化系统不同，Arbor 维护一个显式的评分假设搜索树，作为跨 Agent 的共享工作记忆，随每次测量演化，将失败视为诊断信号以重塑后续探索，并在先前成功转移瓶颈分布时扩展。Arbor 配对编排器 Agent（跨推理栈委派给领域专家）和批评者 Agent（通过根因分析、内省和测量验证保障稳定性），形成制衡架构。Agent 能力分解为硬技能（领域专长）和软技能（协调协议）。在完整 LLM 推理优化上实现高达 193% 的吞吐量-延迟 Pareto 改进，而单 Agent 无 harness 仅 +33% 并在数小时内崩溃。
- **应用场景**：LLM 推理系统自动优化、多 Agent 协作调试、基础设施性能调优
- **为什么值得关注**：将树搜索从传统搜索算法提升为 Agent 认知基础设施，193% 的性能提升远超单 Agent 方案

### AI / LLM 技术（11 项）

#### 10. Pythagoras-Prover - 高效形式化证明（4B 超越 671B）
- **地址**：https://arxiv.org/abs/2606.12594
- **来源**：arXiv (cs.AI)
- **领域**：形式化证明 / LLM
- **技术介绍**：计算高效开源 Lean 定理证明器家族，面向实际计算预算。包含两种生成范式：4B 和 32B 参数的自回归模型，以及首个概念验证扩散证明器（4B），在推理时迭代精炼 Lean 证明。训练方面构建分层（简单/中等/困难）的 Lean 验证语料用于课程 SFT，使模型从短简单证明逐步获取长困难证明的技能。SFT 期间动态证明推理过滤方案保留信息性证明轨迹，同时保持在 8K token 上下文预算内。引入增强 Lean 形式化（ALF），通过自蒸馏将稀缺验证语料扩展为形式语句变体，无需形式化验证每个变异实例。实证结果：Pythagoras-Prover-4B 在 MiniF2F-Test 上 pass@32 超越 DeepSeek-Prover-V2-671B（86.1% vs 82.4%），参数少 ~167 倍；32B 版本在 MiniF2F-Test 上达 93.0%，解决 PutnamBench 672 题中的 93 题。
- **应用场景**：形式化验证、数学证明、定理证明
- **为什么值得关注**：4B 模型超越 671B 模型，证明效率优化的巨大潜力；扩散式证明器是首次概念验证

#### 11. 战略决策支持 AI Agent
- **地址**：https://arxiv.org/abs/2606.12587
- **来源**：arXiv (cs.AI)
- **领域**：Agent 决策 / 人机协作
- **技术介绍**：重新审视决策支持的两个基本原则——寻求支持的成本-价值权衡和不确定性量化的角色——在 AI Agent 作为中心行动者的设置中。提出通过优化问题最小化支持使用，同时控制反事实错失支持误差（Agent 独自行动但支持本可显著改善输出的概率）。在群体层面，最优策略是支持值的阈值规则。开发在线算法自适应阈值，使用随机探索控制错失支持误差而无需分布假设。引入即时校准方法减少不必要的在线支持调用。在信息收集、人机协作和工具使用等多种场景中实例化该框架。
- **应用场景**：Agent 决策优化、人机协作、工具调用策略
- **为什么值得关注**：将经典决策支持理论反转应用于 Agent 中心场景，提供理论保证的在线算法

#### 12. PersonaDrive - 人类风格检索增强 VLA 驾驶 Agent
- **地址**：https://arxiv.org/abs/2606.12616
- **来源**：arXiv (cs.AI)
- **领域**：自动驾驶 / VLA Agent
- **技术介绍**：闭环驾驶仿真器通常使用行为高度一致的自交通 Agent，由基于规则的流量管理器或训练为单一行为模式的 learned 模型生成。PersonaDrive 引入管线，将 VLA 驾驶 Agent 条件化于从风格指令的人类驾驶数据集中检索的演示。参与者在一套驾驶模拟器上以激进、中立和保守指令驾驶 CARLA 排行榜路线。管线三阶段：（i）离线三元组挖掘，使用图像-文本联合相似度评分；（ii）训练轻量检索头，融合冻结视觉特征与小型控制编码器；（iii）微调单一 VLA backbone，将检索上下文点作为 in-context 行为演示用于航路点预测。推理时通过切换检索头查询的每风格数据库实现风格条件化，无需每风格重训练。在 Bench2Drive 上，PersonaDrive 驾驶分数比 SimLingo 高 4.6%，比 HiP-AD 高 2.5%。
- **应用场景**：自动驾驶仿真、VLA Agent、风格多样化驾驶
- **为什么值得关注**：首次使用显式风格指令的人类驾驶数据作为 VLA Agent 的条件化信号

#### 13. "Did you lie?" - LLM 谎言检测评估
- **地址**：https://arxiv.org/abs/2606.12618
- **来源**：arXiv (cs.AI)
- **领域**：AI 安全 / 模型可解释性
- **技术介绍**：针对 LLM 的鲁棒谎言检测器可用于审计、监控和事后调查模型行为，但评估需要模型可验证地相信与其所说相反的测试床。现有训练模型生物往往不满足此要求。本文提供 13 种推理模型生物，其隐藏信念在 chain-of-thought 中验证并泛化到 held-out 任务，以及 Varied Deception（涵盖广泛谎言诱导动机的提示说谎测试床）。评估四种检测器：CoT judge、logprob 分类器和两个激活探针（包括新方法 Did-You-Lie/DYL）。在提示说谎上，跨 31 个开放权重模型（2B 到 1T 参数），四种检测器均显示与模型能力的正缩放。但在训练模型生物上，每个激活和 logprob 基检测器急剧下降，DYL 保留最多信号；仅 CoT judge 保持强劲（0.82 平衡准确率）。当前谎言检测器无法支持关于模型信念的高置信度声明。
- **应用场景**：AI 安全审计、模型诚实性评估、谎言检测
- **为什么值得关注**：首次系统评估 LLM 谎言检测器的真实能力，揭示现有方法的严重局限

### GitHub 项目

#### 1. Arbor - 树搜索作为自主 Agent 认知层

**地址**: https://arxiv.org/abs/2606.12563

**介绍**: Arbor 是一个多 Agent 框架，将结构化树搜索引入自主 Agent 的认知层，适用于大型有状态动作空间。传统自主优化系统在无状态评估的孤立目标上运行，而 Arbor 维护显式的评分假设搜索树作为跨 Agent 的共享工作记忆，随每次测量演化，将失败视为诊断信号重塑后续探索。

**技术架构**: 
- 编排器 Agent（Orchestrator）：驱动优化，跨推理栈委派给领域专家
- 批评者 Agent（Critic）：通过根因分析、内省和测量验证保障稳定性
- 显式搜索树：评分假设的树结构，作为共享工作记忆
- 能力分解：硬技能（领域专长）+ 软技能（协调协议）

**核心算法/系统设计**:
- 树搜索作为认知层：不同于传统 MCTS，Arbor 的树是跨 Agent 共享的工作记忆
- 失败即信号：将失败测量视为诊断信号，而非简单丢弃
- 制衡架构：编排器和批评者均不能单方面驱动系统

**关键实现机制**:
- 每次测量后更新搜索树评分
- 瓶颈分布随成功而转移，树结构相应扩展
- 多日自主优化战役，无需人工干预

**与现有技术对比**:
- 传统优化：单目标、无状态、孤立评估
- Arbor：多目标、有状态、协作优化
- 单 Agent 方案：+33% 吞吐量，数小时内崩溃
- Arbor：+193% Pareto 改进，稳定运行多日

**优势与局限**:
- 优势：193% 性能提升、硬件无关、可重现（方差 <2pp）
- 局限：需要多 Agent 协调基础设施、搜索树维护开销

**未来发展潜力**:
- 可扩展到其他基础设施优化领域（数据库、网络）
- 树搜索认知层可能成为自主 Agent 的标准架构组件

**个人技术理解**:
Arbor 的核心创新在于将"树搜索"从传统算法提升为 Agent 认知基础设施。传统树搜索（如 MCTS）是单一 Agent 的决策工具，而 Arbor 的树是跨 Agent 的共享工作记忆。这种范式转变使得多 Agent 系统能够像人类工程团队一样协作优化——每个 Agent 贡献专业知识，树结构记录集体智慧。193% 的性能提升远超单 Agent 方案，证明了这一架构的有效性。

#### 2. Pythagoras-Prover - 高效形式化证明（4B 超越 671B）

**地址**: https://arxiv.org/abs/2606.12594

**介绍**: Pythagoras-Prover 是计算高效开源 Lean 定理证明器家族，面向实际计算预算。4B 参数模型在 MiniF2F-Test 上 pass@32 超越 DeepSeek-Prover-V2-671B（86.1% vs 82.4%），参数少 ~167 倍。32B 版本达 93.0%，解决 PutnamBench 672 题中的 93 题。首次引入扩散式证明器概念验证。

**技术架构**:
- 自回归模型：4B 和 32B 参数
- 扩散证明器：4B 参数，迭代精炼 Lean 证明
- 课程 SFT：分层（简单/中等/困难）训练语料
- 动态证明推理过滤：保留信息性轨迹，控制在 8K token 内
- ALF（增强 Lean 形式化）：自蒸馏扩展验证语料

**核心算法/系统设计**:
- 课程学习：从短简单证明到长困难证明的渐进式技能获取
- ALF 变异：扰动已知问题同时保持形式特征，减少对表面形式的依赖
- 扩散生成：迭代精炼而非一次性生成

**关键实现机制**:
- MiniF2F-ALF 基准：ALF 变异后的污染敏感基准，所有模型准确率下降
- 8K token 上下文预算内的动态过滤
- 扩散式推理时精炼

**与现有技术对比**:
- DeepSeek-Prover-V2-671B：4B 超越（86.1% vs 82.4%）
- Goedel-Prover-V2-32B：4B 匹配 SOTA
- 32B 版本：开源 SOTA（93.0%）

**优势与局限**:
- 优势：计算效率极高、开源、扩散式证明器首次验证
- 局限：扩散式证明器仍为概念验证、ALF 基准可能引入偏差

**未来发展潜力**:
- 扩散式证明器可能成为形式化证明的新范式
- 小模型高效证明将降低形式化验证的门槛

**个人技术理解**:
Pythagoras-Prover 的核心启示是"效率优于规模"。4B 模型超越 671B 模型，证明通过更好的训练策略（课程学习、ALF 变异、动态过滤）可以大幅减少对模型规模的依赖。扩散式证明器的引入尤其值得关注——它将证明生成从"一次性生成"转变为"迭代精炼"，这与人类数学家的工作方式更相似。

#### 1. ToolSense - 参数化工具知识审计诊断框架

**地址**: https://arxiv.org/abs/2606.12451

**介绍**: SAP 开源的诊断框架，针对 LLM Agent 在大型工具目录上的参数化工具检索能力进行深度审计。首次系统性地揭示 Agent 工具知识中的"知识-检索分离"现象。

**技术架构**:
- 输入：任意工具目录
- 输出：三个自动生成的基准
  - 真实检索基准（RRB）：三级模糊度查询
  - MCQ 探测基准
  - QA 探测基准
- 评估：五种参数化模型训练配置

**核心算法/系统设计**:
- 三级模糊度查询生成：从清晰到模糊的渐进式测试
- 知识 vs 检索分离检测：对比检索性能和事实探测性能
- 开放源码：github.com/SAP/toolsense

**关键实现机制**:
- LLM 驱动的基准生成
- 约束解码 vs 自由解码对比
- 嵌入模型基线对比

**与现有技术对比**:
- ToolBench：使用冗长完全指定查询 + 约束解码
- ToolSense：真实模糊查询 + 自由解码 + 事实探测
- 发现：RRB 查询上性能下降 50-64pp

**优势与局限**:
- 优势：首次揭示知识-检索分离、开放源码、适用于任意工具目录
- 局限：需要 LLM 生成基准（计算成本）、仅针对参数化检索

**未来发展潜力**:
- 可能成为 Agent 工具能力评估的标准诊断工具
- 扩展到更多工具类型和检索方法

**个人技术理解**:
ToolSense 的核心贡献是揭示了 Agent 工具使用中的一个关键问题：检索能力强 ≠ 理解工具。模型可能通过表面模式匹配实现高检索准确率，但对工具的实际功能理解接近随机。这一发现对 Agent 工具评估有深远影响——现有的高准确率基准可能严重高估了 Agent 的真实能力。

#### 2. "Did you lie?" - LLM 谎言检测评估

**地址**: https://arxiv.org/abs/2606.12618

**介绍**: 首次系统评估 LLM 谎言检测器的真实能力。提供 13 种推理模型生物（隐藏信念在 CoT 中验证）和 Varied Deception 基准。发现现有激活和 logprob 基检测器在真实模型生物上急剧下降，仅 CoT judge 保持 0.82 平衡准确率。

**技术架构**:
- 13 种推理模型生物：隐藏信念在 CoT 中验证
- Varied Deception 基准：广泛谎言诱导动机
- 四种检测器：CoT judge、logprob 分类器、DYL 激活探针

**核心算法/系统设计**:
- 信念验证：通过 CoT 验证模型的真实信念
- 泛化测试：held-out 任务上的泛化能力评估
- 检测器对比：四种检测器在提示说谎和训练模型生物上的表现

**关键实现机制**:
- 31 个开放权重模型（2B-1T 参数）的评估
- 提示说谎 vs 训练模型生物的对比
- 缩放规律分析

**与现有技术对比**:
- 提示说谎：四种检测器均显示正缩放
- 训练模型生物：激活/logprob 检测器急剧下降
- DYL：保留最多信号的激活探针
- CoT judge：唯一保持强劲的检测器

**优势与局限**:
- 优势：首次系统评估、验证信念的模型生物、广泛基准
- 局限：CoT judge 的优势部分源于验证过程偏向 CoT 可读信念

**未来发展潜力**:
- 谎言检测器需要新的架构方向
- 可能推动模型可解释性和诚实性研究

**个人技术理解**:
这篇论文的核心发现令人不安：当前主流的激活探针和 logprob 分类器在真实模型生物上几乎失效。这意味着大量基于这些方法的 AI 安全研究可能建立在脆弱的基础上。仅 CoT judge 保持强劲，但这部分是因为验证过程本身偏向 CoT 可读的信念。谎言检测领域需要根本性的新方向。
