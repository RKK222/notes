## 技术扫描（30+）

### AI Testing / Software Testing（9 项）

#### 1. ERTS - 伦理 AI 对抗测试系统
- **地址**：https://arxiv.org/abs/2606.13282
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 伦理安全
- **技术介绍**：封闭式对抗测试管线框架，将伦理困境编码为 22 维伦理后果空间（ECS），基于已建立的伦理理论。应用 17 种语义扰动函数，受 6 类有效性约束（包括新颖的语义连贯约束）。通过 4 组件伦理不稳定性指数（EII）测量决策偏差。在 50 个伦理场景（8 个部署域）上评估 4 个结构化基线模型和 2 个生产 LLM（Gemini 2.0 Flash 和 Llama 3.2），生成 1500 个对抗测试用例。结果：仅 33% 模型通过评估清除，本地 Llama-3.2 模型在公平性腐败和信息降解攻击中特别脆弱（ERS = 0.737）。
- **应用场景**：高风险伦理决策 AI 系统部署前评估、医疗分诊、自动驾驶、就业筛选
- **为什么值得关注**：首个结合有界伦理后果空间、语义连贯约束和域自适应评估的对抗测试管线，为 AI 伦理安全提供可操作的评估框架

#### 2. ARMOR-MAD - 自适应多 Agent 辩论路由
- **地址**：https://arxiv.org/abs/2606.13197
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 多 Agent 推理
- **技术介绍**：训练-free 异构多 Agent 辩论框架，将辩论视为条件计算。三个组件：辩论前协议路由（PAR）决定独立生成的 Round-0 答案是否需要辩论；早期协议停止评估器（EASE）在收敛后停止辩论；语义异常检测（SOD）在聚合期间降权异常最终答案。在 MATH Level 5、GSM8K、MMLU、MMLU-Pro 上分别达到 65.5%、96.5%、90.0%、81.5% 准确率，持续超越固定轮次异构辩论。
- **应用场景**：LLM 推理效率优化、多模型协作推理、资源受限推理场景
- **为什么值得关注**：真正的模型异构性和基于协议的控制在 MAD 中都很重要，条件计算范式避免浪费计算和放大相关错误

#### 3. UXBench - 移动端 UX 推理基准
- **地址**：https://arxiv.org/abs/2606.13192
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / MLLM 评估
- **技术介绍**：首个针对 MLLM 在 UI 推理方面能力的系统评估基准，包含 2000 个 VQA 数据样本，基于真实 UI 截图设计。8 个任务要求对布局关系、视觉层次和内容一致性进行细粒度 UX 问题诊断。对主流 MLLM 的广泛评估显示，它们在基于 UI 的推理能力上仍然根本受限。提出 UI-UX 方法，基于 Qwen3-VL-4B-Thinking，通过 RL 增强，包含奖励路由机制动态平衡感知不足和过度感知。
- **应用场景**：移动端 UI/UX 自动化评估、MLLM 能力边界测量、设计-to-code 生成质量
- **为什么值得关注**：揭示 MLLM 在 UI 推理方面的根本局限，为 CUA 和 GUI Agent 评估提供新维度

#### 4. SciR - 可控科学推理基准
- **地址**：https://arxiv.org/abs/2606.13020
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 科学推理评估
- **技术介绍**：首个在提取和推理两个维度上都有参数控制的多范式科学推理基准。结合演绎、归纳和因果溯因三种推理范式，从形式化对象（演绎树、归纳规则假设、因果图）生成任务以保证可验证答案，然后通过每轨道领域调优的文体渲染为多文档科学论述。可独立变化两个难度轴：提取关键信息的难度和原理推理的难度。测试六种模型，两个轴都影响每个模型且效果复合。渲染甚至伤害神经符号管线（将推理交给已验证求解器）。
- **应用场景**：科学推理能力评估、多范式推理对比、渲染效应对推理影响研究
- **为什么值得关注**：解决科学基准依赖人工标注成本高且缺乏机制真值、合成逻辑推理基准不 resemble 真实科学文档的两难

#### 5. STG - 结构化测试台生成
- **地址**：https://arxiv.org/abs/2606.12983
- **来源**：arXiv (cs.AI)
- **领域**：Software Testing / HDL 验证
- **技术介绍**：利用硬件设计固有结构生成确定性测试台的结构化框架。作为直接验证工具，STG 比迭代 LLM 基线测试台生成流程快 720x，更高的成功编译率，更高的覆盖率，减少不正确 DUT 上的假通过裁决。作为数据策展引擎，在单 CPU 核心上比 LLM 过滤快 11x，能耗低 127x，生成的蒸馏模型在多项基准上提供 SOTA 性能。还帮助识别 RTL 生成基准中的错误，暴露有缺陷的基准测试台。
- **应用场景**：HDL 设计验证、RTL 生成质量评估、硬件验证数据策展
- **为什么值得关注**：将测试台生成从随机代码合成推进到结构化确定性生成，解决 LLM 驱动 RTL 工作流中的关键瓶颈

#### 6. 多模态 Agent 电力配网缺陷检测
- **地址**：https://arxiv.org/abs/2606.12969
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 工业 Agent 评估
- **技术介绍**：针对电力配网缺陷检测的多模态 Agent 框架，系统评估多模态基础模型作为统一认知引擎。从三个关键能力维度评估：(1) 感知——准确识别设备并生成专家级缺陷描述；(2) 推理——解释视觉发现以诊断原因、评估严重性并基于领域知识规划维护策略；(3) 工具使用——作为自主操作员执行查询知识库或生成工单等动作以实现闭环维护。开发领域特定评估数据集和综合基准。
- **应用场景**：工业基础设施自主检测、电力网络维护、高 stakes 工业环境 Agent 部署
- **为什么值得关注**：为高 stakes 工业环境部署自主 Agent 提供实证证据，三维度评估框架可推广到其他工业场景

#### 7. V-RAGBench - 长视频 RAG 基准
- **地址**：https://arxiv.org/abs/2606.13141
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 视频 RAG 评估
- **技术介绍**：解决现有 VideoRAG 基准允许无需视频回答问题从而掩盖检索错误的问题。引入 ⟨查询, 证据块, 答案⟩ 三元组基准，实现检索和生成的忠实解耦评估。CARVE 方法跨配置运行并行检索器，采用块自适应重排识别每个块的获胜配置，然后以获胜配置进入生成器，产生交错证据形式。超越 8 个近期 VideoRAG 基线。
- **应用场景**：长视频理解、多模态 RAG 评估、视频问答系统
- **为什么值得关注**：首次解耦评估视频 RAG 的检索和生成阶段，揭示现有方法的系统性缺陷

#### 8. 医疗影像 AI 幻觉跨模态分析框架
- **地址**：https://arxiv.org/abs/2606.13211
- **来源**：arXiv (cs.AI)
- **领域**：AI Testing / 医疗 AI 安全
- **技术介绍**：跨五种成像模态（X 光、CT、MRI、超声、病理）的结构化叙事综合，统一幻觉分类法、病因学、检测和缓解。三个核心发现：(1) 三个分类框架共同覆盖成像管线，没有单个框架能单独做到；(2) 通用基础模型在幻觉特定基准上超越医疗专用模型，表明窄域微调可能引入过拟合诱导的编造；(3) 物理信息架构约束、CoT 提示和人在环 safeguards 各自解决不同失败模式，组合时有效。所有发现映射到 FDA 总产品生命周期和预定变化控制计划框架。
- **应用场景**：医疗影像 AI 安全评估、FDA 合规、临床部署风险管理
- **为什么值得关注**：揭示"窄域微调反而增加幻觉"的反直觉发现，对医疗 AI 开发实践有直接影响

#### 9. 上下文窗口信任问题
- **地址**：https://garrit.xyz/posts/2026-05-06-dont-trust-large-context-windows
- **来源**：HN（112 points, 82 comments）
- **领域**：AI Engineering / Agent 架构
- **技术介绍**：通过阻止用户顶级对话线程中的所有工具调用来避免上下文大小问题。需要工具调用的任何操作必须在递归调用 Agent 中发生，将结果返回给调用者。可以在百万 LOC+ 代码库上整天保持同一高级对话而不触及有意义的 token 限制。不需要压缩或摘要技巧。可以在递归调用中消耗 5000 万 token 而根对话线程仍不到 10 万 token。外部符号递归似乎不是 Frontier 模型训练过的——它们在上下文内模拟递归方面很棒，但如果试图减少 token 使用则不想要那样。递归深度超过 1 没有观察到提升。
- **应用场景**：大型代码库 Agent 开发、上下文 token 优化、递归 Agent 架构
- **为什么值得关注**：提供实用的上下文管理策略，挑战"越大越好"的上下文窗口迷思

### AI / LLM 技术（11 项）

#### 10. 基因组锚定个性化健康 AI
- **地址**：https://arxiv.org/abs/2606.13556
- **来源**：arXiv (cs.AI)
- **领域**：AI / 个性化健康
- **技术介绍**：解决个性化健康 AI 系统的冷启动问题。个体的基因组谱系作为外源遗传锚点——在受孕时固定的域知情个性化先验，免受反向因果影响，在收集单个行为观察之前就可获得。锚点初始化贝叶斯信念状态：G-hat = μ + Σ(βᵢ × gᵢ)，其中 βᵢ 是 GWAS 导出的效应量，gᵢ 是风险等位基因计数。每个传入的生理测量 P 产生非宪法偏差 δ = P - G-hat，将归因于环境和状态的信号与宪法固定基线分离。随着行为数据累积，先验根据 G-hat_t = w(t)×G-hat_genomic + [1-w(t)]×P-bar_t 衰减，从基因组主导过渡到经验基线主导推理。在六个生理域中开发此架构，按证据强度对基因组先验分级。
- **应用场景**：个性化健康 AI、冷启动生理解释、基因组引导的贝叶斯推理
- **为什么值得关注**：首次将基因组数据作为个性化 AI 先验锚点，解决 ML 模型需要数周个体行为数据才能区分宪法变异和环境驱动偏差的问题

#### 11. LLM 意图性与道德责任哲学分析
- **地址**：https://arxiv.org/abs/2606.13441
- **来源**：arXiv (cs.AI)
- **领域**：AI / 哲学分析
- **技术介绍**：论证将代理或道德代理资格归因于 LLM 是误导的。道德责任要求基于内在意图性和自我归因行动的承诺承载代理，这种代理构成与责任相关的自由意志形式。虽然 LLM 生成连贯且可规范评估的输出，但其操作完全由从数据学习的概率输入输出映射表征。其明显的意图性是衍生的而非内在的，输出既不作为承诺拥有也不由理由引导。随机采样引入的变异性不构成选择或创作。回应来自意向立场、功能主义、兼容主义和模型输出中存在道德推理的反对意见。
- **应用场景**：AI 代理理论、LLM 道德地位讨论、AI 治理哲学基础
- **为什么值得关注**：清晰界定 LLM 代理边界，对抗过度拟人化，为 AI 治理提供哲学基础

#### 12. 神经符号 Agent 合规构建
- **地址**：https://arxiv.org/abs/2606.13405
- **来源**：arXiv (cs.AI) / NILA Workshop @ IJCAI-ECAI 2026
- **领域**：AI / 神经符号 / 合规自动化
- **技术介绍**：LLM 基 Agent 进入受监管行业，自动化判断密集型质量管理流程。论证这些域中已嵌入的符号结构（包括法规、类型化流程模型和合规约束）应被视为核心架构组件而不仅仅是外部监控机制，塑造 Agent 的决策和行为。提出合规构建作为护栏监控的补充范式：防止控制流违规的结构基础，同时护栏对捕获语义错误仍然 essential。识别基础和能力层面的结构化神经符号研究挑战。
- **应用场景**：受监管行业 Agent 自动化、合规构建、金融/医疗/法律 AI
- **为什么值得关注**：从护栏监控推进到合规构建，为受监管行业 Agent 部署提供架构范式

#### 13. MOSAIC - 模态增量持续学习
- **地址**：https://arxiv.org/abs/2606.13258
- **来源**：arXiv (cs.AI)
- **领域**：AI / 持续学习 / 医疗
- **技术介绍**：针对帕金森步态评估的紧凑持续学习框架。识别"毒性教师"现象（新模态蒸馏不稳定），引入模态特定预热在蒸馏前稳定新学习模态表示。提出统计解耦 MSBN 架构隔离传感器统计同时维护共享语义骨干。设计课程引导排斥目标进行可塑性恢复，在保留遗留知识的同时恢复模态特定容量。在三个多模态帕金森步态数据集上验证。
- **应用场景**：多模态医疗 AI、增量传感器部署、临床系统持续学习
- **为什么值得关注**：首次识别增量持续学习中的"毒性教师"问题，为医疗 AI 持续部署提供解决方案

#### 14. TerraBench - 地球系统 Agent 推理
- **地址**：https://arxiv.org/abs/2606.13148
- **来源**：arXiv (cs.AI)
- **领域**：AI / 地球科学 / Agent 推理
- **技术介绍**：地球科学推理基准，基于 TerraAgent（ReAct 风格可执行框架）将 LLM 规划与科学工具耦合，用于环境检索、地理空间处理、模拟和工件后备计算。统一地球观察图像分析、网格数据、GIS 推理和模拟在单个可执行接口中，而先前基准将这些能力隔离为狭窄的单独任务。也是该领域首个将过程级工具使用指标与容差感知数值评分配对的基准。403 个广泛 Agent 任务，三个轨道（基础、模拟接地、文档接地验证），八个应用领域，24500 个验证执行步骤。
- **应用场景**：气候和环境决策、地球科学 Agent、多源数据推理
- **为什么值得关注**：首个统一地球观察图像、网格数据、GIS 推理和模拟的 Agent 基准

#### 15. 预测市场认知单培养问题（Nous）
- **地址**：https://arxiv.org/abs/2606.13038
- **来源**：arXiv (cs.AI)

- **地址**：https://github.com（HN Show 项目）
- **项目介绍**（200~400 字）：Project Brain 2.0 是为 Claude Code 及任何文件读取 Agent 设计的项目记忆管理工具。在 AI 辅助开发日益普及的背景下，Agent 在长期项目中的上下文遗忘问题日益突出。传统方法依赖提示工程或上下文窗口扩展，但这些方法在大型项目中效果有限。Project Brain 2.0 采用结构化的项目记忆管理策略，通过外部文件系统维护项目的关键状态、决策历史、架构选择和待办事项。工具支持自动提取和更新项目记忆，确保 Agent 在多次会话间保持对项目的一致理解。与 projectmem（事件源记忆）不同，Project Brain 2.0 更侧重于"策展"而非"记录"——不是记录所有交互，而是有选择地保留对项目理解最关键的信息。这种策展式记忆方法在 token 成本和记忆完整性之间取得平衡。

- **地址**：https://ataraxy-labs.github.io/weave/
- **项目介绍**（200~400 字）：Weave 是一种基于语言结构而非行的代码合并方法。传统的基于行的合并工具（如 git merge）在 AI 辅助开发时代面临根本性挑战。当多个 Agent 或 AI 辅助工具同时修改同一代码库时，语义相同但文本不同的代码变更会导致大量虚假冲突。Weave 使用语言感知分析来理解代码的抽象语法树（AST）结构，在语义层面进行合并决策。这意味着即使两个 Agent 以不同的方式格式化或组织相同的逻辑，Weave 也能正确识别它们是等效的并自动合并。这种方法特别适用于多 Agent 协作开发场景，其中不同 Agent 可能使用不同的编码风格但实现相同的功能。Weave 目前支持多种主流编程语言，并计划扩展到更多语言。

- **技术架构**：语言解析器 → AST 构建 → 语义等价检测 → 结构化合并 → 冲突解决

- **优势与局限**：优势是减少虚假冲突、支持多 Agent 协作、提高合并准确性。局限是实现复杂度高、需要语言特定的解析器、对动态语言的支持仍在开发中。

- **未来发展潜力**：可能成为 AI 辅助开发的标准合并工具，与主流 IDE 和 CI/CD 系统集成。

- **个人技术理解**：随着 AI 生成代码比例的增加，传统基于行的合并工具将越来越不适应。Weave 代表的语义合并方向可能是未来的必然选择。

#### 1. 基因组锚定个性化健康 AI（2606.13556）
- **地址**：https://arxiv.org/abs/2606.13556
- **论文介绍**（200~400 字）：本文解决个性化健康 AI 系统的冷启动问题——机器学习模型需要数周的个体行为数据才能区分宪法变异和环境驱动偏差。作者提出基于因果推理和贝叶斯先验设计的解决方案：个体的基因组谱系作为外源遗传锚点，在受孕时固定，免受反向因果影响，在收集任何行为观察之前就可获得。锚点初始化贝叶斯信念状态 G-hat = μ + Σ(βᵢ × gᵢ)，其中 βᵢ 是 GWAS 导出的效应量，gᵢ 是风险等位基因计数。每个传入的生理测量 P 产生非宪法偏差 δ = P - G-hat，分离环境和状态信号与宪法固定基线。随着行为数据累积，先验衰减从基因组主导过渡到经验基线主导。在六个生理域中开发此架构，按证据强度对基因组先验分级（从稳健复制的 FTO、FADS1/2、FKBP5 到有争议的 SLC6A4、MAOA、DRD2）。解决关联、孟德尔随机化和个体 token 因果之间的推理边界，定义四个部署约束：证据分级先验、动态衰减、血统匹配效应量和归因而非确定性输出。

- **技术架构**：基因组锚点 → 贝叶斯先验初始化 → 生理测量偏差计算 → 先验衰减过渡 → 六域生理推理

- **核心算法**：G-hat_t = w(t) × G-hat_genomic + [1-w(t)] × P-bar_t，其中 w(t) 随时间衰减

- **关键实现机制**：证据分级基因组先验（按 GWAS 证据强度分级）、动态衰减函数（控制基因组到经验的过渡速率）、血统匹配效应量（避免群体 stratification 偏差）

- **与现有技术对比**：传统个性化健康 AI 依赖数周的行为数据积累才能建立个体基线。本文首次利用基因组数据作为先验锚点，在零行为数据时即可启动推理。

- **优势与局限**：优势是解决冷启动问题、因果推理框架严谨、部署约束明确。局限是依赖 GWAS 数据质量、对非欧洲血统人群的效应量数据不足、基因组解释的伦理风险。

- **未来发展潜力**：可能成为个性化健康 AI 的标准架构，与可穿戴设备、电子健康记录系统集成。

- **个人技术理解**：本文代表了 AI 与基因组学交叉的一个重要方向——不是用 AI 分析基因组，而是用基因组引导 AI。这种"生物引导 AI"的范式可能在个性化医疗、精准营养等领域产生深远影响。

#### 2. 伦理 AI 对抗测试系统 ERTS（2606.13282）
- **论文介绍**（200~400 字）：本文引入伦理鲁棒性测试系统（ERTS），一个封闭式对抗测试管线框架。随着 AI 系统部署在高风险伦理环境中（如医疗分诊、自动驾驶、就业筛选），评估其对抗伦理推理操纵的鲁棒性的形式化方法仍然不足。ERTS：(1) 将伦理困境编码为 22 维伦理后果空间（ECS），基于已建立的伦理理论；(2) 应用 17 种语义扰动函数，受 6 类有效性约束（包括新颖的语义连贯约束）；(3) 通过 4 组件伦理不稳定性指数（EII）测量决策偏差；(4) 生成域自适应部署前鲁棒性评估裁决。在 50 个伦理场景（8 个部署域）上评估 4 个结构化基线模型和 2 个生产 LLM（Gemini 2.0 Flash 和 Llama 3.2），生成 1500 个对抗测试用例。结果：仅 33% 模型通过评估清除，本地 Llama-3.2 模型在公平性腐败和信息降解攻击中特别脆弱（ERS = 0.737）。据作者所知，没有现有框架在单个对抗测试管线中结合有界伦理后果空间、语义连贯约束和域自适应评估。

- **技术架构**：伦理困境编码 → 语义扰动生成 → 决策偏差测量 → 域自适应裁决

- **核心算法**：22 维 ECS 编码 + 17 种扰动函数 + 4 组件 EII + 域自适应阈值

- **关键实现机制**：伦理理论 grounded 的后果空间、语义连贯约束（防止扰动产生无意义输入）、域自适应评估（不同部署域使用不同阈值）

- **与现有技术对比**：现有 AI 安全测试主要关注内容安全（如有害输出检测），缺乏对伦理推理鲁棒性的系统评估。ERTS 首次将伦理理论形式化为可计算的测试框架。

- **优势与局限**：优势是理论 grounded、覆盖多部署域、提供可操作的评估裁决。局限是伦理后果空间的主观性、扰动函数的覆盖范围、对更复杂 LLM 的评估不足。

- **未来发展潜力**：可能成为 AI 伦理安全评估的标准框架，与监管合规流程集成。

- **个人技术理解**：ERTS 代表了 AI 安全评估从"内容安全"向"推理鲁棒性"的重要转变。在 AI 系统越来越多地参与伦理决策的背景下，这种评估框架的必要性将持续增加。
