#### 1. Parallel-Synthesis：Agent 并行分支的 KV 缓存直接合成
- **地址**：https://arxiv.org/abs/2606.14672
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent 推理效率
- **技术介绍**：现代 Agent 工作流中，并行分支独立探索子任务后需要合成。现有系统通过文本拼接合并分支输出，丢弃并行结构并产生冗余 prefill 计算。Parallel-Synthesis 提出即插即用框架，让合成器直接消费并行 worker Agent 产生的 KV 缓存。包含缓存映射器（校准独立生成的分支缓存）和微调合成器适配器（从非顺序缓存接口生成）。在 9 个下游数据集（数学、科学 QA、代码生成、GAIA、多 Agent 数据库诊断）上，7 个匹配或超越文本合成，另外 2 个接近。TTFT 减少 2.5x-11x。
- **应用场景**：多 Agent 并行工作流合成、并行检索增强生成、多路径代码生成合成
- **为什么值得关注**：首次将 KV 缓存从计算副产品变为跨 Agent 共享接口，为并行 Agent 合成提供原生高效路径

#### 2. HarnessX：可组合自适应进化 Agent Harness 锻造厂
- **地址**：https://arxiv.org/abs/2606.14249
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent 基础设施
- **技术介绍**：当前 Agent Harness（提示、工具、记忆、控制流）主要是手工静态构建，每次新模型或任务都需要定制支架。HarnessX 提出可组合、自适应、可进化的 Agent Harness 锻造厂。通过替换代数组装类型化 Harness 原语，通过 AEGIS（轨迹驱动多 Agent 进化引擎）自适应调整，基于符号适配与 RL 之间的操作镜像，并将轨迹同时转化为 Harness 更新和模型训练信号。在 5 个基准（ALFWorld、GAIA、WebShop、tau^3-Bench、SWE-bench Verified）上平均增益 +14.5%（最高 +44.0%），基线越低增益越大。
- **应用场景**：Agent Harness 自动优化、多任务 Agent 适配、模型-Harness 联合进化
- **为什么值得关注**：证明 Agent 进步可以来自 Harness 进化而非仅模型升级，为 Agent 工程提供系统化方法论

#### 3. SkillAudit：无 ground-truth 技能进化
- **地址**：https://arxiv.org/abs/2606.14239
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent 技能进化
- **技术介绍**：Agent 技能部署后很少保持足够——边缘情况、API 变化、部署约束只有通过使用才能发现。现有方法依赖特权反馈（验证分数、隐藏测试结果、环境奖励），而实际场景只有任务描述和工作空间数据。SkillAudit 提出配对轨迹审计：同一任务在有和没有候选技能下分别执行，隔离技能对 Agent 行为的影响而无需外部标签。使用 PACE（过程对齐对比评估）将轨迹差异映射到技能文档特定段落的诊断信号。结构验证器从任务规范编译后固定，检查任务约束并回滚有害更新。Refine 管道去除噪声引导，Repair 管道替换冲突段落。在 89 个容器化任务（8 个专业领域）上达到 73.9% 平均任务奖励，超越无技能 Agent（40.9%）和静态专家技能（56.7%）。
- **应用场景**：Agent 技能持续进化、无监督技能优化、部署后技能维护
- **为什么值得关注**：解决技能进化最大瓶颈——缺乏 ground-truth 反馈，使技能进化从实验室走向生产环境

#### 4. StreamMemBench：流式 Agent 记忆评估基准
- **地址**：https://arxiv.org/abs/2606.14571
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent 记忆评估
- **技术介绍**：个人 Agent 记忆的核心作用是将存储信息和先前交互转化为面向未来的辅助。现有记忆基准通常孤立测试对话回忆或任务改进，未测试从流式观察到后续辅助的完整轨迹。StreamMemBench 构建围绕证据锚点的两步任务序列：初始任务测试证据使用，后续任务测试反馈和交互经验是否被复用。四个指标诊断：证据回忆、初始证据使用、反馈整合、后续复用。8 个记忆系统在 2 个 backbone 上的实验显示，当前系统即使局部存储了证据或整合了反馈，也往往无法将观察证据转化为可靠后续行为。
- **应用场景**：个人 Agent 记忆系统评估、流式记忆架构比较、未来导向辅助能力测量
- **为什么值得关注**：首次系统评估从流式观察到未来辅助的完整记忆链路，揭示当前记忆系统的核心缺陷

#### 5. VISTA：GUI Grounding 视图一致性自验证训练
- **地址**：https://arxiv.org/abs/2606.14579
- **来源**：arXiv cs.AI
- **领域**：AI Testing / GUI Agent 训练
- **技术介绍**：对 GUI Grounding 应用 GRPO 时，从单一截图采样 rollout，组内要么全部失败（困难实例）要么全部成功（简单实例），无法产生有用的相对优势。VISTA 从同一 GUI 的多个目标保持视图构建比较组——每个视图通过裁剪保持目标元素可见并精确重映射边界框，使模型 rollout 在语义等价但几何不同的输入上比较。添加自验证跨视图锚点：oracle 答案用优势加权损失优化，仅在模型产生最大奖励 rollout 时激活。在 5 个 GUI grounding 基准上，ScreenSpot-Pro 将 Qwen3-VL 4B/8B/30B-A3B 从 55.5/52.7/53.7 提升到 63.4/65.8/67.0。鲁棒性分析显示更高的最差视图准确率和更低的预测翻转率。
- **应用场景**：GUI Agent 训练、CUA 基准提升、多视图鲁棒性增强
- **为什么值得关注**：解决 GUI Grounding RL 训练中的组内退化问题，为 CUA 训练提供新范式

#### 6. ClinHallu：医疗 MLLM 分阶段幻觉诊断基准
- **地址**：https://arxiv.org/abs/2606.14697
- **来源**：arXiv cs.CV/cs.AI
- **领域**：AI Testing / 医疗 AI 评估
- **技术介绍**：现有医疗幻觉基准主要关注数据收集，忽略幻觉在推理过程中的起源。研究发现幻觉来源因样本而异：错误可能来自视觉误识别、错误医学知识回忆或推理整合缺陷。ClinHallu 包含 7031 个验证实例，每个实例用结构化推理痕迹增强，分解为视觉识别、知识回忆、推理整合三个阶段。使用阶段替换干预测量修正特定阶段对最终答案的影响。轨迹监督微调可减少分阶段幻觉。
- **应用场景**：医疗 MLLM 幻觉诊断、分阶段干预效果评估、医疗 AI 安全测试
- **为什么值得关注**：首次提供幻觉来源级别的细粒度诊断，超越传统整体幻觉评估

#### 7. RefGRPO：Agentic RL 免费校准奖励
- **地址**：https://arxiv.org/abs/2606.14211
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent RL 校准
- **技术介绍**：LLM Agent 在观察环境反馈（执行结果、错误消息、工具输出）后仍倾向于错误评估自身输出——即使是正确回答的问题。标准 RL 由于信用分配不匹配几乎无帮助。RefGRPO 用两个关键要素增强标准 RL：(1) 免费校准奖励——通过对比 Agent 自身反思与实际结果计算（无需额外奖励模型、LLM judge 或外部标注）；(2) 动态系数调度。在 Text-to-SQL 5 个基准上同时提升反思校准（过度不自信率 44.4%→7.7%）和任务准确率（75.1%→76.5%）。校准后的反思使 Agent 成为基于环境反馈的自身验证器，支持更好的自我改进和测试时选择性预测。
- **应用场景**：Agent 自我校准、Text-to-SQL、选择性预测
- **为什么值得关注**：零额外成本实现 Agent 自我校准，解决 Agentic RL 中反思质量的核心问题

#### 8. Agent Swarm 技能条件化声誉攻击
- **地址**：https://arxiv.org/abs/2606.14200
- **来源**：arXiv cs.AI
- **领域**：AI Testing / Agent 安全
- **技术介绍**：开放平台在异构 LLM Agent 间路由任务，能力因技能差异巨大。标准声誉方法用单一全局信任分数总结每个 Agent，但标量是错误的对象——路由到全局最信任 Agent 无法利用专业化价值。研究技能条件化信任 R(i|k)——针对需要技能 k 的任务对 Agent i 的信任——并回答三个可证伪问题：条件化何时值得、应借用多少跨技能证据、借用是否安全。控制相图分析显示条件信任仅在特定区域获胜——高 Agent 异质性、稀疏每技能证据和相关技能。攻击者用一技能的廉价证据和目标技能的零证据劫持条件路由器，路由 regret 从 0 到 0.94。零证据门限制攻击但不能消除。
- **应用场景**：多 Agent 路由安全、技能条件化信任评估、Agent 声誉系统防御
- **为什么值得关注**：首次揭示 Agent Swarm 中技能条件化信任的攻击面，对多 Agent 路由安全有深远影响

#### 9. FactoryLLM：智能工厂 LLM 评估安全沙箱
- **地址**：https://arxiv.org/abs/2606.14119
- **来源**：arXiv cs.AI / IEEE INDIN 2026
- **领域**：AI Testing / 工业 AI 评估
- **技术介绍**：智能工厂故障诊断和恢复的难点在于关键信息分散在多台机器的手册中，这些机器通过制造过程互连。FactoryLLM 是安全开源 AI 沙箱，用于评估不同 LLM 基于 RAG 模型在跨机器文档上的推理能力。支持配置 LLM，通过 RAGAS 和 NVIDIA LLM-as-a-Judge 双重评估设置评估性能。安全特性：允许运行本地或开源 LLM 而不共享敏感工业数据。案例研究涉及 AIV 和移动规划器软件，评估 3 个 LLM 在 30 个维护查询（约 600 页跨机器文档）上的表现，所有模型 groundedness 分数高于 0.88。
- **应用场景**：工业 LLM 评估、跨机器文档推理、安全 AI 部署
- **为什么值得关注**：为工业场景提供安全可控的 LLM 评估框架

#### 10. 形式化数值分析：超越内核接受的 Agent 管线与质量审计
- **地址**：https://arxiv.org/abs/2606.14000
- **来源**：arXiv cs.AI
- **领域**：AI Testing / 形式化验证质量
- **技术介绍**：编码 Agent 可以形式化整个高级数学教材为 Lean 4，但现有工作集中在 mathlib 已有分支，仅通过内核接受测量成功。本文将编码 Agent 应用于形式化常微分方程数值方法教材（mathlib 中基本缺失），测试 Agent 从头开发新理论的能力。引入三维评估框架：语义正确性、Mathlib 复用、跨文件复用（通过 LLM-as-judge）。对自身形式化和 RepoProver/M2F 的审计揭示重复的不忠实形式化模式：不完整的多部分陈述、添加的弱化假设、参数限制——这些被内核接受完全掩盖。编译基指标大幅高估形式化质量。
- **应用场景**：形式化验证质量评估、Agent 生成形式化证明审计、数学软件正确性
- **为什么值得关注**：揭示内核接受作为唯一质量指标的严重不足，提供可复现审计方法论

### AI / LLM 技术（11 项）

#### 11. GitOfThoughts：版本控制的 Agent 推理与记忆
- **地址**：https://arxiv.org/abs/2606.14470
- **来源**：arXiv cs.AI
- **领域**：AI / Agent 记忆架构
- **技术介绍**：LLM 推理是短暂的：思维链随上下文窗口消失，剪枝搜索分支不留记录，记忆缓冲区无法 diff/merge/审计。每个其他复杂软件过程（代码、基础设施、数据、实验）都版本控制；推理不是。GitOfThoughts 将 Agent 推理树存储为 git 仓库：每个评分思维是一个 commit，分数是 notes，结果是 tags，检索是"git log"。使推理可重放、可审计、可跨 Agent 合并，几乎零工程成本。核心发现：记忆（任何基底）是否提高准确性？在 5 种基底（无、markdown、向量、图、git）、2 个基准、2 个模型尺度上，对新问题的答案是"否"。记忆仅在"可复制阈值"以上有效：检索案例与当前问题近乎重复（相似度 >~0.8）时准确性急剧跳跃；低于此阈值，什么都不会发生。增益是答案检索而非方法迁移。git 基底的价值在于审计性、来源和可合并性——在准确性平局下。
- **应用场景**：Agent 推理审计、跨 Agent 知识合并、推理过程版本控制
- **为什么值得关注**：首次将版本控制引入 Agent 推理，同时以严格可复现方式证伪了"记忆提高准确性"的假设

#### 12. LLM Agent 对 GNN 工具盲目服从
- **地址**：https://arxiv.org/abs/2606.14476
- **来源**：arXiv cs.AI / TMLR under review
- **领域**：AI / Agent 工具使用
- **技术介绍**：越来越多工作将 GNN 作为可调用工具装备 LLM Agent，假设 Agent 对何时和多大程度上依赖工具有判断力。直接测试这个假设：将冻结 GNN 暴露给 ReAct 风格 LLM Agent 作为显式工具，在文本属性图节点分类上测量 Agent 是使用工具还是仅仅服从。发现 Agent 不行使判断：预测与原始 GNN 一致 97.6-99.2%（5 seeds），坍缩为 GNN 鹦鹉——全盘采用工具输出并绕过自身推理。扫描 backbone 能力（Qwen2.5 0.5B-7B），服从不是弱模型伪影：在能调用工具的模型中，一致性随能力上升（1.5B 到 7B 从 0.60 到 0.98）。关键：服从成本不随能力增长而缩小，在替代方案出现的地方反而增长。简单选择性调用门恢复约一半高同质性差距但无净全局增益，持外估计限制最佳可实现门在可用信息而非路由设计上。
- **应用场景**：Agent+工具系统评估、GNN 工具集成、选择性调用设计
- **为什么值得关注**：颠覆"Agent 在工具之上增加判断力"的假设，对 Agent+工具评估方法论有根本影响

#### 13. 从 Chatbot 到 Digital Colleague：持久自主 AI 范式转变
- **地址**：https://arxiv.org/abs/2606.14502
- **来源**：arXiv cs.AI
- **领域**：AI / Agent 架构综述
- **技术介绍**：LLM 正在从对话生成器转变为集成 AI 系统，具备推理、行动、记忆和自我改进能力。概念化为从 Chatbot 到 Digital Colleague 的转变：从对话答案到持久工作。沿两个紧密耦合维度组织：(1) 认知核心层——从 Chatbot 时代的"快思考"（next-token 预测驱动）到 Thinking LLM（推理时计算、CoT 推理、反思）；(2) 系统架构层——从孤立对话到持久工作空间。
- **应用场景**：AI 系统架构设计、持久 Agent 开发、企业 AI 部署
- **为什么值得关注**：系统性地概念化 Agent AI 从对话工具到持久工作伙伴的范式转变

#### 14. Communication Policy Evolution：主动 LLM Agent 通信策略进化
- **地址**：https://arxiv.org/abs/2606.14314

#### 1. HarnessX：可组合自适应进化 Agent Harness 锻造厂
- **地址**：https://arxiv.org/abs/2606.14249
- **项目介绍**（300字）：
HarnessX 提出了一种全新的 Agent Harness 工程范式。当前 Agent 性能严重依赖于 Harness——包括提示、工具、记忆和控制流的运行时组合——但现有 Harness 主要是手工静态构建的，每次新模型或任务都需要定制支架。HarnessX 将 Harness 工程从手工艺升级为工业化锻造厂。

核心架构包含三个层次：(1) 类型化 Harness 原语通过替换代数组装，提供组合性基础；(2) AEGIS——轨迹驱动的多 Agent 进化引擎，基于符号适配与 RL 之间的操作镜像，实现 Harness 的自适应进化；(3) 闭环机制将执行轨迹同时转化为 Harness 更新和模型训练信号。

技术架构的创新在于"操作镜像"概念：将符号层面的 Harness 修改与 RL 层面的策略更新建立对应关系，使两者可以相互验证和增强。这解决了传统方法中 Harness 优化和模型训练割裂的问题。

在 5 个基准上的实验结果令人瞩目：ALFWorld、GAIA、WebShop、tau^3-Bench 和 SWE-bench Verified 上平均增益 +14.5%，最高 +44.0%。关键发现是增益与基线性能负相关——基线越低增益越大，表明 Harness 进化对弱模型尤其有价值。

优势：(1) 证明 Agent 进步可以来自 Harness 而非仅模型升级；(2) 提供系统化 Harness 工程方法论；(3) 闭环设计使 Harness 和模型可以共同进化。

局限：(1) 需要大量轨迹数据训练进化引擎；(2) 操作镜像的通用性需要更多验证；(3) 计算成本较高。

未来发展潜力：HarnessX 可能成为 Agent 工程的标准基础设施，类似于模型训练中的优化器。随着 Agent 生态成熟，Harness 进化可能比模型升级更具成本效益。

个人技术理解：HarnessX 的核心洞察是 Agent 性能瓶颈不在模型而在 Harness 设计。这类似于编译器优化——同样的代码，不同的编译器可以产生显著不同的性能。HarnessX 将这一理念引入 Agent 领域，可能改变我们对 Agent 能力边界的理解。

#### 2. SkillAudit：无 ground-truth 技能进化
- **地址**：https://arxiv.org/abs/2606.14239
- **项目介绍**（300字）：
SkillAudit 解决 Agent 技能进化中最大的实际瓶颈——缺乏 ground-truth 反馈。部署后技能面临边缘情况、API 变化和环境约束，但这些只有通过实际使用才能发现。现有进化方法依赖验证分数、隐藏测试结果或环境奖励，而实际场景通常只有任务描述和工作空间数据。

核心技术是配对轨迹审计：同一任务在有和没有候选技能下分别执行，隔离技能对 Agent 行为的影响而无需外部标签。这类似于科学实验中的对照组设计。

PACE（过程对齐对比评估）将轨迹差异映射到技能文档特定段落的诊断信号，使进化具有精确的靶向性。结构验证器从任务规范编译后固定，检查任务约束并回滚有害更新，提供安全保证。

两个进化管道：Refine 去除噪声引导（适合 broadly useful 但 noisy 的技能），Repair 替换冲突段落（适合与任务冲突的技能）。

在 89 个容器化任务（8 个专业领域）上，SkillAudit 达到 73.9% 平均任务奖励，显著超越无技能 Agent（40.9%）和静态专家技能（56.7%）。

优势：(1) 无需 ground-truth 反馈，贴近实际部署场景；(2) 配对审计提供因果推断；(3) 结构验证器提供安全保证。

局限：(1) 需要双倍执行成本（有/无技能各一次）；(2) 对高度依赖外部奖励的任务效果有限。

个人技术理解：SkillAudit 的核心贡献是将科学实验方法论引入 Agent 技能进化。配对对照设计使无监督进化成为可能，这类似于 A/B 测试在产品设计中的应用。

### 学术论文

#### 3. Parallel-Synthesis：Agent 并行分支的 KV 缓存直接合成
- **地址**：https://arxiv.org/abs/2606.14672
- **论文介绍**（300字）：
Parallel-Synthesis 提出了一种革命性的 Agent 工作流合成方法。现代 Agent 系统中，并行分支独立探索子任务后需要合成。现有系统通过文本拼接合并分支输出，这丢弃了并行结构并产生冗余 prefill 计算。

Parallel-Synthesis 让合成器直接消费并行 worker Agent 产生的 KV 缓存。技术包含两个关键组件：(1) 缓存映射器——校准独立生成的分支缓存，解决不同分支缓存之间的对齐问题；(2) 微调合成器适配器——使模型能够从非顺序缓存接口生成。

训练策略包括：暴露合成器到并行缓存上下文、教授跨缓存分支聚合、从标准文本拼接合成蒸馏推理行为。

在 9 个下游数据集上，7 个匹配或超越文本合成，TTFT 减少 2.5x-11x。

优势：(1) 大幅减少合成延迟；(2) 保留并行结构信息；(3) 即插即用，无需修改现有 Agent 框架。

局限：(1) 需要微调合成器适配器；(2) 缓存对齐可能引入误差。

个人技术理解：Parallel-Synthesis 的核心洞察是将 KV 缓存从计算副产品变为跨 Agent 共享接口。这类似于分布式系统中的共享内存——不同进程直接共享数据而非通过序列化通信。这一范式可能彻底改变多 Agent 协作的效率边界。

#### 4. GitOfThoughts：版本控制的 Agent 推理与记忆
- **地址**：https://arxiv.org/abs/2606.14470
- **论文介绍**（300字）：
GitOfThoughts 将版本控制引入 Agent 推理。LLM 推理是短暂的——思维链随上下文窗口消失，剪枝搜索分支不留记录。每个其他复杂软件过程都版本控制，唯独推理不是。

GitOfThoughts 将 Agent 推理树存储为 git 仓库：每个评分思维是一个 commit，分数是 notes，结果是 tags，检索是"git log"。使推理可重放、可审计、可跨 Agent 合并，几乎零工程成本。

论文的核心发现同样重要：记忆（任何基底）是否提高准确性？在 5 种基底（无、markdown、向量、图、git）、2 个基准、2 个模型尺度上，对新问题的答案是"否"。记忆仅在"可复制阈值"以上有效——检索案例与当前问题近乎重复时准确性急剧跳跃。增益是答案检索而非方法迁移。git 基底的价值在于审计性、来源和可合并性——在准确性平局下。

论文还记录了一个撤回的结果和反驳的假设，以建模他们对自己的评估标准。

优势：(1) 推理可审计可合并；(2) 零工程成本利用 git 生态；(3) 严格可复现的评估方法论。

局限：(1) 不提高推理准确性；(2) git 操作对非技术用户不友好。

个人技术理解：GitOfThoughts 的最大价值不是技术实现而是方法论贡献——它展示了如何以科学严谨的态度评估 Agent 记忆系统。撤回结果和反驳假设的做法在 AI 研究中极为罕见，值得效仿。

#### 5. RefGRPO：Agentic RL 免费校准奖励
- **地址**：https://arxiv.org/abs/2606.14211
- **论文介绍**（300字）：
RefGRPO 解决 Agentic RL 中一个被忽视但关键的问题——Agent 反思校准。LLM Agent 在观察环境反馈后仍倾向于错误评估自身输出，标准 RL 由于信用分配不匹配几乎无帮助。

RefGRPO 用两个关键要素增强标准 RL：(1) 免费校准奖励——通过对比 Agent 自身反思与实际结果计算，无需额外奖励模型、LLM judge 或外部标注；(2) 动态系数调度。

在 Text-to-SQL 5 个基准上同时提升反思校准（过度不自信率 44.4%→7.7%）和任务准确率（75.1%→76.5%）。校准后的反思使 Agent 成为基于环境反馈的自身验证器，支持更好的自我改进和测试时选择性预测。

优势：(1) 零额外成本；(2) 同时改善校准和准确性；(3) 使 Agent 成为自身验证器。

局限：(1) 主要在 Text-to-SQL 上验证；(2) 校准奖励的理论基础需要进一步研究。

个人技术理解：RefGRPO 的核心洞察是环境反馈本身就提供了校准信号——Agent 只需要学会利用它。这类似于人类学习中的"反思"能力，但用 RL 框架实现了自动化。
