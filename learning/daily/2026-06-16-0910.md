#### 1. HarnessX: 可组合、自适应、可进化的 Agent Harness 铸造厂
- **地址**: https://arxiv.org/abs/2606.14249
- **介绍（300字）**: HarnessX 代表了一种范式转变——将 Agent 性能提升的焦点从模型本身转移到运行时 harness。传统 harness 是手工编写、静态不变的：每个新模型或新任务都需要定制的脚手架，而执行过程中产生的丰富 trace 很少被系统性地蒸馏回改进。HarnessX 通过三个核心创新解决了这个问题：(1) 类型化 harness 原语通过替换代数组装，使 harness 成为可组合的模块；(2) AEGIS——一个基于 trace 的多 Agent 进化引擎，在符号自适应和 RL 之间建立操作镜像；(3) 闭环——将轨迹同时转化为 harness 更新和模型训练信号。在 ALFWorld、GAIA、WebShop、tau^3-Bench 和 SWE-bench Verified 五个基准上，HarnessX 实现了平均 +14.5% 的提升（最高 +44.0%），且在基线最低的领域增益最大。
- **技术架构**: 三层架构——原语层（类型化 harness 组件）、进化层（AEGIS 多 Agent 系统）、闭环层（轨迹→harness 更新+模型信号）。
- **核心算法**: 替换代数组合原语；AEGIS 通过 operational mirror 将符号自适应映射到 RL 更新；trace-driven 进化循环。
- **与现有技术对比**: 传统方法每个模型/任务需要手动调优 harness；HarnessX 自动进化 harness，且增益在基线最低处最大。
- **优势与局限**: 优势——不依赖模型升级即可大幅提升性能；局限——需要大量 trace 数据进行进化。
- **未来发展潜力**: Harness 即代码的范式可能成为 Agent 工程的标准实践。

#### 2. SkillAudit: 无真值反馈的技能进化
- **地址**: https://arxiv.org/abs/2606.14239
- **介绍（300字）**: SkillAudit 解决了一个实际但被忽视的问题：部署后的 Agent 技能如何进化？边缘情况、API 变更和部署约束只有在实际使用中才会显现，但现有方法依赖特权反馈（保留验证分数、隐藏测试结果、环境奖励），而这些信号在实际场景中往往不可用。SkillAudit 的核心创新是"配对轨迹审计"——同一任务在有/无候选技能下分别执行，隔离出技能对 Agent 行为的影响，无需外部标签。PACE（过程对齐对比评估）将轨迹差异映射为诊断信号，关联到技能文档的具体段落。结构验证器从任务规范编译一次后固定，检查任务约束并回滚有害更新。在 89 个容器化任务、8 个专业领域上，SkillAudit 达到 73.9% 平均任务奖励，远超无技能（40.9%）和静态专家技能（56.7%）。
- **技术架构**: 四组件——配对执行引擎、PACE 评估器集群、结构验证器、双管道编辑器（Refine/Repair）。
- **核心算法**: 配对轨迹对比隔离技能影响；PACE 将行为差异映射为文档段落的诊断信号；结构验证器编译时生成。
- **与现有技术对比**: 现有方法需要 ground truth 或 reward signal；SkillAudit 完全无需。
- **优势与局限**: 优势——实用性强，适用于真实部署场景；局限——依赖任务规范的质量。
- **未来发展潜力**: 技能自进化可能成为 Agent 技能生态的核心能力。

#### 3. GitOfThoughts: 版本控制推理与 Agent 记忆
- **地址**: https://arxiv.org/abs/2606.14470
- **介绍（250字）**: LLM 推理是短暂的——思维链随上下文窗口消失，剪枝的搜索分支不留记录，记忆缓冲区无法 diff/merge/审计。GitOfThoughts 将 Agent 推理树存储为 git 仓库：每个评分思维是 commit，分数是 notes，结果是 tags，检索是"git log"。在 5 种存储介质、2 个基准、2 个模型规模的系统评估中，发现记忆格式对新颖问题准确性无可靠帮助，仅在"可复制阈值"（相似度 >~0.8）以上才有收益。git 作为基质的价值在于可审计性、溯源性和可合并性，而非准确性提升。
- **技术架构**: git-native 推理存储，commit=思维，notes=分数，tags=结果。
- **与现有技术对比**: 传统记忆系统（向量/图）追求准确性提升；GitOfThoughts 追求可审计性。
- **优势与局限**: 优势——零工程成本实现推理可审计；局限——不提升准确性。

### 学术论文

#### 4. StreamMemBench: Agent 记忆流式评估基准
- **地址**: https://arxiv.org/abs/2606.14571
- **介绍（250字）**: 个人 Agent 记忆的核心作用是将存储信息和先前交互转化为面向未来的辅助。现有记忆基准通常孤立地测试对话回忆或任务改进，而 StreamMemBench 构建了两步任务序列（证据使用 + 后续复用），基于 EgoLife 第一人称流。4 个指标诊断证据回忆、初始使用、反馈整合、后续复用。8 个记忆系统 × 2 个基座模型实验表明，当前系统常无法使用观察到的证据或将反馈转化为可靠后续行为。
- **技术架构**: 两步任务序列 + 四维评估指标。
- **关键发现**: 证据存储 ≠ 证据使用；反馈整合 ≠ 后续行为改进。
- **优势与局限**: 优势——首次系统评估流式记忆到后续辅助的转化；局限——依赖 EgoLife 数据集。

#### 5. VISTA: 视图一致性自验证训练用于 GUI Grounding
- **地址**: https://arxiv.org/abs/2606.14579
- **介绍（250字）**: GRPO 用于 GUI Grounding 时，从单一截图采样 rollout，困难实例全失败或简单实例全成功，无法产生有用的相对优势。VISTA 从同一 GUI 的多个目标保持视图构建对比组，每个视图通过裁剪保持目标元素可见并精确重映射边界框。自验证跨视图锚点：oracle 答案用优势加权损失优化，仅在模型产生最大奖励 rollout 时激活。ScreenSpot-Pro 上 Qwen3-VL 4B/8B/30B-A3B 从 55.5/52.7/53.7 提升至 63.4/65.8/67.0。
- **技术架构**: 多视图 GRPO + 自验证跨视图锚点。
- **核心创新**: 用几何变化但语义等价的视图构建对比组，解决 GRPO 在 GUI grounding 上的退化问题。
- **优势与局限**: 优势——显著提升 GUI grounding 准确性；局限——需要生成多个视图，增加计算开销。

#### 6. RefGRPO: Agent RL 中的免费校准奖励
- **地址**: https://arxiv.org/abs/2606.14211
- **介绍（250字）**: 发现"反射差距"——LLM Agent 在观察环境反馈后仍会错误评估自身输出，标准 RL 因信用分配不匹配几乎无法帮助。RefGRPO 用两个关键要素增强标准 RL：(1) 免费校准奖励——对比 Agent 自身反思与实际结果，无需额外奖励模型/LLM Judge/外部标注；(2) 动态系数调度。Text-to-SQL 上反射校准率 44.4%→7.7%，任务准确率 75.1%→76.5%。校准后的反思使 Agent 成为基于环境反馈的自验证器。
- **技术架构**: RL + 免费校准奖励 + 动态调度。
- **核心创新**: 无需外部标注即可校准 Agent 自我评估。
- **优势与局限**: 优势——零额外成本；局限——仅在环境反馈可用时有效。

## 技术趋势

1. **Agent Harness 进化成为独立研究方向**: HarnessX 和 SkillAudit 同时出现，表明社区开始将 Agent harness 视为可独立进化的工程组件，而非模型的附属品。Harness 进化可能在某些场景下比模型升级带来更大的性能提升。

2. **Agent 记忆从"准确性"转向"可审计性"**: GitOfThoughts 的发现（记忆格式不提升准确性但提供可审计性）与 StreamMemBench（流式记忆到后续辅助的转化失败）共同指向：Agent 记忆的核心价值可能不在于提升准确性，而在于提供可审计、可追溯、可合并的推理历史。

3. **本地编码模型进入实用阶段**: HN 上 653 points 的讨论表明，Qwen 3.6 35B-A3B 配合 Pi harness + llama.cpp 已在 Mac Studio 等消费级硬件上实现 5x 编码加速，且完全离线。这标志着本地编码模型从"玩具"走向"实用工具"的转折点。

## 新领域收获

1. **键寻址 P2P 网络（Iroh 1.0）**: 用密钥替代 IP 地址作为网络寻址基础，支持 QUIC 多路径、NAT 穿透、本地发现，以及 BLE/LoRa/Tor 等自定义传输。30 天内 2 亿+ 端点创建，已用于视频流、LLM 训练、Agent 通信等场景。这是一个可能重塑 P2P 网络基础设施的项目。

2. **对抗 AI 编码技能退化的间隔重复学习（fata.dev）**: AI 编码时代工程师技能退化的新挑战。fata 用科学间隔重复帮助开发者保持全栈基础技能，10,000+ 开发者已使用。这是一个新兴的 Developer Tools 方向。

3. **多维度战略推理评估（Poker Arena）**: 将战略推理分解为 9 个可解释维度（下注规模校准、位置意识等），发现标量排行榜系统性地错误排名。多轴评估可能成为 Agent 评估的新标准。
