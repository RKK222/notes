### AI Testing / Software Testing

1. **Looking Is Not Picking: An Attention-Segment Account of Tool-Selection Failures in LLM Agents**
   - 地址：https://arxiv.org/abs/2606.16364
   - 来源：arXiv
   - 领域：AI Testing
   - 技术介绍：LLM Agent 工具选择失败的传统解释是"工具太多看不到"，但本文通过注意力段分析发现相反结论——模型在 80% 的情况下注意力集中在正确工具上（vs 21% 随机），但仍然选错。失败根源在决策读取端而非工具集拥挤。通过三种方式验证：(1) 提示修复仅恢复≤23% 失败，而读取端干预恢复 59-91%；(2) 两种表示不变性干预恢复几乎相同失败（Jaccard 0.865）；(3) 训练-free gold-free 选择器关闭大部分差距（+11.9 点）。覆盖 3-32B 模型。
   - 应用场景：Agent 工具调用可靠性提升、BFCL 工具选择基准改进
   - 为什么值得关注：颠覆"拥挤工具集导致失败"的直觉假设，将诊断焦点从 Harness 设计转移到模型决策机制

2. **AgentFairBench: Do LLM Agents Discriminate When They Act?**
   - 地址：https://arxiv.org/abs/2606.16723
   - 来源：arXiv
   - 领域：AI Testing
   - 技术介绍：现有 LLM 公平性评估基于答案评分，但 Agent 越来越多地采取实际行动（筛选申请人、推荐信贷、分诊患者）。AgentFairBench 是首个多领域行为公平性基准，覆盖招聘、借贷、医疗分诊三个监管锚定领域。使用反事实匹配集（仅改变姓名编码的种族×性别信号），在四种 Agent 脚手架（直接、CoT、多Agent 协商、工具增强）下评估。NumPy-only harness 计算反事实翻转率、平均绝对分数差异、行动率差异和工具调用差异，每个模型仅需个位数美元。 pilot（864 决策）发现方法学教训：六组分数扩散对比双次运行噪声差异会通过统计 arity 高估差异约 2.4 倍。
   - 应用场景：Agent 行为公平性审计、监管合规验证
   - 为什么值得关注：首次将公平性评估从静态答案推进到动态行为，揭示统计 arity 陷阱

3. **Bayesian Inference and Decision Audits for Public Archives of Frontier AI Evaluations**
   - 地址：https://arxiv.org/abs/2606.17005
   - 来源：arXiv
   - 领域：LLM Evaluation
   - 技术介绍：公共 AI 评估通常被读作终端排行榜，但底层证据是受报告规则、基准修订和缺失值影响的时序数据。本文使用 LiveBench 和 Open LLM Leaderboard v2 的重复公共档案作为主要纵向记录，将评估构建为贝叶斯推理问题：在固定报告惯例下，一个终端示例兼容两个前终端历史，在相同终端尾模型下达到天花板 0.05 内的时间分别为 23.03 或 75.13。提出档案-裁决协议重构公共评估历史，隔离验证时序边界，证伪不支持的前沿声明。
   - 应用场景：AI 评估纵向审计、模型能力趋势分析
   - 为什么值得关注：将 AI 评估从静态排行榜推进到动态纵向审计，揭示报告规则对评估结果的系统性影响

4. **Scaling LLM Reasoning from Minimal Labels: A Semi-Supervised Framework with a Lightweight Verifier**
   - 地址：https://arxiv.org/abs/2606.16811
   - 来源：arXiv
   - 领域：LLM Evaluation
   - 技术介绍：现有伪中间推理生成方法依赖大量正确标注答案来评估推理质量。本文提出半监督框架，将推理验证本身转化为数据创建机制。在少量标注样本上训练轻量级推理正确性分类器，判断 LLM 生成的中间推理轨迹是否有效。熵置信度阈值过滤不可靠样本，剩余高置信度轨迹用于微调模型。在 Orca-Math 和 GQA 上，达到使用 10-15 倍更多标注数据相当的准确性。
   - 应用场景：低资源推理能力评估、自主推理系统构建
   - 为什么值得关注：用轻量推理验证替代昂贵的答案级监督，为自主推理系统提供可扩展路径

9. **METIS: Many-Shot Model Merging with Loss-Gap Balancing**
   - 地址：https://arxiv.org/abs/2606.16501
   - 来源：arXiv (ICML 2026)
   - 领域：AI/LLM
   - 技术介绍：模型合并是构建多任务 LLM 的实用后训练策略，但现有方法依赖 post-hoc 合并（训练后仅合并一次），常因任务干扰导致信息擦除。本文证明用迭代 many-shot 合并协议替代 post-hoc 合并可有效提升多任务性能。METIS 通过任务级 loss-gap 加权和共识掩码解决信息擦除问题，在表现最差的任务上显著提升性能。
   - 应用场景：多任务模型合并、开源模型生态建设
   - 为什么值得关注：ICML 2026 录用，挑战"一次合并足够"的假设

10. **Bidirectional Provability Fingerprinting (BPF) for Autoformalization Faithfulness**
    - 地址：https://arxiv.org/abs/2606.16541
    - 来源：arXiv
    - 领域：AI/LLM
    - 技术介绍：自动形式化（自然语言数学→形式化证明助手）的瓶颈不是翻译流畅性而是忠实度——形式化语句可以通过类型检查并可证明，但仍编码与源意图不同的定理。BPF 通过正向和反向结果邻域特征化每个候选，与来自自然语言语句的探针匹配。包含四个新组件：反事实探针生成、等价谱（连续忠实度分数）、自适应探针预算分配、忠实度引导解码。在 2,183 个 NL/Lean4 对上，BPF+CPG 以 3.0% 误报率检测 89.6% 漂移形式化（vs 类型检查 41.2%、LLM-judge 63.3%）。FGD 将最先进自动形式化器发出漂移语句的比率降低 47%。
    - 应用场景：自动形式化质量控制、数学 AI 验证
    - 为什么值得关注：首次系统解决自动形式化忠实度问题，PAC-忠实度理论保证

11. **PACT: Plan, Align, Commit, Think — RL+SLM Hybrid Planning**
    - 地址：https://arxiv.org/abs/2606.16995
    - 来源：arXiv (LM4Plan Workshop ICML 2026)
    - 领域：AI/LLM
    - 技术介绍：RL 策略在不熟悉环境中常因缺乏显式 deliberation 而退化。PACT 结合快速反应式 RL 策略和慢速 deliberative SLM 规划器。SLM 异步生成和验证候选行动计划，一旦计划通过模拟验证（安全、可行、完整），直接执行绕过 RL 策略，无需重新训练。在三个 FrozenLake 配置上超越所有基线，仅依赖 2B 参数 SLM。
    - 应用场景：RL 策略在未知环境中的鲁棒性提升
    - 为什么值得关注：deliberative planning 和 reactive execution 协同优于各自独立

13. **MA-SBI: Misspecification-Aware Simulation-Based Inference via Side-Channel Guidance**
    - 地址：https://arxiv.org/abs/2606.16923
    - 来源：arXiv
    - 领域：AI/LLM
    - 技术介绍：SBI 中的模拟器误设（模拟与真实观察之间的不匹配）常阻碍潜在参数推断。MA-SBI 是无校准框架，将非结构化侧信道信息（制度标签、指令文本、政策公报）转化为后验校正。主要定理通过误设与侧信道之间的互信息界定可达偏差减少。在 hide-the-calibration 基准上，仅用文本匹配 oracle 后验（TOST 等价），而 RoPE 用更多数据也不匹配。
    - 应用场景：流行病学数据推断、认知科学数据分析
    - 为什么值得关注：将侧信道文本信息转化为后验校正，无需参数真值配对

14. **RAID: Retrieval-Augmented Iterative Diffusion for True Cold-Start Forecasting**

15. **Tensor-Coord: Algebraic Decomposition for Conflict-Free Multi-Agent LLM Planning**
    - 地址：https://arxiv.org/abs/2606.16478
    - 来源：arXiv
    - 领域：AI/LLM
    - 技术介绍：LLM 在多 Agent 规划中因独立生成计划导致协调失败（空间碰撞、资源竞争、时间死锁）。Tensor-Coord 将 N 个 Agent 的联合计划表示为三阶张量，使用 CP 和 Tucker 分解识别潜在协调结构。最小 ε-近似 CP 秩 R* 定义可计算协调复杂度度量。实验显示 2-Agent 情况 100% 在 1.4 次迭代内收敛到无冲突计划。
    - 应用场景：多机器人协调、多 Agent 任务规划
    - 为什么值得关注：首次用多线性代数框架解决多 Agent LLM 规划冲突

16. **AdaSTORM: Scaling LLM Reasoning on Dynamic Graphs via Adaptive Spatio-Temporal Multi-Agent Collaboration**
    - 地址：https://arxiv.org/abs/2606.16328
    - 来源：arXiv
    - 领域：AI/LLM
    - 技术介绍：LLM 在动态图推理中面临扩展瓶颈（仅能处理数十节点）。AdaSTORM 将大规模动态图推理重构为两阶段：(1) 自适应分区，将大图划分为匹配模型推理能力的子区域；(2) 协作推理，将图分区拓扑与时空解耦多 Agent 架构对齐。成功扩展到千节点图，准确率超 90%，显著超越七个竞争基线。
    - 应用场景：社交网络分析、交通网络推理、知识图谱推理
    - 为什么值得关注：首个针对动态图推理的多 Agent 框架

1. **Looking Is Not Picking: Attention-Segment Tool Selection Diagnosis**
   - 地址：https://arxiv.org/abs/2606.16364
   - 项目介绍（200-400字）：
     本文对 LLM Agent 工具选择失败进行了开创性的因果诊断。传统假设认为工具选择失败源于"工具集太拥挤导致模型看不到正确工具"，但本文通过注意力段分析彻底推翻这一假设。研究者在真实 BFCL 失败案例上发现：模型在 80% 的情况下注意力集中在正确工具上（vs 21% 随机基线），黄金工具在仅 10% 情况下是被欠关注的段。这意味着失败发生在决策读取端而非工具集设计端。
     
     研究通过三种独立方法验证这一发现：(1) 提示修复（重新排序或复制黄金工具）仅恢复≤23% 失败，而读取端干预恢复 59-91%；(2) 两种表示不变性干预——加性注意力 logit 偏置和残差流引导向量——恢复几乎相同的失败（任务 Jaccard 0.865 合并，0.79-0.91 每任务），证明瓶颈局部化在读取端且独立于被操作的表示；(3) 训练-free gold-free 选择器：每段注意力关闭大部分 gold-free vs oracle 差距（BFCL +11.9 点合并函数名选择 vs +17.9 点 oracle 头顶空间，Seal-Tools +14.9 点）。
     
     因果注意力偏置剂量-响应在 10 个掩码尊重模型上是双向且单调的（3-32B 完整跨度），而完整的 0.5-32B 跨度仅携带相关诊断。可部署选择器在 5 个单轮模型上评估，尚未转移到多轮循环。
   - 技术架构：注意力段分析框架 + 因果干预实验 + 训练-free 选择器
   - 核心算法：每段注意力 argmax 诊断 + 加性注意力 logit 偏置 + 残差流引导向量
   - 关键实现机制：通过注意力段级别分析定位工具选择失败的确切位置
   - 与现有技术对比：传统方法假设工具集拥挤是根因，本文证明根因在决策读取端
   - 优势与局限：优势是因果诊断精确、干预效果显著；局限是尚未验证多轮循环场景
   - 未来发展潜力：可能成为 Agent 工具选择可靠性诊断的标准方法
   - 个人技术理解：这篇论文的核心价值在于方法论——通过注意力段分析实现精确因果定位，这种思路可以推广到 Agent 其他失败模式诊断

4. **BPF: Bidirectional Provability Fingerprinting for Autoformalization**
   - 地址：https://arxiv.org/abs/2606.16541
   - 项目介绍：
     自动形式化（自然语言数学→形式化证明助手）的瓶颈不是翻译流畅性而是忠实度。BPF 通过正向和反向结果邻域特征化每个候选形式化语句，与来自自然语言语句的探针匹配，认证忠实度。
     
     四个新组件：(1) 反事实探针生成 (CPG)——对比程序合成针对特定漂移方向的探针；(2) 等价谱——连续忠实度分数替代脆弱二元裁决；(3) 自适应探针预算分配 (APBA)——信息论预算路由器；(4) 忠实度引导解码 (FGD)——使用 BPF 信号作为自动形式化期间的奖励。
     
     证明漂移检测定理和 PAC-忠实度结果：在温和假设下，自然语言语句的等价类可从 O(log(1/δ)/ε) 探针学习。发布 DriftBench（2,183 个 NL/Lean4 对，受控漂移标签覆盖 mathlib4 六个子领域）。BPF+CPG 以 3.0% 误报率检测 89.6% 漂移形式化（vs 类型检查 41.2%、LLM-judge 63.3%）。FGD 将最先进自动形式化器发出漂移语句的比率降低 47%。
   - 技术架构：双向可证明性指纹 + 反事实探针 + 自适应预算 + 忠实度引导解码
   - 核心算法：正向/反向结果邻域匹配 + 对比探针生成 + PAC 学习保证
   - 关键实现机制：通过结果邻域特征化而非表面语法匹配认证忠实度
   - 与现有技术对比：从类型检查/LLM-judge 推进到因果忠实度认证
   - 优势与局限：优势是理论保证、高检测率、低误报；局限是依赖 mathlib4 生态
   - 未来发展潜力：可能成为自动形式化质量控制的标准方法
   - 个人技术理解：PAC-忠实度理论将忠实度认证从经验方法推进到可证明保证

5. **Trace-Economic Underwriting: AI Agent Risk Insurance**
   - 地址：https://arxiv.org/abs/2606.16465
   - 项目介绍：
     AI Agent 现在可以在运营系统中采取不可逆行动，但 Agent 造成的损失仍无法明确分配、定价或转移。提供商通常免责声明 consequential damages，用户留下未补偿损失，默认人类审查限制自动化效率增益。
     
     本文提出追踪经济承保，将工具使用轨迹映射到客户暴露和可索赔损失，用于定价、控制和风险转移。使用确定性经济标签而非 LLM judge。在追踪-损失测试台中，追踪经济定价将定价 MAE 从 $17.7K 降至 $569，移除累进交叉补贴。300 条轨迹专家审计接受 295 条标签不变。在 1,000 条真实 SWE-smith 轨迹上，追踪条件控制将 CVaR95 降低 72%。
     
     定理 1 给出有限样本范围条件。自动化在预期收益超过保费、控制成本和剩余风险时经济上可接受。
   - 技术架构：轨迹-损失映射 + 确定性经济标签 + 风险转移机制
   - 核心算法：工具使用轨迹到经济损失的映射 + CVaR 条件风险价值优化
   - 关键实现机制：将 Agent 行为轨迹转化为可定价、可保险的风险单元
   - 与现有技术对比：从 LLM judge 风险评估推进到确定性经济标签
   - 优势与局限：优势是专家审计通过率高、MAE 大幅降低；局限是仅覆盖工具使用轨迹
   - 未来发展潜力：可能成为 autonomous AI 部署的经济基础设施
   - 个人技术理解：将 AI Agent 风险与保险机制结合是一个全新的研究方向——Agent 安全不仅是技术问题也是经济问题

6. **Kairos: Native World Model Stack for Physical AI**
   - 地址：https://arxiv.org/abs/2606.16533
   - 项目介绍：
     世界模型从被动视觉生成器向物理 AI 基础运营基础设施转变。Kairos 是原生世界模型栈，围绕三个核心需求设计：(1) 从异构经验中学习世界知识；(2) 在长时间内维持持久状态；(3) 在真实部署约束下高效执行。
     
     Kairos 代表世界模型从学术研究到工程基础设施的关键一步。与之前主要关注视觉生成的世界模型不同，Kairos 强调知识获取、状态持久化和部署效率。作者团队包括 Kairos Team 和多位知名研究者（Fei Wang, Dacheng Tao, Xiaogang Wang 等）。
   - 技术架构：原生世界模型栈（知识获取 + 状态持久化 + 高效执行）
   - 核心算法：异构经验学习 + 持久状态维护 + 部署优化
   - 关键实现机制：将世界模型从生成模型重新定位为运营基础设施
   - 与现有技术对比：从被动视觉生成推进到主动知识获取和状态维护
   - 优势与局限：优势是面向真实部署；局限是具体实现细节待完整公开
   - 未来发展潜力：可能成为 Physical AI 的标准基础设施
   - 个人技术理解：世界模型从"生成逼真视频"到"维持世界状态"的范式转变是 Physical AI 的关键前提

## 技术趋势
1. **Agent 工具选择从 Harness 设计到决策诊断**：Looking Is Not Picking 颠覆"拥挤工具集导致失败"假设，将诊断焦点从工具集设计转移到模型决策读取端。结合 ToolSense 和 GIST-CMTF，形成完整的工具选择诊断-验证-优化证据链。
2. **AI 评估从终端排行榜到纵向审计**：贝叶斯评估审计揭示终端排行榜的根本歧义性——同一终端结果兼容多个演化轨迹。结合 AgentFairBench 的 arity 匹配零假设方法论，AI 评估正从"谁更好"推进到"我们如何知道谁更好"。
3. **形式化验证从离散到连续**：TNODEV 将形式化验证从离散系统扩展到 Neural ODE 连续时间控制器，BPF 解决自动形式化忠实度问题。形式化方法正从数学证明扩展到安全关键控制系统。

## 新领域收获
1. **Agent 风险保险机制**：Trace-Economic Underwriting 首次将 AI Agent 风险与保险转移机制结合，将 Agent 安全从技术问题扩展为经济问题。追踪经济承保可能成为 autonomous AI 部署的基础设施。
2. **统计 arity 陷阱**：AgentFairBench 揭示公平性评估中的系统性高估问题——比较六组分数扩散与双次运行噪声差异会通过统计 arity 高估差异约 2.4 倍。这一发现可能影响所有多组公平性研究。
3. **Neural ODE 形式化验证**：TNODEV 是首个 sound 的 Neural ODE 形式化验证器，将形式化验证从离散系统扩展到连续时间神经控制器。为安全关键 Neural ODE 部署提供验证基础设施。
