### AI Testing / Software Testing（11个）

1. **All Smoke, No Alarm: Oracle Signals in Agent-Authored Test Code**
   - 地址：https://arxiv.org/abs/2606.18168
   - 来源：arXiv (cs.SE) / IEEE ICAIT 2026
   - 领域：AI Testing
   - 技术介绍：对86,156条Agent-authored测试补丁（来自33,596个PR，2,807个仓库，5个编码Agent）进行实证研究。发现80.2%的测试补丁包含弱或无显式oracle信号。虽然强oracle PR的原始合并率较低，但回归分析显示强oracle显著提高合并可能性（OR=1.28, p<0.001）。提出8类oracle信号语法分类法。
   - 应用场景：Agent生成代码质量评估、PR审查流程优化、测试质量门禁
   - 为什么值得关注：测试文件数量严重高估验证强度，从业者应采用oracle感知质量检查

2. **Coding Benchmarks Are Misaligned with Agentic Software Engineering**
   - 地址：https://arxiv.org/abs/2606.17799
   - 来源：arXiv (cs.SE)
   - 领域：Software Testing
   - 技术介绍：论证当前编码基准与Agentic软件工程不对齐。三个症状：(i)基准分数混淆模型与Harness其余部分；(ii)单一参考答案惩罚同等有效的替代方案；(iii)缺乏Harness组件级别信号使端到端分数难以迭代。编码Agent在实践中不是模型而是系统——模型、Harness、上下文、环境和反馈信号的组合。
   - 应用场景：Agent编码评估框架设计、基准方法论反思
   - 为什么值得关注：直击Agent评估方法论核心问题，任何 Harness 组件的变动幅度可与模型代际差距相当

3. **A Framework for Evaluating Agentic Skills at Scale**
   - 地址：https://arxiv.org/abs/2606.17819
   - 来源：arXiv (cs.SE)
   - 领域：AI Testing
   - 技术介绍：提出Agentic Skills规模化评估框架，让技能作者构建真实任务来严格评估技能最关键方面。在500个真实世界技能上应用，生成1,000个任务，覆盖19种Agent-模型配置（专有和开源）。发现模型在遵循技能编码指令方面差异巨大，导致性能增益显著不同。技能访问显著改变模型行为。
   - 应用场景：Agent技能质量评估、技能市场标准化、模型-技能匹配
   - 为什么值得关注：首个可复用的个体技能评估方法论，为Agent技能生态提供质量基础设施

4. **ReproRepo: Scaling Reproducibility Audits with GitHub Repository Issues**
   - 地址：https://arxiv.org/abs/2606.18237
   - 来源：arXiv (cs.CL)
   - 领域：Software Testing

   - 应用场景：科研可复现性自动化审计、代码质量评估
   - 为什么值得关注：可扩展的复现性评估框架，利用真实世界问题作为监督信号

5. **StepGuard: Guarding Web Navigation via Single-Step Calibration**
   - 地址：https://arxiv.org/abs/2606.17871

### AI / LLM 技术（13个）

12. **Variable-Width Transformers (XL-former)**
    - 地址：https://arxiv.org/abs/2606.18246
    - 来源：arXiv (cs.CL)
    - 领域：LLM架构
    - 技术介绍：提出×形Transformer架构，保持早期和晚期层较宽而中间层较窄，使用无参数残差调整机制。在200M-2B参数（dense）和3B参数（MoE）解码器语言模型上持续优于参数匹配的均匀基线。平均层宽减少使总FLOPs降低22%，KV缓存内存和I/O成本降低15%。分析显示瓶颈结构在残差流中产生定性不同的表示。
    - 应用场景：高效LLM架构设计、资源优化推理
    - 为什么值得关注：非均匀宽度分配实现更资源最优的模型扩展，挑战均匀架构假设

13. **Small Initialization Matters for Large Language Models**
    - 地址：https://arxiv.org/abs/2606.17945
    - 来源：arXiv (cs.AI)
    - 领域：LLM训练
    - 技术介绍：揭示参数初始化是训练和模型能力的基因级决定因素。减小初始化规模持续改善预训练，在推理 demanding 任务上增益最大。发现两个广泛使用的经验设置限制了小初始化的优势。小初始化驱动独特的发展轨迹：参数先凝聚为低复杂度结构，后扩展为更丰富表示——为"压缩=智能"提供具体形式。提出γ初始化规则：将初始化范围作为显式旋钮，默认使用小初始化。
    - 应用场景：LLM预训练初始化策略、推理能力增强
    - 为什么值得关注：几乎零成本的干预，跨模型规模改善预训练和推理能力

14. **ZPPO: Zone of Proximal Policy Optimization**
    - 地址：https://arxiv.org/abs/2606.18216
    - 来源：arXiv (cs.CL)

15. **d-OPSD: On-policy Self-distillation for Diffusion LLMs**

16. **ConSA: Controllable Sparsity in Hybrid Attention**
    - 地址：https://arxiv.org/abs/2606.18056
    - 来源：arXiv (cs.CL)
    - 领域：LLM架构
    - 技术介绍：学习最优FA/SWA分配，在用户指定稀疏度目标下。使用L0正则化学习二进制掩码，增强拉格朗日约束在层或KV-head粒度执行目标稀疏度。学习到的分配持续优于规则基线，KV-head级分配优于层级分配。学习模式将SWA放在底层，FA集中在中间层连续块中。

17. **How Inference Compute Shapes Frontier LLM Evaluation**

    - 来源：arXiv (cs.AI)

- 关键发现：两个广泛使用的经验设置（具体为weight decay和learning rate schedule的某种组合）实际上限制了小初始化的优势。修正这些设置后，小初始化的优势全面显现。
- 与现有技术对比：传统做法将初始化视为需要调优的超参数，本文证明初始化是决定模型发展轨迹的结构性因素。
- 优势与局限：优势是几乎零成本的性能提升。局限是需要同时调整相关超参数设置才能充分发挥优势。
- 未来发展潜力：可能改变LLM训练的标准实践——初始化从"调参细节"变为"架构设计"。
- 个人技术理解："压缩=智能"的实证——小初始化迫使模型先压缩信息到简单结构，再扩展为复杂表示，这与信息瓶颈理论一致。初始化不是超参数，是架构设计。

#### 5. How Inference Compute Shapes Frontier LLM Evaluation
- 地址：https://arxiv.org/abs/2606.17930
- 论文介绍：评估基准分数严重依赖推理计算量。固定预算评估越来越低估frontier能力——新模型在大预算下解锁更难任务并更可靠地解决。基准在哪些推理扩展方法最有帮助上存在差异：重复提交普遍改善性能，但更大token预算、外部反馈和并行尝试的价值因基准而异。
- 技术洞察：基准分数不是模型的固有属性，而是模型+推理计算的联合函数。同一模型在不同推理预算下可能表现出完全不同的能力等级。
- 关键发现：新模型在大预算下解锁更难任务——这意味着基准排名可能随推理预算变化而反转。固定预算排名可能误导模型选择。
- 与现有技术对比：传统评估假设模型能力是固定属性，本文证明能力是模型×推理计算的函数。
- 优势与局限：优势是揭示评估方法论的根本缺陷。局限是需要为每个模型-基准组合测试多个推理预算，评估成本大幅增加。
- 未来发展潜力：可能改变AI基准评估的标准实践——从"单次运行排名"到"推理预算-性能曲线"。
- 个人技术理解：推理计算不是模型的"加速器"，而是模型的"能力解锁器"。评估模型不是测"它有多强"，而是测"给它多少算力它有多强"。
