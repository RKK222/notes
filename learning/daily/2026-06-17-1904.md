### AI Testing / Software Testing

1. **PreAct: Computer-Using Agents that Get Faster on Repeated Tasks**
   - 地址：https://arxiv.org/abs/2606.17929
   - 来源：arXiv
   - 领域：Software Testing / CUA
   - 技术介绍：CUA Agent 每次执行任务都从零开始——重新读屏、重新推理每一步。PreAct 首次让 CUA Agent 在重复任务上变得更快。第一次成功执行后，PreAct 将运行编译为小型状态机程序（检查屏幕的状态 + 执行动作的转移），后续执行直接回放而非调用 LLM，速度提升 8.5-13x。回放不是盲目的：每一步 PreAct 检查屏幕是否与程序预期匹配，一旦出现偏差立即交还控制权给 Agent。编译的程序进入存储前必须通过独立评估器验证——从干净状态重跑确认任务真正完成，防止故障程序累积。
   - 应用场景：CUA Agent 重复任务加速、桌面/移动/Web 自动化
   - 为什么值得关注：首次实现 CUA Agent 的"经验积累"能力，从每次从零推理推进到编译复用，成本降低数量级

2. **Trust the Right Teacher: Quality-Aware Self-Distillation for GUI Grounding**
   - 地址：https://arxiv.org/abs/2606.18101
   - 来源：arXiv
   - 领域：AI Testing / GUI Grounding
   - 技术介绍：GUI Grounding 要求 VLM 从高分辨率截图中定位小目标元素并预测精确屏幕坐标。OPSD（On-policy Self-Distillation）是后训练的有效方法，但 naive OPSD 存在问题：当学生生成的前缀已偏离目标坐标时，教师信号质量退化，导致不可靠的监督。本文提出质量感知自蒸馏，包含两个互补组件：(1) 软正确性感知识门——检查教师当前坐标 token 预测在学生前缀下是否仍能完成为 ground-truth 框，否则降权；(2) 教师概率缩放——用教师置信度作为轻量因子校准监督强度。关键发现：单独任一组件都不提升性能，组合使用才一致改善。
   - 应用场景：GUI 自动化、CUA Agent 视觉定位、屏幕元素识别
   - 为什么值得关注：揭示了 OPSD 在坐标敏感任务中的核心缺陷，两个机制互补的发现对自蒸馏方法论有普遍意义

3. **EComAgentBench: Benchmarking Shopping Agents on Long-Horizon Tasks**

   - 应用场景：电商 Agent 评估、长程任务规划测试
   - 为什么值得关注：购物场景是 Agent 商业化的核心场景之一，该基准填补了长程电商 Agent 评估空白

10. **Agentic AI Framework for Mitigating Premature Diagnostic Handoff and Silent Hallucination**
    - 地址：https://arxiv.org/abs/2606.18068
    - 来源：arXiv
    - 领域：AI / 医疗安全
    - 技术介绍：医疗 Agent 的两个关键失败模式：过早诊断移交和静默临床幻觉。本文提出多 Agent 框架，用确定性编排约束替代 "LLM-as-a-judge" 路由。两个安全机制：(1) 神经符号状态跟踪门——强制执行 OLDCARTS 临床协议完整性（发作、位置、持续时间、特征、加重/缓解因素、辐射、时间、严重度），在所有必需维度收集完成前阻止诊断转移；(2) 认知不确定性量化门——计算 K=5 独立诊断样本的语义熵，在交付前拦截分歧输出。在 llama-3.1-70b 驱动的模拟患者 Agent 上 150 测试用例，完整架构诊断精度 49.3%，比无约束基线绝对提升 11.3pp。OLDCARTS 完整性与语义熵显著负相关（r=-0.181, p<0.05）。
    - 应用场景：医疗诊断 Agent、临床决策支持、AI 医疗安全
    - 为什么值得关注：首次用神经符号门控 + 认知不确定性量化解决医疗 Agent 两大核心失败模式，结构化信息收集降低诊断不确定性的发现有普遍意义

11. **STAR: SpatioTemporal Adaptive Reward Allocation for Text-to-Image RL**
    - 地址：https://arxiv.org/abs/2606.17979
    - 来源：arXiv
    - 领域：AI / 扩散模型 RL
    - 技术介绍：现有 Text-to-Image RL 后训练将最终图像奖励转换为单一标量优势并均匀应用于整个生成轨迹。但文生图天然有时空结构：不同去噪步骤负责不同生成阶段，决定文本对齐的内容通常只出现在图像局部。STAR 利用生成模型内部的文本-图像注意力，从用户真正关心的提示核心内容出发，构建跨去噪步骤和 rollout 动态变化的空间分配图，将相同组相对优势分配给更相关的潜在区域，几乎无额外计算开销。SD3.5-Medium 基线，GenEval 0.9759、OCR 0.9757、PickScore 23.60。
    - 应用场景：文生图模型 RL 后训练、图像生成质量优化
    - 为什么值得关注：首次将 RL 奖励分配从标量均匀推进到时空自适应，揭示扩散模型 RL 训练的粒度匹配问题

12. **Fixed-Point Reasoners: Stable and Adaptive Deep Looped Transformers (FPRM)**
    - 地址：https://arxiv.org/abs/2606.18206
    - 来源：arXiv
    - 领域：AI / Transformer 架构
    - 技术介绍：循环架构为组合推理任务提供逐步学习的归纳偏置，但循环深度导致信号传播问题（类似深层架构）。本文用 pre-norm 层和残差缩放解决信号传播，提出 FPRM——基于 Transformer 的不动点推理模型，用不动点收敛作为端到端停止机制。FPRM 能根据任务难度自适应调整计算量，在 Sudoku、Maze、状态跟踪、ARC-AGI 上有效。
    - 应用场景：组合推理、自适应计算分配、高效推理
    - 为什么值得关注：不动点收敛作为停止机制为循环 Transformer 提供理论保证，自适应计算是推理效率的关键方向

### GitHub 项目

#### 1. PreAct: Computer-Using Agents that Get Faster on Repeated Tasks
- **地址**：https://arxiv.org/abs/2606.17929
- **项目介绍**（200-400 字）：
PreAct 首次为 CUA（Computer-Using Agent）引入了"经验积累"能力。当前所有 CUA Agent 都有一个根本性缺陷：每次执行任务都从零开始——重新读屏、重新推理每一步点击、重新调用 LLM。PreAct 打破了这个模式。当 Agent 第一次成功完成某个任务时，PreAct 将完整的执行轨迹编译为一个小型状态机程序：状态检查屏幕内容，转移执行动作。后续执行直接回放这个状态机，跳过所有 LLM 调用，实现 8.5-13x 加速。

PreAct 的核心创新不在于编译本身，而在于质量控制机制：(1) 回放不是盲目的——每一步都检查屏幕是否与程序预期匹配，偏差时立即交还控制权给 Agent；(2) 编译的程序进入存储前必须通过独立评估器验证——从干净状态重跑确认任务真正完成，防止故障程序累积。实验覆盖移动端、桌面端和 Web 三个基准，存储时检查在所有三个基准上都带来 1.75-2.6 个任务的净增益。

- **技术架构**：编译层（轨迹→状态机）+ 回放层（状态匹配+动作执行）+ 验证层（独立评估器确认）
- **核心算法**：屏幕状态匹配 + 故障回退 + 存储时验证
- **关键实现机制**：状态机编译将 LLM 推理轨迹转换为确定性程序；回放时实时屏幕比对确保环境一致性
- **与现有技术对比**：传统 record-and-replay 缺乏质量控制，PreAct 的存储时验证和动态回退是本质区别
- **优势与局限**：优势是数量级加速 + 质量保证；局限是仅适用于重复任务，对新任务无帮助
- **未来发展潜力**：可与其他 Agent 优化（如技能编译、缓存）结合，构建完整的 Agent 经验积累体系
- **个人技术理解**：PreAct 揭示了一个重要范式——Agent 不应只是"每次从零推理"的黑盒，而应像人类一样从经验中学习。状态机编译是经验积累的最小可行方案，为 CUA Agent 的持续改进提供了工程路径。

#### 2. Trust the Right Teacher: Quality-Aware Self-Distillation for GUI Grounding
- **地址**：https://arxiv.org/abs/2606.18101
- **项目介绍**：
GUI Grounding 要求 VLM 从高分辨率截图中定位小目标元素并预测精确屏幕坐标。OPSD（On-policy Self-Distillation）是后训练的有效方法，但 naive OPSD 在坐标敏感任务中存在核心缺陷：当学生生成的前缀已偏离目标坐标时，教师信号质量退化，产生不可靠的监督。

本文提出质量感知自蒸馏，包含两个互补组件：(1) 软正确性感知识门——检查教师当前坐标 token 预测在学生前缀下是否仍能完成为 ground-truth 框，若不能则降权该教师信号；(2) 教师概率缩放——用教师置信度作为轻量因子校准监督强度。关键发现：单独任一组件都不提升性能，组合使用才一致改善。这揭示了两机制的互补角色：正确性门抑制不可靠信号，概率缩放校准剩余信号强度。六个 GUI 基准实验一致超越强基线。

- **技术架构**：OPSD 框架 + 正确性门控 + 概率缩放
- **核心算法**：坐标可完成性检查 + 教师置信度加权
- **关键实现机制**：在 OPSD 训练循环中插入正确性检查点，动态调整教师信号权重
- **与现有技术对比**：传统 OPSD 假设教师信号始终可靠，本文揭示并解决了信号退化问题
- **优势与局限**：优势是无需额外数据/标注即可提升；局限是两个组件必须组合使用，单独无效
- **未来发展潜力**：质量门控思想可推广到其他自蒸馏场景（如代码生成、数学推理）
- **个人技术理解**：本文的核心贡献是揭示了 OPSD 在坐标敏感任务中的信号退化问题。"教师不一定总是对的"这一洞察对自蒸馏方法论有普遍意义——质量感知可能是自蒸馏从"能用"到"好用"的关键分水岭。

#### 3. IsabeLLM: Automated Theorem Proving for Consensus Verification
- **地址**：https://arxiv.org/abs/2606.18098
- **项目介绍**：
形式化验证传统上 reserved for safety-critical systems，因为需要大量专业知识和人力投入。AI 定理证明正在改变这一局面。本文改进 IsabeLLM（Isabelle 自动定理证明工具），实现三大增强：(1) RAG 框架——从形式化证明库中检索相关引理和证明策略；(2) 错误追踪 + 反例生成——为 LLM 提供更丰富的调试上下文；(3) 兼容最新 Isabelle 和 Sledgehammer。在 Bitcoin PoW 共识验证任务上比较两版本性能。

- **技术架构**：LLM 推理引擎 + RAG 检索 + 反例生成 + Isabelle 集成
- **核心算法**：RAG 引导的证明搜索 + 反例驱动的错误修复
- **关键实现机制**：将形式化证明分解为可检索的子问题，用 LLM 生成证明草稿，用 Isabelle 验证
- **与现有技术对比**：传统 Isabelle 证明依赖人工编写策略，IsabeLLM 用 AI 自动化
- **优势与局限**：优势是降低形式化验证门槛；局限是复杂定理仍需人工干预
- **未来发展潜力**：RAG+反例生成的策略可推广到其他形式化验证场景（如智能合约、分布式系统）
- **个人技术理解**：形式化验证 × AI 是 AI 安全的基础设施方向。IsabeLLM 的 RAG 增强策略揭示了一个重要模式——形式化证明不是"从头推理"，而是"检索+适配"，这与 LLM 的检索增强生成高度契合。

### 学术论文

#### 1. STAR: SpatioTemporal Adaptive Reward Allocation for Text-to-Image RL
- **地址**：https://arxiv.org/abs/2606.17979
- **论文介绍**（200-400 字）：
现有 Text-to-Image RL 后训练方法有一个根本性假设错误：将最终图像奖励转换为单一标量优势，然后均匀应用于整个生成轨迹的所有 token 和所有空间区域。但文生图天然有时空结构——不同去噪步骤负责不同生成阶段（早期构图、中期细节、后期精炼），决定文本对齐的内容通常只出现在图像的局部区域。这种粒度不匹配导致策略更新无法聚焦于真正影响奖励的生成组件。

STAR 利用生成模型内部的文本-图像注意力机制，从用户真正关心的提示核心内容出发，构建跨去噪步骤和 rollout 动态变化的空间分配图。相同组相对优势被分配给更相关的潜在区域，几乎无额外计算开销。然后通过空间解析策略目标对这些区域施加更强的策略更新。SD3.5-Medium 基线，GenEval 0.9759、OCR 0.9757、PickScore 23.60。

- **技术架构**：文本-图像注意力提取 + 空间分配图构建 + 空间解析策略优化
- **核心算法**：基于注意力的空间相关性估计 + 组相对优势分配
- **关键实现机制**：从提示核心内容出发，追踪注意力流向，构建时空自适应的奖励分配图
- **与现有技术对比**：传统 RL 后训练用标量优势均匀更新，STAR 用空间自适应优势精准更新
- **优势与局限**：优势是几乎零额外计算开销；局限是依赖模型内部的注意力机制，黑盒模型不适用
- **未来发展潜力**：时空自适应奖励分配思想可推广到其他生成模型（视频、音频、3D）
- **个人技术理解**：STAR 揭示了一个普遍问题——RL 训练中的奖励粒度匹配。当奖励是粗粒度的（最终图像评分），但生成过程是细粒度的（逐 token 去噪），均匀更新必然低效。时空自适应是解决粒度不匹配的自然方案。

#### 2. Agentic AI Framework for Mitigating Premature Diagnostic Handoff and Silent Hallucination
- **地址**：https://arxiv.org/abs/2606.18068
- **论文介绍**：
医疗 Agent 的两个关键失败模式：(1) 过早诊断移交——Agent 在信息不足时过早给出诊断；(2) 静默临床幻觉——错误诊断在交付前未被检测。本文提出多 Agent 框架，用确定性编排约束替代 "LLM-as-a-judge" 路由。

两个核心安全机制：(1) 神经符号状态跟踪门——强制执行 OLDCARTS 临床协议完整性（发作、位置、持续时间、特征、加重/缓解因素、辐射、时间、严重度），在所有必需维度收集完成前阻止诊断转移；(2) 认知不确定性量化门——计算 K=5 独立诊断样本的语义熵，在交付前拦截分歧输出。

在 llama-3.1-70b 驱动的模拟患者 Agent 上 150 测试用例，完整架构诊断精度 49.3%，比无约束基线绝对提升 11.3pp。OLDCARTS 完整性与语义熵显著负相关（r=-0.181, p<0.05），揭示结构化信息收集降低诊断不确定性。

- **技术架构**：神经符号门控 + 认知不确定性量化 + 确定性编排
- **核心算法**：OLDCARTS 完整性检查 + 语义熵计算 + 分歧拦截
- **关键实现机制**：用临床协议作为结构化约束，用语义熵作为不确定性度量
- **与现有技术对比**：传统医疗 Agent 用 LLM-as-judge 做质量检查，本文用确定性门控替代
- **优势与局限**：优势是结构化约束比 LLM judge 更可靠；局限是依赖预定义的临床协议
- **未来发展潜力**：神经符号门控思想可推广到其他高风险 Agent 场景（金融、法律）
- **个人技术理解**：本文的核心贡献是揭示了一个重要原则——高风险场景中的质量检查不应依赖 LLM judge（因为 judge 本身也可能幻觉），而应依赖确定性约束。结构化信息收集降低不确定性的发现对 Agent 设计有普遍意义。

#### 3. Fixed-Point Reasoners: Stable and Adaptive Deep Looped Transformers (FPRM)
- **地址**：https://arxiv.org/abs/2606.18206
- **论文介绍**：
循环架构为组合推理任务（Sudoku、Maze、状态跟踪）提供逐步学习的归纳偏置，但循环深度导致信号传播问题——类似深层架构的梯度消失。本文用 pre-norm 层和残差缩放解决信号传播，提出 FPRM：基于 Transformer 的不动点推理模型，用不动点收敛作为端到端停止机制。

FPRM 的核心创新是不动点收敛作为停止条件：当连续两次循环的输出变化低于阈值时停止，使计算量自适应任务难度。简单任务少量循环即可收敛，复杂任务自动增加循环次数。在 Sudoku、Maze、状态跟踪、ARC-AGI 上有效。

- **技术架构**：循环 Transformer + pre-norm + 残差缩放 + 不动点收敛检测
- **核心算法**：不动点迭代 + 自适应停止
- **关键实现机制**：用输出变化量作为收敛指标，动态调整计算量
- **与现有技术对比**：传统循环架构用固定循环次数，FPRM 用自适应停止
- **优势与局限**：优势是计算效率自适应；局限是不动点收敛不一定保证正确性
- **未来发展潜力**：自适应计算是推理效率的核心方向，不动点机制可推广到其他推理任务
- **个人技术理解**：FPRM 揭示了循环架构的一个关键洞察——推理不是"越多层越好"，而是"收敛即足够"。不动点收敛作为停止机制为循环 Transformer 提供了理论保证，是自适应计算的重要进展。

## 技术趋势

1. **CUA Agent 从"每次推理"到"经验积累"**：PreAct 首次让 CUA Agent 通过状态机编译实现经验复用，8.5-13x 加速。这标志着 CUA Agent 从"每次从零推理"的黑盒模式推进到"从经验中学习"的持续改进模式。未来 CUA Agent 将像人类一样，重复任务越来越快。

2. **自蒸馏质量门控成为后训练分水岭**：Trust the Right Teacher 揭示了 OPSD 在坐标敏感任务中的信号退化问题，质量门控是解决之道。这一洞察可推广到代码生成、数学推理等自蒸馏场景——"教师不一定总是对的"，质量感知可能是自蒸馏从"能用"到"好用"的关键。
