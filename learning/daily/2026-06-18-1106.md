## 技术扫描（35+）

### AI Testing / Software Testing（10项）

1. **C-Trace** — Agent 运行时 GDPR 合规验证 | arXiv:2606.19242
   - 将 GDPR 要求（同意、目的限制、数据最小化、删除权）表达为 Agent 执行轨迹上的形式化策略谓词
   - 运行时监控器拦截每次工具调用和模型输出，拒绝不合规动作
   - 使用 DSPy 生成攻击对话和红队语料进行测试，完美提取下攻击成功率 0%
   - 解决传统离线红队测试无法保证运行时合规的问题

2. **SkillSpector** — AI Agent 技能安全扫描器 | GitHub 7,530 stars (周增 5,257)
   - 64 种漏洞模式覆盖 16 个类别：提示注入、数据泄露、权限提升、供应链攻击等
   - 两阶段分析：快速静态分析 + 可选 LLM 语义评估
   - 支持 Git/URL/zip/目录/单文件多格式输入，输出 Terminal/JSON/Markdown/SARIF
   - 研究证实 26.1% 的技能包含漏洞，5.2% 显示恶意意图

3. **CEO-Bench** — 长期 Agent 能力评估基准 | arXiv:2606.18543
   - 模拟运营初创公司 500 天，评估导航长程不确定性、噪声信息获取、适应变化世界、协调多组件能力
   - 最强模型（Claude Opus 4.8 / GPT-5.5）仅能维持 $1M 起始余额，均未能持续盈利
   - 填补了 Agent 从短期任务执行到长期战略决策评估的空白

4. **RTSGameBench** — VLM 战略推理 RTS 基准 | arXiv:2606.18950
   - 基于 Beyond All Reason 大规模 RTS 游戏，评估多智能体协调、对手策略适应、长程规划
   - 提供多样化比赛结构评估 + 迷你游戏诊断 + 自进化生成框架
   - 揭示 SOTA VLM 在紧密协调和多智能体任务中表现不佳

5. **SciRisk-Bench** — AI4Science 安全风险维度基准 | arXiv:2606.18936
   - 覆盖 7 个学科、31 个子学科、10 个风险维度
   - 评估 LLM 在高 stakes 科学场景中识别和避免风险的能力
   - 提供细粒度诊断，揭示科学模型在哪些维度仍然不安全

6. **DeFAb** — 可废止归纳推理基准 | arXiv:2606.18557
   - 372,648+ 实例来自 18 个知识库，33.75M 物化规则
   - 最强前沿模型 Level 2 准确率仅 7.8-23.5%，符号求解器 100%
   - 多项式时间可验证黄金标准，可作为精确奖励用于 DPO/RLVR/GRPO

7. **WorldLines** — 长程具身 Agent 基准 | arXiv:2606.18847
   - 项目驱动的长期家庭辅助基准，包含对话、动作、执行反馈、对象状态变化
   - 提出 ObsMem 观察者基础记忆框架，维护可见性感知记忆
   - 揭示部分可观测性、覆盖世界状态、长期记忆到具身规划的持续挑战

8. **CAPRA** — 软件架构交付物多 Agent 评估 | arXiv:2606.18976
   - 多 Agent LLM 系统分析软件架构交付物，生成个性化 LaTeX 反馈
   - 确定性证据锚定（Levenshtein 距离模糊匹配）+ ConsistencyManager 交叉验证
   - 88.8% 评估标准满足率，kappa = 0.582 与人类评估者一致性

9. **Written by AI, Managed by AI** — Index Sickness 研究 | arXiv:2606.19121
   - 391 次连续协作 session 的行动研究，揭示"索引病"失败模式
   - 当符号系统超过复杂度阈值，LLM 放弃真实业务语义理解，退化为自引用推理
   - 提出"基线-日志物理分离"机制，AI Instructions 减少 ~75%

10. **VbR (Variability by Regeneration)** — Vibe Coding 产品线方法 | arXiv:2606.19042
    - 首个产品线方法，LLM 作为派生引擎从声明式规格生成无死码变体
    - 变体调度器透明路由用户请求到匹配二进制
    - 核心观点：AI 生成软件中的变异性属于规格说明而非代码

#### 1. SkillSpector — AI Agent 技能安全扫描器

**地址**: https://github.com/NVIDIA/SkillSpector

**项目介绍**:
SkillSpector 是 NVIDIA 推出的 AI Agent 技能安全扫描器，专门用于检测 Claude Code、Codex CLI、Gemini CLI 等工具中安装的 Agent 技能的安全漏洞和恶意模式。研究数据显示 26.1% 的技能包含漏洞，5.2% 显示可能的恶意意图。SkillSpector 回答的核心问题是："这个技能安全吗？"

**技术架构**:
- 多格式输入：Git 仓库、URL、zip 文件、目录、单文件
- 64 种漏洞模式覆盖 16 个类别
- 两阶段分析管道：快速静态分析 → 可选 LLM 语义评估
- 实时漏洞查询：SC4 查询 OSV.dev 获取 CVE 数据，自动离线回退
- 多输出格式：Terminal、JSON、Markdown、SARIF

**核心创新**:
- 16 个漏洞类别覆盖全面：提示注入、数据泄露、权限提升、供应链攻击、过度代理、输出处理、系统提示泄露、记忆毒化、工具滥用、流氓 Agent、触发器滥用、危险代码（AST）、污点追踪、YARA 签名、MCP 最小权限、MCP 工具毒化
- 风险评分 0-100，带严重性标签和清晰建议
- Docker 支持，无需安装 Python 即可运行

**与现有技术对比**:
传统 Agent 安全工具主要关注模型层面的安全对齐（如 RLHF、红队测试），而 SkillSpector 聚焦于 Agent 技能这一特定攻击面——技能以隐式信任安装且几乎无审查。这是 Agent 安全从"模型安全"向"生态系统安全"扩展的重要标志。

**优势与局限**:
- 优势：覆盖全面、支持 CI/CD 集成（SARIF）、静态分析快速、LLM 语义分析可选
- 局限：依赖 LLM 语义分析时需要 API 密钥、对新型攻击模式需要持续更新规则库

**未来发展潜力**:
随着 Agent 技能生态的爆发式增长（agent-skills 62K+ stars），技能安全扫描将成为 Agent 部署的标配工具。SkillSpector 有望成为 Agent 技能安全的事实标准。

#### 2. Apple Container — Apple Silicon 原生容器

**地址**: https://github.com/apple/container

**项目介绍**:
Apple Container 是 Apple 官方推出的 Linux 容器运行工具，专为 Apple Silicon 优化。它使用 Swift 编写，通过轻量级虚拟机在 Mac 上创建和运行 Linux 容器，消费和产出 OCI 兼容容器镜像。一周内获得 9,735 stars，显示社区对 Apple 容器化方案的高度关注。

**技术架构**:
- 基于 Containerization Swift 包进行底层容器、镜像和进程管理
- 轻量级虚拟机隔离，Apple Silicon 硬件加速
- OCI 兼容，可从任何标准容器注册表拉取/推送镜像
- macOS 26 支持，利用新的虚拟化和网络功能

**核心创新**:
- Apple 首次正式进入容器化领域，代表 Apple Silicon 生态成熟度里程碑
- Swift 原生实现，与 Apple 生态深度集成
- 零配置体验，`container system start` 即可运行

**与现有技术对比**:
相比 OrbStack、Rancher Desktop 等第三方方案，Apple Container 是官方原生实现，性能优化更深。相比 Docker Desktop，它更轻量、更原生，但功能成熟度尚在早期（0.x 版本）。

**优势与局限**:
- 优势：官方支持、Apple Silicon 原生优化、OCI 完全兼容、Swift 生态
- 局限：仅支持 Apple Silicon、macOS 26+、版本尚不稳定（0.x）

**未来发展潜力**:
如果 Apple 持续投入，这可能成为 Mac 上容器开发的事实标准，对 AI/ML 开发者的本地训练和推理环境有重要意义。

#### 3. iroh — 键寻址 P2P 网络栈

**地址**: https://github.com/n0-computer/iroh

**项目介绍**:
iroh 是一个用 Rust 编写的模块化 P2P 网络栈，核心理念是"IP 地址会坏，改用键拨号"。它通过内容寻址和键标识替代传统 IP 寻址，解决网络脆弱性问题。日增 421 stars，周增显著。

**技术架构**:
- Rust 实现，模块化设计
- 内容寻址 + 键标识替代 IP 寻址
- P2P 直连 + 中继 fallback
- 支持离线发现和 NAT 穿透

**核心创新**:
- 从 IP 寻址到键寻址的范式转换
- 对 Agent 间通信、分布式 Agent 网络有直接应用价值
- 零依赖 P2P 通信，适合边缘/移动场景

**优势与局限**:
- 优势：Rust 安全保证、模块化、P2P 原生
- 局限：相对较新、生态系统尚不成熟

**未来发展潜力**:
在 Agent-to-Agent 通信、去中心化 AI 基础设施、边缘计算场景中可能有重要应用。

### 学术论文

#### 1. CEO-Bench: Can Agents Play the Long Game?

**地址**: https://arxiv.org/abs/2606.18543

**论文介绍**:
CEO-Bench 是一个评估 Agent 长期能力的基准，模拟运营初创公司 500 天。它测试四个核心能力：(1) 在不确定性中长程导航；(2) 在噪声环境中获取信息；(3) 适应变化世界；(4) 协调多组件实现一致目标。Agent 通过可编程 Python 接口管理定价、营销、预算等方面，面临与人类 CEO 相同的挑战。

**技术架构**:
- 可编程 Python 接口模拟商业环境
- 噪声互联商业数据库分析
- 信号到策略的翻译 + 编程协调
- 500 天时间跨度的持续评估

**核心算法/系统设计**:
- 环境模拟真实商业不确定性：市场需求波动、竞争动态、客户偏好隐藏
- Agent 需要编写复杂代码：模拟客户队列预测现金流、挖掘谈判历史发现隐藏偏好
- 评估指标：最终资产、盈利能力、适应速度、决策质量

**与现有技术对比**:
现有 Agent 基准（SWE-bench、OSWorld、WebArena）主要评估短期任务执行能力（单轮/少轮交互）。CEO-Bench 首次将评估扩展到 500 天持续运营，要求 Agent 在长程不确定性中保持战略一致性。

**优势与局限**:
- 优势：首次长程 Agent 能力评估、真实商业场景模拟、可编程环境灵活
- 局限：模拟环境简化了真实商业复杂性、500 天计算成本高

**未来发展潜力**:
CEO-Bench 可能成为 Agent 长期能力评估的标准基准，推动 Agent 从"任务执行者"向"战略决策者"演进。

#### 2. C-Trace: Runtime Compliance Verification for AI Agents

**地址**: https://arxiv.org/abs/2606.19242

**论文介绍**:
C-Trace 是一个 Agent 运行时 GDPR 合规验证框架。随着 AI Agent 通过工具使用、函数调用和多轮对话处理个人数据，产生了 GDPR 合规义务。当前测试实践主要依赖离线红队测试或静态提示审查，无法保证运行时行为符合法规。C-Trace 将 GDPR 要求表达为 Agent 执行轨迹上的形式化策略谓词。

**技术架构**:
- 形式化策略谓词：同意、目的限制、数据最小化、删除权
- 运行时监控器拦截每次工具调用和模型输出
- 攻击对话测试：DSPy 生成提示 + 红队语料
- 四案例研究重构为 GDPR 合规验证

**核心算法/系统设计**:
- 策略谓词在 Agent 执行轨迹上实时评估
- 监控器作为中间层，拒绝不合规动作
- 10% 每类别提取噪声下（包括丢弃和过度类型化），攻击成功率 ≤12%
- 完美提取下攻击成功率 0%，误报率 ≤16%

**与现有技术对比**:
传统 Agent 安全方法：(1) 离线红队测试——无法覆盖运行时行为；(2) 静态提示审查——无法检测动态违规。C-Trace 首次将形式化验证引入 Agent 运行时合规，提供实时保证。

**优势与局限**:
- 优势：运行时实时保证、形式化策略表达、低误报率
- 局限：仅覆盖 GDPR 子集、依赖提取器准确性、策略编写需要专业知识

**未来发展潜力**:
随着 AI Agent 在医疗、金融等受监管行业的部署，运行时合规验证将成为刚需。C-Trace 框架可扩展到 HIPAA、PCI-DSS 等其他法规。

#### 3. RODS: Reward-Driven Online Data Synthesis

**地址**: https://arxiv.org/abs/2606.19047

**论文介绍**:
RODS 解决多轮工具使用 RL 中静态数据集信息样本快速耗尽的问题。作者观察到 GRPO 中的梯度信号集中于最高 rollout 奖励方差的任务（Popoviciu 上界的结果），因此能力边界附近的样本（成功和失败大致平衡）贡献不成比例大的策略梯度。随着训练进行，这个边界持续移动，逐渐耗尽静态数据集中的信息样本。

**技术架构**:
- 进度奖励方差作为零成本边界检测器
- 技能对齐重采样管道合成新的多轮变体
- 动态回放缓冲区与策略协同进化
- 从 400 人类种子 + ~800 训练池达到 17K 离线管道性能

**核心算法/系统设计**:
- 利用 GRPO 已有的 rollout 计算，无需额外推理
- 识别能力边界样本 → 合成结构复杂度匹配的新变体（API 拓扑、依赖深度）
- 动态缓冲区管理，样本随策略演化

**与现有技术对比**:
传统方法：(1) 大规模离线数据集——成本高、样本冗余；(2) 环境增强——改变环境而非数据。RODS 首次将 RL 训练和数据生成闭环，利用训练过程中的奖励方差作为数据合成信号。

**优势与局限**:
- 优势：20x 更少轨迹达到相同性能、零额外推理成本、动态适应策略演化
- 局限：仅在受控环境中验证、需要环境支持样本合成

**未来发展潜力**:
RODS 框架可推广到更多 RL 场景，特别是需要大规模高质量数据的 Agent 训练。它代表了从"数据驱动 RL"向"RL 驱动数据合成"的范式转变。

## 技术趋势

1. **Agent 运行时验证从离线到在线**: C-Trace 代表 Agent 安全从离线红队测试向运行时形式化验证的范式转变。随着 Agent 在受监管行业部署，运行时合规验证将成为标配。SkillSpector 则从技能安全角度补充了这一趋势。

2. **长期 Agent 能力评估成为新焦点**: CEO-Bench（500 天模拟）、WorldLines（长程具身）、RTSGameBench（战略推理）共同指向 Agent 评估从短期任务到长程能力的扩展。这反映了 Agent 从"工具使用者"向"自主决策者"演进的现实需求。

3. **推理优化从数据驱动到过程驱动**: Rubric-Conditioned Self-Distillation（规则条件蒸馏）、RODS（奖励驱动数据合成）、ARIADNE（无训练路由）代表推理优化从"更多数据"向"更智能过程"的转变。核心思路是利用训练过程中的内在信号（奖励方差、规则结构、嵌入质心）而非外部数据。

## 新领域收获

1. **Index Sickness（索引病）**: AI 辅助长期软件开发中的语义退化问题。当符号系统超过复杂度阈值，LLM 放弃真实业务语义理解，退化为自引用推理。提出"基线-日志物理分离"解决方案，AI Instructions 减少 75%。这对所有 AI 辅助长期编码项目有重要警示意义。

2. **Vibe Coding 产品线工程（VbR）**: 将 AI 生成软件中的变异性从代码移到规格说明，LLM 作为派生引擎从声明式规格生成无死码变体。这是 Vibe Coding 从"一次性生成"到"可管理产品族"的关键一步。

3. **在轨卫星 VLM（NAVI-Orbital）**: 首次 LEO 卫星自主多模态推理演示，Gemma 3 在硬件加速 GPU 上运行，无微调，88.16% 准确率。代表基础模型在太空边缘计算中的可行性，可能改变卫星数据处理范式（从"采集-下传"到"语义压缩-下传"）。
