## 技术扫描（32+）

### AI Testing / Software Testing（9项）

1. **TxBench-PP: TherapeuticsBench Preclinical Pharmacology** — Agent 药物发现评估基准
   - 地址：https://arxiv.org/abs/2606.19245
   - 来源：arXiv
   - 领域：AI Testing / 药物发现
   - 技术介绍：首个专注于小分子临床前药理学的可验证 Agent 基准，包含 100 项评估，覆盖作用机制（MoA）和药效学（PD）推理、化合物-靶点结合、因果靶点验证、可开发性和安全性、转化 efficacy 五大维度。Agent 接收真实工作流快照，在编码环境中检查文件，返回结构化答案进行确定性评分。16 个模型-Harness 配置（11 个模型，4800 条轨迹）中，最强配置 Claude Opus 4.8/Pi 仅通过 59.3%（178/300），GPT-5.5/Pi 为 55.3%。
   - 应用场景：药物发现 Agent 能力评估、科学推理 Agent 基准
   - 为什么值得关注：揭示前沿 Agent 在专业科学决策中的显著能力差距，最强模型通过率不足 60%

2. **X+Slides: Benchmarking Audience-Conditioned Slide Generation** — 受众条件幻灯片生成基准
   - 地址：https://arxiv.org/abs/2606.19256
   - 来源：arXiv
   - 领域：AI Testing / 内容生成
   - 技术介绍：首个针对受众条件幻灯片生成的基准，覆盖 113 个主题和 7 种演示场景，8133 个去重源 grounding 探针。通过为同一探针分配受众特定效用权重，报告四个互补指标：Audience Coverage（受众必需信息覆盖率）、Domain-wise Coverage（信息类型覆盖）、Efficiency（单位注意力成本交付效用）、Correctness（幻灯片声明是否得到源支持）。DeepPresenter 在 τ_A=0.7 时 Audience Coverage 达 0.714，SlideTailor 为 0.594，NotebookLM 为 0.853。
   - 应用场景：AI 辅助演示文稿生成、Agent 内容生成质量评估
   - 为什么值得关注：首次将目标受众作为关键评估维度，揭示视觉质量和广泛主题覆盖不应被视为证据支持

3. **Multi-Domain Benchmark for Detecting AI-Generated Text-Rich Images** — AI 生成文本丰富图像检测基准
   - 地址：https://arxiv.org/abs/2606.19259
   - 来源：arXiv
   - 领域：AI Testing / 内容真实性
   - 技术介绍：针对 OpenAI GPT Image 2 生成的文本丰富图像的多域检测基准，包含 8602 张图像覆盖 6 个代表性类别：商业海报、信息图、学术海报、收据、表格和 UI 截图。评估 5 个代表性 AI 生成图像检测器的零样本性能，发现检测器性能高度域依赖，最强传统检测器对 JPEG 压缩极度敏感。探索性评估显示 MLLM 在结构化格式上既有潜力也有局限。
   - 应用场景：数字信任、内容真实性验证、AI 生成内容检测
   - 为什么值得关注：揭示现有检测器在文本和布局感知方面的系统性不足，需要专门的检测方法

4. **Correct Yourself, Keep My Trust: Self-Correction and Social Connection in Chatbots** — 社交聊天机器人自我纠正与信任
   - 地址：https://arxiv.org/abs/2606.19286
   - 来源：arXiv
   - 领域：AI Testing / 人机交互
   - 技术介绍：N=120 的被试间实验比较三种错误纠正策略：网页撤回、同一社交聊天机器人自我纠正、专家聊天bot纠正。关键发现：三种策略同样有效地纠正了错误，但只有自我纠正在不损害聊天机器人可信度的情况下做到了——参与者给自我纠正的聊天机器人显著更高的信任度和专业度评分。社交连接强度（通过社交吸引力和自我披露测量）显著预测了信念变化的幅度，但仅在聊天机器人自我纠正时有效。
   - 应用场景：社交聊天机器人设计、AI 错误恢复策略
   - 为什么值得关注：首次实证证明社交聊天机器人应该自我纠正而非外包纠正，社交连接是放大纠正效果的功能机制

6. **DeFAb: Defeasible Abductive Reasoning Benchmark** — 可废止归纳推理基准

   - 应用场景：Agent 推理能力评估、RL 奖励信号生成

7. **WorldLines: Long-Horizon Embodied Agent Benchmark** — 长程具身 Agent 基准

8. **CAPRA: Software Architecture Multi-Agent Evaluation** — 软件架构交付物多 Agent 评估
   - 地址：https://arxiv.org/abs/2606.18976

### AI / LLM 技术（12项）

10. **STARE: Surprisal-Guided Token-Level Advantage Reweighting** — 惊喜引导策略熵稳定
    - 地址：https://arxiv.org/abs/2606.19236
    - 来源：arXiv
    - 领域：AI / RL 训练
    - 技术介绍：GRPO 等 RLVR 算法在复杂推理后训练中面临策略熵崩溃问题。本文对 GRPO 下的 token 级熵动力学进行一阶梯度分析，发现 token 级信用分配不匹配：per-token 熵变化分解为 trajectory-level advantage 和 entropy sensitivity function 的乘积，产生 advantage-surprisal 四象限结构和近临界特性。STARE 通过批次内部惊喜分位数识别熵关键 token 子集，选择性重加权其有效优势，并引入目标熵闭环门控实现稳定熵调节。在 1.5B 到 32B 模型和三个任务族（Short CoT、Long CoT、Multi-Turn Tool Use）上，STARE 在数千步训练中维持稳定 RL 训练，同时保持策略熵在目标带内。AIME24 和 AIME25 上超越 DAPO 等基线 4%-8% 平均准确率。
    - 应用场景：GRPO/RLVR 训练稳定性、复杂推理模型后训练
    - 为什么值得关注：首次从梯度分析角度揭示 GRPO 熵崩溃的 token 级机制，提供可操作的解决方案

11. **MAST: Mechanism-Aligned Selective Targeting for RLVR Unlearning** — 机制引导选择性遗忘
    - 地址：https://arxiv.org/abs/2606.19222
    - 来源：arXiv
    - 领域：AI / 模型编辑
    - 技术介绍：针对 RLVR 诱导推理的选择性遗忘方法。在 Qwen2.5-Math-1.5B 和 Qwen3-1.7B-Base 的匹配 SFT/RLVR 检查点上，SFT-to-RLVR 增量在 token 级 delta-log-probability 上与 SFT 更新显著不同，全参数梯度上升只能通过损坏保留的 MATH 和 GSM8K 来遗忘。MAST 按离主能量、更新幅度和遗忘梯度耦合幅度对注意力投影张量排序，然后仅更新排名靠前的子集。在主模型上，MAST 诱导统计显著的靶向遗忘（MATH forget 45/150 降至 37/150；McNemar p=0.0078），同时保留 GSM8K（+0.8 pp）和 MATH retain（-0.5 pp）。
    - 应用场景：RLVR 推理能力选择性移除、模型安全编辑
    - 为什么值得关注：为 RLVR 诱导的能力提供首个机制引导的选择性遗忘方法，大幅降低附带损伤

12. **UBP2: Uncertainty-Balanced Preference Planning** — 不确定性平衡偏好规划
    - 地址：https://arxiv.org/abs/2606.19328
    - 来源：arXiv
    - 领域：AI / 偏好基 RL
    - 技术介绍：偏好基 RL 提供从行为成对比较中学习奖励模型的方法，绕过显式奖励设计的需求。但现有方法通常依赖被动数据收集，样本效率差，尤其在学习早期。UBP2 引入基于模型的方法，通过联合推理奖励、动态和价值函数的不确定性来主动指导探索。使用奖励、动态和价值函数模型的集成，根据结合期望奖励、终端价值和认知不确定性的统一分数评估候选轨迹。在 Meta-World 基准上，UBP2 实现比无模型偏好基方法和非乐观基于模型基线显著更高的样本效率，并在标准正则性假设下建立了有限和无限视界设置的亚线性遗憾保证。
    - 应用场景：偏好基 RL、机器人控制、奖励学习
    - 为什么值得关注：将主动探索引入偏好基 RL，解决早期学习样本效率低的核心问题

13. **Explaining Attention with Program Synthesis** — 用程序合成解释注意力
    - 地址：https://arxiv.org/abs/2606.19317
    - 来源：arXiv
    - 领域：AI / 可解释性
    - 技术介绍：用可执行程序近似深度网络组件行为的新方法。聚焦 Transformer 语言模型中的注意力头：首先计算给定头在随机选择的训练样本集合上的注意力矩阵，然后提示预训练语言模型生成一组 Python 程序来重现注意力模式，最后根据程序在保留输入上的预测性能重新排序。少于 1000 个生成的程序可以复现 GPT-2、TinyLlama-1.1B 和 Llama-3B 中注意力头的模式，在 TinyStories 上平均 IoU 相似度超过 75%。最佳拟合程序可以替代神经网络注意力头而不显著影响模型行为：在三个模型中用程序化替代替换 25% 的注意力头仅导致 16% 的平均困惑度增长，同时保持各种下游问答基准的性能。
    - 应用场景：Transformer 可解释性、模型压缩、符号透明性
    - 为什么值得关注：首次用人类可读的可执行代码替代注意力头，25% 替换仅 16% 困惑度增长

14. **OneCanvas: 3D Scene Understanding via Panoramic Reprojection** — 全景重投影 3D 场景理解
    - 地址：https://arxiv.org/abs/2606.19253
    - 来源：arXiv
    - 领域：AI / 3D 视觉
    - 技术介绍：OneCanvas 将所有视图的 patch 特征聚合到单个等距全景画布上。每个 patch 使用深度和相机姿势反投影到 3D 世界坐标，然后放置在画布上该点的连续经度和纬度位置，无需光栅化或重叠视图聚合。添加 patch 度量坐标的 3D 位置嵌入以恢复角度画布坐标折叠时丢失的深度。预训练 VLM 将此表示作为普通图像消费。在 SQA3D 和 VSI-Bench 上达到 SOTA 准确率，在 SPBench 上泛化到 OOD 数据，使用比最强竞争方法少一个数量级的训练计算。
    - 应用场景：3D 场景理解、具身 AI、机器人导航
    - 为什么值得关注：无需复杂几何编码器或大量训练预算，用简单全景表示实现 SOTA

15. **ClaMPAPP: LLM as Interface, ML as Predictor for Clinical Decision** — 混合 LLM-ML 医疗系统
    - 地址：https://arxiv.org/abs/2606.19183
    - 来源：arXiv
    - 领域：AI / 医疗 AI
    - 技术介绍：ClaMPAPP（Clinical Language-assisted Machine-learning Pipeline for Appendicitis）混合系统使用 LLM 作为接口而非最终决策者。LLM 从类笔记叙事中提取模式约束的临床特征，应用确定性合理性检查，将验证的特征传递给用临床、实验室和超声变量训练的 XGBoost 分类器。在两个独立的德国医院儿科阑尾炎队列上评估，与端到端 LLM 基线（包括开源和专有模型）比较。ClaMPAPP 在内部和外部验证中都实现了最强的整体诊断性能，同时最小化漏诊阑尾炎病例——急性分诊中的关键安全问题。端到端 LLM 显示不稳定的敏感性-特异性权衡，在叙事重排下更大的退化。
    - 应用场景：临床决策支持、医疗 AI 系统设计
    - 为什么值得关注：实证验证 LLM-as-interface + ML-as-predictor 架构在医疗诊断中优于端到端 LLM

16. **Trade-offs in Medical LLM Adaptation: French QA Case Study** — 医疗 LLM 适应权衡
    - 地址：https://arxiv.org/abs/2606.19266
    - 来源：arXiv
    - 领域：AI / 医疗 NLP
    - 技术介绍：以法语医疗 QA 为案例研究，比较持续预训练（CPT）、监督微调（SFT）及其组合在三个模型族、多个规模和三种初始化类型上的效果。对于 MCQA，CPT+SFT 通常达到最高分，但相对于 SFT 的增益很小且经常不显著，使 SFT 成为强而成本有效的默认选择。对于 OEQA，CPT 持续改善重叠指标，而 SFT 经常降低生成质量。跨语言实验进一步显示从法语适应到英语基准的有效迁移。
    - 应用场景：医疗 LLM 领域适应、多语言医疗 QA
    - 为什么值得关注：为计算约束下的适应策略选择提供实用指南

17. **Decision-Focused RL for EV Charging** — 决策聚焦 RL 用于 EV 充电控制
    - 地址：https://arxiv.org/abs/2606.19199
    - 来源：arXiv
    - 领域：AI / RL 应用
    - 技术介绍：针对未知出发时间的受控 EV 充电决策聚焦 RL 框架。传统方法先训练预测器再训练控制器，但预测误差会传播并阻碍控制器性能。DF-RL 框架中预测器端到端训练，即从 RL 代理的充电策略动作中获取反馈。联合训练预测器和控制器产生更高质量的动作：DF-RL 方法比其他基线产生更优的充电决策，总奖励提升高达 14%，未供应能量（即 EV 已离开而未完成的充电）减少 55%。
    - 应用场景：智能电网、EV 充电优化、决策聚焦预测
    - 为什么值得关注：将预测模型训练从准确性目标转向决策质量目标，避免误差传播

18. **Towards an Agent-First Web: Redesigning the Web for AI Agents** — Agent-First Web 架构
    - 地址：https://arxiv.org/abs/2606.19116

### 学术论文

#### 1. STARE: 惊喜引导 token 级优势重加权解决 GRPO 策略熵崩溃

**地址**: https://arxiv.org/abs/2606.19236
**类型**: 论文

**论文介绍**:
GRPO（Group Relative Policy Optimization）及其变体已成为 LLM 复杂推理后训练的主导范式，但普遍面临策略熵崩溃问题——训练过程中策略分布过度集中，导致探索能力丧失和性能退化。本文首次从一阶梯度分析角度揭示了 GRPO 中 token 级熵动力学的内在机制：per-token 熵变化分解为 trajectory-level advantage 和 entropy sensitivity function 的乘积，产生 advantage-surprisal 四象限结构和近临界特性。基于这一发现，本文提出 STARE（Surprisal-guided Token-level Advantage Reweighting），通过批次内部惊喜分位数识别熵关键 token 子集，选择性重加权其有效优势，并引入目标熵闭环门控实现稳定熵调节。

**技术架构**:
- 一阶梯度分析框架：将 token 级熵变化分解为 advantage × sensitivity
- 四象限分类：根据 advantage 和 surprisal 的符号将 token 分为四类
- 惊喜分位数门控：批次内部动态识别熵关键 token
- 闭环熵调节：目标熵作为控制信号

**核心算法**:
1. 计算每个 token 的 surprisal（-log p）
2. 按批次内部 surprisal 分位数识别熵关键子集
3. 对关键子集的 advantage 进行选择性重加权
4. 目标熵闭环门控确保熵稳定在目标带内

**与现有技术对比**:
- DAPO 等基线方法缺乏对 token 级熵动力学的显式建模
- STARE 在 AIME24 和 AIME25 上超越 DAPO 4%-8% 平均准确率
- 在 1.5B 到 32B 模型和三个任务族上均保持稳定

**优势与局限**:
- 优势：首次从梯度分析角度揭示 GRPO 熵崩溃机制，提供可操作的解决方案
- 局限：需要批次内部统计，可能增加训练复杂性

**未来发展潜力**:
STARE 的发现（advantage-surprisal 四象限结构）可能对 RLVR 训练方法论有普遍意义。未来可能扩展到更多 RL 算法和任务类型。

**个人技术理解**:
STARE 的核心贡献不仅是方法本身，更是从梯度分析角度揭示 GRPO 熵崩溃的 token 级机制。advantage-surprisal 四象限结构是一个基础性发现，可能影响未来 RLVR 训练方法的设计。近临界特性的发现暗示 GRPO 训练可能处于一个微妙的平衡点，微小的扰动可能导致熵崩溃。STARE 的闭环熵调节机制为这一平衡提供了工程化的解决方案。

#### 2. Explaining Attention with Program Synthesis: 用程序合成解释注意力头

**地址**: https://arxiv.org/abs/2606.19317
**类型**: 论文

**论文介绍**:
可解释深度学习的长期目标是用人类有意义的符号描述替代不透明的神经计算。本文提出用可执行程序近似深度网络组件行为的新方法，聚焦 Transformer 语言模型中的注意力头。流程：(1) 计算给定头在随机训练样本上的注意力矩阵；(2) 提示预训练语言模型生成 Python 程序来重现注意力模式；(3) 根据程序在保留输入上的预测性能重新排序。少于 1000 个生成的程序可以复现 GPT-2、TinyLlama-1.1B 和 Llama-3B 中注意力头的模式，在 TinyStories 上平均 IoU 相似度超过 75%。最佳拟合程序可以替代神经网络注意力头：在三个模型中用程序化替代替换 25% 的注意力头仅导致 16% 的平均困惑度增长，同时保持各种下游问答基准的性能。

**技术架构**:
- 注意力矩阵提取：从训练样本计算注意力模式
- LLM 辅助程序生成：提示 LLM 生成 Python 程序
- 性能驱动重排序：根据保留输入上的预测性能选择最佳程序
- 程序化替代：用生成的程序替代神经网络注意力头

**核心创新**:
- 首次用人类可读的可执行代码替代注意力头
- 25% 替换仅 16% 困惑度增长，表明注意力头存在大量冗余
- 生成的程序可作为注意力行为的符号化解释

**与现有技术对比**:
- 传统可解释性方法（如注意力可视化、探针）只提供分析，不提供可执行替代
- CIRCA 等方法需要大量计算和领域专业知识
- 本文方法自动化程度高，无需领域专业知识

**优势与局限**:
- 优势：自动化、人类可读、可执行替代、低性能损失
- 局限：程序生成依赖 LLM 质量，复杂注意力模式可能难以用简单程序描述

**未来发展潜力**:
程序化注意力替代可能成为模型压缩和可解释性的新方向。未来可能扩展到更多网络组件（如 FFN 层）和更多模型架构。

**个人技术理解**:
本文的核心洞察是：注意力头中存在大量冗余，可以用简单程序替代。25% 替换仅 16% 困惑度增长这一发现暗示 Transformer 的注意力机制可能比想象中更"可编程"。这不仅对可解释性有意义，也可能为模型压缩提供新路径——用轻量级程序替代部分注意力头，在保持性能的同时减少计算量。

#### 3. ClaMPAPP: LLM-as-Interface + ML-as-Predictor 医疗诊断架构

**地址**: https://arxiv.org/abs/2606.19183
**类型**: 论文

**论文介绍**:
LLM 可以使临床决策支持更易访问，但直接用作诊断引擎受限于对提示的敏感性、信息顺序和看似合理但不正确的输出。结构化 ML 模型提供更稳定的风险预测，但需要难以集成到叙事临床工作流中的表格输入。ClaMPAPP（Clinical Language-assisted Machine-learning Pipeline for Appendicitis）混合系统使用 LLM 作为接口而非最终决策者：LLM 从类笔记叙事中提取模式约束的临床特征，应用确定性合理性检查，将验证的特征传递给用临床、实验室和超声变量训练的 XGBoost 分类器。在两个独立的德国医院儿科阑尾炎队列上评估，与端到端 LLM 基线比较。ClaMPAPP 在内部和外部验证中都实现了最强的整体诊断性能，同时最小化漏诊阑尾炎病例——急性分诊中的关键安全问题。端到端 LLM 显示不稳定的敏感性-特异性权衡，在叙事重排下更大的退化。

**技术架构**:
- LLM 接口层：从自然语言叙事中提取结构化临床特征
- 确定性验证层：模式约束 + 合理性检查
- ML 预测层：XGBoost 分类器基于验证特征进行预测
- 分离设计：自然语言可用性（LLM）与预测推理稳定性（ML）解耦

**核心创新**:
- LLM-as-Interface + ML-as-Predictor 架构在医疗诊断中的首次系统验证
- 确定性验证层作为 LLM 和 ML 之间的安全屏障
- 在两个独立队列上验证了架构的鲁棒性

**与现有技术对比**:
- 端到端 LLM：不稳定的敏感性-特异性权衡，对叙事重排敏感
- 纯 ML：需要表格输入，难以集成到叙事临床工作流
- ClaMPAPP：结合两者的优势，避免各自的劣势

**优势与局限**:
- 优势：最小化漏诊（关键安全指标），对叙事重排鲁棒，可审计路径
- 局限：需要模式定义和合理性规则，维护成本可能较高

**未来发展潜力**:
LLM-as-Interface + ML-as-Predictor 架构可能成为高风险 AI 应用的标准设计模式。未来可能扩展到更多医疗场景和非医疗领域。

**个人技术理解**:
ClaMPAPP 的核心贡献是实证验证了 LLM-as-Interface + ML-as-Predictor 架构在医疗诊断中的优越性。这一架构的关键洞察是：LLM 擅长理解自然语言但不擅长稳定推理，ML 擅长稳定推理但不擅长理解自然语言。将两者解耦，让 LLM 做它擅长的事（理解语言），让 ML 做它擅长的事（稳定预测），可能成为高风险 AI 应用的标准设计模式。

## 技术趋势

1. **RL 训练稳定性从经验调参到理论诊断**：STARE 从一阶梯度分析揭示 GRPO 熵崩溃的 token 级机制，advantage-surprisal 四象限结构为 RLVR 训练提供新的理论基础。RL 训练正从"调参艺术"走向"可诊断工程"。

2. **Agent 互联网接入从浏览器自动化到语义接口**：Agent-Reach 日增 1,161 stars 和 Agent-First Web 论文同时出现，反映 Agent 互联网接入正从底层的浏览器自动化（browser-use）向高层的语义接口演进。

3. **高风险 AI 应用从端到端 LLM 到混合架构**：ClaMPAPP 实证验证 LLM-as-Interface + ML-as-Predictor 在医疗诊断中优于端到端 LLM。分离自然语言理解和预测推理可能成为高风险 AI 应用的标准设计模式。

## 新领域收获

1. **Agent-First Web**：Web 从人类消费到 Agent 消费的范式转变。三层重构（访问模型、经济学、内容）可能催生新的 Web 标准和协议。与 Agent-Reach 的爆发式增长形成呼应，Agent 互联网接入基础设施正在快速成熟。

2. **注意力程序化可解释性**：用人类可读的可执行代码替代注意力头，25% 替换仅 16% 困惑度增长。这不仅对可解释性有意义，也可能为模型压缩提供新路径。

3. **药物发现 Agent 评估**：TxBench-PP 揭示前沿 Agent 在临床前药理学决策中的显著能力差距（最强模型通过率不足 60%）。药物发现 Agent 从"能生成假设"到"能做真实决策"还有很长的路。
