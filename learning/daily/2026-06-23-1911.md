## 技术扫描（32）

### AI Testing / Software Testing (10)

1. **AOHP: OS-Level Agent Harness for Personalized, Efficient and Secure Interaction**
   - 地址：https://arxiv.org/abs/2606.23449
   - 来源：arXiv (cs.AI/cs.SE)
   - 领域：Testing / Agent Infrastructure
   - 技术介绍：提出AOHP（Android Open Harness Project），基于AOSP构建的OS级Agent Harness。核心设计理念是将Agent作为一等OS Actor，提供自适应UI和Agent友好运行时环境。保留成熟Android软件生态同时，为Agent提供原生OS支持。解决当前OS为应用中心设计导致Agent执行开销和安全风险问题。
   - 应用场景：移动Agent系统、OS级Agent集成
   - 为什么值得关注：首个开源Agent原生OS测试床，从"适配现有OS"推进到"为Agent设计OS"

2. **OptDetect: Source-Free Detection of Compiler Optimization Problems**
   - 地址：https://arxiv.org/abs/2606.23512
   - 来源：arXiv (cs.SE)
   - 领域：Testing / Software Performance
   - 技术介绍：提出从应用二进制文件直接检测编译器优化问题的无源码框架。通过二进制反汇编、块级分类和加权分数聚合处理混合优化级别。对21,972个native库分析发现30.5%使用低优化级别，影响91.7%应用。修复后CPU指令减少10-63%，性能投诉降低42%。发现第三方库本身以低优化级别分发是根本原因。
   - 应用场景：移动应用性能测试、编译器优化验证
   - 为什么值得关注：揭示被忽视的编译器优化问题，30.5% native库未优化影响数亿用户

3. **AgentCIBench: Do Computer-Use Agents Follow Contextual Integrity?**
   - 地址：https://arxiv.org/abs/2606.23189
   - 来源：arXiv (cs.AI)
   - 领域：Testing / Agent Security
   - 技术介绍：评估CUA在跨应用操作中的上下文完整性违规。三种失败模式：视觉共定位（拉入UI相邻禁止项）、任务歧义过度共享（对模糊提示转储密集个人状态）、接收者错位（发送给不合适收件人）。15个frontier Agent中11个在>50%场景泄漏，平均泄漏67.9%。
   - 应用场景：CUA安全评估、隐私泄漏测试
   - 为什么值得关注：揭示CUA跨应用隐私风险被严重低估，11/15 Agent高泄漏率

4. **Litmus: Zero-Label, Code-Driven Metric Specification for Evaluating AI Systems**
   - 地址：https://arxiv.org/abs/2606.23403
   - 来源：arXiv (cs.AI)
   - 领域：Testing / Evaluation
   - 技术介绍：从零标签系统通过源码和目标询问设计AI管道评估指标。先识别"必须测量什么和为什么"，再转换为约束构建合理分阶段指标组合。在金融账户分组、科学QA、固有风险评估三个真实管道上评估，实现最宽覆盖、近零冗余、最高有效性（科学QA Spearman ρ=0.72 vs 基线<0.47）。
   - 应用场景：AI系统评估指标设计、自动化评估规范
   - 为什么值得关注：从"自动实现指标"推进到"自动规范指标"，评估基础设施范式转变

5. **EHR-Complex: Benchmarking Medical Agents for Complex Clinical Reasoning**
   - 地址：https://arxiv.org/abs/2606.23301
   - 来源：arXiv (cs.AI)
   - 领域：Testing / Medical AI
   - 技术介绍：基于MIMIC-IV（365K患者、31表、5亿+记录）构建52K任务交互式临床数据库推理基准。覆盖6种临床意图，支持患者级和群体级查询。平均31.93 SQL结构组件/查询。最佳模型仅62.3%精确匹配，Pass@4一致性<50%。三种失败模式：SQL逻辑错误、医疗代码查找失败、语义误解。
   - 应用场景：医疗Agent评估、临床推理测试
   - 为什么值得关注：最大规模交互式临床推理基准，暴露Agent在真实EHR分析中的差距

6. **HOLMES: Higher-Order Logic Meets Real-World Explainable Symbolic Reasoning**
   - 地址：https://arxiv.org/abs/2606.23238
   - 来源：arXiv (cs.AI)
   - 领域：Testing / LLM Evaluation

   - 应用场景：高阶逻辑推理评估、可验证LLM测试
   - 为什么值得关注：超越一阶逻辑基准，揭示高阶符号推理是关键瓶颈

7. **Automated Input Alphabet Construction for Stateful Protocol Learning**

   - 应用场景：协议模糊测试、状态机学习
   - 为什么值得关注：消除对手工协议专业知识的依赖，自动发现语义缺陷

### AI / LLM 技术 (12)

11. **SPIRAL: Sequential-Parallel-Aggregative Reinforcement Learning**
    - 地址：https://arxiv.org/abs/2606.23595
    - 来源：arXiv (cs.AI)
    - 领域：AI / RL
    - 技术介绍：统一训练LLM使用三种推理原语——序列推理、并行采样和聚合。模型并行采样独立轨迹集，然后生成聚合轨迹；所有组件针对最终聚合响应奖励端到端优化。使用集合RL教模型产生对聚合器有用的轨迹集，标准RL教聚合。在推理任务上比GRPO高11倍扩展效率和15%性能提升。
    - 应用场景：LLM推理优化、测试时计算扩展
    - 为什么值得关注：首次统一三种推理计算原语的RL训练框架

12. **DART: Draft-Agreement Routing for Training-Free Adaptive Thinking Budgets**
    - 地址：https://arxiv.org/abs/2606.23181
    - 来源：arXiv (cs.AI/cs.CL)
    - 领域：AI / Inference Optimization
    - 技术介绍：训练-free路由框架，采样两个低成本无思考草稿，草稿一致时接受直接回答，不一致时用草稿熵预测思考预算。数学推理上Olympiad级问题+9.0点准确率，思考token降15-69%。代码推理+22.5点，思考token降51-63%。跨0.6B-32B模型、多模型家族和API-only设置有效。
    - 应用场景：混合推理模型路由、推理成本优化
    - 为什么值得关注：无需训练数据或梯度更新即可实现自适应推理预算分配

13. **GIF: Geometric Information Flow Control for LLMs**
    - 地址：https://arxiv.org/abs/2606.23277
    - 来源：arXiv (cs.AI)
    - 领域：AI / Security
    - 技术介绍：语义信息流跟踪框架，用LLM Jacobian和局部输出几何上界Shannon互信息。满足局部几何可靠性，提供Lean 4机械化证明。在提示注入和隐私泄漏基准上接近完美召回，结合轻量解分类器匹配或超越GPT-5.5 xhigh F1，token成本降低81倍。小代理模型检测可迁移到更大SOTA模型。
    - 应用场景：LLM安全监控、信息流控制、提示注入检测
    - 为什么值得关注：首个可证明可靠的信息流控制框架，从启发式推进到理论保证

14. **Topology of Ill-Posed Questions: Persistent Homology for Detection and Steering**
    - 地址：https://arxiv.org/abs/2606.23590
    - 来源：arXiv (cs.AI)
    - 领域：AI / Topological Analysis
    - 技术介绍：用零维持久同调表征LLM内部状态几何，每层三个紧凑描述符（平均有限寿命、归一化寿命熵、最大寿命浓度）。跨三层连接得问题拓扑表示。引入拓扑条件激活引导，检索拓扑相似例子构建查询特定激活干预。三模型上平均准确率从67.4%提升到78.9%（AmbigQA），可接受响应率从61.4%提升到70.6%。
    - 应用场景：歧义问题检测、LLM行为引导
    - 为什么值得关注：首次用拓扑数据分析LLM内部状态，提供可解释的引导机制

15. **VeriEvol: Scaling Multimodal Mathematical Reasoning via Verifiable Evol-Instruct**

16. **Abstract Representational Geometry Supports Inference in LLMs**

1. **AOHP: Android Open Harness Project - OS-Level Agent Harness**
   - 地址：https://arxiv.org/abs/2606.23449
   - 项目介绍：AOHP是首个基于AOSP构建的OS级Agent Harness，核心设计理念是将AI Agent作为一等OS Actor。当前移动操作系统为应用中心设计，Agent运行在应用层之上导致执行开销和安全风险。AOHP通过提供自适应UI和Agent友好运行时环境，在保留Android成熟软件生态的同时为Agent提供原生OS支持。
   - 技术架构：基于AOSP构建，包含Agent运行时引擎、自适应UI框架、安全隔离层和工具调用接口。Agent作为一等公民获得系统级资源调度和权限管理。
   - 核心算法：Agent意图理解→OS级资源调度→自适应UI生成→安全执行
   - 关键实现机制：保留Android应用兼容性的同时提供Agent专用API，支持多Agent并发和跨Agent通信
   - 与现有技术对比：区别于CUA（Computer-Use Agent）在UI层操作的方式，AOHP在OS层提供原生支持，减少中间层开销

### 学术论文

1. **SPIRAL: Sequential-Parallel-Aggregative Reinforcement Learning**
   - 地址：https://arxiv.org/abs/2606.23595
   - 论文介绍：首次统一三种推理计算原语（序列、并行、聚合）的RL训练框架。模型并行采样独立轨迹集，然后生成聚合轨迹；所有组件针对最终聚合响应奖励端到端优化。比GRPO高11倍扩展效率和15%性能提升。
   - 技术架构：集合RL（教模型产生有用轨迹集）+ 标准RL（教聚合）
   - 核心算法：SET-RL用于轨迹多样性、标准RL用于聚合质量
   - 关键实现机制：端到端优化三种原语的组合使用
   - 与现有技术对比：现有方法只优化序列推理，SPIRAL统一三种原语
   - 优势与局限：优势是扩展效率显著提升；局限是训练复杂度增加
   - 未来发展潜力：可能成为推理优化的标准RL训练范式
   - 个人技术理解：推理计算从单一原语推进到多原语组合优化

2. **DART: Draft-Agreement Routing for Adaptive Thinking Budgets**
   - 地址：https://arxiv.org/abs/2606.23181
   - 论文介绍：训练-free路由框架，采样两个低成本无思考草稿，一致时直接回答，不一致时用草稿熵预测思考预算。Olympiad级数学+9.0点，代码+22.5点，思考token降51-69%。
   - 技术架构：双草稿采样→一致性检查→熵预算预测
   - 核心算法：草稿一致性作为简单/困难问题分类器，熵作为预算预测器
   - 关键实现机制：无需训练数据或梯度更新
   - 与现有技术对比：现有路由需要标注数据或固定预算，DART完全训练-free
   - 优势与局限：优势是即插即用零成本；局限是草稿质量依赖基础模型
   - 未来发展潜力：可能成为混合推理模型的标配路由策略
   - 个人技术理解：用极低成本实现自适应推理预算分配，工程价值极高

1. **Agent安全从通用对齐走向精细化维度**：AgentCIBench揭示CUA跨应用隐私泄漏（67.9%平均泄漏率）、GIF提供可证明信息流控制、ToolPrivBench揭示过度特权工具选择。Agent安全正在从"是否对齐"推进到"在哪些维度对齐"。

5. **科学ML从强制先验走向自适应先验决策**：SPADE提供闭式框架决定何时以及多强施加物理结构先验，用规范测试和James-Stein收缩提供理论保证。
