### AI Testing / Software Testing (11)

1. **MANGO: Multi-Agent Test Oracle Generation for VLA Models**
   - 地址：https://arxiv.org/abs/2606.24815
   - 来源：arXiv (cs.SE)
   - 领域：Testing / 机器人测试
   - 技术介绍：针对VLA（Vision-Language-Action）机器人模型测试中手动构建符号测试预言成本高昂、领域耦合严重的问题。MANGO通过多Agent框架从自然语言描述自动生成细粒度预言：首先生成可复用原子任务库，然后为每个原子任务生成模拟器 grounded 的预言定义，最后将复杂指令分解为有序原子动作序列和对应预言。Generator、Assessor、Judge三个Agent通过结构化反馈迭代精炼生成物。在LIBERO_10和RoboCasa Humanoid Tabletop基准上验证，生成的可执行细粒度预言能检测与符号预言相似数量的失败，同时精确定位故障并提供更丰富的诊断信息。
   - 应用场景：机器人VLA模型测试、自动化测试预言生成
   - 为什么值得关注：解决机器人测试最大瓶颈——预言构建，从"手工编写预言"推进到"自然语言驱动自动生成"

2. **Cascading Maintenance Burden: Two-Level vs Multi-Level Modelling**
   - 地址：https://arxiv.org/abs/2606.24721
   - 来源：arXiv (cs.SE)
   - 领域：Testing / 模型驱动工程
   - 技术介绍：在模型驱动工程(MDE)中，当核心定义变化时，每个依赖工件都必须更新。主流的两级建模(2LM)将领域知识分散到元模型和模型工件中，导致共进化成为不一致性和工作量的持续来源。多级建模(MLM)统一这些工件，但从未在受控配对比较中测试过。本文提出预注册的基于变异的实证比较框架，从已发表的2LM共进化场景构建语义等价的MLM对应物，应用相同的进化变异，通过自动化一致性检查和预注册假设测试测量结果。
   - 应用场景：MDE工具评估、模型共进化分析
   - 为什么值得关注：首次提供评估范式级结构选择如何影响级联维护负担的实证框架

3. **Bit-Precise Conformance Testing of Simulink Model Checkers**
   - 地址：https://arxiv.org/abs/2606.24719
   - 来源：arXiv (cs.SE)
   - 领域：Testing / 形式化验证
   - 技术介绍：MATLAB/Simulink是CPS开发的事实标准，但Simulink Design Verifier(SLDV)和SMT-based模型检测器(SmtMC)的可靠性因缺乏覆盖每个模型元素和数值行为细节的语义而存疑。本文形式化Simulink基本块类型，定义块类型特征集和比特精确行为，提出测试比特精确一致性关系的方法。对10种块类型的实验发现SmtMC高效通过所有测试，而SLDV在一致性(i)和(ii)上仅达到94-96%和80-90%的通过率。分析了失败原因：错误、边界情况和超时。
   - 应用场景：Simulink模型检测器验证、CPS安全关键系统
   - 为什么值得关注：揭示工业级模型检测器的实际一致性差距，QRS 2026接收论文

   - 应用场景：软件文档自动化、需求分析加速

5. **BigBag: Agentic Generation of AST Transformation Rules for Breaking Updates**
   - 地址：https://arxiv.org/abs/2606.24446
   - 来源：arXiv (cs.SE)
   - 领域：Testing / API兼容性
   - 技术介绍：第三方库持续演进引入破坏性API变更，阻止客户端代码编译。现有修复方法生成项目特定补丁无法复用。BigBag框架生成修复转换——在API级别编码修复逻辑的结构化可执行程序，可迁移到被同一更新破坏的任何客户端。在BUMP基准的157个编译失败破坏性依赖更新上评估，结合4个LLM和2个AST转换引擎(Spoon/JavaParser)。最佳配置达到94.3%可编译转换率和78.6%修复率。生成的转换跨项目迁移，总体跨项目修复率33.3%，在受影响API元素统一调用的更新中达80%+。
   - 应用场景：API兼容性维护、依赖更新自动化
   - 为什么值得关注：从"每个项目独立修复"推进到"一次生成跨项目复用"

7. **AutoSpec: Safety Rule Evolution for LLM Agents via ILP**
   - 地址：https://arxiv.org/abs/2606.24245
   - 来源：arXiv (cs.SE/cs.CR)
   - 领域：Testing / Agent安全
   - 技术介绍：LLM Agent的自主性带来安全风险：可能执行破坏性命令、泄露敏感数据或违反领域约束。现有安全方法面临根本权衡：手工规则可解释但脆弱，过于保守的规则阻止安全操作（高假阳性），宽松规则遗漏不安全行为（高假阴性）。AutoSpec通过反例引导的归纳逻辑合成(CEGIS)结合归纳逻辑编程(ILP)自动演化部署的专家设计安全规则。从专家规则和注释轨迹流开始，迭代评估规则、挖掘假阳性和假阴性反例、用ILP学习区分谓词、生成候选规则编辑、验证选择最佳修订。ILP高效识别在假阴性中频繁出现但在假阳性中罕见的谓词，大幅修剪规则编辑的指数搜索空间。在291个执行轨迹上评估，F1提升至0.98和0.93，假阳性降低高达94%，4-5次迭代收敛。
   - 应用场景：Agent安全规则自动演化、安全关键部署
   - 为什么值得关注：从"静态手工规则"推进到"自动演化可解释规则"，ILP引导是关键创新

8. **LemonHarness: Integrated Execution Framework for Long-Horizon Agents**
   - 地址：https://arxiv.org/abs/2606.24311
   - 来源：arXiv (cs.AI)
   - 领域：Testing / Agent执行框架
   - 技术介绍：长周期Agent在多轮迭代中修改工作区状态，但通常只观察工具输出和日志片段，实际状态变化发生在文件系统中。LemonHarness建立显式执行边界，将状态改变操作约束在明确定义的工作空间内，将模型调用、工具执行和规则知识纳入单一受控边界。引入可复用规则知识库，将重复执行规则和验收标准转化为运行时知识。添加时间感知执行机制，向模型暴露经过和剩余预算，使其在时间压力变化时重新平衡探索、实施和验证工作。在Terminal-Bench 2.0上，LemonHarness_GPT-5.3-CodeX达到84.49%准确率，GPT-5.5骨干达到86.52%。
   - 应用场景：长周期Agent执行、终端任务自动化
   - 为什么值得关注：解决长周期Agent状态管理核心痛点，Terminal-Bench 2.0最高精度

9. **Bayesian Control for Coding Agents**

   - 来源：arXiv (cs.AI)
   - 领域：Testing / 编码Agent控制
   - 技术介绍：现代编码Agent将LLM生成器与各种工具（包括廉价诊断和昂贵验证器）配对。工具使用决策通常由使用固定规则且忽略不确定性的编排器控制。将编排建模为成本敏感的序贯假设检验：贝叶斯控制器维护候选正确性的信念，动态决定收集更多证据、精炼候选、验证或停止。在6个生成器和9个编码基准上评估，证明当验证成本高且批评者信息丰富但不完美时，贝叶斯控制最有价值。信念状态产生可解释的正确性分数，在不确定性量化上超越token概率和原始工具成功基线。

   - 为什么值得关注：首次将序贯假设检验形式化应用于编码Agent编排

10. **Agentic-LTPO: Bilevel Long-Term Optimization for Policy-Driven Physical Layer**

11. **Toward Self-Evolution-Ready Workflow Harnesses: Reversible Migration for Expert LLM Pipelines**

12. **OpenThoughts-Agent: Data Recipes for Agentic Models**

    - 领域：AI / Agent训练数据
    - 技术介绍：Agentic语言模型大幅扩展AI应用，但如何策划训练数据以支持广泛能力的Agent几乎无人公开知晓。现有开源工作如SWE-Smith、SERA和Nemotron-Terminal通常针对单一基准，留下如何训练跨多样Agentic任务泛化的模型的问题。OpenThoughts-Agent(OT-Agent)项目提供完全开放的数据策划管道，进行100+受控消融实验系统调查每个阶段，得出关于任务来源和多样性重要性的洞察。组装100K示例训练集并在Qwen3-32B上微调，在7个Agent基准上平均精度44.8%，比最强现有开源数据Agent模型(Nemotron-Terminal-32B, 40.9%)提升3.9pp。训练数据表现出强扩展特性，在计算控制比较中每个训练集大小都超越替代开源数据集。

13. **World Models in Pieces: Structural Certification for General Agents**
    - 地址：https://arxiv.org/abs/2606.24842
    - 来源：arXiv (cs.AI)
    - 领域：AI / 理论保证
    - 技术介绍：在大世界范式中，Agent不可能普遍能力，其能力不可避免地专业化于分片世界模型。因此标准一致保证无法区分关键瓶颈理解和无关失败。首先形式化此限制，证明一般Agent不是通用的，使标准最坏情况分析失去信息量。引入结构认证——转换局部框架，将有界目标引导性能映射到Agent内部世界模型的逐条目保证。主要贡献是构造性的：提供使用深度组合目标过滤特定转换的算法，证明一般Agent在这些目标上具有O(1/n)+O(δ)误差界的结构世界模型。反之，此界在小δ范式中是紧的，其存在由认证明确保证。这些结果使一般Agent的可认证部署成为可能，通过定位长周期规划可靠的特定转换。ICML 2026 camera-ready。
    - 应用场景：Agent理论保证、可认证部署
    - 为什么值得关注：首次为一般Agent提供结构认证框架，ICML 2026

14. **Grading the Grader: Lessons from Evaluating an Agentic Data Analysis System**

    - 领域：AI / 评估方法论
    - 技术介绍：Agentic数据分析系统产生丰富输出（代码、数值结果和语言诊断），比单轮LLM响应更难评估。必须区分Agent输出与ground-truth答案之间的真正分歧与评分伪影。调查自动化评分器评估此类系统的可靠性及提高评分质量的策略。在DSGYM的153个数值QRData任务上应用LAMBDA多Agent数据分析系统。开发三层人-AI评分级联：严格正则匹配、LLM宽松评分和片段人工检查。两个自动化评分器均达到100%观察精确度(0/70假阳性)。宽松评分器对人类标签的召回率97%。关键词锚定提取管道将严格评分器召回率提高60pp。迭代nudge机制将评分运行成功率从36%提升到97%。
    - 应用场景：Agentic系统评估、自动化评分质量
    - 为什么值得关注：揭示Agentic系统评估中评分器本身的质量问题，三层级联方案

    - 应用场景：AI训练基础设施、CUDA原生框架
    - 为什么值得关注：C++/CUDA原生训练框架，22%内存减少+更高吞吐量

16. **Can Scale Save Us From Plasticity Loss in LLMs?**
    - 地址：https://arxiv.org/abs/2606.24752

OpenThoughts-Agent(OT-Agent)是首个完全开放的Agentic模型训练数据管道。当前开源Agent训练工作（SWE-Smith、SERA、Nemotron-Terminal）普遍针对单一基准，缺乏跨多样Agentic任务泛化的数据策划方法论。OT-Agent填补这一空白，提供从数据收集、过滤、增强到最终训练集构建的完整开放管道。团队进行了100+受控消融实验，系统调查管道每个阶段——包括任务来源多样性、数据质量过滤策略、增强技术选择等——得出关于如何构建有效Agent训练数据的关键洞察。

最终组装的100K示例训练集在Qwen3-32B上微调，在7个Agent基准（覆盖编码、终端操作、网页交互等多领域）上达到平均44.8%精度，比最强现有开源数据Agent模型Nemotron-Terminal-32B（40.9%）提升3.9个百分点。更重要的是，训练数据表现出强扩展特性：在计算控制比较中，每个训练集大小下都超越替代开源数据集。

- 技术架构：数据收集→质量过滤→多样性增强→训练集构建→模型微调→多基准评估
- 核心算法：100+消融实验系统验证每个管道阶段，计算控制的缩放比较
- 关键实现机制：开放数据管道+开放训练集+开放模型+开放实验数据
- 与现有技术对比：超越所有现有开源Agent数据方案，在计算效率和质量上双重领先
- 优势与局限：优势在于完全开放和系统性验证；局限在于当前仅验证Qwen3-32B，其他架构的泛化性待验证
- 未来发展潜力：可能成为开源Agent训练数据的黄金标准，推动开源Agent生态从"模型开放"推进到"数据+模型双开放"
- 个人技术理解：Agent训练数据的质量比数量更重要。OT-Agent的100+消融实验揭示的关键洞察——任务来源多样性和数据质量过滤的相对重要性——对后续Agent训练具有指导意义。

#### 2. AutoSpec: Safety Rule Evolution for LLM Agents via ILP
- 地址：https://arxiv.org/abs/2606.24245
- 项目介绍（300字）:
AutoSpec解决LLM Agent安全规则的核心困境：手工规则可解释但脆弱，神经网络分类器准确但不可解释。现有方法面临根本权衡——过于保守的规则阻止安全操作（高假阳性），宽松规则遗漏不安全行为（高假阴性）。AutoSpec通过反例引导的归纳逻辑合成(CEGIS)结合归纳逻辑编程(ILP)自动演化部署的专家设计安全规则。

工作流程：从专家规则和注释轨迹流开始，迭代评估规则→挖掘假阳性和假阴性反例→用ILP学习区分谓词→生成候选规则编辑→验证选择最佳修订。ILP的关键洞察是高效识别在假阴性中频繁出现但在假阳性中罕见（或反之）的谓词，大幅修剪规则编辑的指数搜索空间。迭代持续直到收敛，产生平衡精确度和召回率的可解释规则。

在291个执行轨迹（覆盖代码执行和具身Agent域）上评估，AutoSpec将规则F1提升至0.98和0.93，假阳性降低高达94%，4-5次迭代收敛。ILP引导方法比启发式CEGIS实现高达4.8x更高的F1。学习的规则人类可读、可审计、泛化到未见场景。

- 技术架构：专家规则→CEGIS循环→ILP谓词发现→候选生成→验证选择→收敛规则
- 核心算法：CEGIS+ILP联合优化，谓词频率分析引导搜索空间修剪
- 关键实现机制：反例引导的迭代规则演化，ILP作为谓词发现引擎
- 与现有技术对比：超越手工规则（F1 0.98 vs ~0.7）和神经网络（可解释性），ILP引导比启发式CEGIS高4.8x F1
- 优势与局限：优势在于可解释+高精度+快速收敛；局限在于依赖初始专家规则质量和注释轨迹覆盖度
- 未来发展潜力：可能成为Agent安全规则管理的标准工具，从"静态规则"推进到"动态演化规则"
- 个人技术理解：AutoSpec的核心创新是用ILP解决规则编辑搜索空间的指数爆炸问题。传统CEGIS在规则编辑空间中盲目搜索，而ILP通过频率分析精准定位区分性谓词，将搜索空间从指数级降低到可管理级别。这是"符号AI+机器学习"融合的优秀案例。

#### 3. BigBag: Agentic Generation of Reusable Fixing Transformations
- 地址：https://arxiv.org/abs/2606.24446
- 项目介绍（250字）:
第三方库持续演进引入破坏性API变更，阻止客户端代码编译。现有修复方法（如Dependabot的自动PR）生成项目特定补丁，无法跨项目复用——每个受影响项目必须独立修复，即使它们被同一库更新破坏。BigBag框架生成修复转换：在API级别编码修复逻辑的结构化可执行程序，可迁移到任何被同一更新破坏的客户端。

在BUMP基准的157个编译失败破坏性依赖更新上评估，结合4个LLM和2个AST转换引擎(Spoon和JavaParser)。最佳配置达到94.3%可编译转换率和78.6%修复率。生成的转换跨项目迁移，总体跨项目修复率33.3%，在受影响API元素统一调用的更新中达80%+。

- 技术架构：AST分析→LLM生成转换规则→跨项目验证→可执行转换输出
- 核心算法：Agentic AST转换规则生成，跨项目迁移验证
- 关键实现机制：API级别修复逻辑编码，Spoon/JavaParser AST引擎
- 与现有技术对比：从"每个项目独立修复"推进到"一次生成跨项目复用"
- 优势与局限：优势在于跨项目可迁移性；局限在于对API调用模式不一致的更新迁移率较低
- 未来发展潜力：可能改变依赖管理范式，从"被动修复"推进到"主动生成可迁移修复"
- 个人技术理解：BigBag揭示了一个重要洞察——大多数破坏性更新的影响是结构化的（相同API元素以相同方式被调用），这使得跨项目修复迁移成为可能。

#### 1. World Models in Pieces: Structural Certification for General Agents (ICML 2026)
- 地址：https://arxiv.org/abs/2606.24842
- 论文介绍（300字）:
在"大世界"范式中（Agent面对开放世界而非封闭环境），Agent不可能普遍能力——其能力不可避免地专业化于分片世界模型。因此标准一致保证（uniform guarantees）无法区分关键瓶颈理解和无关失败。此论文首先形式化此限制，证明一般Agent不是通用的，使标准最坏情况分析失去信息量。

核心贡献是引入"结构认证"——转换局部框架，将有界目标引导性能映射到Agent内部世界模型的逐条目保证。主要技术贡献是构造性的：提供使用深度组合目标过滤特定转换的算法，证明一般Agent在这些目标上具有O(1/n)+O(δ)误差界的结构世界模型。反之，此界在小δ范式中是紧的，其存在由认证明确保证。

这些结果使一般Agent的可认证部署成为可能——通过定位长周期规划可靠的特定转换，而非声称全局可靠性。ICML 2026 camera-ready版本。

- 技术架构：理论证明→结构认证算法→误差界分析→可认证部署指南
- 核心算法：深度组合目标过滤、转换局部保证映射
- 关键实现机制：将全局保证分解为逐条目(entry-wise)保证
- 与现有技术对比：超越标准最坏情况分析，提供实用的局部保证
- 优势与局限：优势在于为Agent部署提供可认证保证；局限在于理论性强，实际系统验证有限
- 未来发展潜力：可能成为Agent安全部署的理论基础，类似形式化验证在传统软件中的地位
- 个人技术理解：此论文的核心理念是"承认不完美，但量化不完美"。与其追求不可能的一般Agent全局保证，不如提供可计算的局部保证——在哪些转换上Agent是可靠的，误差界是多少。这对高 stakes Agent部署至关重要。

#### 2. ReM-MoA: Reasoning Memory Sustains Mixture-of-Agents Scaling
- 地址：https://arxiv.org/abs/2606.24437
- 论文介绍（250字）:
Mixture-of-Agents(MoA)架构通过组织多个LLM Agent到分层推理管道改善推理时扩展。但现有MoA变体无法在深度增加时维持增益——表现出退化、早期平台或饱和。ReM-MoA提出记忆增强MoA框架，通过两种机制维持扩展：(1)排名推理记忆——持久存储和排名所有层的推理轨迹，使用比较性Reviewer Agent；(2)策划多样化记忆路由——暴露不同Agent到成功和失败轨迹的不同组合，在传播高质量推理的同时保持探索多样性。

在数学、形式逻辑、代码、知识和常识的5个推理基准上，ReM-MoA一致超越先前MoA变体，优势随深度扩大。引入可选多域Reviewer蒸馏管道通过前沿模型监督改善排名质量。

- 技术架构：分层Agent管道→推理记忆存储→排名路由→多样化暴露
- 核心算法：排名推理记忆+多样化路由，Reviewer蒸馏
- 关键实现机制：跨层推理轨迹持久化和比较性排名
- 与现有技术对比：解决MoA深度扩展饱和问题，优势随深度扩大
- 优势与局限：优势在于简单有效；局限在于Reviewer质量依赖前沿模型
- 未来发展潜力：可能成为推理时扩展的标准架构
- 个人技术理解：ReM-MoA揭示MoA的核心瓶颈不是Agent数量而是信息流动质量。结构化跨层记忆解决了"深层Agent看不到浅层学到什么"的问题。

#### 3. Detecting AI Coding Agents in Open Source: Census of 180M Repositories
- 地址：https://arxiv.org/abs/2606.24429
- 论文介绍（250字）:
生成式AI编码Agent进入开源供应链但踪迹多样且常常不可见。引入多层检测框架，整合配置文件扫描、commit消息分析、作者身份匹配和bot签名查找，覆盖World of Code(180M+ Git仓库)。将Agent踪迹分为4种行为类型。

关键发现：单一方法只能捕获一小部分活动。多方法检测在一个快照中识别850,157个Claude Code commit，其中bot账户查找（大多数采用研究依赖的信号）仅恢复28,154个(3.3%)——30倍相对召回差距。这意味着单信号采用估计严重低估(至少30x)。495个标签的手工验证，每个单元的精确度和Wilson置信区间。

跨2024年12月到2026年4月的快照，commit归因Agent每月生成超过320,000个commit。Claude Code领先(886,122 commit跨17,295项目)，主导静默的仅配置文件采用(21,078项目)。与独立PR普查(AIDev)比较，两个通道捕获几乎不相交的Agent群体——PR普查遗漏79%的commit检测Claude Code采用者和几乎所有Codex采用者。

- 技术架构：多层检测（配置+commit+作者+bot签名）→交叉验证→普查分析
- 核心算法：多方法融合检测，手工验证精确度
- 关键实现机制：World of Code 180M仓库规模扫描
- 与现有技术对比：首次大规模AI编码Agent采用普查，揭示单信号检测严重低估
- 优势与局限：优势在于规模和验证严谨性；局限在于快照时间覆盖有限
- 未来发展潜力：可能成为开源供应链AI Agent采用追踪的标准方法
- 个人技术理解：此论文揭示AI编码Agent采用被严重低估——30x差距意味着当前所有基于单一信号的研究都严重失真。多层检测是必须的。

## 技术趋势
1. **Agent训练数据管道成熟化**: OpenThoughts-Agent的100+消融实验和100K训练集标志着Agent训练从"模型中心"推进到"数据+模型双中心"。数据质量、多样性和可扩展性成为Agent能力的关键决定因素。
2. **Agent安全规则从静态到动态演化**: AutoSpec用ILP自动演化安全规则，从"手工编写静态规则"推进到"基于反例自动演化可解释规则"。这代表了Agent安全治理的范式转变。
3. **开源AI编码Agent采用被严重低估**: 180M仓库普查揭示单信号检测低估30x，每月320K+ commit由AI Agent生成。开源供应链正在被AI Agent深刻重塑，但追踪方法严重滞后。
4. **推理时扩展架构从MoA到记忆增强**: ReM-MoA解决MoA深度扩展饱和问题，证明结构化跨层推理记忆是维持MoA扩展的关键缺失机制。
5. **去中心化AI训练基础设施兴起**: BlockTrain证明模型可划分为独立训练块在分布式GPU上训练，打破集中式训练垄断。

## 新领域收获
1. **Token经济学(AI Tokenomics)**: 首个系统性框架研究token作为AI服务会计单位的生成、消费、定价、分配和优化。连接token级技术成本到工作流级经济价值，揭示token支出≠经济价值的核心洞察。
