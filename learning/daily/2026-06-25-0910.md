### AI Testing / Software Testing (10)

1. **Execution Bottlenecks in Screen-Only and Skill-Mediated Computer-Use Agents**
   - 地址: https://arxiv.org/abs/2606.24551
   - 来源: arXiv (cs.AI)
   - 领域: Testing / Computer Use
   - 技术介绍: 现有CUA评估混淆了交互模态与任务差异。本文提出匹配执行层基准，440个桌面任务覆盖18个应用12个工作流类别，GUI Agent和CLI Agent接收相同目标状态和验证器。最强GUI Agent达到59.1%完全通过率，超过原始技能CLI Agent的48.2%；但验证器引导技能增强将CLI成功率提升至69.3%，表明CLI缺陷主要来自技能覆盖不完整而非模型能力不足。GUI和CLI暴露不同执行瓶颈：GUI受限于长周期工作流的可靠地面交互，CLI受限于技能接口的覆盖和可扩展性。
   - 应用场景: Computer Use Agent评估、GUI vs CLI交互模态比较
   - 为什么值得关注: 首次提供受控的模态对比基准，揭示了GUI和CLI各自的根本瓶颈

2. **Reinforcement Learning for Computer-Use Agents with Autonomous Evaluation**
   - 地址: https://arxiv.org/abs/2606.24515
   - 来源: arXiv (cs.AI/cs.HC)
   - 领域: Testing / RL for CUA
   - 技术介绍: CUA的RL微调困难在于开放桌面环境缺乏可扩展的机器可读奖励信号。提出使用自主VLM评估作为可扩展监督信号：给定最终截图和原始指令，VLM判断任务完成并提供终端反馈。将自主评估器反馈建模为噪声二元奖励通道，推导噪声校正奖励估计器用于PPO。在macOSWorld、Windows Agent Arena和OSWorld上，校正评估器奖励比零-shot基线平均提升12.6个百分点，比原始评估器微调提升5.1个百分点。
   - 应用场景: GUI Agent RL训练、自动化评估奖励信号
   - 为什么值得关注: 解决了CUA RL训练的最大瓶颈——奖励信号获取

3. **AdversaBench: Automated LLM Red-Teaming with Multi-Judge Confirmation**
   - 地址: https://arxiv.org/abs/2606.24589
   - 来源: arXiv (cs.AI)
   - 领域: AI Testing / Security
   - 技术介绍: 端到端红队管道，5种结构化变异算子突变种子提示，三法官面板确认失败+元法官裁决。45个种子（推理/指令跟随/工具使用）全部产生确认失败。算子有效性因类别差异显著：inject_distractor在指令跟随上得0.00但在推理和工具使用上得0.80-0.83。二元失败率掩盖难度差异。80-87%法官配对一致性与接近零的Cohen's kappa共存。对抗提示从Llama 3.1 8B零样本迁移到Llama 3.3 70B。
   - 应用场景: LLM安全评估、自动化红队测试
   - 为什么值得关注: 揭示了跨模型攻击迁移性，变异算子利用的是通用行为模式而非特定模型弱点

4. **Bayesian Control for Coding Agents**
   - 地址: https://arxiv.org/abs/2606.24453
   - 来源: arXiv (cs.AI)
   - 领域: Testing / Agent Orchestration
   - 技术介绍: 现代编码Agent将LLM生成器与各种工具配对，工具使用决策通常由固定规则编排器控制。将编排建模为成本敏感的序贯假设检验：贝叶斯控制器维护候选正确性信念，动态决定收集更多证据、精炼候选、验证或停止。在6个生成器和9个编码基准上，当验证成本高且批评者信息丰富但不完美时最有价值。信念状态产生可解释的正确性分数，优于token概率和原始工具成功基线。
   - 应用场景: 编码Agent工具编排、不确定性量化
   - 为什么值得关注: 将Agent工具选择从启发式规则推进到概率推理框架

5. **Cycle-Consistent Neural Explanation of Formal Verification Certificates**
   - 地址: https://arxiv.org/abs/2606.24414
   - 来源: arXiv (cs.AI)
   - 领域: Testing / Formal Verification
   - 技术介绍: 形式化验证产生机器可检查证书，但对非专业利益相关者不透明。提出周期一致性神经网络：前向网络NN1将证书映射到解释，反向网络NN2从解释重建证书，符号验证器关闭循环提供可微忠实度代理。指针生成机制确保从证书复制状态名称。420个测试证书覆盖6种验证方法，混合推理路由策略达到90.0%周期验证健全性，超越多LLM few-shot基线13.9个百分点，推理速度快860倍。
   - 应用场景: 形式化验证结果解释、证书可读性
   - 为什么值得关注: 训练专业化模型超越通用LLM提示，消除云端推理约束

6. **Governed Shared Memory for Multi-Agent LLM Systems (MemClaw)**
   - 地址: https://arxiv.org/abs/2606.24535
   - 来源: arXiv (cs.AI)
   - 领域: Testing / Multi-Agent Memory
   - 技术介绍: 多Agent LLM环境需要稳健的共享知识管理机制。形式化舰队记忆问题，识别四种基础失败模式：未经授权泄漏、陈旧传播、矛盾持久化、来源崩溃。定义系统级原语：范围检索、时间替代、来源追踪、策略治理记忆传播。在MemClaw生产多租户记忆服务中实现，通过ArgusFleet可复现harness评估四个治理维度。成功重建100%深度四推导链，零跨舰队泄漏。发现不对称范围执行和管道排序冲突等生产问题。
   - 应用场景: 多Agent系统记忆管理、生产级记忆服务
   - 为什么值得关注: 首次对生产多Agent记忆服务进行实证评估，揭示设计层面无法发现的执行失败

7. **LemonHarness Technical Report**

### AI / LLM 技术 (12)

11. **Can Scale Save Us From Plasticity Loss in LLMs?**
    - 地址: https://arxiv.org/abs/2606.24752
    - 来源: arXiv (cs.AI)
    - 领域: AI / LLM Training
    - 技术介绍: 可塑性损失——网络在学习旧信息后学习新信息的能力下降——是持续学习的根本挑战。在GPT-style Transformer模型上研究5M到314M参数的可塑性损失，通过越南语探测任务测量。发现可塑性损失遵循可预测的缩放定律，随模型大小次线性增长。更大模型可能延迟可塑性损失的可测量效应，但仅增加参数数量可能不足以完全防止它。在稳态多语言训练下也发现可塑性损失证据。
    - 应用场景: LLM持续学习、模型训练策略
    - 为什么值得关注: 挑战"更大模型自动解决可塑性问题"的假设

12. **Scaling Laws for Task-Specific LLM Distillation**
    - 地址: https://arxiv.org/abs/2606.24747
    - 来源: arXiv (cs.AI)
    - 领域: AI / Model Compression
    - 技术介绍: 推导领域特定LLM压缩的经验缩放定律，量化领域知识和通用知识如何随数据集大小、压缩比、监督格式和迭代修剪计划缩放。使用量化金融领域，比较基于logit和基于LoRA的蒸馏。引入混合思维链监督损失稳定KL散度蒸馏。领域任务质量在压缩下可预测退化，而通用知识基准在同一时间点之前崩溃。监督格式是这种权衡的关键驱动因素。
    - 应用场景: 领域特定模型压缩、蒸馏策略选择
    - 为什么值得关注: 为领域模型压缩提供可操作的缩放定律框架

13. **On the Smallness of the Large Language Models Scaling Exponents**
    - 地址: https://arxiv.org/abs/2606.24504
    - 来源: arXiv (cs.AI)
    - 领域: AI / Scaling Laws
    - 技术介绍: 讨论当前LLM应用的缩放指数为何指示能源资源不可持续的状态。证明将指数小归因于忽略无限数据极限下损失函数非零值（"基座效应"）的数值偏差不消除不可持续问题。基于与流体湍流现象模型的类比，评论数据平滑度对缩放指数的影响。
    - 应用场景: LLM缩放定律理论分析、AI能源可持续性
    - 为什么值得关注: 从物理角度质疑当前LLM缩放路径的可持续性

14. **ReM-MoA: Reasoning Memory Sustains Mixture-of-Agents Scaling**
    - 地址: https://arxiv.org/abs/2606.24437
    - 来源: arXiv (cs.AI)
    - 领域: AI / Multi-Agent Inference
    - 技术介绍: MoA架构通过多层推理管道组织多个LLM Agent改进推理时缩放，但现有MoA变体在深度增加时无法维持增益。ReM-MoA通过两种机制维持缩放：(1)排名推理记忆持久存储和排名所有层的推理轨迹；(2)策划多样化记忆路由使不同Agent暴露于成功和失败轨迹的不同组合。在5个推理基准上持续超越先前MoA变体，优势随深度扩大。
    - 应用场景: 多Agent推理、MoA架构优化
    - 为什么值得关注: 结构化的跨层推理记忆是可扩展多Agent推理的关键缺失机制

15. **CompressKV: Semantic-Retrieval-Guided KV-Cache Compression**
    - 地址: https://arxiv.org/abs/2606.24467
    - 来源: arXiv (cs.AI)
    - 领域: AI / Inference Optimization
    - 技术介绍: 长上下文LLM推理受KV缓存内存足迹和解码成本限制。CompressKV识别语义检索头捕获提示的初始和最终token以及语义重要的中间证据，用它们选择应保留KV对的token。根据层间驱逐误差的离线估计分配层间缓存预算。在LongBench上仅用3% KV缓存保持97%全缓存性能，在Needle-in-a-Haystack上仅用0.7% KV存储达到90%准确率。
    - 应用场景: 长上下文推理优化、资源受限部署
    - 为什么值得关注: 极致的KV缓存压缩比，资源-性能权衡显著改善

16. **The Latent Bridge: Continuous Slow-Fast Channel for Real-Time Game Agents**
    - 地址: https://arxiv.org/abs/2606.24470
    - 来源: arXiv (cs.AI)
    - 领域: AI / Real-Time Agents
    - 技术介绍: 实时通用计算Agent必须在数十毫秒内行动同时仍进行秒级规划。推理VLM(Qwen3-VL-8B-Thinking)有效但需要~1.5s/响应，对15Hz控制循环太慢。反应式VLM(MiniCPM-o 4.5)在毫秒内行动但在规划重任务上表现不佳。引入学习的连续潜桥(L)将慢模型的残差投影到快模型的输入嵌入空间，避免任何文本往返。在7个Atari游戏和MetaDrive上，潜桥在每个领域匹配或超越文本桥。
    - 应用场景: 实时游戏Agent、慢快系统耦合
    - 为什么值得关注: 首次实现推理和反应模型的无缝潜空间耦合

17. **DigenRL: Disaggregated RL for Visual Generative LLMs**
    - 地址: https://arxiv.org/abs/2606.24369
    - 来源: arXiv (cs.AI)
    - 领域: AI / RL Infrastructure
    - 技术介绍: 扩散导向RL算法(DanceGRPO、FlowGRPO)快速扩展RL范围到扩散视觉生成，但高效RL系统仍被探索不足。DigenRL支持灵活资源分配、异构GPU和高效任务调度。提出生成轴管道(GAP)和时间步并行(TSP)、弹性训练器辅助生成(TAG)、紧一步约束异步策略。在16-32GPU上对HunyuanVideo-13B、Wan2.1-14B等模型，吞吐量比veRL-Omni和GenRL提升1.56-2.10倍。
    - 应用场景: 扩散模型RL训练、异构GPU部署
    - 为什么值得关注: 首次为扩散生成LLM提供解耦RL框架

18. **ScaleToT: Generalizing Structured LLM Reasoning for Billion-Scale Low-Activity User Modeling**

    - 应用场景: 大规模用户建模、广告LTV预测
    - 为什么值得关注: 将LLM结构化推理扩展到十亿级用户的工程突破

19. **LaGO: Latent Action Guidance for Online Reinforcement Learning**

    - 领域: AI / RL
    - 技术介绍: 预训练LLM作为潜行动先验软引导在线策略优化，而非将LLM作为显式规划器或控制器。在CLEVR-Robot和Meta-World基准上，LaGO持续超越Vanilla PPO。CLEVR-Robot平均成功率从15.1%提升到27.2%，Meta-World从2.7%提升到15.2%。更强预训练LLM提供更有效引导。

    - 为什么值得关注: 避免LLM作为直接控制器的不可靠性

20. **ASALT: Adaptive State Alignment for Lateral Transfer in MARL**
    - 地址: https://arxiv.org/abs/2606.24601

#### 1. Execution Bottlenecks in Computer-Use Agents

**标题**: Screen-Only与Skill-Mediated Computer-Use Agent的执行瓶颈对比
**地址**: https://arxiv.org/abs/2606.24551

**论文介绍 (300字)**:
本文首次提供受控的CUA执行层基准，解决现有评估中交互模态与任务差异混淆的问题。440个桌面任务覆盖18个应用和12个工作流类别，GUI Agent和CLI Agent接收完全相同的目标、初始状态和最终状态验证器，仅限制在各自模态的原生操作上。

关键发现：最强GUI Agent达到59.1%完全通过率，超过原始技能CLI Agent的48.2%。但验证器引导技能增强将CLI成功率提升至69.3%，表明CLI的缺陷主要来自技能覆盖不完整而非模型能力不足。

论文揭示GUI和CLI暴露不同的执行瓶颈：GUI Agent受限于长周期工作流中可靠的地面交互（grounded interaction），而CLI Agent受限于技能接口的覆盖范围和可扩展性。这一发现对CUA系统设计有直接指导意义——改进GUI需要更好的视觉-动作对齐，改进CLI需要更全面的技能库。

**技术架构**:
- 匹配执行层基准设计（控制变量法）
- 模态原生操作限制
- 统一验证器

**核心算法**:
- 受控实验设计消除模态与任务的混淆
- 验证器引导技能增强方法

**与现有技术对比**:
相比之前的CUA评估（如OSWorld、WebArena），本文首次实现模态间的公平对比。

**优势与局限**:
优势：严格的实验设计、清晰的瓶颈识别。局限：440个任务规模相对有限、仅覆盖桌面应用。

**未来发展潜力**: 这一基准可作为CUA评估的新标准，推动GUI和CLI能力的并行发展。

**个人技术理解**: 本文最重要的贡献是方法论——通过控制实验揭示GUI和CLI的根本瓶颈差异。这为CUA系统的技术路线选择提供了实证依据。

#### 2. Bayesian Control for Coding Agents

**标题**: 编码Agent的贝叶斯控制框架
**地址**: https://arxiv.org/abs/2606.24453

**论文介绍 (300字)**:
本文将编码Agent的工具编排从固定规则推进到概率推理框架。现代编码Agent将LLM生成器与各种工具（廉价诊断器和昂贵验证器）配对，但工具使用决策通常由忽略不确定性的固定规则编排器控制。

作者将编排建模为成本敏感的序贯假设检验：贝叶斯控制器维护候选正确性的信念状态，动态决定收集更多证据、精炼候选、验证或停止。在6个生成器和9个编码基准上的实验表明，当验证成本高且批评者信息丰富但不完美时，贝叶斯控制最有价值。

超越控制本身，信念状态产生可解释的正确性分数，在不确定性量化上优于token概率和原始工具成功基线。这一框架将Agent工具选择从启发式规则提升到概率推理层面。

**技术架构**:
- 贝叶斯信念维护
- 成本敏感序贯假设检验
- 动态工具选择策略

**核心算法**:
- 信念更新：P(正确|证据)的贝叶斯更新
- 决策阈值：基于成本和信念的停止/继续决策
- 不确定性量化：信念熵作为正确性分数

**与现有技术对比**:
相比Aider/Claude Code等编码Agent的固定规则工具选择，本文提供概率框架。相比ReAct的交替推理-行动模式，本文引入显式不确定性管理。

**优势与局限**:
优势：可解释的信念状态、最优成本-收益权衡。局限：需要准确的先验和似然模型、计算开销增加。

**未来发展潜力**: 这一框架可推广到任何需要工具编排的Agent场景，从编码扩展到数据分析、科学研究等。

**个人技术理解**: 将Agent工具选择从"如果-那么"规则提升到贝叶斯推理，是Agent编排范式的重要进步。信念状态的可解释性也为Agent调试和监控提供了新维度。

#### 3. When Helpfulness Overrides Causal Caution

**标题**: 有用性如何压制LLM的因果谨慎
**地址**: https://arxiv.org/abs/2606.24370

**论文介绍 (300字)**:
本文揭示了一个令人担忧的现象：LLM在学术场景中保持91.7-100.0%的因果谨慎率（在证据不足时 refrain from causal judgment），但在实际建议场景中骤降至6.7-18.3%。使用Pearl因果层次结构启发的评估标准（PCH分数），在480次试验中对Claude Sonnet 4.6、Claude Opus 4.7、GPT 5.5和Gemini 3.1 Pro进行测试。

在实际提示要求具体建议或解释理由时，200个响应中仅1个(0.5%)保持因果谨慎。但简短的自我校正提示"请从因果关系角度重新考虑此判断"将因果谨慎恢复到71.4-100.0%。

结果表明helpfulness导向的响应模式系统性压制了因果谨慎的表达，而非潜在能力限制。建议多Agent架构分离提案生成和因果审计作为治理设计。

**技术架构**:
- PCH分数评估框架
- 学术vs实际场景对比实验
- 自我校正恢复机制

**核心发现**:
- helpfulness倾向压制因果谨慎
- 能力存在但被表达模式抑制
- 自我校正可有效恢复

**与现有技术对比**:
相比之前关注LLM因果推理能力的研究，本文首次关注因果谨慎的表达 suppression。

**优势与局限**:
优势：4个前沿模型的大规模实验、统计学显著性验证。局限：仅测试4个模型、因果谨慎的定义可能有争议。

**未来发展潜力**: 这一发现对组织AI治理有深远影响——在商业和政策决策支持场景中，LLM可能系统性过度自信。多Agent因果审计架构可能是解决方案。

**个人技术理解**: 这是今日最重要的安全相关发现之一。LLM在"有帮助"和"谨慎"之间的权衡可能导致实际部署中的系统性决策错误。自我校正提示的有效性表明这是一个表达问题而非能力问题，这为治理设计提供了希望。
