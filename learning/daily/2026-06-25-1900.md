1. **SAFARI - 长周期Agent故障归因框架**
   - 地址: https://arxiv.org/abs/2606.24626
   - 来源: arXiv
   - 领域: Agent Testing / 故障诊断
   - 技术介绍: SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation) 解决多步骤多Agent任务中故障归因的扩展性问题。当前方法将完整执行轨迹加载到LLM上下文窗口，面临注意力稀释和上下文限制问题。SAFARI用工具增强的诊断循环替代线性上下文加载，为LLM配备专门的工具箱来读取和搜索轨迹片段，配合持久化短期记忆进行跨轮次推理。实验显示在1M token预算下Who&When数据集上超越SOTA 20%，即使目标故障位于5倍上下文窗口之外仍保持0.58精度。
   - 应用场景: Agent故障诊断 / 多Agent系统调试 / 长周期任务归因分析
   - 为什么值得关注: 首次系统性解决Agent故障归因的上下文扩展问题 对多Agent系统可观测性有直接价值

2. **Computer-Use Agent GUI vs CLI 执行瓶颈对比**
   - 地址: https://arxiv.org/abs/2606.24551
   - 来源: arXiv
   - 领域: Agent Testing / 评估
   - 技术介绍: 研究首次以受控方式对比GUI Agent和CLI Agent的执行瓶颈。在440个桌面任务、18个应用、12个工作流类别的匹配基准上，最强GUI Agent达到59.1%完全通过率，超过原始技能CLI Agent的48.2%；但验证器引导的技能增强将CLI成功率提升至69.3%。关键发现：GUI Agent受限于长周期工作流中可靠的接地交互，CLI Agent受限于技能接口的覆盖范围和可扩展性。
   - 应用场景: Computer-Use Agent评估 / GUI vs CLI交互范式选择
   - 为什么值得关注: 首次在受控条件下分离交互模态与任务差异 为Agent交互设计提供实证依据

3. **RL with Autonomous Evaluation for Computer-Use Agents**
   - 地址: https://arxiv.org/abs/2606.24515
   - 来源: arXiv
   - 领域: Agent Testing / RL训练
   - 技术介绍: 提出用自主视觉语言评估作为GUI Agent RL微调的可扩展监督信号。给定最终截图和原始指令，VLM判断任务完成度并提供终端反馈，无需任务特定启发式或手动标签。由于自主评估器不完美，将反馈建模为噪声二元奖励通道并推导出噪声校正奖励估计器用于PPO。在macOSWorld、Windows Agent Arena和OSWorld上平均提升12.6个百分点。
   - 应用场景: GUI Agent RL训练 / 自动化评估管线
   - 为什么值得关注: 解决CUA训练中最核心的可扩展奖励信号问题 噪声校正方法具有通用性

4. **LemonHarness - Agent工作空间边界管理**
   - 地址: https://arxiv.org/abs/2606.24311
   - 来源: arXiv
   - 领域: Agent Testing / 测试基础设施
   - 技术介绍: LemonHarness解决LLM Agent在长任务中修改工作空间状态的问题。Agent通常只观察工具输出和日志片段，而实际状态变化发生在文件系统中。没有显式工作空间边界时，文件写入和临时工件生成等操作会分散更改，随时间累积导致状态难以追踪。LemonHarness提供工作空间边界管理机制，使状态变化可追踪和可回滚。
   - 应用场景: Agent测试环境隔离 / 工作空间状态管理 / 测试可重复性
   - 为什么值得关注: 直击Agent测试基础设施的核心痛点 状态可追踪性是可靠测试的前提

5. **Themis - RLHF可解释测试评估框架**
   - 地址: https://arxiv.org/abs/2606.24622
   - 来源: arXiv
   - 领域: RL Testing / 可解释AI
   - 技术介绍: Themis是首个结合可解释AI(XAI)和人类反馈对齐的RL测试评估框架。支持200+广泛使用的环境，可配置用于RL、透明度和对齐实验。结果证明可以训练匹配或超越环境真实奖励信号的奖励模型。提供基于云的平台收集人类反馈和管理实验，支持在适度商业机器上同时运行千用户实验。
   - 应用场景: RLHF测试评估 / 奖励模型训练 / 人类反馈收集
   - 为什么值得关注: 填补XAI与RLHF结合的工具空白 对RL训练透明度有直接价值

7. **AdversaBench - 多法官确认LLM红队框架**
   - 地址: https://arxiv.org/abs/2606.24589
   - 来源: arXiv
   - 领域: LLM Testing / 安全红队
   - 技术介绍: AdversaBench是端到端LLM红队管线，用5种结构化操作符变异种子提示，查询目标模型，通过三法官面板加元法官决胜确认失败。45个种子每个都产生了确认失败。关键发现：操作符有效性因类别而异；二元失败率掩盖难度差异；80-87%的成对法官一致性与接近零的Cohen's kappa共存；对抗提示从Llama 3.1 8B零样本迁移到Llama 3.3 70B。
   - 应用场景: LLM安全评估 / 红队测试自动化 / 跨模型脆弱性分析
   - 为什么值得关注: 揭示对抗提示的跨模型迁移性 对安全评估有深远影响

13. **Can Scale Save Us From Plasticity Loss?**
    - 地址: https://arxiv.org/abs/2606.24752
    - 来源: arXiv
    - 领域: LLM训练 / 持续学习
    - 技术介绍: 研究可塑性损失（网络学习新信息能力的退化）在现代Transformer LLM中是否仍是问题。在5M到314M参数的GPT风格模型上研究多语言持续学习。发现可塑性损失遵循可预测的缩放定律，随模型大小次线性增长。更大模型可能延迟可塑性损失的测量效应，但仅增加参数数量可能不足以完全阻止它。在平稳多语言训练下也发现可塑性损失证据。
    - 应用场景: 持续学习 / 模型可扩展性分析 / 训练策略优化
    - 为什么值得关注: 挑战"规模解决一切"的假设 对持续学习策略有根本性影响

14. **Scaling Laws for Task-Specific LLM Distillation**
    - 地址: https://arxiv.org/abs/2606.24747
    - 来源: arXiv
    - 领域: LLM压缩 / 缩放定律
    - 技术介绍: 推导领域特定LLM压缩的经验缩放定律，量化领域知识和通用知识性能如何随数据集大小、压缩比、监督格式和迭代修剪计划变化。比较基于logit和基于LoRA的蒸馏，引入混合CoT监督损失来稳定KL散度蒸馏。发现监督格式是权衡的关键驱动因素，CoT监督积极恢复修剪删除的通用知识。

15. **BlockTrain - 去中心化AI训练和推理**
    - 地址: https://arxiv.org/abs/2606.24722
    - 来源: arXiv
    - 领域: 分布式训练 / 去中心化AI
    - 技术介绍: Spheroid BlockTrain是去中心化训练协议，将模型划分为可独立训练的块，每个块在本地目标上优化并在推理时组合为一个模型。在byte级WikiText上达到CE 1.359（perplexity 3.89），与相同设置的端到端Transformer参考仅差0.04 CE。每个活跃工作者只训练一个块，避免完整模型优化器状态。推理路径使用一个块栈遍历完成完整输出。
    - 应用场景: 去中心化训练 / 分布式推理 / 开放AI基础设施
    - 为什么值得关注: 挑战 hyperscaler 对AI训练的中心化垄断 可能改变AI基础设施格局

16. **CompressKV - 语义检索引导的KV缓存压缩**
    - 地址: https://arxiv.org/abs/2606.24467
    - 来源: arXiv
    - 领域: LLM推理优化 / KV缓存
    - 技术介绍: CompressKV是为GQA基LLM设计的资源高效KV缓存压缩框架。识别捕获提示初始/最终token和语义重要中间证据的语义检索头(SRHs)，用它们选择应保留的token KV对。根据离线估计的逐层驱逐误差分配缓存预算。在LongBench QA任务上仅用3% KV缓存保持97%全缓存性能，在Needle-in-a-Haystack上仅用0.7% KV存储达到90%准确率。
    - 应用场景: 长上下文LLM推理 / 资源受限部署 / 缓存优化
    - 为什么值得关注: 极致的KV缓存压缩比 对长上下文推理成本优化有直接价值

18. **ReM-MoA - 推理记忆维持MoA扩展**

19. **LaGO - 在线RL的潜动作引导**
    - 地址: https://arxiv.org/abs/2606.24669
    - 来源: arXiv (ICML 2026 Workshop)
    - 领域: 强化学习 / 机器人控制

    - 应用场景: 机器人控制 / 连续决策 / LLM辅助RL
    - 为什么值得关注: 避免LLM直接控制的不可靠性 软引导范式更实用

20. **Token经济学 - Foundation模型的Token、计算和定价**
    - 地址: https://arxiv.org/abs/2606.24616

#### 1. SAFARI - 长周期Agent故障归因框架

**论文介绍**
SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation) 解决多步骤多Agent任务中故障归因的扩展性问题。当前方法将完整执行轨迹加载到LLM上下文窗口，面临注意力稀释和上下文限制。SAFARI用工具增强的诊断循环替代线性上下文加载，为LLM配备专用工具箱读取和搜索轨迹片段，配合持久化短期记忆进行跨轮次推理。

**技术架构**
- 工具增强诊断循环：LLM配备专用工具箱（读取/搜索轨迹片段）
- 持久化短期记忆(STM)：跨轮次推理状态维护
- 主动调查策略：动态选择需要调查的轨迹片段
- 解耦诊断精度与架构上下文限制

**核心算法**
- 轨迹分段搜索：不加载完整轨迹，而是按需读取相关片段
- STM维护：跨诊断轮次的状态持久化
- 主动调查：基于不确定性动态选择下一步调查目标

**与现有技术对比**
- 传统方法：完整轨迹加载 → 注意力稀释 + 上下文限制
- SAFARI：工具增强诊断循环 → 解耦诊断精度与上下文限制

**关键数据**
- Who&When数据集：1M token预算下超越SOTA 20%
- TRAIL GAIA子集：25K token预算下超越19%
- 目标故障位于5倍上下文窗口之外仍保持0.58精度

**优势与局限**
- 优势：突破上下文限制、主动调查策略、持久化记忆
- 局限：依赖轨迹可搜索性、诊断精度受工具箱能力限制

**发展潜力**
对多Agent系统可观测性和调试有直接价值。随着Agent任务复杂度增加，故障归因将成为关键基础设施需求。

#### 2. Structural Certification for General Agents (ICML 2026)

**论文介绍**
证明通用Agent不可能是万能的，其能力不可避免地专业化。引入结构认证框架，将受限目标引导的性能映射到Agent内部世界模型的逐条目保证。提供算法使用深度组合目标过滤特定转换，证明通用Agent在这些目标上具有O(1/n)+O(δ)误差界的结构世界模型。

**技术架构**
- 大世界假设：Agent无法普遍能力，能力必然专业化
- 结构认证框架：转换局部框架，映射有界目标引导性能到世界模型逐条目保证
- 深度组合目标过滤：算法过滤特定转换
- 误差界证明：O(1/n)+O(δ)误差界，小δ范围内紧致

**核心贡献**
- 理论证明：通用Agent不是万能的（形式化证明）
- 框架创新：结构认证替代标准一致性保证
- 算法构造：可计算的认证算法

**优势与局限**
- 优势：首次为通用Agent提供可认证理论框架、ICML 2026录用
- 局限：理论性强、实际应用需要进一步工程化

**发展潜力**
为Agent安全部署提供理论基础。随着Agent在关键领域的应用增加，可认证性将成为刚需。

#### 3. ReM-MoA - 推理记忆维持MoA扩展

**论文介绍**
ReM-MoA是记忆增强的Mixture-of-Agents框架，解决MoA深度扩展中的性能退化问题。通过排名推理记忆和策划多样化记忆路由两种机制维持扩展性。引入可选多领域Reviewer蒸馏管线提高排名质量。

**技术架构**
- 排名推理记忆：持久存储和排名所有层推理痕迹
- 策划多样化记忆路由：不同Agent暴露于成功/失败痕迹的不同组合
- 多领域Reviewer蒸馏：前沿模型监督提高排名质量
- 跨层推理记忆传递机制

**核心创新**
- 结构化跨层推理记忆：MoA中缺失的关键机制
- 多样化路由：保持探索多样性同时传播高质量推理
- 深度扩展优势：优势随深度增加而扩大

**关键数据**
- 5个推理基准（数学、形式逻辑、代码、知识、常识）
- 一致超越先前MoA变体
- 优势随深度增加而扩大

**优势与局限**
- 优势：解决MoA深度退化、跨基准一致性、可扩展架构
- 局限：增加记忆管理开销、Reviewer蒸馏需要前沿模型

**发展潜力**
多Agent推理是scaling law之外的关键方向。ReM-MoA提供的结构化记忆机制可能成为多Agent推理的标准架构组件。

## 技术趋势

1. **Agent GUI自动化从后端走向前端**：Alibaba Page Agent（纯JS页面内）和Apple Container（原生Mac容器）代表Agent工具从后端/服务端向用户端/前端迁移的趋势。Agent不再需要浏览器扩展或Python后端，直接在页面内或原生OS中运行。

2. **多Agent推理从简单混合到结构化记忆**：ReM-MoA证明结构化跨层推理记忆是MoA深度扩展的关键。多Agent推理正在从"简单混合输出"进化为"有记忆的结构化协作"。

3. **Agent安全从行为层到内部层**：PHANTOM（VLM对抗数据集）和AdversaBench（多法官红队）代表Agent安全评估从外部行为测试向内部结构分析深化。同时SAFARI从故障归因角度提供Agent可观测性。

4. **LLM训练从规模驱动到效率驱动**：Plasticity Loss研究挑战"规模解决一切"，CompressKV实现极致KV缓存压缩，Scaling Laws for Distillation提供实用压缩决策框架。训练和推理效率成为与规模同等重要的维度。

5. **云厂商+AI Agent融合加速**：AWS Agent Toolkit代表云厂商正式将Agent工具纳入官方产品矩阵。云+Agent将成为下一代开发者平台的标准配置。
