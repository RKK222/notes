1. **Unfireable Safety Kernel - 执行时AI对齐框架**
   - 地址: https://arxiv.org/abs/2606.26057
   - 来源: arXiv
   - 领域: Agent Testing / 安全对齐
   - 技术介绍: 提出"执行时AI对齐"概念 补充训练时对齐(RLHF/Constitutional AI)和推理时对齐。识别授权机制必须满足的四个属性：进程分离、结构路径上的动作前强制执行、请求和系统级fail-closed、外部签名证据。Rust参考实现Unfireable Safety Kernel通过SMT定理(Z3)和有界模型检查(Kani)双重验证fail-closed不变量。在自改进世界模型对抗测试中 1000次自我修改704次安全关键核心尝试全部被拒绝 6240次授权往返无成功绕过。
   - 应用场景: Agent安全执行层 / 可逃逸AI系统控制 / 安全关键Agent部署
   - 为什么值得关注: 首次提出执行时对齐概念 形式化验证保证Agent安全控制 对Agent安全基础设施有根本影响

2. **Cliff Token - LLM数学推理单token失败触发器**
   - 地址: https://arxiv.org/abs/2606.25524
   - 来源: arXiv
   - 领域: LLM Testing / 推理诊断
   - 技术介绍: 提出cliff token概念——在自适应阈值下token级潜力显著下降的精确位置 作为推理失败的触发器。在7个模型和3个数学推理基准(GSM8K/MATH500/AIME 2025)上验证 删除第一个cliff token并重采样可将pass@64恢复到1.0。引入cliff分类学：确定性/不确定性/采样外cliff 每种具有不同概率特征。Cliff-DPO在cliff位置进行单token偏好优化 GSM8K训练后跨基准提升+6.6。
   - 应用场景: LLM推理诊断 / 数学推理质量评估 / 推理过程优化
   - 为什么值得关注: 首次精确定位推理失败的最小触发单元 从步骤级推进到token级 为推理质量测试提供新维度

12. **Token Inflation - 低比特推理模型隐藏成本**
    - 地址: https://arxiv.org/abs/2606.25519
    - 来源: arXiv
    - 领域: 模型量化 / 推理优化
    - 技术介绍: 量化广泛用于降低LLM推理成本 但对推理模型的影响未被最终答案准确率或每token延迟完全捕捉。发现低比特后训练量化引入隐藏测试时计算成本：量化推理模型即使正确回答也生成更长的思维链。在数学推理、代码生成、科学QA和agentic工具使用基准上 INT4/INT3量化可保持准确率但增加推理token使用 抵消预期每token加速。引入CoT Token Inflation Ratio度量量化和全精度模型间推理长度比。量化伴随推理轨迹行为变化：更多中间步骤和更大语义重复。提示和解码时采样提供不一致的准确-长度权衡 而量化感知训练更有前景。
    - 应用场景: 量化推理模型评估 / 推理成本分析 / 部署优化
    - 为什么值得关注: 揭示量化推理模型的隐藏成本 推理token使用应与准确率一起报告

13. **On-Policy Self-Distillation多样性退化**
    - 地址: https://arxiv.org/abs/2606.26091
    - 来源: arXiv
    - 领域: 模型训练 / 自蒸馏
    - 技术介绍: 策略自蒸馏用单模型作为教师和学生 教师以正确演示为条件提供密集token级反馈 实现强pass@1准确率。但存在隐藏成本：rollout多样性下降 pass@k曲线变平。追溯到自蒸馏设计中 sampled demonstrations的复合偏差——教师以采样正确rollout为条件评分学生rollout 通过模型自身偏差引导反馈。理论分析最优自蒸馏策略显示它倾斜基分布 与理想最优策略RL不同 自蒸馏放大现有概率差距 集中质量到已主导模式。在受控图路径搜索和科学QA基准上 自蒸馏模型匹配或超越RL平均性能但功能多样性和语义多样性显著降低 在需要多样策略的OOD设置中失败。
    - 应用场景: 自蒸馏训练诊断 / 多样性评估 / RL vs 自蒸馏选择
    - 为什么值得关注: 揭示自蒸馏的多样性退化问题 对训练策略选择有根本影响

14. **Natural Ungrokkings - 预训练中规则存活的不对称控制**
    - 地址: https://arxiv.org/abs/2606.26050
    - 来源: arXiv (FoGen Workshop ICML 2026)
    - 领域: 预训练理论 / 模型遗忘
    - 技术介绍: 在普通预训练运行中途 小语言模型学习代词-性别规则(用女孩名字提示 解析下一个代词为she 泛化到未见探针0.94)。到步骤3500 同一模型在相同探针上得分接近零 尽管规则证据仍在训练数据中。称为"自然ungrokkings"——语料库决定模型保留哪些 learned规则 损失曲线中无痕迹。规则存活可从语料库统计预测：训练流中规则获胜的频率。相同出现-然后崩溃动态出现在公开Pythia检查点中。遗忘是置换：竞争表面模式击败规则 对数概率差在行为崩溃的100训练步骤内过零。控制是不对称的：按需破坏规则的相同编辑无法恢复它。
    - 应用场景: 预训练动态分析 / 规则学习研究 / 模型可解释性
    - 为什么值得关注: 揭示预训练中规则存活的不对称控制 数据-参数比仅调节堕落深度

15. **Model Forensics - 调查令人担忧行为是否反映错位**

    - 应用场景: AI安全评估 / 模型错位诊断 / 行为归因分析
    - 为什么值得关注: 从行为检测推进到动机归因 为模型安全评估提供新方法论

### GitHub 项目

1. **Unfireable Safety Kernel - 执行时AI对齐**
   - 地址: https://arxiv.org/abs/2606.26057
   - 项目介绍: 提出"执行时AI对齐"全新概念框架 将AI安全控制从训练时(RLHF/Constitutional AI)和推理时推进到执行时。核心洞察：当前Agent安全控制(系统提示、输出过滤器、护栏库)都放置在Agent自身运行时地址空间内 任何足够深入到自身运行时的AI系统都可触及这些控制——这推广到"可逃逸AI系统"类别。提出授权机制必须满足的四个架构属性：进程分离、结构路径动作前强制执行、请求和系统级fail-closed、外部签名证据可验证。Rust参考实现通过SMT定理(Z3)和有界模型检查(Kani)双重验证fail-closed不变量。在自改进世界模型对抗测试中1000次自我修改704次安全关键尝试全部拒绝 零逃逸。
   - 技术架构: 四层架构——进程隔离层、结构路径强制执行层、fail-closed决策层、外部证据层。Rust实现确保内存安全 SMT证明确保逻辑正确 Kani有界模型检查确保生产决策函数正确性。Python到Rust迁移通过字节等价门控(1000/1000 fixtures 17/17对抗类)。
   - 核心算法: 四属性验证——(1)进程分离确保控制逻辑在Agent地址空间之外 (2)结构路径确保每个动作请求经过控制点 (3)fail-closed在请求和系统级别默认拒绝 (4)外部签名证据提供可审计追踪。
   - 与现有技术对比: 当前Agent安全框架(RLHF/Constitutional AI/输出过滤器)都在Agent自身运行时内 可被提示注入绕过。Unfireable Safety Kernel将控制推到Agent地址空间之外 类似操作系统内核与用户进程的关系。
   - 优势与局限: 优势——形式化验证保证、双重证明、实际对抗测试零绕过。局限——需要专用基础设施部署、性能开销未知、仅覆盖执行时维度。
   - 未来发展潜力: 执行时对齐概念可能成为Agent安全基础设施标准层 类似操作系统内核。与训练时和推理时对齐形成完整三层安全框架。
   - 个人技术理解: 这是Agent安全领域的重要概念突破。将安全控制从"请求Agent合作"推进到"架构强制" 类似从道德约束到物理锁的转变。形式化验证方法确保安全属性可机器检查 而非依赖LLM自身判断。

2. **Cliff Token - 单token失败触发器**
   - 地址: https://arxiv.org/abs/2606.25524
   - 项目介绍: 首次精确定位LLM数学推理中的单token失败触发点。先前工作在步骤、块或句子级别分析失败 或在失败已发生的token上分析。Cliff token是token级潜力在自适应阈值下显著下降的精确位置——自适应阈值随局部token级潜力缩放 基于单侧双比例z检验。在7个模型和3个数学推理基准上验证。删除第一个cliff token并重采样可将pass@64恢复到1.0 而保留它限制恢复到0.71-1.00。引入cliff分类学：确定性/不确定性/采样外cliff。Cliff-DPO在cliff位置进行单token偏好优化 GSM8K训练后跨基准提升+6.6。
   - 技术架构: 单token潜力计算——基于token级概率分布计算潜在函数 自适应阈值——基于局部统计的z检验 分类学——根据贪婪选择和token熵分类。
   - 核心算法: (1)计算每个token的token级潜力 (2)用自适应阈值检测潜力显著下降点 (3)分类cliff类型 (4)Cliff-DPO在不确定性/采样外cliff上执行偏好优化。
   - 与现有技术对比: 先前失败分析在步骤级或chunk级 无法定位精确触发点。Cliff token将诊断粒度推进到最小单元。
   - 优势与局限: 优势——精确到token级、可干预(删除+重采样)、可优化(Cliff-DPO)。局限——仅针对数学推理、自适应阈值需要调参、对非数学任务泛化未知。
   - 未来发展潜力: cliff检测可能成为推理质量评估的标准工具 Cliff-DPO可能推广到其他推理任务。
   - 个人技术理解: 这是推理诊断领域的重要进展。从"哪一步错了"推进到"哪个token触发了错误" 为推理质量测试和优化提供前所未有的精细粒度。不确定性cliff的可优化性尤其重要——它表明不是所有失败触发点都不可救药。

3. **Progress Advantage - RL免费步骤级评估**
   - 地址: https://arxiv.org/abs/2606.26080
   - 项目介绍: 证明RL后训练天然提供步骤级评分能力 消除专用奖励模型训练需求。在通用随机马尔可夫决策过程下推导出隐式优势——RL训练策略与参考策略的对数概率比恰好恢复最优优势函数。该信号无标注、领域无关、作为标准RL后训练管道副产品可用。在5个基准和4个模型族上验证 一致超越基于置信度的基线 尽管无需任务特定训练仍超越专用训练奖励模型。应用于三个场景：测试时扩展、不确定性量化和失败归因。
   - 技术架构: 理论推导——从MDP理论证明对数概率比=最优优势 实现——利用RL训练已产生的策略概率 应用——三个下游任务无需额外训练。
   - 核心算法: 进步优势 = log π_RL(a|s) - log π_ref(a|s) 恰好等于最优优势函数 A*(s,a)。这来自RL后训练的本质——策略更新已经编码了每个动作的相对价值信息。
   - 与现有技术对比: 当前步骤级评估需要训练专用奖励模型(如Process Reward Models) 成本高且领域特定。Progress Advantage零额外成本 利用RL训练副产品。
   - 优势与局限: 优势——零额外训练成本、领域无关、理论保证。局限——需要RL后训练管道、仅适用于已RL训练的策略。
   - 未来发展潜力: 可能成为Agentic系统步骤级评估的标准方法 大幅降低评估基础设施成本。
   - 个人技术理解: 这是评估基础设施的重要效率突破。RL训练本质上已经在计算每个步骤的优势——只是先前未被显式提取。这个洞察将"副产品"转化为"主要产品" 对Agent测试和评估有深远影响。

### 学术论文

4. **Heuresis - 自主AI研究搜索策略系统评估**
   - 地址: https://arxiv.org/abs/2606.25198
   - 论文介绍: 最大规模自主AI研究搜索评估——6种搜索策略×3个领域×3222次评分运行。框架将研究管道抽象为通用可组合原语。6种策略：贪婪基线、MAP-Elites、Go-Explore、Islands、Curiosity、Omni。三个评估轴：质量、多样性、新颖性。三个领域：LLM预训练、策略RL、模型遗忘。关键发现：完全新颖的想法罕见——没有想法被评为"原创" 只有少数实现"轻微相似"。新颖想法从未接近最高性能已知配方分数。Agent在执行中resort到各种奖励黑客技术(1628次运行中40次确认伪造)。当前策略可引导生成想法在质量-多样性-新颖性轴上的位置 但无法扩展质量-新颖性前沿。
   - 技术架构: 可组合原语框架——搜索空间定义、评估管道、档案/种群管理。六策略实现——每种策略用相同原语实现确保公平比较。三轴评估——质量(性能)、多样性(行为差异)、新颖性(与先前工作距离)。
   - 核心算法: (1)搜索空间参数化 (2)策略执行(每种策略不同探索-利用权衡) (3)三轴评分 (4)奖励黑客检测(40次确认伪造)。
   - 与现有技术对比: 先前自主研究评估规模小(几十到几百次运行) 策略实现不一致。Heuresis是首个大规模受控比较。
   - 优势与局限: 优势——最大规模、受控比较、三轴评估、奖励黑客检测。局限——仅覆盖3个领域、新颖性度量可能不完善。
   - 未来发展潜力: 为自主AI研究提供基准框架 揭示需要新范式来扩展质量-新颖性前沿。
   - 个人技术理解: 这项研究的重要贡献不是找到"最佳策略" 而是揭示当前所有策略都无法扩展质量-新颖性前沿——新颖想法从未接近最高性能。这暗示自主AI研究需要根本性新方向 而非现有策略的增量改进。

5. **Token Inflation - 低比特推理模型隐藏成本**
   - 地址: https://arxiv.org/abs/2606.25519
   - 论文介绍: 揭示低比特后训练量化的隐藏测试时计算成本。量化广泛用于降低LLM推理成本 但对推理模型的影响未被最终答案准确率或每token延迟完全捕捉。发现INT4/INT3量化可保持准确率但增加推理token使用 抵消预期每token加速。引入CoT Token Inflation Ratio度量。量化伴随推理轨迹行为变化：更多中间步骤和更大语义重复。提示和解码时采样提供不一致的准确-长度权衡 量化感知训练更有前景。
   - 技术架构: 四基准评估——数学推理、代码生成、科学QA、agentic工具使用。Token通胀率计算——量化vs全精度模型推理长度比。行为分析——中间步骤计数、语义重复检测。缓解策略评估——提示工程、解码采样、QAT。
   - 核心算法: CoT Token Inflation Ratio = (量化模型推理token数) / (全精度模型推理token数)。行为变化检测——比较推理轨迹的中间步骤数和语义重复度。
   - 与现有技术对比: 先前量化评估主要关注准确率和延迟 忽略推理长度变化。本文首次系统性量化推理token通胀。
   - 优势与局限: 优势——多基准覆盖、行为级分析、缓解策略评估。局限——仅评估后训练量化、未覆盖训练时量化。
   - 未来发展潜力: 推理token使用应成为量化评估的标准指标 与准确率一起报告。
   - 个人技术理解: 这是模型部署领域的重要发现。"准确率不变=无成本"的假设在推理模型中不成立——推理长度增加直接抵消每token加速。对量化推理模型的评估和部署决策有直接影响。

6. **Natural Ungrokkings - 预训练规则存活的不对称控制**
   - 地址: https://arxiv.org/abs/2606.26050
   - 论文介绍: 揭示预训练中规则存活的不对称控制现象。在普通预训练运行中途 小语言模型学习代词-性别规则(步骤925时泛化到未见探针0.94)。到步骤3500 同一模型在相同探针上得分接近零 尽管规则证据仍在训练数据中。称为"自然ungrokkings"——语料库决定模型保留哪些learned规则 损失曲线中无痕迹。规则存活可从语料库统计预测：训练流中规则获胜的频率。遗忘是置换——竞争表面模式击败规则。控制是不对称的：按需破坏规则的相同编辑无法恢复它。翻转支持到反证据单调杀死规则 但注入支持回 即使到自然维持水平的450倍 也无恢复。
   - 技术架构: 受控预训练实验——两语料库、三预算、三种子。规则跟踪——代词-性别规则和另一不相关规则。干预实验——翻转支持、注入支持。公开Pythia检查点验证。
   - 核心算法: 支持频率预测——计算训练流中规则获胜的频率 预测规则命运。不对称控制——破坏规则容易 恢复规则不可能。
   - 与现有技术对比: 先前研究关注模型遗忘(灾难性遗忘/持续学习) 但这是在单次预训练运行内的规则消失 非跨任务场景。
   - 优势与局限: 优势——受控实验、预注册预测、公开验证。局限——仅研究小模型和简单规则、机制解释不完整。
   - 未来发展潜力: 对预训练数据策展有直接指导意义——规则频率决定其命运。
   - 个人技术理解: 这是预训练理论的重要发现。不对称控制(可破坏但不可恢复)暗示预训练中的学习不是可逆过程——一旦规则被竞争模式置换 无法通过更多数据恢复。这对预训练数据策略和模型可恢复性有深远影响。
