## 深度研究

### GitHub 项目

#### 1. Murmur: Shared Communication Bus for Coding Agents
- **地址**: https://github.com/instavm/murmur
- **项目介绍**: Murmur 是一个为编码Agent设计的共享通信总线，解决多Agent协作中的信息传递瓶颈。在Agentic编程范式下，多个Agent需要协调工作、共享上下文、传递中间结果。传统方式依赖文件系统或消息队列，但缺乏针对Agent工作流优化的专用通信层。Murmur提供统一的通信抽象，支持发布-订阅模式、事件流、状态同步等核心能力。
- **技术架构**: 基于事件驱动的发布-订阅架构，支持多Agent间的实时消息传递。采用轻量级协议设计，避免传统消息队列的复杂性。
- **核心创新点**: 首次为Agent间通信提供专用基础设施，而非复用通用消息队列。针对Agent工作流特征（如上下文共享、状态同步、结果聚合）优化。
- **为什么值得关注**: 多Agent协作是Agentic编程的核心模式，通信基础设施的质量直接决定多Agent系统的可扩展性和可靠性。

#### 2. no-mistakes: Git Push Safety Guard
- **地址**: https://github.com/kunchenguid/no-mistakes (3,434 stars, +398/day)
- **项目介绍**: AI编码时代的关键安全基础设施。随着AI生成代码的普及，有缺陷代码被推送到主分支的风险急剧增加。no-mistakes 提供Git push前的安全守卫，通过静态分析、语义检查和AI辅助审查，防止有缺陷的代码进入生产环境。
- **技术架构**: 作为Git hook集成到工作流中，在push前自动触发安全检查。支持多种检查策略：语法正确性、语义一致性、安全漏洞检测。
- **核心创新点**: 将AI辅助代码审查从PR阶段推进到push阶段，在代码进入仓库之前就进行拦截。填补了AI编码安全链条的最后一环。
- **与现有技术对比**: 传统CI/CD在push后检查，no-mistakes在push前拦截，更早发现问题。
- **优势与局限**: 优势是前置安全屏障，减少有缺陷代码的传播；局限是可能增加push延迟，需要平衡安全性和开发效率。
- **未来发展潜力**: 随着AI编码Agent的普及，push安全守卫可能成为标准基础设施，类似CI/CD的普及路径。

#### 3. ai-berkshire: 价值投资研究框架
- **地址**: https://github.com/xbtlin/ai-berkshire (3,119 stars, +1,274/day)
- **项目介绍**: 基于Claude Code构建的价值投资研究框架，整合巴菲特、芒格、段永平、李录四大师的方法论，通过多Agent并行研究实现对抗性分析。将AI编码能力与价值投资哲学结合，创建系统化的投资研究工具。
- **技术架构**: 多Agent架构，每个Agent代表不同的投资哲学视角。通过对抗性分析框架，让不同视角的Agent相互质疑和补充。
- **核心创新点**: 将投资哲学形式化为可执行的Agent策略，实现AI驱动的系统化价值投资研究。
- **为什么值得关注**: 代表AI Agent在垂直领域的深度应用，从通用编码工具走向专业领域专家系统。

### 学术论文

#### 1. SKILL-DISCO: Distilling Agent Traces into Reusable Procedural Skills
- **地址**: https://arxiv.org/abs/2606.26669
- **论文介绍**: 提出从Agent执行轨迹中自动蒸馏和编译可复用程序技能的方法。传统Agent技能需要手工编写和维护，SKILL-DISCO通过分析Agent的成功执行轨迹，自动提取可复用的技能模式，并编译为可重用的程序表示。
- **技术架构**: 轨迹分析→模式提取→技能编译→验证优化四阶段管道。使用序列建模识别关键决策点，用程序综合生成可执行技能代码。
- **核心算法**: 基于轨迹聚类的技能模式发现算法，结合程序综合技术将自然语言轨迹转化为可执行代码。
- **关键实现机制**: 轨迹相似度度量、技能边界检测、代码生成验证三核心模块。
- **与现有技术对比**: 传统技能编写依赖人工经验，SKILL-DISCO实现自动化技能提取，大幅降低技能构建成本。
- **优势与局限**: 优势是自动化程度高，可从大量轨迹中学习；局限是依赖高质量轨迹数据，对复杂技能的泛化能力有待验证。
- **未来发展潜力**: 可能成为Agent技能生态的基础设施，实现技能的自动积累和共享。

#### 2. Ask, Don't Judge: Binary Questions for Interpretable LLM Evaluation
- **地址**: https://arxiv.org/abs/2606.27226
- **论文介绍**: 提出用二元问题替代传统黑盒评分的LLM评估新范式。传统LLM评估依赖单一分数或复杂评分标准，缺乏可解释性。该方法将评估标准分解为原子二元问题，每个问题独立回答，最终聚合为可解释的多维度评分。
- **技术架构**: 评估标准分解→二元问题生成→独立回答→结果聚合四阶段。元提示生成细粒度评估问题，LLM独立回答每个输出。
- **核心算法**: 基于元提示的二元问题生成算法，确保问题原子性和独立性。聚合算法将二元答案映射为多维度评分。
- **关键实现机制**: 问题生成器、回答评估器、结果聚合器三核心组件。
- **与现有技术对比**: 相比UniEval和G-Eval，Binary Questions提供更透明的评估过程，避免天花板效应。
- **优势与局限**: 优势是可解释性强，支持迭代优化；局限是问题生成质量依赖元提示设计。
- **未来发展潜力**: 可能成为LLM评估的标准方法，推动评估从黑盒走向透明。
