#### 1. Agent-Native Immune System (ANIS)
**地址**: https://arxiv.org/abs/2606.28270
**类型**: 论文

**项目介绍（300字）**:
ANIS 是首个生物启发的 Agent 内生防御架构，将安全从外部护栏推进到 Agent 认知循环内部。当前防御机制（ perimeter security、training-time alignment）都位于 Agent 活跃推理循环之外，导致完全对齐的 Agent 仍然极易受到运行时劫持——通过记忆毒化、工具链操纵或多 Agent 协议攻击。ANIS 的核心洞察是：Agent 安全需要像生物免疫系统一样，成为 Agent 的内在属性而非外部附加物。

ANIS 提出六层免疫塔（L0-L5），其中 L1 Barrier Immunity 作为非认知的物理和逻辑隔离层。框架建立了 Agent Virus 和 Agent Vaccine 的统一分类法，形式化区分表面的非参数防御和鲁棒的参数疫苗。提出 Harness Triad（Meta/Self/Auto）作为自监控元认知自动化骨干，驱动持续免疫学习（CIL），使疫苗能够动态适应新威胁。

**技术架构**:
- L0: 基础运行时隔离
- L1: Barrier Immunity - 非认知物理-逻辑隔离
- L2-L3: 威胁检测和响应
- L4-L5: 免疫学习和适应
- Harness Triad: Meta Harness（自监控）+ Self Harness（自诊断）+ Auto Harness（自修复）

**核心算法**:
- 持续免疫学习（CIL）：疫苗动态适应新威胁
- Autoimmunity Rate：衡量假阳性干预率
- 病原体-疫苗共进化动力学建模

**与现有技术对比**:
- 传统对齐：静态"宪法"价值基础，ANIS：动态"执法"机制
- 外部护栏：推理循环之外，ANIS：嵌入认知循环内部
- 规则驱动：手工规则，ANIS：参数疫苗自动演化

**优势与局限**:
- 优势：内生安全、动态适应、理论框架完整
- 局限：评估指标尚不成熟、实际部署经验缺乏

**未来发展潜力**:
免疫协议标准化、新型评估指标、集体智能生态中的共进化动力学。可能成为 Agent 安全基础设施的标准范式。

**个人技术理解**:
ANIS 代表 Agent 安全从"外部约束"到"内在免疫"的范式转变。六层免疫塔的设计借鉴了生物免疫系统的多层次防御思想，但将认知层和非认知层明确分离是关键创新。Harness Triad 的自监控-自诊断-自修复闭环与 Agent 自我改进研究形成互补。Autoimmunity Rate 作为评估指标很有价值——过度免疫（假阳性）和免疫不足（假阴性）需要平衡。

#### 2. Tandem Reinforcement Learning (TRL)
**地址**: https://arxiv.org/abs/2606.28166
**类型**: 论文

**项目介绍（300字）**:
TRL 将 Tandem Training 范式引入 RLVR（Reinforcement Learning with Verifiable Rewards）。RLVR 显著提升了 LLM 的推理能力，在竞赛数学等领域达到专家甚至超人水平。然而，较弱的 Agent 和人类能否真正利用这种能力远不确定——RLVR 被记录到会漂移推理 toward 特殊模式，如可读性差和语言混合。

Tandem Training 是一个新范式：训练好的强 senior 与冻结的弱 junior 共生成每个 rollout，两者作为团队获得奖励，从而推动 senior 以 junior 可理解的方式推理。TRL 将这一范式推进到 RLVR 中：senior 和冻结的 junior 随机交替共生成推理，生成结果获得奖励，标准 GRPO loss 应用于 senior。

**技术架构**:
- Senior-Junior 交替共生成机制
- 团队奖励信号（而非个体奖励）
- GRPO loss 仅应用于 senior
- 随机交替策略控制 co-generation 比例

**核心算法**:
- Tandem co-generation：senior 和 junior 随机交替生成 token
- Team reward：基于最终结果的团队奖励
- Distributional drift measurement：测量 senior 输出相对于 junior 的分布漂移

**与现有技术对比**:
- Vanilla GRPO：仅优化 solo 性能，TRL：同时优化 handoff 鲁棒性和可读性
- Standard Tandem：仅证明概念，TRL：扩展到 RLVR 长推理链
- 传统蒸馏：单向知识转移，TRL：双向协同进化

**优势与局限**:
- 优势：同一 rollout 结构同时获得三个属性（handoff 鲁棒性、分布漂移减少、CoT 可读性）
- 局限：需要两个模型协同训练、计算成本增加

**未来发展潜力**:
多模型通信协议、人机兼容性、RLVR 的可部署性。可能成为 RLVR 训练的标准范式。

**个人技术理解**:
TRL 的核心洞察是：RLVR 的强大推理能力如果不能传递给较弱模型和人类，其实际价值就大打折扣。Tandem co-generation 巧妙地将"可读性"作为训练信号而非后处理——senior 必须用 junior 能跟随的方式推理才能获得奖励。这与人类教学中"用学生能理解的方式解释"的直觉一致。三个涌现属性来自同一 rollout 结构的设计非常优雅。

#### 3. GILP: Grounded Iterative Language Planning
**地址**: https://arxiv.org/abs/2606.27806
**类型**: 论文

**项目介绍（250字）**:
GILP 解决 LLM Agent 在图结构规划任务中的幻觉传播问题。Agent-based 世界模型（LLM API）灵活但错误以幻觉状态变化的形式出现，难以用回归损失评分。Parameterized 世界模型（训练好的转移预测器）错误更容易测量（NodeMSE、delta accuracy），但作为独立规划器通常较弱。

GILP 训练一个小参数 backbone 并结合 API-based Agent 推理。Backbone 提供有效动作、预测状态增量、风险和值；LLM 起草动作和想象增量；一致性门控在两者不一致时要求修订。在真实 GPT-4o-mini 调用上将幻觉状态率从 17.6% 降至 3.5%，在校准模拟器中将成功率从 66.8% 提升至 83.8%，仅增加约 22% 额外 LLM 调用。

**技术架构**:
- Parameterized backbone：小模型预测状态转移
- LLM planner：生成动作和想象增量
- Consistency gate：比较两者输出，不一致时触发修订

**核心算法**:
- 一致性门控：backbone 预测 vs LLM 想象的比较
- 风险感知的动作选择
- 迭代修订循环

**与现有技术对比**:
- 纯 LLM 规划：幻觉率高，GILP：通过一致性门控大幅降低
- 纯参数规划：规划能力弱，GILP：结合 LLM 灵活性
- 传统验证：后验检查，GILP：实时一致性检查

**优势与局限**:
- 优势：大幅降低幻觉、仅增加 22% 额外调用、适用于真实 API
- 局限：需要训练参数 backbone、一致性门控可能过度保守

**未来发展潜力**:
Agent 规划可靠性、世界模型与 LLM 的协同、幻觉检测基础设施。

**个人技术理解**:
GILP 的核心价值在于将"幻觉检测"从后验分析推进到实时控制。一致性门控的设计类似于控制系统中的反馈回路——backbone 提供"ground truth"参考，LLM 提供"creative"探索，两者不一致时触发修订。这种设计避免了纯 LLM 规划的盲目自信和纯参数规划的创造力不足。22% 额外调用成本换取 80% 的幻觉降低是非常划算的 trade-off。

### 学术论文

#### 1. HORIZON: Agentic Hardware Design
**地址**: https://arxiv.org/abs/2606.28279
**类型**: 论文

**项目介绍（250字）**:
HORIZON 将硬件设计作为仓库级代码进化问题。Markdown harness 编译为项目包（领域知识 + 可执行评估器 + 接受谓词 + git/runtime 策略），hands-free Agent 循环进化隔离的 git worktree，使用仓库操作进行状态管理、追踪和重放。在 ChipBench、RTLLM、Verilog-Eval 和 9 个 CVDP 类别上 100% 完成。

**技术理解**:
将硬件设计从 EDA 工具链推进到 Agent 原生范式。git worktree 作为状态管理单元的设计巧妙——每个进化尝试在隔离环境中进行，成功时合并，失败时丢弃。100% 基准完成但作者坦诚这些基准是"受控代理"，真实芯片设计仍是开放挑战。

#### 2. NormAct: Hidden Social Norm Compliance
**地址**: https://arxiv.org/abs/2606.27826
**类型**: 论文

**项目介绍（250字）**:
NormAct 是首个具身社交规范交互基准，评估 Agent 在隐式社会规范下的规划能力。GPT-5.4/Claude/Gemini 在显式目标上达到 67.3%，但在隐藏规范合规上仅 26.4%。差距源于激活和 grounding 相关规范的挑战而非缺乏社会知识。NormPerceptor 将 Task Success 从 24.2% 提升至 46.7%。

**技术理解**:
揭示具身 Agent 的"社会智能"缺口——模型知道社会规范但无法在具体情境中激活和应用。这对 AI 测试有重要启示：评估需要区分"知识拥有"和"知识应用"。

#### 3. ATOD: Annealed Turn-aware On-policy Distillation
**地址**: https://arxiv.org/abs/2606.27814
**类型**: 论文

**项目介绍（250字）**:
ATOD 是混合在线蒸馏算法，利用 OPD 和 RL 的互补性。退火 OPD-RL 调度：OPD 主导早期训练接近教师水平，RL 逐渐增强驱动奖励探索。Turn-level Disagreement-Uncertainty Reweighting 放大高效用 turns。在 ALFWorld/WebShop/Search-QA 上超越教师模型 2.16 点。

**技术理解**:
关键洞察是 OPD 和 RL 在不同训练阶段各有优势——早期 OPD 快速接近教师，后期 RL 探索超越教师。退火调度自动平衡两者。超越教师模型 2.16 点证明学生可以"青出于蓝"。
