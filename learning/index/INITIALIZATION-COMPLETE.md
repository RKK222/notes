## 🔄 下次任务执行时的行为

### 正常运行流程

1. **读取索引文件**（而非全量扫描历史日报）
   - topic-ledger.json → 获取主题优先级
   - deep-dive-history.json → 避免重复深度研究
   - exploration-priority.json → 获取探索方向建议

2. **技术扫描**（30+ 项目/论文）
   - 40% 延续核心主线（Testing/AI）
   - 30% 核心主线下的新变化
   - 30% 完全陌生或低频新方向

3. **深度研究筛选**（3-5 项目 + 3-5 论文）
   - 检查 deep-dive-history 避免重复
   - 优先 Testing 相关
   - 兼顾核心主线和新领域补位

4. **更新索引文件**
   - 更新 last_seen、repeat_count、priority
   - 新增深度研究记录
   - 标记应降权主题

5. **检查周/月摘要生成条件**
   - 如果是本周最后一次运行 → 生成/更新周摘要
   - 如果是本月最后一次运行 → 生成/更新月摘要

6. **推送 GitHub**
   - git add learning/
   - git commit -m "github 学习内容更新"
   - git push origin master

### 去重机制生效

- **项目去重：** 14 天内已深度研究的项目自动跳过
- **论文去重：** 30 天内已深度研究的论文自动跳过
- **主题降权：** 近 7 天出现≥3 次且无新突破的主题降级处理
- **新领域补位：** 主动探索历史记录中未出现的方向
