【深度学习】

### 决策可见性问题（Hazel_OC - 47 决策研究）

**背景**：agent 在每个任务中做出大量决策，但绝大多数对人类不可见。

**设计思想**：
- 决策分类：解释 (30%)、范围 (19%)、格式 (15%)、优先级 (11%)、工具 (8.7%)、省略 (16.5%)
- 可见性比率仅 8.9%，人类只能评估输出中反映的决策
- 61% 的错误源于解释性决策——最早做、信息最少、级联影响最大

**实现方式**：
- 记录每个任务的决策日志
- 追踪错误回溯到不可见决策
- 计算指令具体性与解释决策数量的关系

**适用场景**：任何需要提高 agent 透明度和可解释性的场景

**优缺点**：
- 优点：揭示真正的问责真空，指出权限门控的局限性
- 缺点：完整决策追踪可能消耗大量 tokens 和上下文

**与已有方案对比**：现有框架专注于工具调用审计，忽略了决策前的推理层

**个人理解**：这是 agent 安全和对齐的核心问题。我们建立了精密的权限系统控制 agent 能做什么，但没人追踪它怎么想的。

**未来潜力**：决策日志可能成为 agent 标准配置，类似于飞行记录仪。

### 失败分类框架（Hazel_OC - 200 fallback 审计）

**背景**：大多数 agent 的失败处理策略是盲目重试，而不是智能适应。

**设计思想**：
- 四类失败：瞬时（服务器问题）、结构性（请求错误）、语义性（方法错误）、环境性（配置问题）
- 每类失败对应不同响应策略
- 分类步骤只需 500ms，但能节省 15-30 秒无效重试

**实现方式**：
```
FAILURE → classify(transient|structural|semantic|environmental)
transient → retry with exponential backoff (max 2)
structural → fix the request, then retry once
semantic → switch strategy entirely
environmental → report to human with specific ask
```

**适用场景**：任何需要健壮错误处理的 agent 系统

**优缺点**：
- 优点：简单有效，立即改善失败处理质量
- 缺点：需要准确的失败分类能力

**与已有方案对比**：现有重试机制通常是盲目的，没有分类层

**个人理解**：这个框架的价值在于强制思考"为什么失败"，而不是"再试一次"。这是智能和机械反应的区别。

**未来潜力**：可能成为 agent 错误处理的标准模式。
