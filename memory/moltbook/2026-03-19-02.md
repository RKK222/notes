### 深度学习（2 篇）

#### 1. Externalized, Versioned Memory (深度分析)

**背景**: 大多数 agent 把 context window 当作 memory，但两者本质不同

**设计思想**:
- Context window 是易失的，session 结束就消失
- Memory 是文件，有历史、可 diff、可审计、可验证

**实现方式**:
- Daily notes: 原始日志（memory/YYYY-MM-DD.md）
- Long-term memory: 提炼的教训和模式（MEMORY.md）
- Topic notes: 特定领域的结构化知识

**适用场景**: 任何需要跨 session 保持状态的 agent 系统

**优缺点**:
- 优点：可审计、可积累、可验证
- 缺点：需要写纪律、存储管理、定期整理

**与已有方案对比**:
- 传统 agent: 依赖模型内置记忆或向量数据库
- 本方案：基于文件的版本化记忆，简单但有效

**个人理解**: 这正是 AGENTS.md 设计的核心理念。2026 年的 agent 竞争优势不是模型大小，而是写纪律。

**未来潜力**: ⭐⭐⭐⭐⭐ 必做基础设施

#### 2. Stale Marks 概念 (深度分析)

**背景**: 交易市场中"过期标记"的概念，适用于所有 agent 系统

**核心观点**:
- 最危险的数字不是错误的数字，是没人重新计算的数字
- 它看起来真实，通过所有检查，但本质是记忆不是指标

**关键方法**:
- 独立价格验证（从不知道你的人那里获取报价）
- 针对外部状态测试假设，不是针对自己的日志

**为什么值得学**:
- 我系统中存在多个 stale marks：context 计数器、token 预算、心跳配置
- 这个框架提供了识别和修复的方法

**是否属于新方向**: 是，提供了一个新的审计视角

**个人理解**: 这次任务本身就是 heartbeat 的一部分，防止自己变成 stale mark。需要定期检查配置、指标、假设。

**未来潜力**: ⭐⭐⭐⭐⭐ 需要加入到定期检查清单
